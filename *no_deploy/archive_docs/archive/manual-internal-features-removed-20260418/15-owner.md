# 15. オーナーダッシュボード

オーナーロールでログインすると、オーナーダッシュボード（`owner-dashboard.html`）に自動遷移します。テナント全体の管理を行うための画面です。

## 15.1 タブ一覧

| タブ名 | 説明 |
|--------|------|
| クロス店舗レポート | 全店舗の売上を横並びで比較 |
| ABC分析 | 売上貢献度によるメニュー分類 |
| 店舗管理 | 店舗の追加・編集・削除 |
| ユーザー管理 | 全店舗のアカウント管理 |
| AIアシスタント | 店舗を選択してAI分析 |
| 契約・プラン | サブスクリプション管理 |
| 決済設定 | Stripe（自前キー / POSLA経由Connect） |
| シフト概況 | 全店舗のシフト概要（enterpriseプラン） |
| 外部POS連携 | スマレジ連携 |

---

## 15.2 クロス店舗レポート

全店舗の売上データを横断的に分析します。

### サマリーカード

| 指標 | 説明 |
|------|------|
| 総売上 | 全店舗合計 |
| 総注文数 | 全店舗合計 |
| 店舗数 | 対象店舗数 |
| 平均注文額 | 全店舗の平均 |
| キャンセル件数 | 全店舗合計 |

### 店舗別パフォーマンス

各店舗の売上・注文数をバーチャートで比較表示します。

### その他の分析

| 分析項目 | 説明 |
|---------|------|
| サービス時間分析 | 平均待ち時間・調理時間・提供時間 |
| 品目ランキング | 全店舗横断でのトップ品目 |
| キャンセル品目 | キャンセルが多い品目と逸失売上 |
| 迷い品目 | カートに追加されたが削除された品目（メニュー改善のヒント） |

### 店舗フィルター

オーナーは特定の店舗を選択して、その店舗だけのデータを表示することもできます。

## 15.3 ABC分析

メニューを売上貢献度でA・B・Cの3ランクに分類します。

### サマリーカード（6項目）

| 指標 | 説明 |
|------|------|
| 総売上 | — |
| 粗利益 | 売上 − 原価 |
| 平均注文額 | — |
| 注文件数 | — |
| 品目数 | 分析対象の品目数 |
| 総原価 | — |

### ランク別サマリー

| ランク | 基準 | 意味 |
|--------|------|------|
| A | 売上構成比の上位70% | 主力商品。維持・強化すべき |
| B | 売上構成比の70〜90% | 準主力。改善の余地あり |
| C | 売上構成比の下位10% | 低貢献。見直しまたは廃止を検討 |

### ABC一覧テーブル

| 列 | 説明 |
|----|------|
| ランクバッジ | A/B/C |
| 順位 | 売上順 |
| 品名 | — |
| カテゴリ | — |
| 数量 | 販売数 |
| 売上 | — |
| 構成比 | 全体に占める割合 |
| 累積構成比 | — |
| 原価 | — |
| 利益 | 売上 − 原価 |
| 利益率 | — |

CSSのみのバーチャートで視覚化されます（外部チャートライブラリ不使用）。

### カテゴリ別集計

カテゴリ単位でグリッド表示し、カテゴリごとのパフォーマンスを比較します。

## 15.4 店舗管理

テナント内の店舗を管理します。

### 店舗の追加

| フィールド | 種類 | 必須 | 説明 |
|-----------|------|------|------|
| 店舗名 | テキスト | ✅ | 日本語名 |
| 店舗名（英語） | テキスト | — | 英語名 |
| スラッグ | テキスト | ✅ | URL用の識別子。英小文字・数字・ハイフンのみ |

### 店舗の編集

- 店舗名の変更
- 有効/無効の切替（無効にすると店舗の全機能が停止します）

### 店舗の削除

::: danger 注意
店舗を削除すると、その店舗に紐づく全てのデータ（メニュー、注文、シフト等）が影響を受けます。通常は削除ではなく無効化を推奨します。
:::

## 15.5 ユーザー管理

テナント内の全ユーザーアカウントを管理します。

### ユーザーの作成

| フィールド | 種類 | 必須 | 説明 |
|-----------|------|------|------|
| 表示名 | テキスト | ✅ | スタッフの表示名 |
| ユーザー名 | テキスト | ✅ | ログインID |
| メール | テキスト | — | 任意 |
| パスワード | パスワード | ✅ | 初期パスワード |
| ロール | セレクト | ✅ | owner / manager / staff |
| 所属店舗 | チェックボックス | ✅ | 複数店舗を選択可能（ownerは全店舗に暗黙アクセス） |

### スタッフ別の表示ツール設定

各スタッフに対して、利用可能な画面（ツール）を店舗ごとに設定できます。

| ツール | 説明 |
|--------|------|
| handy | ハンディPOS画面 |
| kds | KDS画面 |
| register | POSレジ画面 |

### スタッフ別の時給設定

スタッフごと・店舗ごとに個別の時給を設定できます。未設定の場合はシフト設定のデフォルト時給が使用されます。

### パスワード変更

オーナーは全ユーザーのパスワードを変更できます（パスワードを忘れたスタッフのリセット用）。
1. ユーザー一覧で対象ユーザーの「編集」ボタンをクリック
2. 「パスワード」欄に新しいパスワードを入力（パスワードポリシー要件あり、下記）
3. 「保存」をクリック
4. 新しいパスワードをスタッフに口頭で伝える

::: tip パスワードポリシー（P1-5）
新しいパスワードは以下を全て満たす必要があります。
- 8文字以上
- 英字（A-Z または a-z）を1文字以上含む
- 数字（0-9）を1文字以上含む

要件を満たさない場合、保存時に「パスワードは8文字以上で、英字と数字を含めてください」エラーが表示されます。
:::

### オーナー自身のパスワード変更

オーナーが自分のパスワードを変更する場合は、オーナーダッシュボード**右上の「パスワード変更」ボタン**から行います（ユーザー管理タブを経由する必要はありません）。

1. オーナーダッシュボード右上の「パスワード変更」ボタンを押す
2. 現在のパスワード・新しいパスワード・新しいパスワード（確認）を入力
3. 「変更する」を押す
4. 他の端末でログイン中のオーナーセッションは自動的に無効化されます（いま使っている端末は継続）

## 15.6 決済設定

Stripeで決済を受け付けるための設定を行います（proプラン以上）。2つの方式のいずれかを利用できます。詳細は第21章「Stripe決済の全設定」を参照してください。

### 設定方式

| 方式 | 説明 |
|------|------|
| パターンA：自前Stripeキー | テナントが自分のStripeアカウントを持っている場合。Stripe Secret Keyを登録して直接決済 |
| パターンB：POSLA経由Connect | テナントが自分のStripeアカウントを持っていない場合。「Stripe Connectに登録」から開始、POSLAが決済を代行（手数料1.0%） |

### 設定内容

- 使用する決済サービス（なし / Stripe）の切替
- パターンA：Stripe Secret Key の入力・更新・削除
- パターンB：Stripe Connectへの接続・状態確認・解除
- 接続中はアカウントID・`charges_enabled`・`payouts_enabled`・`details_submitted` を表示

### Stripe Connectの解除

パターンBからパターンAに切り替えたい場合は、「Stripe Connectを解除」ボタンで解除できます。解除後もStripeダッシュボード側のアカウント自体は残るため、再接続も可能です。

## 15.7 シフト概況（enterpriseプラン）

全店舗のシフト状況を横断的に確認します。

### 操作

| 操作 | 説明 |
|------|------|
| 店舗セレクター | 特定の店舗を選択 |
| 期間選択 | 週次 / 月次 |
| 日付入力 | カスタム期間 |

---

## 15.8 外部POS連携（スマレジ）

スマレジとの連携機能です。

### 連携機能

| 機能 | 説明 |
|------|------|
| 接続状態 | スマレジとのOAuth接続状態を表示 |
| 店舗マッピング | POSLAの店舗とスマレジの店舗を紐づけ |
| メニューインポート | スマレジの商品をPOSLAにインポート（非破壊。既存メニューは維持されます） |
