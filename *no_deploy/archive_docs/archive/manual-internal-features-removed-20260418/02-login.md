# 2. ログインとアカウント

## 2.1 初回ログインの手順

### 事前準備

ログインするには、オーナーまたはマネージャーが事前にアカウントを作成している必要があります。アカウント作成時に以下が決まります。

- **ユーザー名**（店舗内で一意。例：tanaka、suzuki、yamada01）
- **初期パスワード**（作成者が設定し、口頭やメモでスタッフに伝えます）
- **ロール**（owner / manager / staff）
- **所属店舗**（ownerは全店舗、manager/staffは指定された店舗のみ）

::: tip メールアドレスは不要です
POSLAのログインにメールアドレスは必要ありません。飲食店スタッフがメールアドレスを持っていない・教えたくないケースに配慮した設計です。ユーザー名とパスワードだけでログインできます。
:::

### ログイン手順

1. ブラウザで管理画面のURLにアクセスします
   - URL例：`https://あなたのドメイン/admin/`
2. ログイン画面が表示されます
3. **ユーザー名**欄に、アカウント作成時に決めたユーザー名を入力します
   - 例：`shibuya-staff`
4. **パスワード**欄に、初期パスワードを入力します
5. **ログイン**ボタンを押します
6. ログインに成功すると、自動的にダッシュボードに遷移します
   - オーナー → オーナーダッシュボード
   - マネージャー・スタッフ → 管理ダッシュボード

### ログインに失敗した場合

| エラーメッセージ | 原因 | 対処法 |
|----------------|------|--------|
| 「ユーザー名またはパスワードが正しくありません」 | ユーザー名かパスワードの入力ミス | 入力内容を確認してやり直す。大文字/小文字の区別に注意 |
| 「このアカウントは無効化されています」 | アカウントが無効化された | オーナーまたはマネージャーに連絡してアカウントを有効化してもらう |
| 「このテナントは無効化されています」 | テナント全体が無効化された | POSLA運営に連絡する |
| 「通信エラーが発生しました」 | インターネット接続の問題 | ネットワーク接続を確認して再試行する |
| 「サーバーが空のレスポンスを返しました」 | サーバー側の問題 | 少し待ってから再試行する。繰り返す場合はPOSLA運営に連絡 |

## 2.2 パスワード変更

パスワード変更には3つの経路があります。

### 自分のパスワードを変更する（推奨）

各ユーザーは画面右上の**「パスワード変更」ボタン**から自分のパスワードを変更できます。

1. ログイン後、画面右上のユーザー名の横にある**「パスワード変更」**ボタンを押す
   - 管理ダッシュボード（manager / staff）：右上の「ログアウト」ボタンの隣
   - オーナーダッシュボード（owner）：右上の「ログアウト」ボタンの隣
2. パスワード変更モーダルが開きます：
   - **現在のパスワード**：いま使っているパスワードを入力
   - **新しいパスワード**：新しいパスワードを入力（パスワードポリシー要件あり、下記）
   - **新しいパスワード（確認）**：もう一度同じものを入力
3. **「変更する」**を押す
4. 成功メッセージが表示され、画面はそのまま使えます
5. **他の端末でログイン中のセッションは自動的にログアウトされます**（いま使っている端末はそのまま継続）

::: tip パスワードポリシー
新しいパスワードは以下の要件を全て満たす必要があります：

- **8文字以上**
- **英字を1文字以上**含む（A-Z または a-z）
- **数字を1文字以上**含む（0-9）

例：`tanaka123`、`Yamada2026`、`shibuya2024`

要件を満たさない場合は「パスワードは8文字以上で、英字と数字を含めてください」というエラーが表示されます。
:::

::: warning 現在のパスワードが分からない場合
何度試してもログインできない場合や、現在のパスワードを忘れてしまった場合は、自分でパスワードを変更することはできません。下記の「オーナー/マネージャーが他のユーザーのパスワードを変更する」手順で、上司にリセットを依頼してください。
:::

### オーナーがスタッフのパスワードを変更する場合

スタッフがパスワードを忘れた場合や、新規スタッフの初期パスワードを設定する場合の手順です。

1. オーナーダッシュボードの「ユーザー管理」タブを開く
2. 対象のユーザーの**編集ボタン**を押す
3. 編集モーダルが開く
4. 「パスワード」欄に新しいパスワードを入力する（パスワードポリシーを満たすこと）
5. **保存**ボタンを押す
6. パスワードが更新される（新しいパスワードを口頭でスタッフに伝えてください）

### マネージャーがスタッフのパスワードを変更する場合

1. 管理ダッシュボードの「店舗設定」→「スタッフ管理」サブタブを開く
2. 対象のスタッフの**編集ボタン**を押す
3. 「パスワード」欄に新しいパスワードを入力する（パスワードポリシーを満たすこと）
4. **保存**を押す

::: warning 注意
マネージャーが変更できるのは自店舗のスタッフのパスワードのみです。他店舗のスタッフや他のマネージャーのパスワードは変更できません。
:::

### パスワード設定の運用ルール

- 推測されやすいパスワード（`12345678`、`password`、`tanaka2024` のような単純なパターン）は避けてください
- スタッフ間でパスワードを使い回さないでください
- 他人にパスワードを教えないでください（POSLA運営も問い合わせません）
- 退職者のパスワードは速やかにオーナーがリセット、またはアカウント自体を無効化してください

## マルチデバイスログイン制御

POSLAでは、同じアカウントで複数の端末（PC、タブレット、スマートフォン等）から同時にログインできます。ただし、以下の制御が動作します。

### 複数端末ログイン時の動作

1. ログイン時に、同じアカウントが他の端末でもログイン中かをシステムが自動チェックします
2. 他の端末でもログイン中の場合、ログイン直後に**警告メッセージ**が表示されます
   - 「他の端末でもログインしています」
3. この警告は情報提供のみです。ログイン自体はブロックされません
4. アクティブなセッション数も表示されます（例：「アクティブセッション: 3」）

### セッションタイムアウト

POSLAのセッションには有効期限があります。

- **セッション有効時間：8時間**
- **操作なし25分で警告モーダル表示：**「セッションタイムアウト — 5分以内に操作がないとログアウトされます」
- 警告モーダルの**「操作を続ける」ボタン**を押すと、セッションがリセットされます
- 5分以内に操作がない場合、自動的にログアウトされ、ログイン画面にリダイレクトされます
- 通常の操作（クリック、キー入力、スクロール、タッチ）を行っている限り、タイマーは自動リセットされます

### アクティブセッション管理（リモートログアウト）

マネージャー以上のロールは、他の端末のセッションをリモートでログアウトさせることができます。

1. 管理ダッシュボードの「店舗設定」→「監査ログ」タブ、または専用のセッション管理画面を開く
2. アクティブセッション一覧が表示されます
   - セッションごとに：端末情報（ブラウザ/OS）、IPアドレス、ログイン日時、最終操作日時
3. 不要なセッションの**ログアウト**ボタンを押す
4. 確認ダイアログで「はい」を押す
5. 対象セッションが無効化され、その端末は次回操作時にログイン画面にリダイレクトされます

::: tip 使いどころ
- スタッフが退勤後にログアウトし忘れた場合
- 紛失した端末のセッションを無効化したい場合
- 不審なセッションを検出した場合
:::

## 2.4 ログアウト

ログアウトは画面右上の**ログアウトボタン**から行います。

1. 画面右上のユーザー名の横にある**ログアウト**ボタンを押す
2. セッションが終了し、ログイン画面に戻ります

::: warning 退勤時は必ずログアウトしてください
特に共有タブレット（KDSやPOSレジ端末）を使用している場合、退勤時にログアウトしないと次のスタッフの操作が前のスタッフのアカウントで記録されます。勤怠打刻や監査ログに影響するため、退勤時のログアウトを習慣化してください。
:::
