17. セルフオーダー画面（お客さん向け） ​

セルフオーダー画面（/customer/menu.html）は、お客さんがスマートフォンでQRコードを読み取って注文する画面です。テーブルのQRコードを読み取ると、そのテーブル専用の注文画面が開きます。

17.2 注文の流れ ​

ステップ1：QRコードを読み取る ​

  1. テーブルに設置されたQRコードをスマートフォンのカメラで読み取ります
  2. セルフオーダー画面が開きます
  3. 店舗のブランドカラーとロゴが適用された画面が表示されます

ステップ2：メニューを選ぶ ​

  1. カテゴリタブでカテゴリを選択します
  2. メニュー品目をタップします
  3. オプションがある場合はオプション選択画面が表示されます
  4. 品目がカートに追加されます

ステップ3：カートを確認する ​

画面下部にカートバーが表示されます。

要素説明
品目数カート内の品目数
合計金額—
「注文を確認」ボタンカート詳細を開く

ステップ4：注文を送信する ​

  1. 「注文を確認」ボタンをタップします
  2. チェックアウトモーダルが開きます
  3. 注文内容を最終確認します
  4. ひとことメモ： 最大200文字のメモを追加できます
  5. アレルゲン選択チップ： 以下のアレルゲンを選択して申告できます
アレルゲン

  1. 「注文する」ボタンをタップして注文を確定します

注文のリトライ機能 ​

ネットワークエラーが発生した場合：

  • 自動的に最大2回リトライされます（2秒間隔）
  • 冪等性キー（ランダムな32文字の16進数）により、重複注文が防止されます
  • 完全にオフラインの場合、注文はブラウザのlocalStorageに保存され、接続復旧時に自動送信されます
  • 24時間以上経過した保留注文は自動破棄されます

17.3 注文ステータスのリアルタイム追跡 ​

注文送信後、画面上部に注文の進捗がリアルタイムで表示されます。

ポーリング間隔 ​

5秒ごとに注文ステータスが更新されます。

ステータス表示 ​

ステータスバッジ意味
pending受付済み注文がキッチンに届きました
preparing調理中調理を開始しました
ready完成料理ができました
served提供済テーブルに届けました
cancelledキャンセル注文がキャンセルされました

プログレスバー ​

注文全体の進捗を視覚的に表示します。全品目が「served」になると完了です。

17.4 スタッフ呼び出し ​

画面のヘッダーにある🔔アイコンをタップすると、スタッフ呼び出し機能が利用できます。

呼び出しの種類 ​

選択肢説明
注文したい注文に関する呼び出し
お会計をお願いします会計リクエスト
食器を下げてください下げ物の依頼
その他自由入力

呼び出しの流れ ​

  1. 🔔アイコンをタップします
  2. 呼び出し理由を選択します
  3. スタッフのハンディPOSとKDSにアラートが表示されます
  4. スタッフが「対応済み」をタップすると、お客さんの画面に効果音（440Hz→523Hz）で通知されます

クールダウン ​

呼び出し後、3分間は次の呼び出しができません（カウントダウン表示）。これにより連打による誤発信を防止します。

17.5 カートの永続化 ​

カートの内容はブラウザのlocalStorageに保存されます。

  • ページをリロードしてもカートの内容は維持されます
  • セッショントークンが変わった場合（テーブルリセット後）、カートは自動クリアされます

迷い品目トラッキング ​

カートに追加した後に削除した品目は「迷い品目」として記録され、オーナーダッシュボードのクロス店舗レポートで分析できます。これにより、お客さんが迷ったメニューの改善に活用できます。

17.6 ラストオーダー ​

ラストオーダーが発動されると、セルフオーダー画面では新規注文がブロックされます。

  • 画面にラストオーダーバナーが表示されます
  • 「注文する」ボタンが無効化されます
  • 注文送信時にサーバー側でもラストオーダーチェックが行われ、LAST_ORDER_PASSEDエラーが返されます
