# POSLA 機能一覧・網羅資料

作成日: 2026-05-01

## この資料の位置付け

この資料は、POSLAの機能を網羅的に説明するための資料です。

営業提案資料では、売上増・コスト削減・店舗運営の改善に絞って説明します。一方、この資料では「POSLAには具体的に何が入っているのか」を、店舗経営者、店長、現場責任者が確認できるように整理します。

## 記載ルール

POSLAは機能範囲が広いため、誤解を避けるために以下の区分で記載します。

| 区分 | 意味 |
|---|---|
| 標準機能 | POSLAの標準機能として説明できるもの |
| 設定・連携が必要 | POSLA側の機能はあるが、Stripe、LINE、スマレジ、プリンタなど外部設定が必要なもの |
| 運用設計が必要 | 機能はあるが、店舗側のルール作りが必要なもの |
| 拡張構想 | AIカメラなど、現行標準機能としては断定せず、追加開発・PoCとして説明するもの |

## POSLA全体像

POSLAは、飲食店の以下の業務を一つにつなげる店舗運営システムです。

- お客様のセルフオーダー
- スタッフのハンディ注文
- 厨房のKDS
- POSレジ・会計
- テイクアウト
- 予約管理
- 在庫・レシピ
- シフト・勤怠
- レポート・分析
- AIアシスタント
- 複数店舗管理
- 店舗設定・外部連携

単体のPOSレジ、単体のモバイルオーダー、単体の予約台帳ではなく、飲食店の一日の流れをつなげることを目的にしています。

## 利用者別の主な画面

| 利用者 | 主な画面 | 主な目的 |
|---|---|---|
| お客様 | セルフオーダー、テイクアウト注文、Web予約、予約詳細 | 注文、予約、決済、注文状況確認 |
| ホールスタッフ | ハンディPOS、スタッフ画面 | 注文補助、配膳、呼び出し対応、勤怠、シフト確認 |
| キッチンスタッフ | KDS | 調理開始、完成、提供待ち、品切れ確認 |
| レジ担当 | POSレジ | 会計、個別会計、返金、レジ開閉 |
| 店長・副店長 | 管理画面 | メニュー、予約、テイクアウト、シフト、在庫、レポート、店舗設定 |
| オーナー・本部 | オーナーダッシュボード | 複数店舗、契約、ユーザー、本部メニュー、決済、外部連携 |
| POSLA運営 | POSLA管理画面・運用基盤 | テナント、監視、設定、サポート |

## プラン・提供範囲

POSLAは、原則として全機能を標準提供する単一プランです。

| 項目 | 内容 |
|---|---|
| 基本料金 | 1店舗目 月額20,000円 |
| 追加店舗 | 2店舗目以降 月額17,000円/店舗 |
| 無料期間 | 30日間無料 |
| 標準提供 | セルフオーダー、ハンディ、KDS、レジ、予約、テイクアウト、在庫、シフト、レポート、AIなど |
| 任意アドオン | 本部一括メニュー配信 |

本部一括メニュー配信以外は、基本的に機能差別化せず、全契約者に提供する方針です。

## 1. 管理画面

### 1-1. 運営管理

店長が毎日最初に確認する入口です。

| 機能 | 内容 |
|---|---|
| 運営コックピット | 未会計、未締め、予約、シフト不足、品切れ/低在庫、低評価、障害、テイクアウト遅延を一覧化 |
| 全体検索 | 注文、予約、顧客、商品、スタッフ、エラーを横断検索 |
| 売上ドリル | 日別、商品別、注文、会計履歴まで深掘り |
| 初期設定チェックリスト | 店舗、カテゴリ、メニュー、卓、スタッフ、KDS、予約、テイクアウト、在庫、レシピ、プラン、コースの設定状況を確認 |
| 端末稼働 | KDS、ハンディ、レジ、セルフメニューの最終通信時刻を確認 |
| カート離脱 | カートに入れた後で外された商品、注文内キャンセルを確認 |

### 1-2. 店舗切替

複数店舗に所属するユーザーは、管理画面上部から操作対象店舗を切り替えられます。

切り替え対象:

- メニュー
- 注文
- 予約
- テイクアウト
- シフト
- 在庫
- レポート
- 店舗設定

## 2. メニュー管理

### 2-1. カテゴリ管理

メニューを分類するためのカテゴリを管理します。

主な機能:

- カテゴリ作成
- 表示順設定
- カテゴリ名管理
- セルフオーダー、ハンディ、KDS、レジへの反映

### 2-2. 商品管理

本部メニュー、店舗メニュー、限定メニューを管理します。

主な項目:

- 商品名
- 価格
- 税区分
- 説明文
- 画像
- 表示順
- 販売状態
- 売り切れ
- 非表示
- 店舗別価格
- 店舗別画像

### 2-3. セルフメニュー用属性

お客様が選びやすいよう、商品に属性を設定できます。

| 属性 | 用途 |
|---|---|
| 辛さ | 辛い商品を探しやすくする |
| 早く出る | すぐ出せる商品を見せる |
| ベジ | ベジ対応品を見せる |
| 子ども向け | 家族連れに提案しやすくする |
| 提供目安分 | 提供時間の目安を表示する |

### 2-4. オプション管理

サイズ、トッピング、辛さ、麺量などを管理できます。

主な機能:

- 必須オプション
- 任意オプション
- 複数選択
- 選択上限
- 追加料金
- 価格差表示
- 店舗別オプション提供可否
- 売り切れオプション表示

### 2-5. 品切れ・代替提案

商品が売り切れた場合に、代替候補を表示できます。

例:

- 生ビールが売り切れたら瓶ビールを提案
- 人気定食が売り切れたら近い定食を提案
- デザートが売り切れたら別デザートを提案

手動候補がない場合は、同カテゴリ、今日のおすすめ、人気、早く出る商品などから自動候補を出します。

### 2-6. メニュー整備チェック

セルフオーダーでお客様が迷う原因を事前に確認します。

チェック対象:

- 写真なし
- 説明なし
- アレルゲン未設定
- タグ未設定
- 価格未設定

### 2-7. CSV入出力

大量の商品を一括管理できます。

| 対象 | 内容 |
|---|---|
| 本部メニューCSV | 複数店舗共通の商品マスタ |
| 店舗メニューCSV | 店舗別価格、非表示、品切れ、画像URL |
| 限定メニューCSV | 日替わり、季節商品、店舗限定商品 |
| プラン・コースCSV | 食べ放題、コース料理 |
| 在庫・レシピCSV | 原材料、レシピ |

### 2-8. 多言語表示

セルフメニューでは複数言語表示に対応します。

対象:

- メニュー名
- カテゴリ名
- 商品説明
- オプション名
- 画面文言

日本語、英語、簡体中文、韓国語を想定した運用ができます。

## 3. セルフオーダー

### 3-1. QR注文

お客様はテーブルのQRコードを読み取り、自分のスマートフォンで注文できます。

主な機能:

- テーブルQRからメニュー表示
- QR開放認証
- セッション管理
- メニュー閲覧
- カート投入
- 数量変更
- オプション選択
- メモ入力
- アレルゲン入力
- 注文送信
- 注文後の状況確認

### 3-2. 注文しやすくする表示

主な表示:

- おすすめ
- 人気商品
- 早く出る商品
- 品切れ代替候補
- 検索
- カテゴリ切替
- 多言語切替
- 提供目安
- オプション価格差

### 3-3. AIウェイター

お客様からのメニュー質問にAIが回答する補助機能です。

用途:

- おすすめを聞く
- 辛くない商品を聞く
- 子ども向け商品を聞く
- ベジ対応を聞く
- メニュー説明を聞く

注意:

- アレルギーや価格など重要情報は店舗設定を優先します
- AI回答は補助情報であり、店舗側の確認が必要です

### 3-4. 呼び出し

お客様はセルフメニューからスタッフを呼び出せます。

通知先:

- ハンディ
- KDS/呼び出し画面
- PWA通知対象端末

### 3-5. 満足度評価

お客様の評価を収集し、満足度分析に反映できます。

確認できる内容:

- 評価点

- 対象品目
- 時間帯
- コメント

## 7. POSレジ・会計

### 7-1. 通常会計

店内注文やテイクアウト注文の会計を行います。

主な機能:

- テーブル別会計
- 注文内容確認
- 割引
- 支払い方法記録
- 支払い詳細記録
- 担当スタッフPIN
- レシート
- 領収書

### 7-2. 支払い方法

POSLAでは、オンライン決済と通常レジ会計を分けて扱います。

| 区分 | 内容 |
|---|---|
| POSLA決済 | Stripe Connectを使ったセルフ会計・テイクアウト事前決済 |
| 通常レジ会計 | 現金、外部カード端末、外部QR決済、電子マネーの記録 |

通常レジ会計では、POSLAがカード決済を直接実行するのではなく、外部端末で完了した結果をPOSLAに記録します。

### 7-3. 個別会計

同じテーブルの注文を品目単位で分けて会計できます。

主な機能:

- 精算対象品目選択
- 金額再計算
- 支払い方法記録
- 会計済み品目の除外
- セルフ決済金額との整合

### 7-4. レジ開閉

現金管理を記録できます。

主な機能:

- レジ開け
- 前回締め現金で開始
- 0円開始
- 金種入力
- 現金移動
- 釣銭補充
- 現金回収
- 小口購入
- 消耗品購入
- レジ締め
- 差額確認
- 締め忘れ警告

### 7-5. 返金・取消

返金や取消の記録を残せます。

主な機能:

- 返金記録
- 返金理由
- 会計取消
- 外部決済控え番号
- レジ分析への反映

オンライン決済の実返金は、決済事業者側の状態も確認する必要があります。

### 7-6. 緊急レジ

通信障害時に、端末内へ会計記録を残し、後から同期するための機能です。

主な機能:

- 未同期会計記録
- 外部決済控え番号記録
- 同期済み表示
- 重複可能性の要確認表示
- 緊急会計台帳

## 8. テイクアウト管理

### 8-1. お客様向け注文

お客様が事前に注文できます。

主な機能:

- 商品選択
- 名前、電話番号入力
- 受取時刻選択
- メモ入力
- オンライン決済
- 注文詳細確認
- 受取到着連絡
- 再注文候補

### 8-2. 受取時間・厨房キャパ制御

店舗設定により、受けられる注文数を制御できます。

設定項目:

- 最短準備時間
- 受付開始時刻
- 受付終了時刻
- 時間枠あたり最大注文数
- 時間枠あたり最大品数
- ピーク時間帯
- ピーク最大注文数
- ピーク最大品数
- 混雑時追加待ち時間

### 8-3. 店舗側注文管理

店舗側で当日のテイクアウト注文を管理します。

確認できる内容:

- 本日の注文数
- SLA警告
- 遅延中
- 梱包未完了
- 通知失敗
- 返金対応
- 到着待機

### 8-4. ステータス管理

主なステータス:

- 支払待ち
- 受付済み
- 調理中
- 準備完了
- 受渡済み
- キャンセル

### 8-5. 梱包チェック

注文ごとに梱包確認ができます。

チェック項目:

- 商品
- 箸
- 袋
- ソース
- 領収書

### 8-6. 準備完了通知

準備完了時にお客様へ通知できます。

通知経路:

- LINE連携

注意:

- LINE公式アカウント設定とお客様のLINE紐付けが必要です
- テイクアウトSMS通知は現行標準機能としては説明しません

### 8-7. キャンセル・返金運用

運用ステータスとして、キャンセル依頼、返金待ち、返金済みなどを管理できます。

POSLAは、全スタッフに個別アカウントを作成する前提です。

理由:

- 勤怠を正しく記録するため
- 監査ログを残すため
- スタッフ評価を正しく行うため
- シフト確認を本人単位で扱うため

### 20-3. 端末専用アカウント

KDS、ハンディ、レジなど店舗備品端末は、人のスタッフアカウントではなく端末専用アカウントで運用します。

以下は、現行標準機能として断定せず、追加開発、PoC、個別連携として扱います。

- AIカメラで視線や表情を分析する
- AIカメラでメニュー閲覧行動を自動解析する
- AIカメラが毎時在庫を自動読取する
- 顔認識で来店客を個人特定する
- 来店客ごとに完全自動でメニューを個別最適化する
- 発注を完全自動化する
- テイクアウトSMS通知を標準機能として提供する
- LINEリッチメニューやLIFFを標準管理する
- POSLAだけでオンライン決済返金がすべて完結する

機能を全部説明するのではなく、相手の課題に合わせて見る場所を変えます。

| 相手の関心 | 見せる章 |

| 人手不足 | セルフオーダー、ハンディ、KDS、レジ、シフト |
| 客単価アップ | メニュー、おすすめ、併売、カート離脱、AIウェイター |

| 予約を伸ばしたい | 予約管理、顧客台帳、リマインド、予約金 |
