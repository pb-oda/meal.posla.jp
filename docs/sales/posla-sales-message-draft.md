# POSLA 営業資料たたき台

作成日: 2026-05-01

## この資料の目的

POSLAを「機能が多いシステム」として説明すると、飲食店経営者には伝わりにくいです。経営者が知りたいのは、最終的には次の2つです。

- 売上が増えるのか
- コストや手間が減るのか

そのため営業資料では、POSLAを仕様一覧ではなく、**飲食店の毎日の運営を、売上増とコスト削減につなげる仕組み**として説明します。

## 一言で言うと

POSLAは、注文、厨房、会計、予約、テイクアウト、在庫、シフト、レポートをひとつにつなげて、**店長が経験と勘で判断していたことを、毎日のデータで改善できるようにする飲食店向け運営システム**です。

単なるセルフオーダーやPOSレジではありません。

お客様が何を見て、何を注文し、厨房でどこが詰まり、会計後に何が売れ、どの時間に人が足りず、どの商品が品切れやロスにつながったのかを、店舗運営の流れとしてつなげます。

## 営業で刺すべき大きな軸

| 経営課題 | POSLAで変わること | 経営効果 |
|---|---|---|
| 客単価を上げたい | おすすめ、セット提案、追加注文、常連向け提案を出しやすくする | 売上増 |
| 人手不足 | 注文、会計、予約確認、シフト確認、厨房連携を省力化する | 人件費・教育コスト削減 |
| 品切れ・廃棄ロス | 注文、会計、レシピ、在庫、需要予測を連動させる | 原価率改善 |
| 店長の判断が属人化 | 売上、予約、在庫、シフト、低評価、障害を管理画面で見える化する | 店舗運営の安定化 |
| リピーターを増やしたい | 予約情報、常連情報、好み、アレルギー、注意点を現場に出す | 再来店率・満足度向上 |
| テイクアウトを伸ばしたい | 受付枠、受取時間、SLA、準備完了通知、梱包チェックを管理する | 機会損失とクレーム削減 |

## 経営者に伝えるべきメッセージ

### 悪い伝え方

POSLAには、セルフオーダー、KDS、POSレジ、シフト管理、予約管理、テイクアウト管理、在庫管理、AIアシスタント、レポート機能があります。

### 良い伝え方

POSLAを入れると、店長が毎日バラバラに見ていた注文、厨房、会計、予約、在庫、シフトがつながります。

たとえば、売れ筋商品が伸びた時に「売れた」で終わらず、どの時間に売れたのか、厨房は詰まったのか、在庫は足りたのか、人員は足りたのか、次回はどの商品をおすすめすべきかまで見えるようになります。

その結果、売上を伸ばす打ち手と、ムダを減らす打ち手を、同じ画面から判断できます。

## 店舗の一日で見るPOSLA

### 開店前

#### 今まで

- 店長が予約表、シフト表、在庫表、メニューの品切れを別々に確認する
- 昨日の売上や在庫状況を見ながら、今日の仕込み量を勘で決める
- 欠勤や遅刻連絡があると、LINEや電話で個別に調整する
- テイクアウト受付数や厨房キャパは、忙しくなってから現場判断になる

#### POSLA導入後

- 今日の運営コックピットで、予約、シフト不足、低在庫、品切れ、未会計、未締め、テイクアウト遅延リスクをまとめて確認
- 予約人数、過去売上、商品構成、在庫状況から、仕込みや人員配置の判断がしやすくなる
- シフト不足や欠勤は、候補スタッフ確認、空きシフト募集、他店舗ヘルプへつなげられる
- テイクアウトは時間帯ごとの受付上限を設定し、厨房キャパを超えにくくする

#### 経営効果

- 開店前確認の抜け漏れを減らす
- 店長の朝の確認時間を短縮する
- 品切れ、過剰仕込み、シフト不足を事前に見つける

### ランチ・ディナーのピーク中

#### 今まで

- 注文がホール、厨房、レジで分断される
- 口頭伝達や紙伝票でミスが出る
- 混雑時に追加注文やおすすめ提案が弱くなる
- 品切れが出ても、メニュー表示やスタッフへの共有が遅れる
- テイクアウトの受取遅れや準備遅れに気づくのが遅い

#### POSLA導入後

- お客様はスマホから注文でき、スタッフはハンディで注文を補助できる
- 注文はKDSに流れ、厨房ではステーション別に調理状況を確認できる
- 商品単位で調理開始、完成、提供済みを管理できる
- 品切れや低在庫はメニュー表示や運営画面に反映しやすい
- テイクアウトは受取時間SLA、梱包チェック、準備完了通知で管理できる

#### 経営効果

- 注文取りの人手を減らす
- 注文ミス、提供漏れ、伝達漏れを減らす
- 追加注文を取り逃しにくくする
- テイクアウトの遅延クレームを減らす

### 会計時

#### 今まで

- 会計待ちが発生し、回転率が落ちる
- 個別会計や割引、外部決済の確認に時間がかかる
- 会計後に在庫や売上分析へ反映する作業が別になる

#### POSLA導入後

- POSレジで注文内容をもとに会計できる
- 個別会計や割引、支払い方法の記録を管理できる
- 会計完了時にレシピに基づいて在庫を減算できる
- 注文、会計、在庫、レポートが同じデータでつながる

#### 経営効果

- 会計時間を短縮する
- レジ締め、差額確認、売上確認を楽にする
- 在庫反映の二重入力を減らす

### 閉店後

#### 今まで

- 売上、レジ締め、在庫、シフト実績を別々に確認する
- 何が売れたかは分かっても、次に何を改善すべきか分かりにくい
- メニュー変更や価格変更の判断が感覚になりやすい

#### POSLA導入後

- 売上、商品別、注文別、会計履歴まで追える
- カートに入ったが注文されなかった商品、注文後キャンセルされた商品を分析できる
- 在庫、レシピ、原価、品切れ状況を見ながら、メニュー改善の材料にできる
- シフト予定と実績を見て、曜日・時間帯ごとの人員調整に活かせる

#### 経営効果

- メニュー改善、価格見直し、仕込み量、人員配置をデータで判断できる
- 店長の報告作業を減らす
- 複数店舗でも同じ基準で比較しやすくなる

## 売上増につながる訴求

### 1. おすすめ提案で客単価を上げる

お客様がセルフオーダー画面を開いたときに、おすすめ商品、人気商品、追加注文候補を出せます。

たとえば、唐揚げを注文したお客様にドリンクや小鉢を提案する、締めの時間帯にデザートやご飯ものを見せるなど、スタッフが忙しい時でも追加注文のきっかけを作れます。

#### 経営者に刺さる言い方

スタッフが毎回おすすめを言わなくても、メニュー画面が自然に追加注文を促します。

### 2. 常連・予約客に合わせた接客で再来店を増やす

予約管理では、前回来店、好み、注意点、アレルギー、VIP、no-show履歴などを現場に出せます。

ハンディ、KDS、レジに予約文脈が出ることで、ホール、厨房、会計が同じ情報を見ながら対応できます。

#### 経営者に刺さる言い方

常連対応を「覚えているスタッフ」だけに頼らず、店舗全体で共有できます。

### 3. カート離脱・キャンセル分析でメニューを改善する

お客様が一度カートに入れたのに注文しなかった商品、注文後にキャンセルされた商品を見れば、メニュー写真、価格、説明文、提供時間、品切れ表示に問題がないかを考えられます。

#### 見るべきポイント

- カート投入は多いが注文されない商品
- 注文後キャンセルが多い商品
- よく一緒に買われる商品
- 時間帯によって伸びる商品
- テイクアウトで伸びる商品

#### 経営者に刺さる言い方

売れた商品だけでなく、売れそうだったのに売れなかった商品が見えるので、メニュー改善の材料になります。

### 4. 予約・テイクアウトを機会損失にしない

予約は遅刻、no-show、高リスク、キャンセル待ちを管理できます。テイクアウトは受付上限、受取時間、準備状況、梱包チェックを管理できます。

満席時やピーク時に「取れる注文だけ取る」「遅れそうな注文を事前に気づく」ことで、売上機会と顧客満足を両立しやすくなります。

#### 経営者に刺さる言い方

忙しい時に売上を取りにいって現場が崩れるのではなく、厨房キャパに合わせて受けられる注文を増やします。

## コスト削減につながる訴求

### 1. 注文取り・伝達・会計の手間を減らす

セルフオーダー、ハンディ、KDS、POSレジがつながるため、注文を聞く、厨房へ伝える、会計に打ち直すという作業を減らせます。

#### 削減できる作業

- ホールの注文取り
- 紙伝票の受け渡し
- 厨房への口頭確認
- レジへの再入力
- 提供漏れ確認
- 会計待ち対応

### 2. 在庫とレシピの連動で棚卸し・品切れ対応を減らす

商品ごとに原材料と使用量を登録しておくと、会計完了時に在庫が減ります。低在庫や品切れ候補も見えるため、営業中に「実は材料が足りない」と気づくリスクを下げられます。

#### 現行で訴求できること

- 原材料登録
- レシピ登録
- 会計時の在庫減算
- 低在庫アラート
- 品切れ連動
- AI需要予測の参考値

#### 経営者に刺さる言い方

売上と在庫が別々の表ではなく、注文された商品から原材料在庫までつながるので、仕込みと発注の判断が早くなります。

### 3. シフト不足と過剰人員を減らす

シフト管理では、希望提出、公開前チェック、欠勤、交代、穴埋め、当日ヘルプ、担当作業、スキル、休憩不足を見られます。

売上や予約の見込みと合わせて見れば、忙しい時間に人が足りない、暇な時間に人が多いというムダを見つけやすくなります。

#### 経営者に刺さる言い方

人件費をただ削るのではなく、売上が取れる時間に人を寄せ、暇な時間のムダを減らします。

### 4. 店長の確認作業をまとめる

今日の運営コックピットで、未会計、未締め、予約、シフト不足、品切れ、低評価、障害、テイクアウト遅延を一覧できます。

#### 経営者に刺さる言い方

店長が毎朝いろいろな画面や紙を見に行くのではなく、まず見るべき異常が一画面に集まります。

## AIカメラを使ったメニュー改善案

::: warning 現行実装との切り分け
AIカメラで視線、滞在時間、メニュー選択直前の迷いを分析する機能は、営業上の追加提案・将来拡張として扱います。現行POSLAの標準機能として断定しないでください。
:::

### 何が分かると価値があるか

お客様がメニューを選ぶとき、実際には「注文された商品」だけでは分からない情報があります。

- どの商品で迷ったか
- どのカテゴリを長く見たか
- 価格を見て離脱した可能性があるか
- 写真は見られているのに注文されない商品はどれか
- おすすめ表示が本当に見られているか
- 高単価商品が見られているのに注文されない理由があるか

これが分かると、単なる売上ランキングではなく、**売れる可能性があった商品**を改善できます。

### 営業トーク例

今のPOSや注文システムでは、最終的に注文された商品しか分かりません。

しかし経営判断で本当に知りたいのは、売れなかった商品の中に「見られていたのに選ばれなかった商品」があるかです。

AIカメラや画面行動分析を組み合わせると、写真、価格、説明文、配置、セット提案のどこを変えるべきかが見えます。

### 活用例

| 見えること | 改善アクション |
|---|---|
| 写真は見られているが注文されない | 価格、説明文、提供時間、量の表現を見直す |
| 高単価商品が見られない | メニュー配置、カテゴリ名、おすすめ表示を見直す |
| セット候補を見た後に単品注文が多い | セット価格、見せ方、写真を見直す |
| デザートを見ているが注文されない | 食後のタイミングで再提案する |
| 子連れ席で特定カテゴリの閲覧が多い | キッズ向け表示やアレルギー表示を強化する |

### 注意点

顔認識や個人特定を前提にすると、導入ハードルが上がります。営業上は、まずは個人を特定しない匿名分析として説明する方が安全です。

## AIカメラ・画像認識を使った在庫自動読取案

::: warning 現行実装との切り分け
在庫をAIカメラが毎時読み取り、自動で棚卸しする機能は、追加開発・別デバイス設置が必要な拡張案として扱います。現行POSLAでは、レシピ登録と会計データに基づく在庫減算、棚卸し、低在庫、品切れ連動が中心です。
:::

### 経営者に刺さる理由

飲食店の在庫管理で大変なのは、単に数える作業ではありません。

- 忙しい時間に在庫を見に行けない
- 仕込み済みと未仕込みの区別が難しい
- 冷蔵庫、冷凍庫、バックヤードで在庫が分かれる
- 手入力すると誤差が出る
- 棚卸しが閉店後の負担になる
- 発注が店長の経験に依存する

AIカメラや画像認識が使えると、棚や冷蔵庫の状態を定期的に読み取り、在庫の変化をPOSLAのレシピ在庫と照合できます。（未実装・将来的な話）

### 活用イメージ

| 店舗作業 | AI化後のイメージ | 効果 |
|---|---|---|
| 開店前に冷蔵庫を確認 | カメラ画像から主要食材の残量を推定 | 確認時間短縮 |
| ピーク前に材料不足を確認 | 売上予測と在庫推定から不足候補を通知 | 品切れ防止 |
| 閉店後に棚卸し | 画像推定と会計ベース在庫を突合 | 棚卸し負担軽減 |
| 発注量を決める | 過去売上、予約、天候、在庫から提案 | 発注ミス削減 |
| ロスを確認する | 想定消費と実在庫差分を検出 | 廃棄・持ち出し・入力漏れ発見 |

### 営業トーク例

現行POSLAでも、注文された商品から原材料在庫を減らすことはできます。

さらにAIカメラを組み合わせると、システム上の在庫と実際の冷蔵庫の在庫を定期的に照合できます。

つまり、店長が毎回数えに行くのではなく、ズレが大きい食材だけ確認すればよくなります。

## パーソナライズされたメニュー提案

::: warning 現行実装との切り分け
現行POSLAでは、おすすめ、人気商品、AIウェイター、常連情報、予約情報、注意点、好み、アレルギーなどを扱えます。一方で、来店客ごとに完全自動でメニュー表示を最適化する機能は、営業上は追加拡張として切り分けて説明してください。
:::

### 目指す価値

毎回来るお客様に、毎回同じメニューを見せる必要はありません。

常連、家族連れ、宴会、ランチ客、テイクアウト客では、見せるべき商品が違います。

POSLAに来店履歴、注文履歴、好み、アレルギー、時間帯、人数、予約目的が蓄積されれば、その人に合ったおすすめを出しやすくなります。

### 活用例

| お客様 | 出すべき提案 |
|---|---|
| よく唐揚げを頼む常連 | 新しい唐揚げソース、相性の良いドリンク |
| ワインを頼むお客様 | 前菜、チーズ、肉料理のセット |
| 子連れ予約 | キッズメニュー、取り分けしやすい商品 |
| アレルギー登録あり | 該当食材を避けた候補 |
| テイクアウト常連 | 前回注文、受取時間に合う商品 |
| 宴会予約 | コース、飲み放題、追加料理 |

### 経営者に刺さる言い方

常連の好みをスタッフの記憶に頼るのではなく、メニュー画面そのものが「その人に合う商品」を出せるようになります。

## 需要予測・売上予測で運営を変える

### 現場での使い道

売上予測は、単に「明日の売上はいくらか」を見るためだけではありません。

本当に使うべき場面は、仕込み、人員配置、品切れ防止、テイクアウト受付数、予約枠の調整です。

| 予測するもの | 現場で変えられること |
|---|---|
| 時間帯別売上 | シフト人数、休憩タイミング |
| 商品別販売数 | 仕込み量、発注量 |
| 予約人数 | 席配置、コース準備、厨房人員 |
| テイクアウト件数 | 受付枠、受取時間、梱包人員 |
| 品切れリスク | 代替メニュー、おすすめ商品の切替 |
| 低評価リスク | 提供遅れ、会計待ち、混雑対応 |

### 経営者に刺さる言い方

売上予測はレポートを見るためではなく、明日のシフト、仕込み、発注を間違えないために使います。

## POSLAの価値を数字に変える考え方

営業資料では、効果を断定しすぎるよりも、店舗ごとの現状数字を入れて試算できる形にする方が安全です。

### 売上増の試算項目

| 項目 | 計算式 |
|---|---|
| 追加注文増 | 月間客数 × 追加注文率改善 × 平均追加単価 |
| 客単価改善 | 月間客数 × 客単価上昇額 |
| テイクアウト増 | 月間テイクアウト件数増 × 平均単価 |
| no-show削減 | no-show削減件数 × 平均予約単価 |
| リピート増 | 再来店増加人数 × 平均単価 |

### コスト削減の試算項目

| 項目 | 計算式 |
|---|---|
| 注文取り削減 | 削減時間/日 × 時給 × 営業日数 |
| レジ締め短縮 | 削減時間/日 × 時給 × 営業日数 |
| 棚卸し短縮 | 削減時間/月 × 時給 |
| 廃棄ロス削減 | 現在の月間廃棄額 × 削減率 |
| 過剰シフト削減 | 削減人時/月 × 時給 |
| クレーム対応削減 | 削減件数 × 対応時間 × 時給 |

### 商談で聞くべき質問

- 月の客数はどれくらいですか
- 客単価はいくらですか
- 追加注文をもっと取りたい商品はありますか
- 注文取りに何人くらい使っていますか
- レジ締めに毎日何分かかっていますか
- 棚卸しは誰が何時間かけていますか

## 営業トークの短縮版

POSLAは、注文を取るだけのシステムではありません。

注文、厨房、会計、予約、在庫、シフトをつなげて、売上を上げる打ち手と、ムダを減らす打ち手を見えるようにします。

たとえば、売れた商品だけではなく、カートに入ったのに注文されなかった商品、品切れで売れなかった商品、厨房が詰まった時間、シフトが足りなかった時間まで見えるようになります。

そのため、メニュー開発、価格見直し、仕込み量、発注、人員配置を、勘ではなく店舗データで改善できます。
