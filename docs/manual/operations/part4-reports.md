# Part 4 — 売上とレポート

このページでは、お店の売上・品目別ランキング・スタッフ評価・顧客分析などの各種レポートの見方と活用方法を説明します。

::: tip レポートの種類と見る画面
レポートは以下の 2 つの画面に分かれています。

- **dashboard.html の「レポート」タブ**（manager 以上）: 基本レポート全般
- **owner-dashboard.html のタブ**（owner 専用）: ABC 分析、クロス店舗レポート
:::

---

## 4-1. 日次売上の見方

### 4-1-1. 売上レポートを開く

1. dashboard.html にマネージャー以上でログイン
2. 上部タブ **【レポート】** → サブタブ **【売上レポート】**（`data-tab="sales"`）
3. 画面が開く

### 4-1-2. 表示される項目

| 項目 | 説明 |
|---|---|
| 売上合計 | 期間内の税込売上合計 |
| 注文件数 | 注文数 |
| 客単価 | 売上合計 ÷ 注文件数 |
| 時間帯別売上 | 1時間ごとの売上推移（棒グラフ） |
| 日別売上推移 | 過去 7 / 30 / 90 日の折れ線グラフ |
| 支払方法別 | 現金・カード・QR の内訳（円グラフ） |

### 4-1-3. 期間選択

画面上部に期間切替ボタン:

- **日次**: 本日の売上
- **週次**: 今週月〜日
- **月次**: 今月
- **カスタム**: 任意の開始日〜終了日

### 4-1-4. 活用例

- 毎朝「昨日の売上」確認
- 週次ミーティングで「先週の合計」報告
- 月次決算で「月間売上」集計
- 同じ曜日の比較（例: 先週金曜と今週金曜）

## 4-2. 商品別ランキング

### 4-2-1. 見方

1. 【売上レポート】 内の **「商品別ランキング」** セクション（同じタブ内の下スクロール）
2. 売上順に品目が表示
3. 各品目の情報:

| 列 | 説明 |
|---|---|
| 商品名 | メニュー名 |
| 販売数 | 期間内の注文数 |
| 売上合計 | 円単位 |
| 粗利 | 売上 − 原価（レシピ設定時のみ表示） |

### 4-2-2. 活用例

- **上位 10 品**: 看板メニューの把握、在庫発注優先度
- **下位 10 品**: 廃止候補・値引き候補
- **粗利率ランキング**: 利益率の高い品目を積極プッシュ

## 4-3. ABC 分析（オーナー専用）

ABC 分析は **owner-dashboard の専用タブ** に移動済み（`dashboard.html` L92 コメント参照）。manager・staff には表示されません。

### 4-3-1. 開く手順

1. owner アカウントで `https://eat.posla.jp/public/admin/owner-dashboard.html` を開く
2. 上部タブ **【ABC 分析】**（`data-tab="abc-analytics"`、`owner-dashboard.html` L52）

### 4-3-2. A / B / C の定義

売上累積シェアで 3 ランクに分類:

| ランク | シェア | 意味 | 対応方針 |
|---|---|---|---|
| **A ランク** | 売上上位 70% | 主力商品 | **品切れ厳禁**、メイン訴求に使う |
| **B ランク** | 70〜90% | 安定商品 | 現状維持、プラスアルファの訴求 |
| **C ランク** | 90〜100% | 下位商品 | **廃止候補** / メニュー見直し / 値下げ検討 |

### 4-3-3. 活用例

- 全 50 品のうち C ランク 25 品を見直し
- A ランクの原材料を優先発注
- A ランクの新規バリエーション開発（例: 「醤油ラーメン」が A → 「辛味噌ラーメン」を追加）

## 4-4. バスケット分析（併売分析）

「○○を注文した人は △△ も買う」傾向を分析。

### 4-4-1. 見方

1. dashboard.html → 【レポート】 → サブタブ **【併売分析】**（`data-tab="basket"`、manager 以上）
2. 品目の組み合わせがリフト値順に表示

### 4-4-2. リフト値とは

「Aを買った人がBも買う確率 ÷ 全注文でBが買われる確率」

- リフト値 > 1: 一緒に買われやすい
- リフト値 = 1: 関係なし（独立）
- リフト値 < 1: むしろ一緒には買われにくい

例: **ラーメン + 餃子** のリフト値 = 2.5 → ラーメンを食べる人は通常の 2.5 倍の確率で餃子も頼む

### 4-4-3. 活用例

- **セットメニュー提案**: 「ラーメン + 餃子」セットを作って単価アップ
- **レコメンド**: セルフメニューで「一緒にいかがですか？」で餃子を推薦
- **メニュー配置**: 一緒に頼まれる品目を隣接ページに配置

## 4-5. 回転率・客層レポート

### 4-5-1. 開く手順

1. dashboard.html → 【レポート】 → サブタブ **【回転率・客層】**（`data-tab="turnover"`、manager 以上）

### 4-5-2. 表示される指標

| 指標 | 説明 | 活用 |
|---|---|---|
| 平均滞在時間 | 着席から会計までの時間（分） | 15 分以下なら回転率高い、90 分以上ならゆっくり利用 |
| 1 日の回転数 | 1 テーブルあたり何回お客様が入れ替わったか | 高いほど効率的、低いなら滞在時間長い客が多い |
| 時間帯別回転傾向 | 時間帯ごとの客数推移 | ピーク時間の把握、ラストオーダーの最適化 |
| 曜日別傾向 | 月〜日の平均客数 | シフト配置の最適化 |

## 4-6. スタッフ評価レポート

### 4-6-1. 開く手順

1. dashboard.html → 【レポート】 → サブタブ **【スタッフ評価】**（`data-tab="staff-report"`、manager 以上）

### 4-6-2. スタッフごとに表示

| 列 | 説明 |
|---|---|
| スタッフ名 | 登録されている `display_name` |
| 注文入力件数 | ハンディで取った注文数 |
| 売上貢献 | 担当した注文の金額合計 |
| 客単価 | 担当注文の客単価 |
| 満足度評価 | お客様からの星評価平均（5 段階） |
| 勤務日数 | 期間内の出勤日数 |

### 4-6-3. 活用例

- **月次評価・ボーナス査定**: 数値ベースで公平に評価
- **研修対象の発見**: 客単価が低いスタッフは接客改善研修
- **優秀スタッフの昇格判断**: 全指標が高いスタッフはマネージャー候補

## 4-7. 予約レポート（L-9）

### 4-7-1. 開く手順

1. dashboard.html → 上部タブ **【予約管理】** → サブタブの予約台帳内に統計セクション
2. または API: `/api/store/reservation-stats.php` を運営側で叩く

### 4-7-2. 指標

| 指標 | 説明 |
|---|---|
| 予約総数 | 期間内の予約数 |
| 来店人数 | 実際に来店した予約の人数合計 |
| no-show 率 | 来店しなかった予約の割合 |
| キャンセル率 | キャンセルされた予約の割合 |
| ソース別 | web / 電話 / walk-in / AI の内訳 |
| 日別・時間帯別・曜日別 | 需要予測に活用 |

## 4-8. クロス店舗レポート（オーナー向け）

複数店舗の売上・客数・業績を横並びで比較。

### 4-8-1. 開く手順

1. owner アカウントで owner-dashboard
2. 上部タブ **【クロス店舗レポート】**（`data-tab="cross-store"`、`owner-dashboard.html` L51）

### 4-8-2. 表示項目

- 店舗ごとの売上合計
- 店舗ごとの客数
- 店舗ごとの客単価
- 全店平均・最大・最小のベンチマーク
- 期間別（日次・週次・月次）

### 4-8-3. 活用例

- 全 3 店舗のうち 1 店舗だけ売上低い → 原因調査（立地 / スタッフ / メニュー）
- 業績トップ店舗の成功要因を他店舗に横展開

## 4-9. レジ分析（レジ締め精算レポート）

### 4-9-1. 開く手順

1. dashboard.html → 【レポート】 → サブタブ **【レジ分析】**（`data-tab="register"`、manager 以上）

### 4-9-2. 表示項目

| 項目 | 説明 |
|---|---|
| レジ開閉履歴 | いつ・誰が開けた / 閉めたか |
| 日別キャッシュログ | 入金・出金・過不足の履歴 |
| 予想在高 vs 実際 | 差額がプラスなら過多、マイナスなら不足 |
| 過不足アラート | 閾値（デフォルト ¥500）を超える差額は警告 |

### 4-9-3. 活用例

- 毎日のレジ締めで過不足チェック
- 過不足が続くスタッフの現金取扱い研修
- 入金・出金記録漏れのチェック

## 4-10. 注文履歴

### 4-10-1. 開く手順

1. dashboard.html → 【レポート】 → サブタブ **【注文履歴】**（`data-tab="orders"`、manager 以上）

### 4-10-2. 表示項目

- 全注文の一覧（日時・テーブル・品目・金額・ステータス）
- 検索フィルタ（日時・テーブル・品目・ステータス）
- CSV エクスポート可

### 4-10-3. 活用例

- 「先週金曜の 19 時の注文を確認したい」等の個別調査
- クレーム対応: お客様から「○○を頼んだはず」と言われた時の確認
- 税務監査対応: 過去の取引記録

---

## 4-11. CSV エクスポートの方法

各レポートの **【CSV 出力】** ボタンから:

1. 期間を選択
2. **【CSV ダウンロード】**
3. ダウンロードフォルダに保存
4. Excel / Numbers / Google Sheets で開く

### 文字化け対処
- Excel の場合: 「データ」→「テキストから」でインポート時に UTF-8 指定
- Numbers / Google Sheets: デフォルト UTF-8 で OK

## 4-12. 活用ストーリー例

### 例 1: 月次決算（毎月 1 日の作業）

1. 先月の **売上レポート（月次）** で合計売上を確認
2. **ABC 分析** で主力商品の推移を見る
3. **スタッフ評価** で最優秀スタッフにボーナス
4. **レジ分析** で過不足合計を確認

### 例 2: 新メニュー導入判断（週次）

1. **バスケット分析** で既存メニューの組み合わせ傾向を把握
2. **ABC 分析** で C ランク品目を特定
3. C ランク廃止 → 新メニュー追加の判断

### 例 3: スタッフ配置の最適化（月 1 回）

1. **回転率レポート** の時間帯別で、ピーク時間を特定
2. **スタッフ評価** で客単価の高いスタッフを把握
3. ピーク時間に優秀スタッフを配置

## 4-13. よくある質問

**Q1. レポートの数字が合わない**

A1. 期間指定のミスが多い。「営業日区切り時刻」（デフォルト 05:00）を確認。深夜 1 時の売上が「翌日」扱いになっていないか。

**Q2. 粗利が表示されない**

A2. 粗利表示には **レシピ設定**（品目と原材料の紐付け）が必要。在庫・レシピ管理タブで設定してください。

**Q3. スタッフ評価の「担当注文」とは？**

A3. `orders.staff_id` に記録されたユーザーの注文。ハンディで注文入力した個人スタッフ、または POSレジで会計担当に指定されたスタッフ。

**Q4. ABC 分析が見えない**

A4. ABC 分析は **owner 専用**（manager / staff には非表示）。オーナーアカウントで owner-dashboard にログインしてください。

**Q5. CSV ダウンロードしたら改行がおかしい**

A5. Windows の改行コード問題。Notepad++ 等で「LF」に変換するか、Excel で直接「データ→テキストから」を使用してください。
