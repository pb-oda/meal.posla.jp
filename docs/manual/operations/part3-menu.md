# Part 3 — メニュー管理

このページでは、お店のメニューを POSLA に登録・編集する方法を、**ボタンの押し方 1 つずつ** 説明します。

::: tip メニューの 3 層構造（重要）
POSLA のメニューは 3 層構造です。最初に理解すべきポイント。

1. **`menu_templates`（本部マスタ）** — テナント全体で共有する品目マスタ（名前・価格・カテゴリ）
2. **`store_menu_overrides`（店舗上書き）** — 店舗ごとの個別カスタマイズ（価格だけ変えたい・非表示にしたい・売切にしたい）
3. **`store_local_items`（店舗限定品）** — その店舗だけにある品目（他店舗にはない）

この構造のおかげで、同じブランド内の複数店舗で **「全店共通メニュー + 店舗限定メニュー」** が両立できます。
:::

::: warning 本部一括配信アドオン契約時の特殊運用
`tenants.hq_menu_broadcast=1`（本部一括配信アドオン契約）のテナントでは:
- 本部マスタ（`menu_templates`）の編集は **owner-dashboard の「本部メニュー」タブからのみ**
- 各店舗の dashboard「店舗メニュー」タブでは本部マスタ項目（name, name_en, base_price, calories, allergens, image_url）は **readonly**
- 店舗側で編集できるのは `store_menu_overrides` の項目のみ（価格上書き・非表示・売切）

アドオン未契約のテナント（通常）では、店舗側から `menu_templates` を直接編集できる従来運用。
:::

## 3-1. 新しいメニューを追加する

### 3-1-1. 通常（アドオン未契約）の場合

1. dashboard.html にログイン
2. 上部タブ **【メニュー管理】** → サブタブ **【店舗メニュー】**（`data-tab="store-menu"`）
3. 画面右上の **【+ メニュー追加】** ボタン（`btn-add-store-menu`、`dashboard.html` L151）
4. モーダルが開く
5. 以下を入力:

| 項目 | 必須 | 説明 |
|---|---|---|
| **品名** | ✅ | 日本語名（例: `鶏の唐揚げ`） |
| **品名（英語）** | — | 任意（例: `Fried Chicken`） |
| **カテゴリ** | ✅ | ドロップダウンから選択 |
| **基本価格** | ✅ | 円（例: `900`） |
| **カロリー** | — | kcal 単位（例: `380`） |
| **アレルゲン** | — | 卵 / 乳 / 小麦 / 落花生 / えび / かに / そば のチェックボックス |
| **画像URL** | — | HTTPS の URL（`https://...`） |

6. **【保存】**
7. セルフメニュー・ハンディに即反映（お客様側は次回リロード、または 3 分キャッシュ）

### 3-1-2. アドオン契約時の場合

本部一括配信アドオンを契約中のテナントでは、店舗側から本部マスタは追加できません。

1. owner-dashboard.html にオーナーアカウントでログイン
2. **【本部メニュー】** タブ（`data-tab="hq-menu"`）
3. **【+ 本部メニュー追加】** ボタン（`btn-add-hq-menu`、`owner-dashboard.html` L89）
4. 本部マスタとして追加（全店共通メニューになる）

店舗側で **「この店舗だけのメニュー」** を追加したい場合は、3-6「限定メニュー」を参照。

## 3-2. メニューの価格を変更する

### 3-2-1. 全店共通の価格を変える（本部マスタ）

アドオン未契約: dashboard「店舗メニュー」→ 編集で直接変更
アドオン契約中: owner-dashboard「本部メニュー」→ 編集

### 3-2-2. 店舗別の価格を上書きする（店舗オーバーライド）

同じ商品を店舗ごとに別価格で売りたい場合。

1. dashboard.html → 【メニュー管理】 → 【店舗メニュー】
2. 該当メニューの **【編集】** ボタン
3. **「店舗価格（上書き）」** 欄に新しい価格を入力
   - 例: 本部価格 ¥900 / 渋谷店（価格上書き）¥1,000 / 新宿店（価格上書きなし）→ ¥900
4. **【保存】**
5. `store_menu_overrides.price` に保存される

**クリアするには**: 同じ欄を空白にして保存 → `store_menu_overrides.price = NULL`（本部価格を継承）

### 3-2-3. 単純な価格変更の手順（本部価格変更）

1. 【店舗メニュー】で該当メニューの **【編集】**
2. **基本価格** 欄を変更
3. **【保存】**

## 3-3. メニューの写真（画像）を登録する

### 3-3-1. 画像の準備

- 推奨サイズ: **800×600px** または **1000×750px**（4:3 比率）
- フォーマット: JPG または PNG
- ファイルサイズ: 200KB 以下推奨（軽いほうが表示が早い）
- 明るく・清潔感のある写真

### 3-3-2. 画像を公開 URL にアップロード

POSLA 側に画像アップロード機能はないため、**別サービスで公開アップロード**してから URL を取得してください。

推奨サービス:
- **Imgur**（無料、`imgur.com`）— 一番簡単
- **AWS S3**（有料、プロ向け）
- **Cloudinary**（無料枠あり、画像最適化も）
- **店舗のホームページ**（WordPress 等）

手順（Imgur の場合）:

1. `imgur.com` にブラウザでアクセス
2. **「New post」** ボタン → 画像をドラッグ&ドロップ
3. アップロード完了後、画像を右クリック → **「画像アドレスをコピー」**
4. URL は `https://i.imgur.com/xxxxx.jpg` のような形式

### 3-3-3. POSLA に URL を登録

1. 該当メニューの **【編集】**
2. **画像URL** 欄にコピーした URL を貼り付け
3. **【保存】**
4. プレビュー画像がサムネイル表示される

### 3-3-4. よくあるエラー

| 症状 | 原因 | 対処 |
|---|---|---|
| 画像が表示されない | URL が HTTP（http://）になっている | HTTPS（`https://`）の URL に変更 |
| 404 エラー | URL が無効 / 画像が削除された | Imgur で画像を確認、URL 再取得 |
| 崩れて表示 | 極端なアスペクト比 | 4:3 または正方形推奨 |
| 画像が重くて遅い | ファイルサイズが大きすぎ | 200KB 以下に圧縮（`tinypng.com` 等で） |

## 3-4. カテゴリの追加・並べ替え

### 3-4-1. カテゴリとは

メニューを分類するグループです。セルフメニュー画面で横タブとして表示されます。

例:
- 前菜 / メイン / ご飯もの / デザート / ドリンク
- 鮮魚 / 寿司 / 天ぷら / ご飯もの / デザート
- 串 / 刺身 / サラダ / ご飯 / ドリンク

### 3-4-2. カテゴリを追加する

1. dashboard.html → 【メニュー管理】 → サブタブ **【カテゴリ管理】**（`data-tab="hq-categories"`）
2. **【+ カテゴリ追加】** ボタン
3. モーダル:
   - **カテゴリ名**（例: 「デザート」）
   - **カテゴリ名（英語）**（例: `Dessert`）
   - **並び順**（数値、小さいほど上に表示。例: 5）
4. **【保存】**

### 3-4-3. 並べ替え

1. 該当カテゴリの **【編集】**
2. **並び順** の数値を変更
3. **【保存】**

::: info ドラッグ&ドロップ並び替えは今後実装予定
現状は数値入力方式。視覚的な並び替えは将来のアップデートで対応予定。
:::

### 3-4-4. カテゴリの削除

1. 該当カテゴリの **【削除】** ボタン
2. 確認ダイアログ
3. **削除前の警告**: そのカテゴリに紐付く品目がある場合、削除できない（品目のカテゴリを先に変更する必要あり）

## 3-5. オプション（トッピング等）の設定

### 3-5-1. オプションとは

メニューに追加で選べる項目。例:

- 「唐揚げ」に対して **【サイズ選択】**（小 / 中 / 大、+価格）
- 「パスタ」に対して **【ソース選択】**（トマト / クリーム / ペペロンチーノ）
- 「ラーメン」に対して **【トッピング】**（煮卵 +100 / チャーシュー +200 / コーン +50）

### 3-5-2. オプショングループを作る

1. dashboard.html → 【メニュー管理】 → サブタブ **【オプション管理】**（`data-tab="option-groups"`、`data-role-max="manager"`）
2. **【+ オプショングループ追加】** ボタン
3. モーダル:

| 項目 | 説明 |
|---|---|
| **グループ名** | 例: 「トッピング」 |
| **グループ名（英語）** | 例: `Toppings` |
| **選択必須** | トグル ON/OFF（必須にするとお客様が必ず選ばなければならない） |
| **複数選択可** | トグル ON/OFF（ON なら客が複数選べる） |
| **並び順** | 数値 |

4. **【保存】**
5. グループが一覧に追加される

### 3-5-3. グループに選択肢を追加する

1. 作成したグループの行で **【＋ 選択肢追加】** をタップ
2. モーダル:
   - **選択肢名**（例: 「チーズ増し」）
   - **選択肢名（英語）**（例: `Extra Cheese`）
   - **追加価格**（例: `200` で +¥200、`0` なら価格変動なし）
3. **【保存】**
4. 繰り返して選択肢を増やす（例: チーズ増し、コーン、煮卵）

### 3-5-4. 選択肢をメニューに紐付ける

1. 【メニュー管理】 → 【店舗メニュー】
2. 該当メニューの **【編集】**
3. **「オプショングループ」** セクション
4. 使いたいグループのチェックボックスを ON
5. **【保存】**
6. セルフメニューで該当メニューをお客様が選ぶと、オプション選択モーダルが自動で開く

## 3-6. 限定メニュー（店舗独自品）の追加

本部マスタにない、その店舗だけのメニュー。

### 3-6-1. 手順

1. dashboard.html → 【メニュー管理】 → サブタブ **【限定メニュー】**（`data-tab="local-menu"`）
2. **【+ 限定メニュー追加】** ボタン（`btn-add-local`、`dashboard.html` L176）
3. モーダル:
   - 品名
   - 品名（英語）
   - カテゴリ（本部マスタのカテゴリを流用）
   - 価格
   - カロリー・アレルゲン（任意）
   - 画像URL（任意）
4. **【保存】**
5. セルフメニューに **「限定」** ピンクバッジ付きで表示される

### 3-6-2. 限定メニューの使いどころ

- **季節メニュー**: 冬の鍋メニュー、春のお花見弁当など
- **店舗独自開発品**: 渋谷店シェフのオリジナル料理
- **試作メニュー**: 特定店舗でテスト販売してから本部マスタに昇格

## 3-7. 多言語翻訳（英語・中国語・韓国語等）

### 3-7-1. 自動一括翻訳

メニュー全体を一度に翻訳。

1. dashboard.html → 【メニュー管理】 → 【店舗メニュー】
2. 画面右上の **【🌐 一括翻訳】** ボタン（`btn-translate-menu`、`dashboard.html` L152）
3. モーダルで翻訳先言語をチェック:
   - ☑ English
   - ☑ 中文（簡体字）
   - ☑ 繁體中文
   - ☑ 한국어
4. **【翻訳開始】**
5. AI（Gemini）が数十秒で全品目・カテゴリを一括翻訳
6. 「翻訳完了」トースト表示

### 3-7-2. 強制再翻訳

既に翻訳済みのものを上書きしたい時。

1. 一括翻訳ダイアログで **「🔄 強制再翻訳」** チェックボックス ON
2. **【翻訳開始】**
3. 既存の翻訳を上書き

### 3-7-3. 個別修正

AI の訳が気に入らない場合。

1. 該当メニューの **【編集】**
2. **品名（英語）** 欄に手動入力（例: `Fried Chicken Curry Rice`）
3. **【保存】**

### 3-7-4. お客様側での言語切替

- セルフメニュー画面右上の言語ボタン（🌐）でお客様が切替
- 日本語 / English / 中文 / 繁體 / 한국어 の 5 言語

## 3-8. CSV でメニューを一括登録する

大量のメニューを一度に登録する場合は CSV 一括登録が便利。

### 3-8-1. エクスポート（バックアップ・雛形取得）

1. 【店舗メニュー】 → **【CSV エクスポート】** ボタン（`btn-override-csv-export`）
2. ダウンロードフォルダに CSV 保存
3. Excel / Numbers で開く

### 3-8-2. CSV の編集

CSV の列（カラム）:

| カラム名 | 必須 | 説明 |
|---|---|---|
| `name` | ✅ | 品名（日本語） |
| `base_price` | ✅ | 価格（円） |
| `category` | ✅ | カテゴリ名（既存カテゴリと一致すること） |
| `name_en` | — | 英語名 |
| `calories` | — | カロリー（kcal） |
| `allergens` | — | カンマ区切り（例: `egg,milk`） |
| `image_url` | — | HTTPS の URL |

### 3-8-3. インポート（一括登録）

1. 【店舗メニュー】 → **【CSV インポート】** ボタン
2. CSV ファイルを選択
3. プレビュー画面で内容確認
4. エラーがあれば修正指示が表示される
5. **【インポート実行】**
6. 一括登録完了

### 3-8-4. 文字化け対処

日本語が文字化けする場合:

- **Windows Excel**: 保存時に **「CSV UTF-8（コンマ区切り）」** を選択
- **Mac Numbers**: デフォルトで UTF-8 なので OK
- **Google Sheets**: ファイル → ダウンロード → カンマ区切りの値（UTF-8）

## 3-9. スマレジからメニューをインポートする

スマレジを使っているお店が POSLA に移行する場合、メニューを自動移行できます。

詳細は **22 章「スマレジ連携の全設定」** 参照。

### 簡易手順

1. オーナーアカウントで owner-dashboard → **【外部POS連携】** タブ（`data-tab="external-pos"`）
2. **【OAuth 接続】** → スマレジに既存アカウントでログイン
3. 許可を与える
4. **【商品インポート】** ボタン
5. 非破壊インポート（既存 POSLA メニューは維持、スマレジの商品だけ追加）

## 3-10. 本部メニューの管理（チェーン店のみ）

本部一括配信アドオン（`tenants.hq_menu_broadcast=1`、+¥3,000 / 月 / 店舗）契約時のみ利用可能。

### 3-10-1. 本部メニュータブの場所

1. オーナーアカウントで owner-dashboard
2. 上部タブ **【本部メニュー】**（`data-tab="hq-menu"`、`owner-dashboard.html` L53）

### 3-10-2. 本部メニューの追加・編集

1. **【+ 本部メニュー追加】**（`btn-add-hq-menu`）
2. モーダルで品名・価格・カテゴリ入力
3. **【保存】**
4. 全店舗に自動配信される

### 3-10-3. 本部 CSV 一括管理

1. **【CSV エクスポート】**（`btn-hq-menu-csv-export`）
2. **【CSV インポート】**（`btn-hq-menu-csv-import`）

### 3-10-4. カテゴリフィルタ

本部メニューが多くなったら、画面上部のカテゴリドロップダウン（`hq-menu-category-filter`）で絞り込み。

### 3-10-5. 店舗側での本部メニュー編集制限

アドオン契約中、各店舗のダッシュボードでは本部マスタ項目（name, base_price, calories 等）が **readonly**（灰色表示、編集不可）になります。店舗側で変えられるのは:

- 店舗価格上書き（`price`）
- 非表示（`is_hidden`）
- 売切（`is_sold_out`）

### 3-11-1. 機能概要

特定のメニューを「今日のおすすめ」として、セルフメニュー画面の上部に目立つように表示する機能。

### 3-11-2. 設定手順

1. dashboard.html → 【メニュー管理】 → 【店舗メニュー】
2. 画面下部の **「今日のおすすめ」** セクション（折りたたみ式）を展開

4. **【保存】**
5. 日付指定で予約設定も可（翌日の「おすすめ」を事前登録）

## 3-13. よくある質問

**Q1. カテゴリを削除できない**

A1. そのカテゴリに紐付く品目がある場合は削除できません。先に該当品目のカテゴリを変更してから削除してください。

**Q2. 画像URL を公開サービスに置きたくない**

A2. POSLA 側のアップロード機能は現在未実装です。店舗の WordPress などに置いて URL を設定するか、後の POSLA アップグレードで対応予定。

**Q3. AI 翻訳が不正確**

A3. 固有名詞（「牛丼」→「Gyudon」等）は AI が一律翻訳しきれないので、手動修正してください。

**Q4. メニュー画像は必須？**

A4. 必須ではないですが、写真があるとお客様の注文率が大幅に上がります。最低でも主要メニューには画像を付けましょう。

**Q5. CSV インポートで日本語が文字化け**

A5. Excel の保存時に **「CSV UTF-8」** を選択してください（通常の CSV だと Shift-JIS で保存され文字化けの原因）。
