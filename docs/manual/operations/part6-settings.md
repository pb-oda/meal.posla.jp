# Part 6 — 店舗の設定

このページでは、店舗運営に必要な各種設定を、**実際に押すボタン** 1 つずつ順番に説明します。

::: tip このページの範囲
本ページで触る画面は以下の 3 つです。

- **owner-dashboard.html**（オーナー専用、`https://eat.posla.jp/public/admin/owner-dashboard.html`）: 店舗管理・ユーザー管理・決済設定・本部メニュー
- **dashboard.html**（マネージャー以上、`https://eat.posla.jp/public/admin/dashboard.html`）: 店舗設定タブ内の各種設定
- **店舗設定サブタブ**: 営業時間・レジ・税率・ウェルカムメッセージ・プリンタ・外観カスタマイズ
:::

## 6-1. 店舗情報の変更

### 6-1-1. 店舗名を変更する（オーナーのみ）

1. オーナーアカウントで `https://eat.posla.jp/public/admin/owner-dashboard.html` を開く
2. 上部タブから **【店舗管理】** をクリック（`data-tab="stores"`、`owner-dashboard.html` L54）
3. 店舗一覧が表示される
4. 該当店舗の行にある **【編集】** ボタン
5. モーダルが開く
6. **店舗名** 欄を書き換え
7. **【保存】**
8. 数秒で反映（管理画面のヘッダーにも新しい名前が表示）

### 6-1-2. 住所・電話番号を登録する（レシート印字用）

お店のレシートに印字される住所・電話番号は別の画面で設定します。

1. マネージャー / オーナーで `dashboard.html` にログイン
2. 上部タブの **【店舗設定】** をクリック
3. サブタブから **【店舗設定】** を選択（`data-tab="settings"`）
4. 画面を下にスクロールして **「レシート設定」** セクションを探す
5. 以下を入力:
   - **店舗名**（レシートに印字される名前、owner-dashboard の店舗管理と別物）
   - **住所**（例: 「東京都渋谷区渋谷 1-2-3 XX ビル 5F」）
   - **電話番号**（例: 「03-1234-5678」）
6. **【設定を保存】** ボタン

::: tip レシート用の店舗名と、システム上の店舗名は別
システム上の「店舗名」は管理画面のヘッダーや店舗選択に使われる内部識別用。レシート印字用の店舗名は、お客様に見える形式（例: 「松乃家 渋谷店」）を設定できます。
:::

### 6-1-3. 新しい店舗を追加する（オーナーのみ）

多店舗経営で新店舗をオープンする場合。

1. owner-dashboard → **【店舗管理】**
2. **【+ 店舗を追加】** ボタン（`btn-add-store`、`owner-dashboard.html` L113）
3. モーダル:
   - 店舗名
   - 店舗 Slug（URL 用の識別子、半角英小文字とハイフン）
   - 英語名（任意）
4. **【保存】**
5. Stripe サブスクリプションの数量が自動で +1 され、次回請求から追加料金が発生

::: warning 店舗追加の料金
- 2 店舗目以降は **¥17,000 / 月 / 店舗**（1 店舗目は ¥20,000）
- 次回請求日から反映（日割り計算されるケースあり）
- 本部一括メニュー配信アドオン契約中の場合、追加店舗分も **+¥3,000 / 月 / 店舗** 上乗せ
:::

## 6-2. テーブルの追加・削除・レイアウト変更

### 6-2-1. テーブルを追加する

1. dashboard.html → 上部タブ **【テーブル・フロア】** → サブタブ **【テーブル管理】**（`data-tab="tables"`）
2. **【+ テーブル追加】** ボタン（`btn-add-table`、`dashboard.html` L186）
3. モーダル:
   - **テーブルコード**（例: `T06`、店内で一意）
   - **座席数**（`capacity`、例: 4）
   - **フロア**（例: 1F、テラス）
4. **【保存】**
5. 一覧に追加される

### 6-2-2. テーブルを削除する（推奨: 無効化）

過去の注文履歴との紐付けを壊さないため、**完全削除ではなく「無効化」** を推奨。

1. 該当テーブルの **【編集】** ボタン
2. **「有効」トグル** を OFF
3. **【保存】**
4. 一覧で「無効化」バッジ表示
5. セルフメニュー・ハンディから見えなくなる

### 6-2-3. 完全削除（非推奨）

どうしても完全削除したい場合:

1. 該当テーブル → **【削除】** ボタン（赤）
2. 確認ダイアログで **【OK】**
3. 削除される（注意: 過去の注文の `table_id` が不整合になるリスクあり）

### 6-2-4. フロアマップでのレイアウト変更

1. サブタブ **【フロアマップ】**（`data-tab="floor-map"`）
2. 画面にテーブルのカード一覧が表示される
3. 各テーブルの位置座標（pos_x / pos_y）・サイズ（width / height）を編集
4. **【保存】**

::: info ドラッグ＆ドロップ対応は今後実装予定
現在のフロアマップはカード一覧形式。ドラッグ&ドロップによるビジュアル配置機能は今後のアップデートで。
:::

## 6-3. QR コードの印刷

### 6-3-1. テーブル QR の印刷

1. dashboard.html → 【テーブル・フロア】 → 【テーブル管理】
2. 各テーブルの行にある **【QR 印刷】** ボタン
3. 新しいタブで印刷用ページが開く
4. **Cmd+P**（Mac）または **Ctrl+P**（Windows）で印刷
5. テーブルごとに 1 枚ずつ印刷される

### 6-3-2. テイクアウト QR の印刷

1. dashboard.html → 【店舗設定】 → 【店舗設定】サブタブ
2. 画面下部の **「テイクアウト QR」セクション**
3. **【🖨 印刷】** ボタン → 印刷

### 6-3-3. 予約 LP QR の印刷

1. 同じく【店舗設定】 → 【店舗設定】サブタブ
2. 下部の **「📅 予約LP」セクション**
3. **【🖨 印刷】** ボタン → A4 ポスターとして印刷

## 6-4. 営業時間・消費税率・注文上限の設定

### 6-4-1. 設定項目

1. dashboard.html → 【店舗設定】 → 【店舗設定】サブタブ
2. **「営業基本設定」セクション** に以下の項目:

| 項目 | 推奨値 | 説明 |
|---|---|---|
| **営業日区切り時刻** | 通常 `05:00` | 深夜営業店の売上集計日を決める時刻。24:00 を過ぎても当日扱い |
| **税率（店内）** | `10%` | 標準税率 |
| **税率（テイクアウト）** | `8%` | 軽減税率 |
| **注文上限数** | `10` | セルフオーダーの 1 注文あたり品目数上限 |
| **注文上限金額** | `¥30,000` | セルフオーダーの 1 注文あたり金額上限 |
| **ラストオーダー時刻** | 閉店 30 分前 | 以降セルフ注文がブロックされる |

### 6-4-2. 消費税率の注意

- 日本の消費税は **店内 10% / テイクアウト軽減税率 8%** が標準
- 輸出店舗・免税店は別設定が必要（運営にご相談ください）

## 6-5. 決済方法の有効化・無効化

お店で使う支払方法（現金・カード・QR 決済）を個別に ON/OFF できます。

1. dashboard.html → 【店舗設定】 → 【店舗設定】サブタブ
2. **「レジ設定」セクション** に以下のトグル:

| 方法 | デフォルト | 用途 |
|---|---|---|
| **現金** | ON | 現金決済を受け付けるか |
| **クレジットカード** | ON | Stripe Terminal / QR 以外のカード決済 |
| **QRコード** | ON | PayPay / d払い等 |

3. **【設定を保存】**

## 6-6. Stripe 決済の設定（カードリーダー接続）

詳細は **21 章「Stripe 決済の全設定」** 参照。ここでは要約のみ。

### 手順

1. オーナーアカウントで owner-dashboard にログイン
2. 上部タブから **【決済設定】**（`data-tab="payment"`、`owner-dashboard.html` L58）
3. パターン選択:
   - **パターンA（自前 Stripe キー）**: 既に Stripe アカウントを持っている場合
   - **パターンB（POSLA Connect 経由）**: POSLA が仲介して Stripe を使う（推奨）
4. 必要情報入力
5. 保存
6. カードリーダー Stripe Reader S700 を購入
7. S700 を店舗 Wi-Fi に接続
8. POSLA 管理画面でペアリング

## 6-7. ウェルカムメッセージの設定

お客様がセルフメニューを開いた時の挨拶文。

1. dashboard.html → 【店舗設定】 → 【店舗設定】サブタブ
2. **「ウェルカムメッセージ」セクション**
3. **日本語版** を入力（例: 「いらっしゃいませ！今日のおすすめは唐揚げ定食です」）
4. **英語版** を入力（任意、例: `Welcome! Today's recommendation is Karaage Teishoku.`）
5. **【設定を保存】**

## 6-8. テイクアウトの設定

テイクアウト受付の各種設定。

1. dashboard.html → 【店舗設定】 → 【店舗設定】サブタブ
2. **「テイクアウト設定」セクション**:

| 項目 | 説明 |
|---|---|
| **テイクアウト受付** | トグル ON/OFF |
| **受付開始時刻** | 例: `10:00` |
| **受付終了時刻** | 例: `20:00` |
| **スロット容量** | 1 時間あたりの同時受付数（例: 5 件） |
| **最低準備時間** | 注文から受取までの最短時間（例: 15 分） |
| **オンライン決済必須化** | 2026-04-18 以降デフォルト ON、**店頭支払いは廃止** |

3. **【設定を保存】**

## 6-9. プリンタ（Star / Epson）の設定（U-2 新機能）

詳細は **26 章** 参照。ここでは要約のみ。

### 6-9-1. プリンタを Wi-Fi に接続する

1. プリンタの電源を入れる
2. プリンタ本体の **FEED ボタン** を長押し（約 5 秒）
3. 自動印字される紙に **IP アドレス** が書いてある（例: `192.168.1.234`）
4. 紙をメモしておく

### 6-9-2. POSLA にプリンタを登録

1. dashboard.html → 【店舗設定】 → 【店舗設定】サブタブ
2. 画面を下にスクロール
3. **「プリンタ設定」セクション**
4. 以下を入力:

| 項目 | 値 | 説明 |
|---|---|---|
| **プリンタ種別** | Star / Epson / ブラウザ | メーカーを選択 |
| **IP アドレス** | `192.168.1.234` | 上記でメモした値 |
| **ポート** | `80` | 通常は 80 でOK |
| **用紙幅** | 58mm / 80mm | プリンタの仕様 |
| **自動印刷（キッチン伝票）** | ON | 注文時に厨房に伝票を印刷 |
| **自動印刷（レシート）** | ON | 会計時にレシートを印刷 |

5. **【🖨 テスト印刷】** ボタン → テスト用紙が出る
6. **【設定を保存】**

## 6-10. 外観カスタマイズ（ロゴ・色）

ブランドの世界観を反映。

1. dashboard.html → 【店舗設定】 → 【店舗設定】サブタブ
2. **「外観カスタマイズ」セクション**:

| 項目 | 説明 |
|---|---|
| **テーマカラー** | カラーピッカーまたは HEX 値（例: `#ff6f00`）でブランドカラー指定 |
| **表示名** | ヘッダー等に表示される店舗名（多言語対応） |
| **ロゴ画像URL** | ロゴ画像の URL（**HTTPS 必須**） |

3. **【設定を保存】**

### ロゴ画像の準備

- 画像は事前に別サービスでアップロード（Imgur / AWS S3 / 店舗 HP 等）
- URL は HTTPS でなければならない（セキュリティ上の理由）
- 推奨サイズ: 400×100px 以下、PNG 透過推奨

## 6-12. KDS ステーション設定

大規模キッチンで、料理ジャンルごとに KDS 画面を分ける設定。

1. dashboard.html → 【店舗設定】 → サブタブ **【KDSステーション】**
2. **【+ ステーション追加】** ボタン（`btn-add-station`、`dashboard.html` L287）
3. 入力:
   - ステーション名（例: 「和食ステーション」）
   - 英語名（オプション）
   - 担当カテゴリ（チェックボックスで選択）
4. **【保存】**
5. KDS 画面上部のステーション選択バーに追加される

詳細は 7 章「KDS」参照。

## 6-13. 監査ログの閲覧

誰が何をしたか履歴を確認できる機能（manager 以上）。

1. dashboard.html → 【店舗設定】 → サブタブ **【監査ログ】**
2. 日付範囲・ユーザー・イベントタイプでフィルタ可能
3. 表示される情報:
   - 日時
   - ユーザー名・ロール
   - 操作内容（ログイン / メニュー編集 / 売り切れ切替 等）
   - 対象リソース

監査ログは **全 API 呼び出し** ではなく、設定変更・削除・ログインなど **重要操作** のみ記録。

## 6-14. よくある質問

**Q1. 店舗を削除したい**

A1. owner-dashboard の店舗管理で **「論理削除」** のみ可能（物理削除しない）。`is_active=0` に設定されるだけで、過去データは保持されます。

**Q2. 税率を変更したら既存の注文はどうなる？**

A2. 過去の注文には影響しません。変更後の新規注文から新税率が適用されます。

**Q3. プリンタが反応しない**

A3. 26 章のトラブルシューティング参照。主な原因: Wi-Fi 接続切れ / IP アドレス変更 / ファームウェア古い。

**Q4. 営業時間外でも管理画面は使える？**

A4. 使えます。管理画面の操作は 24 時間いつでも可能。営業時間はお客様のセルフオーダーの受付時間に関する設定です。
