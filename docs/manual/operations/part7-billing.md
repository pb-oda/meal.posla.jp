# Part 7 — 契約・お支払い

このページでは、POSLA の月額料金契約・アドオン追加・解約の方法を、**ボタンの押し方 1 つずつ** 順番に説明します。

::: tip 対象者
このページはすべて **オーナー（owner）ロール専用** です。マネージャー・スタッフは契約・決済の操作はできません。

実装: `api/subscription/*` はすべて `require_role('owner')`
:::

---

## 7-0. POSLA の料金体系（α-1 構成、2026-04-09 確定）

### 7-0-1. 単一プラン + アドオン

POSLA は **単一プラン + アドオン** 構成です。機能判定（`check_plan_feature`）は α-1 ベースで一本化済（`hq_menu_broadcast` のみ判定、それ以外は常に true）。ただし POSLA 管理画面のテナント編集 API（`api/posla/tenants.php`）では旧 3 プラン文字列（`standard` / `pro` / `enterprise`）も互換のため受付可能。新規テナントには α-1 構成のみ推奨。

| 項目 | 金額 |
|---|---|
| **基本料金（1 店舗目）** | ¥20,000 / 月 |
| **追加店舗（2 店舗目以降）** | ¥17,000 / 月 / 店舗（1 店舗目から 15% 引き） |
| **本部一括メニュー配信アドオン** | +¥3,000 / 月 / 店舗（任意） |
| **トライアル** | 初回 30 日間無料（Stripe カード登録必須） |

### 7-0-2. 料金計算例

| 構成 | 計算 | 月額合計 |
|---|---|---|
| 1 店舗のみ | ¥20,000 | ¥20,000 |
| 2 店舗 | ¥20,000 + ¥17,000 | ¥37,000 |
| 3 店舗 | ¥20,000 + ¥17,000 × 2 | ¥54,000 |
| 3 店舗 + アドオン | ¥54,000 + ¥3,000 × 3 | ¥63,000 |
| 5 店舗 | ¥20,000 + ¥17,000 × 4 | ¥88,000 |
| 5 店舗 + アドオン | ¥88,000 + ¥3,000 × 5 | ¥103,000 |
| 10 店舗 + アドオン | ¥173,000 + ¥30,000 | ¥203,000 |

### 7-0-3. 全機能が標準提供

基本料金を払っている限り、以下はすべて利用可能（アドオン契約の有無に関わらず）:

- セルフオーダー・KDS・ハンディPOS・POSレジ
- Stripe Terminal カード決済・QR 決済
- 予約管理（L-9）・テイクアウト・多言語
- 在庫・レシピ・AI 需要予測
- シフト管理・勤怠連携・AI シフト提案
- 全レポート（売上・ABC 分析・回転率・スタッフ評価・併売分析）
- クロス店舗分析・統合シフトビュー
- 監査ログ・セキュリティ機能

### 7-0-4. 本部一括配信アドオンを契約する理由

アドオン（`tenants.hq_menu_broadcast=1`）契約時のみ:
- owner-dashboard に **【本部メニュー】** タブが出現
- 本部マスタのメニューを owner 側で一括編集
- 店舗側ダッシュボードでは本部マスタ項目が readonly（店舗独自編集を禁止）
- メニュー変更が全店舗に自動配信される

**想定ユーザー**: 5 店舗以上のチェーン本部、本部で統一メニュー管理したい企業。

## 7-1. サブスクリプション（月額契約）の開始

### 7-1-1. 新規申込のフロー

1. POSLA の LP（ランディングページ）を開く
   - **現行**: `https://eat.posla.jp/public/`
   - **将来本番**: `meal.posla.jp`（予定）
2. **【30 日無料で始める】** ボタン
3. サインアップフォームに入力:

| 項目 | 必須 | 説明 |
|---|---|---|
| 企業名 (屋号) | ✅ | 法人名 / 屋号（例: 株式会社○○ / 松の屋）。`tenants.name` に保存 |
| 1 店舗目の店舗名 | ✅ | 1 店舗目の名称（例: 渋谷店 / 本店）。`stores.name` に保存 |
| 代表者名 | ✅ | オーナー氏名 |
| 電話 | ✅ | 連絡先電話 |
| メール | ✅ | 契約関連の通知先 |
| ユーザー名 | ✅ | オーナーアカウントのユーザー名（半角英数） |
| パスワード | ✅ | オーナー用パスワード（8 文字以上 + 英字 + 数字） |
| 店舗数 | ✅ | 契約する店舗数（1 以上） |
| 本部一括配信アドオン | — | チェックボックス（チェーン店のみ ON） |

4. **【お申込み → Stripe 決済画面へ】**
5. Stripe Checkout 画面に遷移
6. **クレジットカード情報** を入力
   - カード番号
   - 有効期限
   - セキュリティコード
7. **【申し込み】**
8. **30 日間無料トライアル開始**（この時点ではまだ課金されない）
9. サーバー側で自動処理:
   - テナント作成
   - 店舗レコード作成（指定店舗数分）
   - オーナーアカウント作成
   - ウェルカムメール送信（ログイン情報付き）
10. ウェルカムメール受信（通常 5 分以内）
11. メール内の **ログイン URL** からログイン → 運用開始

### 7-1-2. トライアル期間中の注意

- **31 日目から自動で月額課金** が開始される（Stripe が自動処理）
- トライアル期間中でも全機能が使える
- 30 日以内にキャンセルすれば請求なし（7-5 参照）

### 7-1-3. 詳細は

→ [internal/08-signup-flow.md](../internal/08-signup-flow.md)（サインアップフロー完全解説）

## 7-2. 契約内容の確認

### 7-2-1. Stripe Customer Portal を開く

Stripe の顧客ポータルで請求情報を確認できます。

1. owner アカウントで `owner-dashboard.html` を開く
2. 上部タブ **【決済設定】**（`data-tab="payment"`、`owner-dashboard.html` L58）
3. **【Customer Portal を開く】** ボタン
4. 新しいタブで Stripe のポータル画面が開く
5. 以下が確認できる:

| 項目 | 説明 |
|---|---|
| **請求履歴** | 過去の請求明細・PDF ダウンロード可 |
| **次回請求日** | いつ次の課金があるか |
| **プラン詳細** | 契約中のプラン・アドオン |
| **カード情報** | 登録カードの確認・変更 |
| **請求書ダウンロード** | 各月の領収書を PDF で取得 |

### 7-2-2. POSLA 管理画面での確認

一部情報は owner-dashboard の【契約・プラン】タブ（`data-tab="subscription"`）でも確認可能:

- 現在のプラン
- 次回請求予定額
- 店舗数
- アドオン契約状況

## 7-3. 本部一括配信アドオンの追加

途中からアドオンを契約したい場合。

### 7-3-1. 手順

1. owner-dashboard → 【決済設定】タブ
2. **【アドオン管理】** セクション
3. **「本部一括配信 (+¥3,000 / 月 / 店舗)」** トグル **ON**
4. **【Stripe Checkout で確定】** ボタン
5. Stripe の確認画面で最終確定
6. Stripe サブスクリプションに `stripe_price_hq` アイテムが追加される
7. 次回請求から料金が変わる（日割り計算あり）

### 7-3-2. 実装の裏側

`api/lib/stripe-billing.php` の `reconcile_subscription()` が Stripe サブスクリプション items を現在の構成（店舗数 + アドオン ON/OFF）に合わせて自動調整します。

### 7-3-3. アドオン有効化後の変更

- owner-dashboard に **【本部メニュー】** タブが即座に出現
- 既存の店舗メニューデータは引き継がれる（`menu_templates` はそのまま）
- 店舗側ダッシュボードで本部マスタ項目が readonly になる

## 7-4. 解約の方法

### 7-4-1. Stripe Portal 経由で解約

1. owner-dashboard → 【決済設定】 → **【Customer Portal を開く】**
2. Stripe Portal 画面で **【Cancel Plan】** または **【プランをキャンセル】**
3. 理由を選択（任意）
4. **【キャンセルを確定】**
5. 解約処理完了

### 7-4-2. 解約後の動き

- **現在の請求期間末まで利用可能**（例: 4/15 に解約 → 4/30 まで使える）
- 期間末で自動停止（`is_active=0` に移行）
- データは **30 日間保持**（再契約時の復旧用）
- 30 日経過後に物理削除

### 7-4-3. データのエクスポート

解約する前に、必要なデータは CSV でエクスポートしてください:

- メニュー: 【メニュー管理】 → 【CSV エクスポート】
- 注文履歴: 【レポート】 → 【注文履歴】 → CSV
- スタッフ情報: 【スタッフ管理】 → エクスポート（将来対応）
- 売上レポート: 各レポートの CSV エクスポート

解約後 30 日以内ならデータ持ち出しできます。31 日以降は不可。

## 7-5. 30 日無料トライアル中のキャンセル

### 7-5-1. 条件

- 申込から **30 日以内** ならキャンセルで請求なし
- 31 日目以降にキャンセルすると、その月の料金は発生する（月割り / 日割り）

### 7-5-2. 手順

7-4 と同じ手順で Customer Portal から **【Cancel】**。期間末（31 日目）に自動停止。期間中は全機能使える（試用期間として）。

## 7-6. よくある質問

**Q1. プラン変更（アップグレード・ダウングレード）はできる？**

A1. **単一プラン + アドオン構成なので「プラン変更」という概念はありません**。変えられるのは以下のみ:
- 店舗数（Stripe Subscription の quantity で自動）
- 本部一括配信アドオンの ON/OFF

**Q2. 支払方法を変えたい**

A2. Stripe Customer Portal で **【カード情報変更】**。新しいカード情報を入力して保存。

**Q3. 領収書が欲しい**

A3. Stripe から自動発行メールが毎月の請求時に送信されます。また Customer Portal から過去の請求書 PDF を随時ダウンロード可能。

**Q4. 税務書類（インボイス対応）は？**

A4. **インボイス対応予定（Phase 2）**。現在は Stripe の請求書で代用してください。

**Q5. 複数の店舗を違うカードで決済したい**

A5. **不可**。POSLA では **1 テナント = 1 カード登録** の仕様です。同じテナント内の全店舗は 1 枚のカードで一括請求されます。

**Q6. 解約したら即データ消える？**

A6. **30 日間は保持されます**。この間に再契約すれば全データ復旧。30 日経過後は物理削除。

**Q7. 一時休業したい。契約を一時停止できる？**

A7. 現状「一時停止」機能はありません。解約 → 再契約の流れになります（30 日以内なら簡単に戻せる）。

**Q8. 月の途中で店舗を追加した。請求はどうなる？**

A8. Stripe が **日割り計算** します。その月の残り日数分の料金が次回請求に加算されます。

**Q9. クレジットカード以外で払いたい**

A9. 現状 Stripe 経由の **クレジットカードのみ**。銀行振込・請求書払いは非対応。

**Q10. Stripe アカウント持っていない / Stripe に直接登録したくない**

A10. Customer Portal を使わなくても、Stripe 側で自動的にカスタマーアカウントが作られます（ユーザー側の登録作業は不要）。

## 7-7. 契約周りのトラブルシューティング

### 7-7-1. 請求できなかったとき

Stripe がカード決済を試みて失敗した場合:

1. 自動的に 3 回リトライされる（日を置いて）
2. それでも失敗したら POSLA 運営に通知
3. アカウントが一時的に停止される可能性あり
4. オーナーのメールに通知が届く
5. 対処: Customer Portal でカード情報を更新

### 7-7-2. カードが期限切れ

1. 期限切れの 30 日前にカスタマーからメール通知（Stripe から）
2. Customer Portal で新しいカードに更新
3. 期限切れまで放置すると自動的に再試行失敗

### 7-7-3. 請求書の金額が合わない

1. Customer Portal で請求書 PDF をダウンロード
2. 明細を確認（日割り計算が含まれていることが多い）
3. それでも合わない場合 `info@posla.jp` に連絡

## 7-8. 契約書類

POSLA の契約は **利用規約** と **プライバシーポリシー** に同意することで成立します。

- 利用規約: `https://eat.posla.jp/terms`（LP から参照可）
- プライバシーポリシー: `https://eat.posla.jp/privacy`
- 特定商取引法表記: 契約時の LP に記載

## 7-9. 関連 API（参考）

オーナーが Customer Portal を開くときの裏側 API（自動で呼ばれるので、通常は意識しなくてOK）:

- `POST /api/subscription/portal.php` — Stripe Customer Portal セッション作成 → Portal URL を返す
- `POST /api/subscription/checkout.php` — 新規申込時の Stripe Checkout セッション作成
- `GET /api/subscription/status.php` — 現在のサブスク状態取得
- `POST /api/subscription/webhook.php` — Stripe からの自動通知受信（請求成功・失敗・更新）

これらは画面の「Customer Portal を開く」「アドオン追加」等のボタンが裏で呼んでいます。直接叩く必要はありません。

## 7-9-S3. activate.php の Stripe 検証必須化（2026-04-19 S3 修正）

POSLA 運営向けの注意事項です（オーナーは通常意識しません）。

- 2026-04-19 以降、`POST /api/signup/activate.php`（webhook フォールバック）は **Stripe Subscription を必ず確認**してから有効化します
- カード未登録のテナント（`tenants.stripe_subscription_id` が NULL、もしくは Stripe 側 status が `incomplete` / `past_due` / `canceled`）は `PAYMENT_NOT_CONFIRMED` (HTTP 402) で拒否
- 救済策として activate.php を叩く場合も、対象テナントが Stripe 側で `trialing` または `active` 状態であることを事前確認すること
- 詳細: [internal/02-onboarding.md § 2.1.5](../internal/02-onboarding.md)
