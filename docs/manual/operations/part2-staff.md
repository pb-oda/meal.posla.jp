# Part 2 — スタッフ管理

このページでは、スタッフのアカウント作成・シフト管理・勤怠打刻を、**指の動きを 1 つずつ** 順番に説明します。

::: tip POSLA の鉄則: 1 人 1 アカウント
**絶対に「キッチン」「ホール」のような共有アカウントを作らないでください**。以下が全部壊れます。

- 勤怠打刻（誰が出勤したか分からない）
- 監査ログ（誰が操作したか記録されない）
- スタッフ評価レポート（全員のデータが混ざる）
- シフト管理（個別割当不可）

必ず **スタッフ 1 人につき 1 つのアカウント** を作成してください。
:::

::: warning 個人スマホは業務に使わない
POSLA の運用前提は、ハンディ・KDS・レジは **全て店舗が用意する備品端末** です。スタッフ個人のスマートフォンを業務に使う BYOD 運用は推奨しません（労務・セキュリティ・紛失時補償の問題を避けるため）。ハンディ端末は店舗が複数台購入して device アカウントでログインしっぱなしで運用するのが標準です。
:::

## 2-0. 作成できるアカウントの種類（4 種類）

| ロール | 役割 | 作成可能者 |
|---|---|---|
| **オーナー (owner)** | 代表者・全店舗統括 | 通常は POSLA 運営が契約時に発行。既存 owner から追加作成も可 |
| **マネージャー (manager)** | 店長・店舗運営 | **オーナーのみ作成可** |
| **スタッフ (staff)** | アルバイト・社員 | **オーナーまたはマネージャーが作成可** |
| **デバイス (device)** | 端末専用アカウント（KDS / レジ / ハンディ） | **オーナーまたはマネージャーが作成可** |

## 2-1. 新しいスタッフのアカウントを作る

### 2-1-1. 手順（マネージャーの場合）

1. 管理画面にマネージャーでログイン
2. 上部タブ **【店舗設定】** → サブタブ **【スタッフ管理】**
3. 画面右上の **【+ 追加】** ボタン
4. 「スタッフ追加」モーダルが開く
5. 以下を入力:

| 項目 | 必須 | 説明 |
|---|---|---|
| **表示名** | — | 「田中 太郎」等（後でも変更可） |
| **ユーザー名** | ✅ | 半角英数・ハイフン・アンダースコア 3〜50 文字（例: `tanaka01`） |
| **メール** | — | 任意（通知送信用） |
| **パスワード** | ✅ | 8 文字以上 + 英字 + 数字必須 |
| **時給** | — | 人件費レポート用（円 / 時、1〜10000） |
| **ビジブルツール** | — | 通常は空欄（staff には不要） |

6. **【保存】** ボタン
7. スタッフに **ユーザー名 + パスワード** を口頭またはメモで伝える
8. スタッフは初回ログイン後、自分で好きなパスワードに変更（推奨）

### 2-1-2. 手順（オーナーの場合）

オーナーはオーナー管理画面からマネージャー・スタッフを作成できます。

1. オーナーでオーナー管理画面にログイン
2. 上部タブ **【ユーザー管理】**
3. **【+ ユーザー追加】** ボタン
4. モーダル:
   - 表示名
   - ユーザー名
   - メール（任意）
   - パスワード
   - **ロール**: `manager` または `staff` のどちらか選択
   - **担当店舗**: 複数チェックボックス（所属店舗）
5. **【保存】**

### 2-1-3. ユーザー名の命名ルール

- **半角英数字 + ハイフン + アンダースコア** のみ
- **3〜50 文字**
- **POSLA 全体で一意**。同じ店舗内だけでなく、他店舗・他テナントと重複する場合も登録できない

::: warning ユーザー名はPOSLA全体で一意です
他社・他店舗で同じユーザー名が使われている場合も登録できません。店舗名や会社名の略号を付けると重複しにくくなります（例: `matsunoya_tanaka01`, `momonoya_suzuki-hall`）。
:::

**推奨パターン**:
- `<テナント略号>_tanaka01`, `<テナント略号>_suzuki-hall`（人間の場合、名前 + 番号 or 役割）
- `kds-shibuya-01`, `register-ginza-01`, `handy-shinjuku-02`（device の場合、役割 + 店舗 + 番号）

### 2-1-4. 失敗時のエラー

| メッセージ | 原因 | 対処 |
|---|---|---|
| 「ユーザー名は既に使用されています」 | 同名アカウントあり | 別のユーザー名（例: tanaka01 → tanaka02） |
| 「パスワードは8文字以上で入力してください」 | 短い | 8 文字以上にする |
| 「パスワードには英字を1文字以上含めてください」 | 数字のみ | 英字を混ぜる |
| 「パスワードには数字を1文字以上含めてください」 | 英字のみ | 数字を混ぜる |
| 「ユーザー名は半角英数字・ハイフン・アンダースコア（3〜50文字）で入力してください」 | 形式違反 | 正しい形式に |
| 「メールアドレスの形式が正しくありません」 | メール入力ミス | 正しいメール形式に（任意なので空欄 OK） |

## 2-1-b. スタッフのレジ担当 PIN を設定する

POSレジで「誰が操作したか」を特定するため、各スタッフに **4〜8 桁の PIN** を設定します。

::: warning マネージャーも全員 PIN 設定が必須
**マネージャー / オーナーであっても自分の PIN 設定が必須**になりました。レジ金庫操作（レジ開け / レジ閉め / 入金 / 出金）にも PIN が要求されるため、PIN を設定していないマネージャーは開店作業ができません。

「会計用だけ」と思って未設定のままにしていたマネージャーは、**今すぐ自分の PIN を設定**してください。
:::

### PIN が必要な操作の一覧

| 操作 | PIN 必須 |
|---|---|
| 会計確定（現金 / カード / QR / Terminal） | ✅ |
| 返金処理（部分・全額） | ✅ |
| **レジ開け** | **✅** |
| **レジ閉め** | **✅** |
| **入金（Cash In）** | **✅** |
| **出金（Cash Out）** | **✅** |

### 設定手順（店長向け）

1. 管理画面 → 【店舗設定】 → 【スタッフ管理】
2. 対象スタッフの **【編集】** ボタン
3. 編集モーダル内の **「レジ担当 PIN」** 欄に 4〜8 桁の数字を入力
4. **【保存】** ボタン
5. スタッフに PIN を口頭で伝える
6. スタッフは初回ログイン時に自分で別の PIN に変更推奨

### スタッフ自身による PIN 変更

1. スタッフ（マネージャー含む）が管理画面にログイン
2. 同じ手順で自分の **【編集】** ボタン
3. 「レジ担当 PIN」欄に新しい PIN を入力 → **【保存】**

### 注意事項

- ✅ **退勤時は必ず退勤打刻**: 退勤したスタッフの PIN は **自動無効化** される（不正悪用防止）
- ✅ **マネージャーも開店前に出勤打刻が必須**（出勤打刻していないと自分の PIN でレジ開けできない）
- ✅ 弱い PIN は登録不可（`1234`、`0000`、ゾロ目などはエラー）
- ✅ PIN は他人に教えない、付箋でレジに貼らない
- ✅ 半年〜1 年で変更推奨

詳細は [2 章 2.6 担当スタッフ PIN](../tenant/02-login.md) / [8 章 8.6 レジ開閉操作](../tenant/08-cashier.md#86-レジ開閉操作) 参照。

## 2-2. スタッフのパスワードをリセットする

スタッフが「パスワードを忘れた」と言ってきた時。

### 手順

1. 管理画面 → 【店舗設定】 → 【スタッフ管理】
2. 該当スタッフの行を探す
3. **【編集】** ボタン
4. 編集モーダルが開く
5. **【新しいパスワード】** 欄に新しいパスワード（8 文字以上 + 英字 + 数字）を入力
6. **【保存】**
7. スタッフに新パスワードを口頭で伝える
8. スタッフは次回ログイン後、自分で変更してもらう

::: warning 新パスワードを平文でメモ・メール送信しない
セキュリティリスクがあるので、口頭で伝えるのが基本。メモでも必ず破棄。メール送信時は暗号化メッセージで。
:::

## 2-3. スタッフを削除する（退職時）

### 推奨: 削除ではなく **無効化**

監査ログの履歴を維持するため、**アカウントを物理削除せず、無効化する運用** を推奨します。

### 2-3-1. 無効化の手順

1. 【スタッフ管理】 で該当スタッフの **【編集】**
2. **「有効/無効」トグル** を **【無効】** にする
3. **【保存】**
4. 以降、このスタッフはログインできなくなる
5. 過去の注文・勤怠・操作履歴はすべて保持される

### 2-3-2. 完全削除（非推奨）

どうしても完全削除したい場合:

1. 編集画面で **【削除】** ボタン（赤）
2. 確認ダイアログで **【OK】**
3. アカウントが削除される

::: danger 注意
完全削除すると、過去の注文・勤怠・操作履歴とのつながりが確認しづらくなる場合があります。税務監査・労務管理の観点から **無効化を推奨** します。
:::

## 2-4. デバイスアカウント（KDS / レジ / ハンディ端末専用）の作り方

「人」ではなく「端末」のためのアカウントです。

### 2-4-1. デバイスアカウントの特徴

- ログインしっぱなし前提（端末再起動時のみ再ログイン）
- シフト・人件費・スタッフ評価レポートの対象外（人間ではないため）
- 監査ログには端末用アカウントとして記録される
- 管理画面には入れない（KDS / レジ / ハンディのいずれかに直接遷移）

### 2-4-2. 作成手順

1. 管理画面 → 【店舗設定】 → 【スタッフ管理】
2. 画面を下にスクロール → **「デバイスアカウント」セクション**
3. **【+ デバイス追加】** ボタン
4. モーダル:

| 項目 | 必須 | 説明 |
|---|---|---|
| **表示名** | ✅ | 「厨房KDS 1番」「レジ-渋谷-01」等の人間向けの名前 |
| **ユーザー名** | ✅ | `kds-shibuya-01` 等（英数字・ハイフン、3〜50 文字） |
| **パスワード** | ✅ | 8 文字以上 + 英字 + 数字 |
| **用途** | ✅ | チェックボックス: KDS / POSレジ / ハンディ から **少なくとも 1 つ** |

5. **【保存】**
6. 一覧に追加される

### 2-4-3. 用途の組み合わせによる遷移先

デバイスアカウントがログインした後の遷移先は、作成時に選んだ用途で決まります。

| 用途 | ログイン後の画面 |
|---|---|
| ハンディ | ハンディ画面 |
| POSレジ | POSレジ画面 |
| KDS | KDS画面 |
| 複数用途を選んだ場合 | ハンディ、POSレジ、KDSの順で優先表示 |

::: warning 1 端末 1 用途を強く推奨
複数選択は技術的に可能ですが、**運用上は 1 端末 1 用途**（KDS のみ / POSレジ のみ / ハンディのみ）にしてください。KDS 画面表示中に会計が来て切替えたらピックアップ画面が消える、といった事故が起きます。
:::

### 2-4-4. ワンタイムトークンで端末を自動セットアップする

新しい KDS / レジ / ハンディタブレットを設置する時、手動でユーザー名・パスワードを入力する代わりに **ワンタイムトークン**で自動ログインできます。

#### 手順

1. 管理画面 → 【スタッフ管理】 → デバイスアカウントセクション
2. **【トークン発行】** ボタン
3. ダイアログで端末の「表示名」と「用途」を入力
4. セットアップ用URLが表示される
5. **URL を新しい端末で開く**（手動コピー、または QR にして読み取り）
6. POSLAが端末用アカウントを自動作成する
7. 用途に合わせて、KDS / レジ / ハンディの画面へ自動で移動する

#### 注意点
- セットアップURLは1回だけ使えます
- 期限が切れた場合は、もう一度【トークン発行】を押してください
- URLを店外や関係者以外に共有しないでください

### 2-4-5. スタッフアカウントとの違い

| 項目 | スタッフアカウント | デバイスアカウント |
|---|---|---|
| 想定用途 | 人間 1 人 | 端末 1 台 |
| シフト管理対象 | ✅ | ❌ |
| 人件費計算対象 | ✅ | ❌ |
| スタッフ評価レポート | ✅ | ❌ |
| GPS 打刻必須 | 店舗設定による | **常に必須**（セキュリティ） |
| 管理画面アクセス | 限定（予約・シフト 2 タブのみ） | ✗ 不可 |

## 2-5. シフトの作成と公開

シフトの作成は **「シフト管理」タブ** から行います（manager 以上）。

### 2-5-1. シフトを作る手順

1. 管理画面 → 上部タブ **【シフト管理】**
2. サブタブ **【週間カレンダー】**
3. 画面上部の日付切替ボタンで該当週を表示
4. スタッフ名 × 日付のグリッドが表示される
5. セル（スタッフ × 日）をタップ
6. シフト入力モーダル:

| 項目 | 説明 |
|---|---|
| **開始時刻** | 例: `10:00` |
| **終了時刻** | 例: `18:00` |
| **休憩** | 分単位。6 時間超、8 時間超の勤務は公開前チェックでも警告される |
| **持ち場** | ホール、キッチン、ドリンク、洗い場など店舗で設定した持ち場。指定なしも可 |
| **メモ** | 任意（例: 「研修日」） |

7. **【保存】**
8. セルに色付きでシフトが表示される

### 2-5-2. 全員分入力できたら公開前チェック

週間カレンダーの上部に **公開前チェック** が表示されます。ここでは、確定前に以下をまとめて確認します。

| 確認項目 | 内容 |
|---|---|
| 時間帯ごとの人数不足 | テンプレート必要人数、過去 8 週間の同曜日・同時間帯売上、対象日の予約人数を使って必要人数を出し、予定人数と比較 |
| 持ち場不足 | テンプレートで持ち場を指定した時間帯について、予定シフトの持ち場数と比較 |
| 休憩未設定/不足 | 6 時間超は 45 分、8 時間超は 60 分を目安に警告 |
| 希望未提出 | 対象週の希望提出がない、または一部の日だけ未提出のスタッフを表示 |
| 未確認シフト | 確定したがスタッフがマイシフトで **確認** を押していないシフトを表示 |
| 予定人件費率 | 予定シフトの人件費目安を過去売上平均と比較。店舗設定の目標人件費率を超えると警告 |
| 労務リスク | 予定上の週 40 時間超、1 日 8 時間超、6 日以上連続勤務を表示 |
| 実績差分 | 過去日について、予定 vs 勤怠打刻の遅刻・早退・残業・打刻なしを表示 |
| ヘルプ候補 | 不足時間帯について、他店舗スタッフ候補を表示 |

赤または黄色の警告が残っていても、現場判断で公開できます。ただし、赤の警告は公開前に必ず内容を確認してください。

### 2-5-3. 公開する

1. 公開前チェックを確認
2. 必要に応じてシフト、休憩、持ち場を修正
3. 画面上部の **【確定する】** ボタン
4. 確認ダイアログ → **【OK】**
5. 以降、スタッフは **マイシフト** で自分のシフトを確認できる
6. スタッフは内容を見て **確認** を押す
7. 店長は公開前チェックの未確認一覧で、まだ確認していないスタッフを確認する

この操作だけでは外部メッセージ通知は送られません。希望未提出者への連絡は、公開前チェックの **リマインド文コピー** を使い、店舗の通常連絡手段で送ってください。

### 2-5-4. AI シフト提案を使う

過去の売上データとスタッフの希望から、最適なシフトを AI が自動生成する機能。

1. カレンダー画面上部の **【AI 提案】** ボタン
2. AI が数秒で全スタッフのシフト案を生成
3. 内容を画面で確認
4. **【全て採用】** ボタンで一括反映（既存の手動シフトは上書きされない）
5. 内容を調整し、公開前チェックを確認
6. **【確定する】** で公開

AI 提案を使わなくても、公開前チェック、必要人数表示、希望未提出、実績差分、予定人件費率、労務リスク、ヘルプ候補は動きます。

### 2-5-5. 持ち場を整える

持ち場は **シフト設定** で管理します。初期状態はホール、キッチンです。

店舗に合わせて、ドリンク、洗い場、レジ、仕込みなどを追加できます。

1. 管理画面 → 【シフト管理】 → **【シフト設定】**
2. 画面内の **持ち場** を見る
3. code と表示名を入力する
   - code: `drink`、`wash` など英数字・`_`・`-` の 20 文字以内
   - 表示名: 画面に出る名前（例: ドリンク、洗い場）
4. **追加** を押す
5. テンプレートや週間カレンダーの持ち場選択に出る

使わなくなった持ち場は **非表示** にします。過去シフトの履歴は残ります。

### 2-5-6. 目標人件費率を設定する

公開前チェックと集計サマリーの警告基準として、店舗の目標人件費率を設定できます。

1. 管理画面 → 【シフト管理】 → **【シフト設定】**
2. **目標人件費率** に数値を入力（例: 30）
3. 店舗デフォルト時給も未設定なら入力
4. **保存** を押す

公開前チェックの予定人件費率は、予定シフトの人件費を過去売上平均で割って計算します。予約人数は必要人数の判定に使いますが、予約金額として自動換算はしません。

## 2-6. 出退勤の打刻方法と確認

### 2-6-1. スタッフの打刻

スタッフ側の操作。

1. 管理画面（`/admin/`）に staff アカウントでログイン
2. デフォルト表示される「スタッフ画面」の **【🟢 出勤】** ボタンをタップ
3. **GPS 許可ダイアログ** が出たら **【許可】**
4. 店舗から半径 200m 以内（デフォルト）にいるか判定
5. OK なら **「出勤しました 10:05」** とメッセージ
6. 退勤時は **【退勤】** ボタン（赤）をタップ

### 2-6-2. 店長の勤怠確認

1. 管理画面 → 【シフト管理】 → サブタブ **【勤怠一覧】**
2. 日別にスタッフの打刻記録が一覧表示
3. 打刻漏れや時刻誤りがある場合は **【修正】** で手動修正
4. 公開前チェックと集計サマリーで、予定（シフト）vs 実績（打刻）の遅刻・早退・残業・打刻なしを確認

---

## 2-7. GPS 打刻の設定

### 2-7-1. 店舗の GPS 設定

1. 管理画面 → 【シフト管理】 → **【シフト設定】** サブタブ
2. 以下を入力:

| 項目 | 説明 |
|---|---|
| **店舗緯度** | Google Maps で店舗の住所を検索 → 右クリック → 座標をコピー（例: 35.6585） |
| **店舗経度** | 同上（例: 139.7454） |
| **GPS 許容半径** | デフォルト 200m |
| **GPS 打刻必須** | トグル ON/OFF |

3. **【保存】**

### 2-7-2. 店舗座標の調べ方（Google Maps）

1. [Google Maps](https://www.google.com/maps) を開く
2. 店舗の住所を検索
3. 地図上の店舗アイコンを右クリック
4. **「座標をコピー」** または表示された緯度経度の数字をメモ
5. 例: `35.6585, 139.7454` → 緯度 35.6585 / 経度 139.7454

### 2-7-3. device ロールは常に GPS 必須

スタッフは店舗設定により GPS 打刻の必須/任意が変わりますが、**デバイスアカウントは常に GPS 確認が必要**です。店舗備品の端末が店外に持ち出された場合のリスクを下げるためです。

### 2-7-4. GPS がずれる場合

- 室内だと GPS 精度が落ちる（5〜50m ずれることも）
- ビル内の店舗は特に誤差が大きい
- 対処: 許容半径を **300〜500m** に広げる
- それでもダメなら **GPS 必須を OFF** にして、代わりに監査ログで出退勤を確認

## 2-8. AI シフト提案の詳細

### 2-8-1. 仕組み

- 過去 30〜90 日の売上データ分析
- 各時間帯の必要人員を推定
- スタッフごとの希望シフト（シフト希望提出機能）との合致
- 労基法遵守（週40時間以下、1日8時間以下）

### 2-8-2. 使い方

1. シフト希望を集める（スタッフが 【希望提出】 サブタブで送信）
2. 【週間カレンダー】で **【AI 提案】** ボタン
3. 提案内容を確認
4. 修正が必要なら手動調整
5. 公開前チェックを確認
6. **【確定する】** で確定

### 2-8-3. 注意点

- AI 提案は「あくまで案」。最終判断はマネージャー
- 新人スタッフの研修シフト等、AI には見えない要素は手動調整
- 既存の手動シフトは上書きされない

## 2-9. 複数店舗のシフトヘルプ

owner / manager のみが使える機能。他店舗から人員を借りる仕組み。

### 2-9-1. ヘルプ要請を送る

1. 管理画面 → 【シフト管理】 → **【当日・ヘルプ】** サブタブ
2. **【+ ヘルプ要請】** ボタン
3. 入力:
   - 送信先店舗（同テナント内の他店舗から選択）
   - 希望日時
   - 持ち場（ホール / キッチン / 店舗で追加した持ち場 / 指定なし）
   - 必要人数
   - 理由・メモ
4. 入力欄の下に候補スタッフが表示される
5. 候補を見て、依頼先店舗と時間帯が妥当か確認する
6. **【送信】**
7. 相手店舗のマネージャーは当日・ヘルプタブで要請を確認する

### 2-9-2. ヘルプ要請を受ける / 承認する

1. 相手店舗のマネージャーが【当日・ヘルプ】サブタブで要請を確認
2. 承認できる場合 → スタッフを選択 → **【承認】**
3. 選ばれたスタッフは要請元の店舗で勤務できるようになる（その日だけ）

### 2-9-3. スタッフが他店舗で働く

1. スタッフ本人は管理画面で自分のシフトを確認する
2. 派遣先店舗のハンディ、KDS、POSレジは、派遣先店舗が用意したデバイスアカウント端末を使う
3. 会計など人の責任を記録する操作は、スタッフ本人の担当スタッフ PIN を入力する
4. その日は派遣先店舗のシフトで稼働

## 2-9-b. 当日の不足・交代・欠勤を見る

店長が当日見る画面は **当日・ヘルプ** です。ここに「今日対応が必要なもの」がまとまります。

画面の最上部には **今日の店長チェック** が出ます。これは、下の一覧を全部読まなくてもよいように、未打刻、未対応申請、欠勤穴埋め、時間帯不足、休憩不足、未完了作業、変更未確認だけを先頭に集める欄です。

| 店長チェック | 先に見る理由 | 対応 |
|---|---|---|
| 遅刻/未打刻 | 予定開始後に打刻がない | 本人確認、打刻漏れ修正、欠勤判断 |
| 交代・欠勤未対応 | 承認待ちで現場人数が確定していない | 承認または却下 |
| 欠勤穴埋め | 欠勤だけ承認され、代わりが入っていない | 穴埋めボタンから候補、募集、他店舗ヘルプへ進む |
| 時間帯不足 | 必要人数に足りない時間がある | シフト追加、店長フォロー、ヘルプ要請 |
| 休憩不足 | 休憩が未設定または不足している | 休憩時間を調整 |
| 未完了作業 | 今日の担当作業が残っている | 担当者へ声かけ、完了確認 |
| 変更未確認 | 変更済みシフトをスタッフがまだ見ていない | 通常連絡手段で声かけ |

未打刻の行には、店長がその場で押せるボタンがあります。

| ボタン | 使う場面 | 結果 |
|---|---|---|
| 連絡済み | 電話や通常連絡手段で本人に連絡した | 未打刻のまま、対応済み表示が残る |
| 遅刻連絡あり | 本人から遅刻連絡があった | 出勤するまで未打刻のまま、遅刻連絡あり表示が残る |
| 欠勤にする | 本人が来られないことが確定した | 承認済み欠勤へ移り、未打刻から外れる |
| 穴埋め | 欠勤確定後すぐ人を探したい | 欠勤処理後、候補確認、空きシフト募集、他店舗ヘルプへ進める |

| 表示 | 意味 | 対応 |
|---|---|---|
| 開始済み未打刻 | 予定開始から 15 分超過しても出勤打刻がない | 本人確認、打刻漏れ修正、必要なら欠勤扱い |
| 未対応の交代/欠勤 | スタッフから申請が来ているが、店長がまだ対応していない | 承認または却下 |
| 承認済み欠勤 | 欠勤だけ承認され、交代スタッフが入っていない | 呼べる候補または店舗間ヘルプで穴埋め |
| 時間帯不足 | テンプレート必要人数に対して当日の予定人数が不足 | シフト追加、ヘルプ要請 |
| 今日呼べる候補 | 今日シフトが入っていない同店舗スタッフ | 電話や通常連絡手段で確認 |

### スタッフから交代依頼が来た場合

1. 【当日・ヘルプ】 → **交代・欠勤申請** を見る
2. 対象行の **承認** を押す
3. 交代スタッフ候補から選ぶ
4. **承認** を押す
5. 元のシフト担当者が交代スタッフに差し替わる

候補がいない場合、交代なしで承認することもできます。その場合は欠員として当日不足に残ります。

### スタッフから欠勤連絡が来た場合

1. 【当日・ヘルプ】 → **交代・欠勤申請** を見る
2. 理由を確認する
3. 交代スタッフを入れられる場合は選んで承認
4. 交代なしなら、そのまま承認
5. 必要なら今日呼べる候補、または店舗間ヘルプを使う

欠勤連絡を承認しても、外部メッセージ通知は自動送信されません。本人への連絡や最終確認は店舗の通常連絡手段で行ってください。

## 2-9-c. 空きシフト募集で穴を埋める

急な欠勤、予約増、仕込み増で人が足りない時は、いきなり週間カレンダーへ誰かを入れるのではなく、まず **空きシフト募集** を使えます。

### 店長が募集を作る

1. 管理画面 → 【シフト管理】 → **【当日・ヘルプ】**
2. **現場シフト運用** の **【募集作成】**
3. 入力:
   - 日付
   - 開始時刻
   - 終了時刻
   - 休憩
   - 持ち場
   - 必要スキル（任意）
   - メモ
4. **【作成】**

### スタッフが応募する

1. スタッフが管理画面 → 【シフト管理】 → **【マイシフト】**
2. **募集シフト** を見る
3. 入れる募集の **【応募】**
4. 店長へのメモがあれば入力
5. 応募済みになる

### 店長が承認する

1. 【当日・ヘルプ】 → 現場シフト運用 → 空きシフト募集
2. 応募者の **【承認】**
3. POSLA がそのスタッフのシフトを自動作成
4. 募集は締め切りになり、他の応募は却下される

注意点:

- 応募だけではシフトになりません。店長の承認が必要です。
- 同時間帯に既存シフトがあるスタッフは応募・承認できません。
- 必要スキル付き募集は、そのスキルを持つスタッフだけが応募できます。
- 承認後のシフトは通常の週間カレンダーで修正できます。

## 2-9-d. 今日の担当作業を割り振る

「誰がレジ開けをやったか」「中間清掃は誰か」「閉店清掃が終わったか」を口頭だけで回すと抜けます。担当作業は、1 タップ完了で現場の迷いを減らすための機能です。

### 店長が担当作業を追加する

1. 【シフト管理】 → **【当日・ヘルプ】**
2. 現場シフト運用 → **今日の担当作業**
3. **【作業追加】**
4. 作業テンプレート、担当スタッフ、メモを入力
5. **【追加】**

初期テンプレート:

| 作業 | 用途 |
|---|---|
| レジ開け | 開店時のレジ準備 |
| 開店前チェック | 店内・端末・予約確認 |
| 中間清掃 | 営業中の清掃 |
| レジ締め | 閉店時のレジ締め |
| 閉店清掃 | 閉店作業 |

### スタッフが完了する

1. スタッフが 【マイシフト】 を開く
2. **担当作業** を見る
3. 終わった作業の **【完了】**
4. 間違えたら **【戻す】**

端末アカウントではなく、スタッフ本人のアカウントでマイシフトを開いて完了します。ハンディ/KDS/レジの device アカウントは、シフトや作業完了の「人」としては扱いません。

## 2-9-e. スキル不足・休憩不足を見る

当日・ヘルプの現場シフト運用には、次の不足がまとまります。

| 表示 | 店長が見ること | 対応 |
|---|---|---|
| スキル不足 | 持ち場に必要なスキル人数が足りない | スキル保有者を入れる、店長が入る、持ち場条件を見直す |
| 休憩不足 | 6 時間超/8 時間超の休憩が足りない | 休憩時間を入力、休憩に出せる時間を作る |
| 経験者なし | その時間帯が新人・勤務実績少なめの人だけ | 経験者を入れる、店長がフォローする |
| 人件費率超過 | 今日の予定人件費が売上に対して重い | シフト時間、予約、売上見込みを確認 |
| 希望未提出 | 直近 7 日の希望が未提出 | 通常連絡手段で提出依頼 |

POSLA は不足を見つけますが、勝手にシフトを入れ替えません。飲食店では「今日は予約がある」「新人研修中」「店長がキッチンに入る」など画面外の事情があるため、最終判断は店長が行います。

## 2-10. シフト希望の提出（スタッフ側）

スタッフが自分の希望シフトを提出する機能。

### 手順

1. スタッフが管理画面にログイン
2. 【シフト管理】 → サブタブ **【希望提出】**
3. カレンダー表示
4. 希望日をタップ → 時間帯入力（例: 10:00〜15:00 希望）
5. **【保存】**
6. マネージャーがシフト作成時に参考にする

## 2-11. シフト集計（労働時間・人件費）

### 手順

1. 【シフト管理】 → サブタブ **【集計サマリー】**
2. 期間を指定（日次 / 週次 / 月次）
3. 以下が表示:

| 指標 | 説明 |
|---|---|
| 総労働時間 | 全スタッフの合計実働時間 |
| 人件費 | `hourly_rate × 実働時間` の合計 |
| 人件費率 | 実績人件費 ÷ 期間内の実売上 |
| スタッフ別労働時間 | 個別の実働時間 |
| 労基法警告 | 週 40 時間超過、月 80 時間残業等があればアラート |
| 深夜労働時間 | 22:00〜05:00 の時間（深夜割増対象） |

## 2-12. スタッフ向けマイシフト閲覧

スタッフが自分の確定シフトを見る機能です。自分以外のシフトは見えません。

1. スタッフが管理画面にログイン
2. 【シフト管理】 → サブタブ **【マイシフト】**
3. 自分のシフトだけがカレンダーで見える
4. 問題なければ **確認** を押す
5. ステータスが **確認済** になる

### 2-12-1. 交代依頼を出す

1. 【マイシフト】で対象シフトを探す
2. **交代依頼** を押す
3. 候補スタッフが表示される
4. 候補を選ぶ、または候補を指定しない
5. 店長へのメモを入力
6. **送信**

申請後は対象シフトに **申請中** と表示されます。店長が承認するまでは、シフト担当者は変わりません。

### 2-12-1-b. シフトが変更された場合

スタッフが一度 **確認済** にしたシフトでも、店長があとから時間、休憩、持ち場、メモを変更すると、その人だけ **変更あり** に戻ります。

スタッフの操作:

1. 【マイシフト】を開く
2. **変更あり** のシフトを見る
3. `10:00 → 10:30` のような変更前後を確認する
4. 問題なければ **確認** を押す

店長が「再確認」ボタンを押す必要はありません。POSLA が変更を検知して自動で対象スタッフだけに出します。

### 2-12-2. 欠勤連絡を出す

1. 【マイシフト】で対象シフトを探す
2. **欠勤連絡** を押す
3. 理由や店長へのメモを入力
4. 送信

欠勤連絡を出しただけでは、シフトが自動削除されるわけではありません。店長が当日・ヘルプで承認または却下します。

### 2-12-3. 欠勤が承認された後の穴埋め

店長が欠勤を **交代なし** で承認した場合、当日・ヘルプに **穴埋め** ボタンが出ます。

1. 【当日・ヘルプ】 → **交代・欠勤申請**
2. 承認済み欠勤の **穴埋め** を押す
3. 同店舗候補を見る
4. 候補がいない場合:
   - **空きシフト募集を作成**
   - または **他店舗ヘルプ要請**

空きシフト募集を作る場合は、欠勤したシフトと同じ日付、時間、持ち場が自動で入ります。店長は内容を確認するだけで済みます。

## 2-13. よくある質問

**Q1. 既存のスタッフアカウントを KDS タブレットでログインしっぱなしにしてはダメ？**

A1. **強く非推奨**。シフト・勤怠・スタッフ評価レポートが全部そのスタッフ名義になってしまい、データが壊れます。必ず device アカウントを作って運用してください。

**Q2. device アカウントは何個まで作れる？**

A2. 上限なし。店舗の KDS 台数 + レジ台数 + ハンディ台数分作ってください。例: 3 店舗 × 各 KDS 2 台・レジ 1 台・ハンディ 3 台 = 18 個。

**Q3. device アカウントはシフト管理に影響しない？**

A3. 影響しません。デバイスアカウントはシフト・勤怠・人件費・スタッフ評価の対象外です。

**Q4. 退職したスタッフのアカウント、削除すべき？**

A4. 無効化推奨（トグル OFF）。過去の勤怠・注文履歴を守るため。完全削除は非推奨。

**Q5. GPS 打刻の精度が悪い**

A5. 室内では GPS が 5〜50m ずれることがあります。許容半径を 300〜500m に広げるか、GPS 必須を OFF にして監査ログで確認してください。

**Q6. スタッフが勤怠打刻を忘れた**

A6. マネージャーが【勤怠一覧】の該当日を **【編集】** で手動追加できます。ただし監査ログには「編集されたシフト」と記録されます。

**Q7. AI シフト提案の質が悪い**

A7. 過去データの蓄積が少ないと AI 精度が低くなります。最低 30 日分のデータが溜まってから使うのが現実的です。

**Q8. 交代依頼を承認したら、元のスタッフのシフトはどうなる？**

A8. 交代スタッフを選んで承認すると、対象シフトの担当者が交代スタッフに差し替わります。交代なしで承認した場合は欠勤だけが承認され、当日の不足として残ります。

**Q9. マイシフトの「確認済」は外部メッセージの既読ですか？**

A9. 違います。POSLA のマイシフト画面でスタッフが **確認** を押した状態です。LINE やメールの既読とは連動しません。
