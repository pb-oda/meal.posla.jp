# Part 5 — 予約管理（L-9 機能）

このページでは、お客様からの予約受付・管理を、**実際にマウスで押すボタン** を 1 つずつ順番に説明します。

::: tip この章の対象
- **24 章「予約管理」** の完全詳細版（1508 行）もありますが、このページは **よく使う操作** を小学生でもわかるレベルで説明したクイックガイドです
- 飲食店で **電話予約を受ける** / **お客様がオンラインで予約を入れる** / **予約客が来店したときの着席** / **no-show 管理** の一連を網羅
:::

---

## 5-0. 予約管理とは？

POSLA の予約管理は、お客様が来店する前日までに「何時に何人で行きます」と申し込む仕組みです。以下の機能があります。

1. **オンライン予約**: お客様が自分のスマホで予約 LP（ランディングページ）から予約できる
2. **電話予約**: マネージャー以上がハンディまたは管理画面から手動で予約を登録できる（staff には管理画面の予約タブは表示されない、2026-04-19 仕様変更）
3. **AI 予約**: お客様が自由文（例: 「明日 19 時 2 名田中」）で予約できる、AI が解析
4. **予約金徴収**: お客様にカードを登録してもらい、来店したら解除、no-show で請求
5. **自動着席**: 予約客が QR を読むとスタッフ操作なしで自動着席
6. **ガント台帳**: 時間 × テーブルのマトリクスで予約状況を見える化

---

## 5-1. 予約 LP（お客様向けページ）の URL 取得と掲載

### 5-1-1. URL / QR を確認する

1. 管理画面にログイン
2. 上部タブから **【店舗設定】** をクリック
3. サブタブから **【店舗設定】** を選択
4. 画面を下にスクロール
5. **「📅 予約LP (L-9)」** セクションが見える
6. 以下の 4 つのボタンと情報が表示されている:

| 要素 | 用途 |
|---|---|
| **URL 表示** | `https://eat.posla.jp/public/customer/reserve.html?store_id=...` のような URL |
| **QR コード画像** | 上記 URL を QR に変換したもの |
| **【URL をコピー】** | クリップボードに URL を保存（LINE / メールで送信用） |
| **【新タブで開く】** | 動作確認用 |
| **【QR 画像コピー】** | Instagram 等に貼る用 |
| **【🖨 印刷】** | A4 ポスターとして印刷 |

### 5-1-2. どこに掲載する？

- **Instagram / X / Facebook** などの SNS プロフィール欄に URL 貼り付け
- **Google マイビジネス** の「予約」リンク
- **お店の公式ホームページ**
- **店頭ポスター**（印刷ボタンで出力して店頭に貼る）
- **LINE 公式アカウント** のメッセージ
- **食べログの口コミ返信** に予約 URL を入れる

### 5-1-3. お客様が URL から予約する流れ（5 ステップ）

1. お客様が URL / QR から予約 LP を開く
2. **日付選択**（カレンダーから希望日をタップ）
3. **人数選択**（1〜20 名程度）
4. **時刻スロット選択**（空きのある時刻のみ緑で表示）
5. **お客様情報入力**（名前・電話・メール・特記事項）
6. **確認画面** → **【予約する】** ボタン
7. 予約金を設定している店舗なら Stripe Checkout で決済へ
8. 予約完了メールが自動送信される

---

## 5-2. ガント台帳の見方

### 5-2-1. 開く手順

1. 管理画面の上部タブから **【予約管理】** をタップ
2. サブタブから **【予約台帳】** を選択
3. 画面が開く
4. 画面上部に **「今日」「‹」「日付入力」「›」** のボタンで日付切替
5. 本体は **時間（縦軸）× テーブル（横軸）** のマトリクス

### 5-2-2. 画面構成

```
┌────────────────────────────────────────────┐
│ 日付: 2026-04-19  [‹] [今日] [›]  予約数: 8  │
├────────┬────────┬────────┬────────┬────────┤
│ 時刻  │  T01   │  T02   │  T03   │  T04   │
├────────┼────────┼────────┼────────┼────────┤
│ 17:00 │  🟦田中 │        │        │        │
│ 17:30 │  (2名) │  🟦鈴木 │        │        │
│ 18:00 │        │  (4名) │        │ 🟨決済 │
│ 18:30 │        │        │  🟩在  │  待ち │
│ 19:00 │  🟥no  │        │  席中  │        │
│       │  show  │        │        │        │
└────────┴────────┴────────┴────────┴────────┘
```

### 5-2-3. 予約ブロックの色分け

| 色 | 意味 | 実装 |
|---|---|---|
| 🟦 青 | 確定（`confirmed`） | 通常の予約 |
| 🟨 黄 | 予約金決済待ち（`pending`） | Stripe Checkout を閉じてしまった客など、決済未完 |
| 🟩 緑 | 着席中（`seated`） | QR 読み込み済み / 手動着席済み |
| 🟥 赤 | no-show（`no_show`） | 時刻過ぎても来店なし |
| ⚫ グレー | キャンセル（`cancelled`） | お客様 or 店舗がキャンセル |

### 5-2-4. ブロックをクリックすると？

予約の詳細モーダルが開き、以下の操作ができます:

- **【🪑 着席】** — 手動で着席状態にする
- **【❌ no-show】** — no-show として記録（予約金があれば請求）
- **【✏️ 変更】** — 時刻・人数・メモ変更
- **【🗑 キャンセル】** — 予約を取り消し

## 5-3. 電話で予約を受けたときの登録手順

お客様が電話で予約を入れたい場合、スタッフが手動で登録します。

### 5-3-1. ガント画面から登録

1. 管理画面 → 【予約管理】 → 【予約台帳】
2. 画面上部の **【+ 新規予約】** ボタンをクリック
3. モーダルが開く
4. 以下を入力:

| 項目 | 必須 | 内容 |
|---|---|---|
| 名前 | ✅ | 電話で聞いた氏名 |
| 電話番号 | ✅ | 10〜11 桁、ハイフンなし |
| メール | — | 任意。予約確認メール送信用 |
| 日時 | ✅ | 予約日と時刻 |
| 人数 | ✅ | 来店予定人数 |
| テーブル割当 | — | チェックボックスで指定テーブル選択（未指定なら後で） |
| メモ | — | 「アレルギー: 卵」「誕生日」等 |
| タグ | — | VIP / 大事な接待 等のマーカー |

5. **「予約金徴収をスキップ」** チェックボックス（電話受付では通常 ON）
6. **【作成】** ボタン
7. 予約が作成され、ガント台帳にブロックが表示される

::: tip 電話予約では予約金徴収をスキップする理由
電話越しにお客様にカード番号を伝えてもらうのは現実的でないため、電話予約では予約金徴収をスキップするのが普通です。ただし大型パーティー（10 名以上）などは、念のため予約金を徴収するよう検討してください。
:::

### 5-3-2. ハンディから登録

外出先や混雑時はハンディからも予約登録できます。

1. ハンディ画面の **【📅 予約】** タブ
2. 右下の **【+ 新規】** ボタン
3. 同じくモーダルで入力
4. 保存

## 5-4. Walk-in（予約なしで来店したお客様）の登録

予約なしのお客様をすぐに着席・注文開始させる場合。

### 手順

1. ハンディ画面の **【テーブル状況】** タブ
2. 空きテーブル（T03）をタップ
3. 画面に **【+ Walk-in】** ボタン → タップ
4. 人数入力（例: 2 名）
5. **【着席】**
6. テーブルがセッション確立、メニュー注文開始できる

Walk-in 客はシステム上は「予約なしの即着席」として記録されます。

---

## 5-5. 予約客が来店したとき（自動着席の仕組み）

POSLA では予約客が店に来て QR を読むと、**スタッフ操作なしで自動的に着席状態**になります（L-9 新機能）。

### 5-5-1. 自動着席の条件（3 つすべて必要）

1. **テーブル割当済み**（予約登録時に特定のテーブルを指定してあること）
2. **予約時刻 ±30 分以内**（18:00 予約なら 17:30〜18:30 の間に QR を読む）
3. **ステータスが `confirmed` または `pending`**

### 5-5-2. 流れ

1. 予約客が来店
2. スタッフが該当テーブル（T03）に案内
3. お客様が T03 の QR をスマホで読む
4. サーバー側で自動マッチング（予約情報と照合）
5. マッチすれば即セッション確立 → お客様のメニュー画面が開く
6. ガント台帳のブロックが 🟦 青 → 🟩 緑（着席中）に変わる

**お客様が注文を始められるまでスタッフの作業は 0 秒**です。

---

## 5-6. 自動マッチングが成立しないときの手動着席

以下の場合、自動着席が失敗します:

- テーブル未割当で予約された
- 予約時刻を 30 分以上過ぎて来店
- お客様が違うテーブルの QR を読んだ

### 手動着席の手順

1. ガント台帳で該当予約ブロックをタップ
2. モーダルに **【🪑 着席】** ボタン
3. タップ
4. テーブルを選択（未割当の場合）
5. 着席状態に変わる

---

## 5-7. no-show の登録

予約時刻を過ぎても来店がない場合の対応。

### 5-7-1. タイミング

通常、予約時刻から 30 分経過しても来ない場合に no-show として処理します。

### 5-7-2. 手順

1. ガント台帳で該当予約ブロックをタップ
2. モーダルに **【❌ no-show】** ボタン
3. タップ
4. 確認ダイアログで **【OK】**
5. ステータスが 🟥 赤（no-show）に変わる

### 5-7-3. 予約金の自動請求

予約金ありの店舗で no-show を登録すると、**自動的に Stripe で予約金が請求**されます（事前に カード `authorize` したものを `capture` へ切替）。お客様のカードに課金される日は Stripe の処理に従います（通常数日以内）。

---

## 5-8. 予約の変更・キャンセル

### 5-8-1. お客様側からの変更

お客様が予約確認メールに記載された URL を開くと、自分で変更・キャンセルできます。

- 時刻変更
- 人数変更
- メモ変更
- キャンセル

### 5-8-2. 店舗側からの変更

1. ガント台帳で予約ブロックをタップ
2. **【✏️ 変更】** または **【🗑 キャンセル】**
3. 変更内容を入力 or キャンセル確認

---

## 5-9. コース予約の管理

コース料理（懐石・フランス料理のプリフィックス等）の予約設定。

### 5-9-1. コース定義

1. 【予約管理】 → 【コース管理】サブタブ
2. **【+ コース追加】**
3. 以下を入力:
   - コース名（例: 「スペシャルディナーコース」）
   - 価格（例: ¥8,800）
   - 所要時間（例: 120 分）
   - 人数範囲（例: 2〜8 名）
   - 含まれる品目（カテゴリ別に設定）
4. **【保存】**

### 5-9-2. お客様視点

予約 LP でコース選択肢として表示される → お客様が予約時に選択 → 来店時はコースの順番通りに料理が流れる（KDS でコース自動発火）

---

## 5-10. 顧客台帳（リピーター管理）

過去に来店したお客様の情報を記録する機能。

### 手順

1. 【予約管理】 → 【顧客台帳】サブタブ
2. 検索フィルタ（名前・電話・タグ）
3. 顧客カードをタップ → 詳細画面
4. 表示される情報:
   - 来店履歴（直近 50 件）
   - 好み / アレルギー
   - タグ（VIP / リピーター / ブラックリスト）
   - 内部メモ（スタッフ間の申し送り）
   - 累計来店回数・累計金額

---

## 5-11. VIP / ブラックリストの設定

### 5-11-1. VIP 設定

1. 顧客詳細画面 → **VIP** チェックボックス ON
2. 保存
3. 予約時にガントで **★VIP** バッジ表示
4. スタッフが特別対応しやすくなる

### 5-11-2. ブラックリスト（manager 以上のみ）

2 回以上 no-show を出した客や迷惑客をブロック。

1. 顧客詳細画面 → **ブラックリスト** チェックボックス ON
2. **理由**を入力（例: 「過去 3 回 no-show」）
3. 保存
4. 以降、この電話番号からの予約 LP からの申込は自動拒否

---

## 5-12. 予約金（デポジット）の設定

大人数予約や、no-show 多発店舗向け。

### 5-12-1. 有効化

1. 【予約管理】 → 【予約設定】サブタブ
2. **「予約金を徴収する」** チェックボックス ON
3. 1 人あたりの金額を入力（例: ¥2,000 / 人）
4. 徴収開始人数を入力（例: 6 名以上）
5. **【保存】**

::: warning Stripe 設定済みであることが前提
予約金徴収は Stripe 経由で処理するため、**先に Stripe 設定（21 章）を完了**してください。Stripe 未設定だと予約金 ON にできません。
:::

### 5-12-2. 予約金の動き

| イベント | POSLA の動作 |
|---|---|
| 予約完了時 | Stripe で **authorize**（カード認証のみ、まだ課金されない） |
| 来店・着席時 | authorize を **release**（課金されずに解除） |
| no-show 時 | authorize を **capture**（課金確定） |
| 期限内キャンセル | release（課金なし） |
| 期限外キャンセル | capture（課金） |

## 5-13. AI 予約（お客様の自由文入力）

お客様が「明日 19 時 2 名田中」のように自由文で予約できる機能。

### 使い方

1. お客様が予約 LP を開く
2. 画面右下の **【💬 AI で予約】** ボタン
3. 自由文で希望を入力
4. Gemini AI が解析 → 日時・人数・名前を抽出
5. 確認画面で間違いがないかチェック
6. 【予約する】

### 店舗側設定

- 【予約管理】 → 【予約設定】 → **「AI チャット予約」** トグル ON/OFF で有効化

実装: `api/customer/reservation-ai-parse.php`（レートリミットあり）

## 5-14. 予約リマインダーメール

予約日時の前にお客様にメール通知。

### 設定

1. 【予約管理】 → 【予約設定】
2. **24 時間前リマインダー** トグル ON
3. **2 時間前リマインダー** トグル ON
4. 【保存】

### 動作

POSLA 運営側で cron が稼働し、自動送信されます（初回セットアップ時に運営が対応）。

## 5-15. よくある質問

**Q1. お客様が予約後に来ないとき、どう対応する？**

A1. 時刻から 30 分経ったらガントで予約ブロックをタップ → **【❌ no-show】**。予約金設定があれば自動で Stripe で請求されます。

**Q2. 大人数予約（10 名以上）はどうする？**

A2. 複数テーブル連結予約が可能（最大 3 テーブル）。予約登録時にテーブル複数チェックで割当。予約金を徴収するのがおすすめ（no-show リスクが高いため）。

**Q3. AI 予約で誤解析されたら？**

A3. お客様が確認画面で修正できます。送信後は店舗側で修正も可。

**Q4. 電話予約でテーブルを決めずに予約受けた。後からどう割当てる？**

A4. ガント画面で予約ブロック →【✏️ 変更】でテーブル追加。または、お客様が来店時に手動着席でテーブル指定。

**Q5. 予約金を取りたくないが、no-show が多くて困る**

A5. ブラックリスト機能で問題客をブロックできます。2 回以上 no-show の客は電話番号でブラック登録。
