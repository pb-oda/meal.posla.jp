# Part 1 — 毎日の営業（開店〜閉店）

このページは、**朝お店を開ける時から夜閉める時まで**、POSLA を使って 1 日を運営する流れを、**指の動き 1 つずつ** 順番に書いています。

::: tip このページの使い方
- **役割ごとに読むページが違います**:
  - **キッチンスタッフ（厨房）**: 1-1-1、1-3、1-7、1-9 を中心に
  - **ホールスタッフ（フロア）**: 1-2、1-4、1-6 を中心に
  - **レジ担当**: 1-1-2、1-5、1-9-1 を中心に
  - **店長・オーナー**: 全部読んで把握してください
- **初めての日は全部を上から読む** と流れが分かります
- 分からない言葉は Part 0 の「0-7. 画面の見方」と「0-8. ロール」を見直してください
:::

## 1-1. 開店準備（営業時間の 15〜30 分前）

お店の扉を開ける前に、POSLA の機材を起動してログインします。

### 1-1-1. KDS（キッチンディスプレイ）の電源を入れてログインする

**対象者**: キッチンスタッフ / 店長

::: tip 標準環境
KDS は **Android タブレット 10 インチ以上 + Google Chrome 最新版** を標準環境としています。iPad/Safari は音声コマンド・通知音が安定しないためサポート対象外です（詳細は「23. KDSデバイス端末ガイド」）。
:::

#### 必要なもの
- キッチンに設置されている **Android タブレット**（10 インチ以上、Chrome 最新版インストール済み）
- Wi-Fi が繋がっていること
- オーナー/店長から教えてもらったキッチン用ログイン情報（例: ユーザー名 `kds-shibuya-01`）

#### 手順
1. キッチンの壁やカウンターに設置されているタブレットを確認
2. タブレットの **電源ボタン**（本体側面または上部の細長いボタン）を 1 秒押す
3. 画面が明るくなる
4. ロック画面（時計だけの画面）が出たら、上にスワイプ（指でなぞる）して解除
5. ホーム画面で **「Chrome」** アイコン（赤・黄・緑・青の丸）をタップして起動
6. 画面上部の **アドレスバー**（細長い白い枠）をタップ
7. 次の URL を半角で入力（または事前にブックマーク済みなら選ぶだけ）:
   ```
   /kds/
   ```
8. キーボードの **「開く」** または **「Enter」** をタップ
9. 「管理画面ログイン」と書かれた画面が開く
10. **ユーザー名** 欄にキッチン用ログイン名（例: `kds-shibuya-01`）を入力
11. **パスワード** 欄にパスワードを入力
12. 青い **【ログイン】** ボタンをタップ

#### ログイン成功後の動作
- **複数店舗所属の場合**: 店舗選択のオーバーレイ（小さなウィンドウ）が表示される → 該当店舗をタップ
- **黒い画面「音声を有効にする」というオーバーレイが表示**:
  - **【タップして開始】**（黒いボタン）を必ず 1 回タップ
  - これをしないと **新規注文の通知音が鳴りません**
  - ブラウザの自動再生制約で、音声を鳴らすには 1 回ユーザー操作が必要（Android Chrome / iOS Safari 共通）
  - このタップで、対応端末では **画面が暗くなりにくくする補助機能（Wake Lock）** も有効になります（KDSヘッダー右に「画面維持: 有効」と表示）。端末の省電力設定によっては「画面維持: 端末設定を確認」と表示されることがあります。詳細は「23. KDSデバイス端末ガイド」
- タップ後、KDS カンバン画面（3 つの縦カラムに分かれた画面）が表示される

#### KDS 画面の見方（基本）
画面が以下の 3 つの縦カラムに分かれています:

```
┌──────────┬──────────┬──────────┐
│  受付    │ 調理中   │ 提供待ち │
│ (pending)│(preparing)│(ready)  │
│          │          │          │
│[注文 1]  │[注文 3]  │[注文 5]  │
│[注文 2]  │[注文 4]  │          │
│          │          │          │
└──────────┴──────────┴──────────┘
```

- **受付**: お客様が注文送信したばかりで、まだ調理を始めていない注文
- **調理中**: 「調理開始」ボタンを押して調理している注文
- **提供待ち**: 調理完了（「完成」ボタンを押した）して、ホールが取りに来るのを待っている注文

各カラムの上に数字（件数）が表示されます。カラムに注文カードが並びます。

#### カラムの色分け
- **受付**: 左枠オレンジ（`#e65100`）
- **調理中**: 左枠黄色（`#f9a825`）
- **提供待ち**: 左枠緑（`#2e7d32`）

#### 時間経過の色警告
- **注文から 10 分経過**: カードが警告色（オレンジ背景）に変わる
- **注文から 20 分経過**: カードが危険色（赤背景）に変わり、点滅する

#### ログインが失敗した場合

| 画面表示 | 原因 | 対処 |
|---|---|---|
| 「ユーザー名またはパスワードが正しくありません」 | 入力ミス | 大文字小文字、0 と O、1 と l の見間違い確認 |
| 「このアカウントは無効化されています」 | 店長がアカウントを停止した | 店長に連絡 |
| 「通信エラーが発生しました」 | Wi-Fi が切れている | Wi-Fi アイコンを確認、必要ならルーター再起動 |
| 画面真っ白 | ブラウザの問題 | Chrome を閉じて再起動、またはキャッシュクリア |
| 「店舗外からのアクセス」赤いオーバーレイ | GPS チェックで店外判定 | GPS を許可し、店内にいることを確認 |

### 1-1-2. POSレジの電源を入れてログインする

**対象者**: レジ担当 / 店長

#### 必要なもの
- レジカウンターに設置されているタブレット
- 店舗契約のカード端末・QR決済端末（カード/QR決済を受ける場合）
- レジ用ログイン情報（例: `register-shibuya-01`）

#### 手順
1. レジタブレットの電源を入れる
2. Chrome を起動
3. 以下の URL を開く（ブックマーク推奨）:
   ```
   /kds/cashier.html
   ```
4. レジ用ユーザー名・パスワードでログイン
5. 複数店舗なら店舗を選択
6. 「音声を有効にする」オーバーレイで **【タップして開始】** をタップ
7. POSレジ画面が表示される

#### POSレジ画面の見方
画面が 2 つの縦カラムに分かれています:

```
┌────────────────┬────────────────────┐
│ 会計待ちテーブル │   会計画面          │
│ (左カラム)      │   (右カラム)        │
│                │                     │
│ [T01 ¥3,400]   │  テーブル未選択時は │
│ [T02 ¥5,800]   │  「左から会計する   │
│ [T03 ¥2,200]   │  テーブルをタップ」 │
│                │                     │
│ 本日売上合計   │                     │
│ ¥48,500        │                     │
└────────────────┴────────────────────┘
```

#### ヘッダー
- **POSレジ** タイトル
- **選択中の店舗名**
- **スタッフ名**（誰が今レジを担当しているか）
- **時計**
- **📋 取引履歴**: 過去の会計を見る
- **💰 レジ開閉**: レジ開け・レジ締め
- **🔊 音声**: レジ操作音の ON/OFF

#### 会計待ちアラート
- 左カラムの会計待ちテーブルで、着席から 60 分以上経過した未会計テーブルは **会計注意** と表示される
- 90 分以上経過した未会計テーブルは **長時間** と表示され、赤系の強調になる
- ヘッダーには注意対象の件数が表示されるため、混雑時の会計声かけや締め漏れ防止に使う

#### 店舗決済端末 / QR決済の確認
1. 店舗で契約しているカード端末・QR決済アプリを起動する
2. 端末側で通信状態・バッテリー・ロール紙を確認する
3. カード/QR支払いは端末・アプリ側で完了させてから、POSLAで会計記録を確定する
4. POSLAのレジ画面は決済を実行しないため、決済端末のエラーは端末側で確認する

### 1-1-3. ハンディ端末の準備

**対象者**: ホールスタッフ / 店長

#### 必要なもの
- ハンディ用タブレット/スマホ（店舗備品）
- 首掛けストラップまたは腰ホルスター

#### 手順
1. ハンディ端末の電源を入れる
2. Chrome を起動
3. 以下の URL を開く:
   ```
   /handy/
   ```
4. ハンディ用アカウント（例: `handy-shibuya-01`）でログイン
5. 店舗選択
6. GPS オーバーレイが出たら **【位置情報を許可】** をタップ
7. ハンディ画面が表示される

#### ハンディ画面の上部タブ
画面上部に以下の 6 つのタブが横並びに表示されます。画面幅が狭い端末では横スクロールできます。

| タブ名 | 用途 |
|---|---|
| **店内注文** | 店内席のお客様からの注文を入力 |
| **テイクアウト** | テイクアウト注文を入力 |
| **テーブル状況** | 全テーブルの使用状況（着席中、空席、予約済みなど）を確認 |
| **運用** | 未送信注文、QR開放中、料理完成、呼び出し、会計待ち、清掃待ち、時間超過、未注文をまとめて確認 |
| **📅 予約** | 当日の予約一覧、着席操作、QR 開放 |
| **注文履歴** | 過去の注文を見る・編集する |

#### 端末モードの選び方
同じハンディ画面でも、持ち歩き端末、壁掛け端末、レジ横端末では見るべき情報が違います。開店時に端末の置き場所に合わせてモードを選びます。

| モード | 置き場所 | 使い方 |
|---|---|---|
| **ハンディ** | ホールスタッフが持ち歩くスマホ/小型端末 | 注文入力、卓操作、QR開放、履歴確認まで全部使う |
| **壁掛け** | ホールや厨房近くの固定タブレット | 呼び出し、料理完成、会計待ち、清掃待ちを大きめに表示する |
| **レジ横** | レジ横の補助端末 | 会計待ち、清掃待ち、テーブル状況、呼び出しを優先して見る |

壁掛け端末は、注文入力よりも **今すぐ対応すること** を見るための端末です。文字サイズと密度を上げ、操作ボタンは少なめにして、誤タップを避ける運用にします。

#### 通知音と再通知
ハンディ画面の右上にある **音ON/OFF** で、料理完成、呼び出し、会計待ち、清掃待ちなどの通知音を切り替えます。

| 設定 | 推奨 |
|---|---|
| ホール持ち歩き端末 | 音ON。ただし接客中に音量が大きすぎる場合は端末側の音量で調整 |
| 壁掛け端末 | 音ON。聞き逃しを防ぐため、店舗の騒音に合わせて音量を確認 |
| レジ横端末 | 店舗ルール次第。レジ担当が常時画面を見るなら音OFFでもよい |

未対応の呼び出しや料理完成を放置すると、運用タブで強調表示されます。呼び出しは 3 分超、配膳待ちは 5 分超、会計待ちは放置時間が長いものから優先して対応します。

### 1-1-4. 店頭決済端末の接続確認

POSLAの通常レジはカード決済・QR決済を実行しません。店舗で契約している外部決済端末・QRアプリを使います。

1. カード端末 / QR決済端末の電源を入れる
2. 通信状態、バッテリー、ロール紙、決済アプリのログイン状態を確認する
3. テスト会計では、外部決済端末側の完了表示を確認してからPOSLAで「カード」または「QR」を選び、支払い詳細も選ぶ
4. レジ締めでは、外部決済端末の集計額とPOSLAのカード/QR/電子マネー売上記録を突き合わせる

### 1-1-5. レシートプリンタの接続確認

お店でプリンタを導入している場合、レシート・キッチン伝票が紙で印刷されます。

#### 手順
1. プリンタの電源を入れる（本体の電源ボタン）
2. **緑色のランプ** が点灯することを確認
3. POSLA 管理画面の「店舗設定」→「プリンタ設定」で設定済みかチェック
4. **【🖨 テスト印刷】** ボタンでテスト用紙が出てくるか確認

#### 紙がなくなったら
1. プリンタのカバーを開ける（本体の爪を押して開く）
2. 新しいロール紙をセット
3. 紙の先端を少し出してカバーを閉める
4. プリンタ側面のランプが緑に戻ることを確認

### 1-1-6. レジを開ける（開店作業）

レジの準備金（つり銭用の硬貨・紙幣）を登録して、1 日の営業をスタートする操作です。

::: warning 開店前の準備
レジ開け操作には **担当スタッフの PIN 入力が必須** です。**マネージャーも例外なく**、操作前に下記を済ませておいてください。

1. 自分の **個人スタッフアカウント**で管理画面にログイン
2. **【🟢 出勤】** で出勤打刻（GPS 許可）
3. レジ担当 PIN が未設定なら、店長に依頼するか自分で設定

これをやらないと「PIN が一致しないか、対象スタッフが出勤中ではありません」エラーで開けられません。
:::

#### 手順
1. POSレジ画面の右上 **【💰 レジ開閉】** ボタンをタップ
2. **【レジを開ける】** というパネルが開く
3. 通常は大きなクイックボタンから選ぶ
   - **前回締め現金でレジ開け**: 前日の締め現金をそのまま使う場合
   - **0円で開始**: 現金を入れずに営業する場合
4. 紙幣・硬貨を数えて残したい場合は **【詳細に数えて開始】** を開く
5. 金種ごとの枚数を入力すると、開始時在高が自動計算される
6. **担当スタッフ PIN 入力モーダル**が表示される
7. 自分の 4〜8 桁 PIN をテンキーで入力 → **【確定】**
8. 画面に「レジを開けました」と緑色のメッセージ
9. 以降、売上・現金移動・締め操作がその営業日のレジログに残る

::: tip 開店前の準備金の金額は店舗ルール次第
多くのお店で ¥10,000〜¥30,000 を準備金として入れています。店長に必ず確認してください。
:::

::: tip 忙しい朝の基本運用
前日と同じ釣銭準備金で営業する店は、**前回締め現金でレジ開け** が一番早いです。差額が出やすい店舗、担当者を厳密に残したい店舗は、週に数回だけでも **詳細に数えて開始** を使うと原因調査が楽になります。
:::

#### お釣り準備で現金を足す場合
営業中に両替してきた小銭をレジに足す、釣銭を補充するなど、現金が増える操作も PIN 必須です。

1. **【💰 レジ開閉】** → **現金移動** を確認
2. よく使う場合は **【釣銭補充 ¥10,000】** などのクイックボタンを押す
3. 任意の金額なら詳細入力を開き、金額とメモを入力
4. **PIN モーダル** → 自分の PIN 入力 → 確定

#### 営業中に現金を出す場合
売上金の一部を金庫へ移す、両替のために一時的に持ち出す、小口購入や消耗品購入でレジ現金を使う場合も PIN 必須です。**現金を出す操作は不正抑止のため最も厳しく追跡されます**。誰が・いつ・いくら・何のために動かしたかが監査ログに残ります。

1. **【💰 レジ開閉】** → **現金移動** を確認
2. **現金回収 / 両替持出 / 小口購入 / 消耗品購入** など、近いクイック操作を選ぶ
3. 金額や理由が違う場合は詳細入力で記録する
4. **PIN モーダル** → 自分の PIN 入力 → 確定

::: warning レジから現金を出したら必ず理由を残す
「あとで書く」は締め時の差額原因になります。現金回収、両替、小口購入、消耗品購入のどれに近いかを選び、必要ならメモを追加してください。
:::

## 1-2. お客様が来たとき（ホール接客）

**対象者**: ホールスタッフ

### 1-2-1. テーブルに案内する（空席確認）

1. ハンディ画面を開く
2. **【テーブル状況】** タブをタップ
3. 全テーブルの状況が色別のカードで表示される:
   - 🟢 緑: 空席
   - 🟡 黄: 着席中（注文あり）
   - 🔴 赤: 会計待ち
   - ⚪ グレー: 無効化されたテーブル
4. 空席のテーブル（T03 など）をお客様に案内

### 1-2-2. テーブル開放認証（📲 QR 開放ボタンの押し方）

POSLA では **スタッフが物理的にテーブルを「開放」した後でしかお客様が QR から注文できない仕組み** を使っています。これにより **店外から QR の写真を撮って遠隔注文される攻撃** を防ぎます。

#### 手順
1. ハンディ画面の **【テーブル状況】** タブを開く
2. お客様を案内したテーブル（T03）のカードを探す
3. カードの下部にある **緑色の【📲 QR 開放（5 分）】** ボタンをタップ
4. 「開放しました」とメッセージ
5. そのテーブルは **5 分間だけ** お客様が QR から注文できる状態になる
6. お客様がテーブルの QR コードをスマホで読む
7. 読んだ瞬間にテーブルセッションが確立して、4 時間（デフォルト）有効になる

::: tip 5 分以内に誰も QR を読まなかった場合
再度 **【📲 QR 開放】** を押してください。過去に QR 開放したトークンは自動無効化されます。
:::

#### QR開放中に必ず確認する表示
QR開放中の卓は、テーブル状況タブと運用タブの両方で確認します。

| 表示 | 意味 | 現場で見ること |
|---|---|---|
| **QR開放中** | スタッフが開放し、お客様の読込待ち | 案内した卓と一致しているか |
| **残り時間** | QR開放トークンの有効期限までの時間 | 時間切れ前に読めているか |
| **開放者** | 誰がQR開放したか | 誤開放時に確認する |
| **QR取消** | 読込前の開放を取り消す | 間違った卓を開放した時に使う |
| **QR再発行** | 古い開放を無効にして新しい開放を作る | 残り時間不足や読込失敗時に使う |

#### 間違ってQR開放した場合
間違った卓を開放したら、すぐに取り消します。お客様が読み込む前なら、その開放は無効になります。

1. ハンディの **【運用】** タブを開く
2. **QR開放中** のカードから該当卓を探す
3. **【QR取消】** をタップ
4. 卓番号を確認
5. 確認ダイアログで **【取消】**
6. 正しい卓をあらためて **QR開放**

#### QRを再発行する場合
以下の場合は、取り消しではなく **QR再発行** を使います。

| ケース | 操作 |
|---|---|
| お客様のスマホでリンクが開かない | QR再発行 |
| 残り時間が少ない | QR再発行 |
| 開放後にスタッフが別対応へ行き、読めたか分からない | 運用タブで確認し、必要ならQR再発行 |
| 紙QRを読み込んだが古い画面が出ている | ブラウザを閉じてもらい、QR再発行後に再読込 |

QR再発行は古い開放を無効にしてから新しい開放を作ります。複数の有効な開放が残らないため、どの開放で入ったか分からなくなる状態を避けられます。

### 1-2-3. お客様に QR コードの読み方を説明する

お客様は自分のスマホで:

1. スマホの **カメラアプリ** を起動（大体ホーム画面にアイコンがある）
2. テーブルに置かれた **QR コード**（紙）にカメラを向ける
3. カメラが QR を認識すると、画面上部に URL 通知が表示される
4. 通知をタップ
5. POSLA のメニュー画面（セルフオーダー）が開く

お客様のスマホに説明しないといけない場合の言い方:
- 「カメラをこの QR に向けてください」
- 「画面の上の方に小さくリンクが出るので、それをタップしてください」

### 1-2-4. ハンディで注文を入れる（スタッフがお客様の代わりに注文を取る場合）

セルフオーダーではなく、スタッフが注文を取る場合の手順です。

#### 手順
1. ハンディの **【店内注文】** タブをタップ
2. 画面上部の **「テーブル」** ドロップダウンから該当テーブル（T03 など）を選ぶ
3. 下に表示されるメニュー一覧から注文された品をタップ
4. オプションがあるメニューならオプション選択モーダルが開く → 選んで【カートに追加】
5. 画面下部の **カートバー** に選んだ品が追加される
6. 数量を変更したい場合は **+/−** ボタン、削除なら **🗑** アイコン
7. メモ欄があれば「アレルギー: 卵抜き」「○○席」などを記入（最大 200 文字）
8. 全品選び終えたら **【注文送信】** ボタン（画面右下の緑の大きなボタン）をタップ
9. 確認ダイアログで **【OK】**
10. 送信完了 → キッチンの KDS 画面に自動で注文が流れる

#### 注文送信の画面表示
- 成功: 緑色の「注文を送信しました」トースト
- 失敗: 赤いエラーメッセージ。通信失敗時は未送信注文として端末に残り、運用タブから中身確認、再送、破棄ができます

### 1-2-5. 注文直後の取消・数量変更

注文直後にお客様から「やっぱり変更したい」と言われることはよくあります。調理状況によって対応を分けます。

| 状態 | 対応 | 理由 |
|---|---|---|
| 送信前のカート | 数量変更、削除 | まだKDSに送っていないためその場で修正 |
| 送信直後、KDS未着手 | 注文履歴から品目取消または数量変更 | 厨房作業前なので影響が小さい |
| 調理開始後 | 取消理由を入力して店長判断 | 原価、廃棄、会計額に影響する |
| 完成後 | 原則店長確認 | 提供済み/廃棄/クレーム対応の判断が必要 |
| 会計後 | レジ側の修正・返金扱い | 売上と締めに影響する |

#### 現場で残す情報
取消や数量変更では、最低限以下を残します。

| 項目 | 内容 |
|---|---|
| 担当者 | 誰が取消/変更したか |
| 対象 | 注文全体、品目、数量のどれを変えたか |
| 理由 | 入力ミス、お客様都合、品切れ、調理不可、提供遅延など |
| 調理状況 | KDS送信前、受付、調理中、完成、提供済み |

### 1-2-6. 未送信注文の復旧

ハンディで注文送信中にWi-Fiが切れた場合、二重送信を避けるため、未送信注文として端末に一時保存します。

1. **【運用】** タブを開く
2. **未送信注文** のカードを見る
3. 卓番号、品目、数量、合計、作成時刻を確認
4. KDSや注文履歴に同じ注文が入っていないか確認
5. 入っていなければ **【再送】**
6. 既に送れている、またはお客様が不要と言った場合は **【破棄】**
7. 破棄時は理由を残す

::: warning 未送信注文はむやみに再送しない
通信エラー直後は、サーバ側に注文が届いているのに端末の応答だけ失敗した可能性があります。再送前に注文履歴とKDSを確認してください。
:::

### 1-2-7. 席移動

お客様から「別の席に移りたい」と言われた場合は、卓Aから卓Bへ移動します。口頭だけで席を変えると、注文、会計、呼び出し、QRの紐付けがずれるため、必ずハンディで席移動します。

#### 移動される情報

| 情報 | 移動内容 |
|---|---|
| セッション | お客様の利用中セッション |
| 注文 | 未会計注文、調理中注文、提供済み注文 |
| QR | セルフ注文のセッション情報、必要に応じたQR表示 |
| メモ | アレルギー、席注意、接客メモ |
| 呼び出し | 未対応の呼び出し |
| サブセッション | 相席や個別QRのグループ |

#### 手順
1. **【テーブル状況】** または **【運用】** から移動元の卓を開く
2. **【席移動】** をタップ
3. 移動先の空席を選ぶ
4. 移動元と移動先を確認
5. **【移動する】** をタップ
6. 移動先卓で注文履歴と会計金額を確認

移動先が使用中、会計待ち、清掃待ち、QR開放中の場合は、そのまま移動しません。移動先の状態を解消するか、別の空席を選びます。

### 1-2-8. 呼び出し対応

お客様がセルフメニューからスタッフ呼び出しを送ると、ハンディの **運用** タブに表示されます。

| 表示 | 対応 |
|---|---|
| 呼び出し直後 | 担当スタッフが卓へ向かう |
| 3分超 | 強調表示。近いスタッフが優先対応 |
| 対応開始 | 必要なら担当者を確認 |
| 対応済み | 卓で用件を聞いた後に押す |

呼び出し対応済みを押す前に、実際に卓へ行ったかを確認します。画面だけ消すと、お客様側では放置された印象になります。

## 1-3. キッチン（厨房）での操作

**対象者**: キッチンスタッフ / 店長

### 1-3-1. KDS 画面で新規注文を確認

1. KDS 画面の **左カラム「受付」** を見る
2. 新規注文が来ると **ピコン音** と共に、受付カラムに新しいカードが追加される
3. 注文カードには以下が表示される:
   - テーブル番号（例: T03）または顧客名（テイクアウト時）
   - 経過時間（注文から何分経ったか）
   - メモ（アレルギー情報など）
   - 品目一覧（メニュー名・個数・オプション）

### 1-3-2. 注文を「調理開始」にする

調理を始める時の操作です。

#### 方法 A: 注文全体を調理開始（簡単）
1. 注文カードの下部の **【全品調理開始】** ボタンをタップ
2. 注文カード全体が「調理中」カラムに移動
3. 経過時間タイマーが継続

#### 方法 B: 品目ごとに調理開始（細かく管理したい場合）
1. 注文カード内の特定の品目の **【調理開始】** 小ボタンをタップ
2. その品目だけが緑（準備中）色に変わる
3. 全品目が調理開始になると、注文全体が調理中カラムに移動

### 1-3-3. 調理が終わったら「提供待ち」にする

1 品完成したら:

#### 方法 A: 注文全体を完成
1. 注文カードの下部の **【全品完成】** ボタン
2. 注文全体が「提供待ち」カラムに移動
3. **同時に、ホール側のハンディ画面に「料理ができました」アラートが自動送信される**

#### 方法 B: 品目ごとに完成
1. 注文カードの特定品目の **【完成】** 小ボタン
2. その品目だけ完成状態
3. 全品目完成したら注文全体が提供待ちカラムへ移動

### 1-3-4. 音声コマンドで操作する方法

キッチンは両手が塞がっていることが多いので、**声で KDS を操作** できます。

#### 前提
- **Android タブレット + Google Chrome 最新版**（標準環境）。iPad/Safari は音声コマンド非対応のためサポート対象外
- マイクが有効（Chrome のサイト設定で「マイク」を許可）
- 初回の **【タップして開始】** で音声を有効化済み

#### 基本コマンド

| 言葉 | 動作 |
|---|---|
| 「T01 調理開始」 | T01 の注文を pending → preparing |
| 「T01 完成」 | T01 の注文を preparing → ready |
| 「T01 提供」 | T01 の注文を ready → served |
| 「1番 調理開始」 | 同上（「T」なしでも可） |
| 「テーブル 01 調理開始」 | 同上 |

#### 品目単位コマンド
「T01 カツ丼 できた」のように、品目名を含めると特定の品目だけ完成にできます。

#### 注意
- **造語はウェイクワードに使えません** — 「POSLA！」のような呼び出し言葉は不要
- 騒音が多い厨房では誤認識する場合あり
- 音声が認識されないときは、画面の指ボタンで操作できるので安心してください

## 1-4. 料理を提供する（ホール）

**対象者**: ホールスタッフ

### 1-4-1. 料理完成アラートの確認

1. キッチンで料理が完成すると、ホールのハンディ画面に **ピコン音** と共にアラートが表示される
2. 画面上部に **「T03 カツ丼できました」** のようなバナー
3. タップすると詳細が開く

### 1-4-2. 料理を運ぶ

1. キッチンに行って料理を受け取る
2. 該当テーブルに運ぶ
3. ハンディ画面のアラートで **【配膳完了】** をタップ
4. または KDS 画面の「提供待ち」カラムで、該当注文の **【全品提供済み】** をタップ
5. 注文カードが消える（「完了」状態）

#### 配膳完了を間違って押した場合
実際にはまだ運んでいないのに配膳完了を押した場合は、すぐに **配膳取消** で戻します。

1. ハンディの **【運用】** タブ、またはKDSの提供待ち/完了一覧を確認
2. 該当注文を開く
3. **【配膳取消】** をタップ
4. 理由を入力
5. 担当者が残ることを確認
6. 料理が実際に提供されたら、あらためて **配膳完了**

配膳完了の取消は、ホールと厨房の状態を戻す操作です。押し間違いに気づいた時点ですぐ戻すことで、「厨房は提供済みと思っているが、ホールではまだ運んでいない」というズレを防ぎます。

#### 配膳待ちの優先順位

| 状態 | 優先度 | 対応 |
|---|---|---|
| 完成直後 | 通常 | 手が空いたスタッフが運ぶ |
| 5分超 | 高 | 運用タブで強調。近いスタッフが優先 |
| 複数卓で完成 | 高 | 冷めやすい料理、クレームになりやすい卓を先に運ぶ |
| 呼び出しと重なった | 店舗判断 | 料理が冷める場合は配膳優先、急ぎの呼び出しは近い別スタッフへ |

### 1-4-3. テーブルの状態を確認

- ハンディ画面の **【テーブル状況】** タブでそのテーブルの状態を確認
- 着席中 → お客様がまだ食事中

## 1-5. お会計

**対象者**: レジ担当

お客様が食事を終えて会計をする時の操作です。POSレジ画面で行います。

### 1-5-0. 担当スタッフ PIN 入力（毎回必要）

会計確定時に **担当スタッフの 4〜8 桁 PIN** 入力が必要です。誰がレジを操作したかを正確に記録するためです。

#### 必要な前提
- スタッフ各自が事前に **PIN 設定済み**（管理画面で店長が設定 or 自分で設定）
- スタッフが朝、個人 staff アカウントで管理画面にログイン → **出勤打刻** している
- （退勤打刻したスタッフの PIN は自動無効化される）

#### 入力フロー
1. 会計実行時、確認ダイアログで「OK」を押すと **PIN 入力モーダル** が表示
2. テンキーで自分の 4〜8 桁 PIN を入力
3. **【確定】** ボタン
4. PIN 一致 → 会計実行・レシート発行
5. PIN 不一致 → 「PIN が一致しないか、対象スタッフが出勤中ではありません」エラー → 再入力

スタッフ交代時はその場で別のスタッフが自分の PIN を入力するだけで OK（ログアウト不要）。

### 1-5-1. 会計待ちテーブルを選ぶ

1. POSレジ画面の **左カラム** に会計待ちテーブルが表示されている
2. お客様が会計したいテーブル（T03）のカードをタップ
3. **右カラム** にそのテーブルの注文詳細（品目・金額）が表示される

### 1-5-1-a. ハンディで会計待ちにする

お客様から「会計お願いします」と言われたら、ハンディで卓を会計待ちにします。

1. ハンディの **【テーブル状況】** または **【運用】** タブを開く
2. 対象卓をタップ
3. **【会計待ち】** をタップ
4. 卓が会計待ち表示になる
5. レジ画面の会計待ちリストに表示される

会計待ち中は、追加注文の扱いを店舗ルールで統一します。原則は **会計待ち後の追加注文は確認してから行う** です。お客様が「やっぱり追加」と言った場合は、会計待ちを解除して食事中へ戻してから注文する運用にすると、レジ担当とホールの認識がずれにくくなります。

### 1-5-1-b. 間違って会計待ちにした場合

間違って会計待ちにした卓は、ハンディから食事中へ戻します。

1. ハンディの **【運用】** タブを開く
2. **会計待ち** のカードから該当卓を探す
3. 卓をタップ
4. **【食事中へ戻す】** をタップ
5. レジ画面の会計待ちリストから消えたことを確認
6. 必要なら注文や呼び出し対応を続ける

会計待ち解除は、会計がまだ完了していない時だけ使います。会計完了後に戻す操作ではありません。会計完了後の修正は、レジ側の訂正や返金対応として扱います。

### 1-5-1-c. 会計完了後のセッション終了

会計が完了したら、前のお客様が持っていたセルフメニューURLを使えない状態にします。これをしないと、同じURLやブラウザ履歴から前の卓セッションに入れてしまうためです。

会計完了後にシステム側で行うこと:

| 処理 | 内容 |
|---|---|
| セッション終了 | 会計済みセッションを終了状態にする |
| QR無効化 | 既存の卓セッションURLから注文できないようにする |
| 呼び出し終了 | 残っている呼び出しを終了対象にする |
| 会計済み判定 | 同じ伝票の二重会計を防ぐ |
| 次回利用分離 | 次のお客様は新しいQR開放から始める |

次のお客様を案内する時は、必ず新しくQR開放します。前のお客様のスマホ画面をそのまま使わせたり、URLを共有したりしないでください。

### 1-5-1-d. 会計時にエラーが出たら

会計確定ボタンを押した後、画面に以下のいずれかが表示された場合の対処です。

| 表示メッセージ | 何が起きたか | 対処 |
|---|---|---|
| `[ALREADY_PAID]` または「すでに会計済みです」 | 別の端末（別のレジ・別のスタッフ・あるいは同じ端末で 2 回押した） が **0.数秒前にすでに同じ伝票を会計済み** | **慌てず画面リロード** → テーブルが空席になっていれば既に会計完了済み。**重複課金は発生していません** |
| `[CONFLICT]` または「処理が競合しました」 | 同時に別操作（例: 別端末で品目追加 + こちらで会計）が発生 | 画面リロード → 最新状態を確認 → 必要なら再会計 |
| `[AMOUNT_MISMATCH]` | 端末の表示金額とPOSLA側の再確認金額が違う（直前にメニュー価格更新あり） | 画面リロード → メニュー価格再取得 → 再会計 |
| `[PAYMENT_NOT_AVAILABLE]` | 通常レジでPOSLA決済を実行しようとした | 外部決済端末・QRアプリで支払い完了後、POSLAには記録のみ残す |

::: warning 慌てて何度も会計ボタンを押さない
同じ伝票を複数端末で同時に操作した場合、POSLA は重複会計を防ぐために片方の処理を止めます。エラーが出たら画面をリロードし、テーブルが空席になっているか、会計待ちのままかを確認してください。
:::

解決しない場合は、決済を続けずPOSLA 運営に連絡してください。

### 1-5-2. 現金でのお会計手順

1. **右カラム** の注文詳細の下にある支払方法選択で **【現金】** をタップ
2. 画面に **受取金額** 入力欄が表示される
3. お客様から受け取った金額を **テンキー** で入力（例: ¥5,000）
4. 画面下に **合計金額** と **お釣り** が自動計算される（例: 合計 ¥3,400、お釣り ¥1,600）
5. **【会計する】** ボタン（緑色の大きなボタン）をタップ
6. 確認ダイアログで **【OK】**
7. レジが開く（プリンタが繋がっている場合、レシートが自動印刷される）
8. お客様にお釣り、レシートを渡す

### 1-5-3. クレジットカードでのお会計手順

1. 店舗契約の外部カード端末で、先にカード決済を完了する
2. 端末画面・控えレシートで承認完了を確認する
3. POSLAでテーブルを選び、支払方法で **【カード】** をタップ
4. 支払い詳細で **クレジット / デビット / その他カード** のいずれかを選ぶ
5. **【会計する】** をタップ
6. 「外部カード端末で決済完了を確認済み」と表示されたら **【OK】**
7. 担当スタッフPINを入力する
8. POSLAにカード会計の記録が残り、レシートを発行できる

### 1-5-4. QR 決済（PayPay、d払い等）でのお会計手順

1. 店舗契約のQR決済アプリ・端末で、先に支払いを完了する
2. 入金完了画面または通知を確認する
3. POSLAで支払方法 **【QR】** を選択
4. 支払い詳細で **PayPay / 楽天ペイ / d払い / 交通系IC / iD / QUICPay** などを選ぶ
5. **【会計する】** をタップ
6. 「店舗契約のQR決済で支払い完了を確認済み」と表示されたら **【OK】**
7. 担当スタッフPINを入力する
8. POSLAにQR/電子マネー会計の記録が残り、レシートを発行できる

### 1-5-5. セルフレジ（お客様がスマホで決済）

お客様が自分のスマホから Stripe Checkout で決済する場合。

1. お客様がメニュー画面の **【お会計に進む】** をタップ
2. Stripe のクレジットカード入力画面が開く
3. カード情報を入力して決済
4. 決済完了後、自動でテーブルセッションがクリアされる
5. POSレジには通知が入らないので、テーブルが空いたらスタッフが物理確認

::: warning セルフ決済の制限
セルフ決済は **「テーブル全体の合計金額を一括で支払う」** 仕組みです。特定の商品だけを選んで支払う「個別会計」には対応していません。お客様から個別精算を希望された場合は、以下の「1-5-6. 個別会計」の手順でレジ対応を行ってください。
:::

### 1-5-6. 個別会計（割り勘）

「私のだけ別で払います」というケース。

1. 左カラムで該当テーブル選択
2. 右カラムの注文品目リストで、**個別会計したい品目だけ** のチェックボックスを ON
3. **【会計する】** で選択した品目だけ会計
4. 残りの品目は後から別会計できる

### 1-5-6-b. 相席の別会計（サブセッション / F-QR1）

相席で別グループが別々に会計したい場合。ハンディで **個別 QR** を発行する仕組み。

#### 状況例
- T05 に Aグループ 2 名（既に着席・注文済）
- 後から B グループ 3 名が相席で着席（同じテーブル）
- A / B で別々に会計したい

#### 手順
1. ハンディの **【テーブル状況】** タブを開く
2. T05 カードの **【📟 個別QR】**（オレンジボタン）をタップ
3. 新しいサブセッション用 QR が発行される（画面表示 or 紙に印刷）
4. B グループにその QR を渡してスマホで読んでもらう
5. B グループは独立して注文可能
6. 会計時は POSレジ側でサブセッション別に処理

複数グループが同じテーブルを使う場合は、グループごとに会計を分けられるよう、個別QRを発行してから注文を受け付けます。

### 1-5-7. レシート発行

- 自動印刷がデフォルト（プリンタ設定済みの場合）
- 手動再発行: **取引履歴** から該当取引を選んで **【再発行】**

## 1-9. 閉店作業

### 1-9-1. レジを閉める（レジ締め）

1 日の売上を締める操作です。

::: warning レジ閉めにも担当スタッフ PIN が必須
レジ閉めも開け・入金・出金と同じく **担当スタッフ PIN 入力が必須**。退勤打刻する前に締めること（先に退勤打刻すると自分の PIN が無効化されて締められなくなる）。マネージャーも例外なし。
:::

1. POSレジ画面の右上 **【💰 レジ開閉】** をタップ
2. **【レジを閉める】** パネルを開く
3. 画面上部の締め前チェックを確認
   - 未会計の注文
   - 会計取消・返金
   - 外部決済控えメモ未入力
   - 予想現金と実際現金の差額
   - 締め後に入った会計や取消
4. 問題がなければ **【問題なければこのまま締める】** を使う
5. 差額や未処理があれば **【詳細に確認して締める】** を開く
6. POSLA が表示する **予想現金 / 現金売上 / カード売上 / QR・電子マネー売上** を確認
7. レジ内の現金を数え、必要なら金種ごとの枚数を入力する
8. 差額がある場合は「差額理由・締めメモ」を入力
9. 閉店時の共有事項があれば「閉店引き継ぎメモ」に残す
10. **閉める** → **担当スタッフ PIN 入力モーダル**
11. 自分の 4〜8 桁 PIN を入力 → **【確定】**
12. 画面に以下が記録される:
   - 開始時在高: ¥10,000
   - 現金売上: ¥38,500
   - カード売上: ¥0
   - QR/電子マネー売上: ¥0
   - 入金: ¥0
   - 出金: ¥0
   - **予想在高**: ¥48,500
   - **実際**: ¥48,500
   - **差額**: ¥0
13. 差額が 0 ならそのまま完了
14. 差額がある場合は現金を数え直し、外部カード端末・QR端末の日締め控えとも突き合わせる
15. レジが締まり、営業終了

::: tip レジ締めを早く終わらせるコツ
普段は **問題なければこのまま締める** を使い、差額・未処理・外部決済控え未入力が出た日だけ詳細確認に入ります。毎日すべてを細かく入力する運用にすると現場負担が増えるため、POSLAでは「問題がある時だけ深掘りする」設計にしています。
:::

### 1-9-2. 売上の確認方法

1. 管理ダッシュボードの **【レポート】→【売上レポート】**
2. 本日の売上、時間帯別、支払方法別、品目別の内訳が見られる

### 1-9-3. KDS / ハンディ / POSレジのログアウト

人間スタッフが使うハンディは必ずログアウト。

- **個人スタッフアカウント**（人間）は **必ず退勤前にログアウト**
- **デバイスアカウント**（KDS-1 など）は **ログアウト不要**（常時ログインしっぱなしが前提）

#### ログアウト手順
1. 画面右上の **【ログアウト】** ボタンをタップ
2. ログイン画面に戻る

#### 端末の電源
- 次の日も使うなら **電源は入れっぱなし** でも OK（ロック画面にはなる）
- 省電力のため切りたいなら、電源ボタン長押しで電源オフ

## 1-10. よくある質問（Part 1 向け）

**Q1. ハンディでの注文送信中に Wi-Fi が切れたら？**

A1. 送信が失敗してエラー表示になります。カートの内容は残るので、Wi-Fi が復旧したらそのまま再送信してください。

**Q2. 同じテーブルに追加で料理を頼まれた**

A2. ハンディの **【店内注文】** タブで、そのテーブルを選んで追加品目を送信すれば、既存注文に追加されます。

**Q3. お客様が注文を取り消したい**

A3. KDS 画面で該当注文の **【取消】** ボタンをタップ。キャンセルされます。既に会計済みの場合は 8 章「返金処理」を参照。

**Q4. テーブルを間違って選んで注文送信してしまった**

A4. ハンディの **【注文履歴】** タブから該当注文を探して **【編集】** → テーブルを修正。ただし会計済みは編集不可。

**Q5. 料理の提供を忘れて 20 分以上経ってしまった**

A5. KDS 画面で該当注文カードが赤く点滅しているはず。すぐ提供してお客様にお詫びください。自動で「遅延アラート」も POSLA 運営に通知されています。
