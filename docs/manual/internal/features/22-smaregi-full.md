# 22. スマレジ連携の全設定（完全ガイド）

POSLAは株式会社スマレジが提供するクラウドPOS「スマレジ」とOAuth認証で連携できます。本章ではテナント側でやること・POSLA運営側でやることを、どちらも完全な手順で記載します。

---

## 22.1 連携の概要

### 何ができるのか

| 機能 | 説明 |
|------|------|
| 店舗マッピング | POSLAの店舗とスマレジの店舗を紐付ける |
| 商品インポート | スマレジに登録済みの商品をPOSLAメニューとしてインポート |
| 商品マッピング | POSLAのメニューとスマレジの商品を紐付け |
| 売上連携 | 将来予定：POSLAの売上データをスマレジに同期 |

### 前提条件

- テナントがスマレジの契約を持っていること
- スマレジ側に店舗と商品マスタが登録済みであること
- POSLA運営がスマレジ開発者コンソールでOAuthクライアントを登録済みであること

---

## 22.2 テナント側の接続手順

### 22.2.1 スマレジアカウントの準備

1. スマレジにログイン（管理者権限が必要）
2. 契約ID（`contract_id`）を控える
3. スマレジに対象店舗と商品が登録されていることを確認

### 22.2.2 POSLAでの接続

1. POSLAのオーナーダッシュボードにログイン
2. 「外部POS連携」タブを開く
3. 「スマレジと接続」ボタンをクリック
4. スマレジのOAuth認証画面にリダイレクト
5. スマレジアカウントでログインし、POSLAからのアクセスを許可
6. POSLAにリダイレクトされ、接続完了
7. 「接続状態」が「接続済み」に変わります

### 22.2.3 店舗マッピング

1. 「外部POS連携」タブの「店舗マッピング」セクション
2. POSLAの各店舗について、対応するスマレジ店舗をドロップダウンで選択
3. 「保存」をクリック

| POSLA店舗 | スマレジ店舗 |
|-----------|-------------|
| 本店 | → 選択 |
| 支店A | → 選択 |

### 22.2.4 商品インポート

1. 店舗マッピング完了後、「商品インポート」ボタンをクリック
2. スマレジから商品一覧を取得
3. インポート対象の商品にチェック
4. 「インポート開始」をクリック
5. 以下の情報がPOSLAメニューとして登録されます：
   - 商品名
   - 価格
   - カテゴリ
   - バーコード（あれば）

::: warning 非破壊インポート
既存のPOSLAメニューは**維持**されます。スマレジからの商品は新規追加または既存メニューへのマッピングとして登録されます。
:::

### 22.2.5 商品マッピング

既存のPOSLAメニューとスマレジ商品を手動で紐付ける場合：

1. 「商品マッピング」セクション
2. POSLAメニューの各項目について、対応するスマレジ商品を選択
3. 「保存」をクリック

### 22.2.6 接続解除

1. 「外部POS連携」タブ
2. 「接続解除」ボタンをクリック
3. 確認ダイアログで「はい」をクリック
4. トークンが削除され、スマレジとの連携が解除されます

## 22.3 POSLA運営側の設定手順

### 22.3.1 スマレジ開発者コンソールでアプリ登録

1. [スマレジ開発者コンソール](https://developers.smaregi.dev/) にログイン
2. 「アプリ登録」から新規アプリを作成
3. 以下を入力：
   - アプリ名：POSLA
   - リダイレクトURI：`https://eat.posla.jp/api/smaregi/callback.php`
   - スコープ：`pos.products:read`, `pos.stores:read`, `pos.transactions:read` など
4. クライアントID（`client_id`）とクライアントシークレット（`client_secret`）を取得

### 22.3.2 POSLA管理画面への登録

POSLA管理画面の「API設定」タブで以下を入力：

| キー名 | 値 |
|--------|-----|
| smaregi_client_id | スマレジのクライアントID |
| smaregi_client_secret | スマレジのクライアントシークレット |

### 22.3.3 OAuthトークン管理

POSLAはテナントごとに以下のトークンを保持します。

| フィールド | 説明 |
|-----------|------|
| smaregi_contract_id | スマレジの契約ID |
| smaregi_access_token | アクセストークン |
| smaregi_refresh_token | リフレッシュトークン |
| smaregi_token_expires_at | トークン有効期限 |
| smaregi_connected_at | 接続日時 |

**トークンリフレッシュ：**
- 有効期限の**5分前**に自動リフレッシュ
- API呼び出し時に期限切れが検知された場合も自動リフレッシュ
- リフレッシュ失敗時は`smaregi_access_token`をNULLに戻し、テナントに再接続を促す

### 22.3.4 マッピングテーブル

| テーブル | 説明 |
|---------|------|
| smaregi_store_mapping | POSLA店舗 ↔ スマレジ店舗 |
| smaregi_product_mapping | POSLAメニュー ↔ スマレジ商品 |

**smaregi_store_mapping：**

| フィールド | 説明 |
|-----------|------|
| tenant_id | テナントID |
| posla_store_id | POSLA店舗ID |
| smaregi_store_id | スマレジ店舗ID |
| created_at | 作成日時 |
| updated_at | 更新日時 |

**smaregi_product_mapping：**

| フィールド | 説明 |
|-----------|------|
| tenant_id | テナントID |
| posla_menu_id | POSLAメニューID |
| smaregi_product_id | スマレジ商品ID |
| smaregi_price_division_id | 価格区分ID（店舗別価格） |
| created_at | 作成日時 |

### 22.3.5 APIエンドポイント

POSLA側のスマレジ連携APIは `/api/smaregi/` 配下に実装されています。

| エンドポイント | 説明 |
|--------------|------|
| `/api/smaregi/connect.php` | OAuth開始。スマレジへリダイレクト |
| `/api/smaregi/callback.php` | OAuth完了。トークン保存 |
| `/api/smaregi/status.php` | 接続状態取得 |
| `/api/smaregi/stores.php` | スマレジ店舗一覧取得 |
| `/api/smaregi/products.php` | スマレジ商品一覧取得 |
| `/api/smaregi/import.php` | 商品インポート実行 |
| `/api/smaregi/mappings.php` | マッピング保存・取得 |
| `/api/smaregi/disconnect.php` | 接続解除 |

## 22.4 動作確認

### テナント接続テスト

1. テストテナントでスマレジに接続
2. 以下を確認：
   - 接続ボタンを押すとスマレジへリダイレクト
   - スマレジで認証後、POSLAに戻ってくる
   - `smaregi_access_token`が保存される
   - 「接続状態」が「接続済み」と表示される

### トークンリフレッシュテスト

1. `smaregi_token_expires_at`を手動で5分後に更新
2. APIを呼び出す
3. 自動的にトークンがリフレッシュされることを確認
4. 新しい`smaregi_token_expires_at`が保存されていることを確認

### 商品インポートテスト

1. スマレジに10件程度の商品を登録
2. POSLAで商品インポートを実行
3. POSLAメニューに10件追加されていることを確認
4. `smaregi_product_mapping`に10行のマッピングが作成されていることを確認

## 22.5 トラブルシューティング

| 問題 | 原因 | 対処 |
|------|------|------|
| OAuth接続時に「アプリが見つかりません」 | POSLA管理画面の`smaregi_client_id`が未設定 | POSLA管理画面で設定 |
| 接続後すぐに切断される | リダイレクトURIが不一致 | スマレジ開発者コンソールでURIを確認 |
| 商品インポート時に「権限がありません」 | OAuthスコープが不足 | スマレジ開発者コンソールでスコープを追加し、再接続 |
| トークンが失効する | リフレッシュ処理のエラー | `error_log`を確認。手動で再接続 |
| 店舗マッピングが保存されない | POSLA店舗ID/スマレジ店舗IDの不一致 | ブラウザ開発者ツールでネットワークエラーを確認 |
