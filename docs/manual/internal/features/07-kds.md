# 7. KDS（キッチンディスプレイ）

KDS（Kitchen Display System）は、キッチンスタッフが注文をリアルタイムで確認し、調理の進捗を管理するための画面です。`/kds/` のURLで新しいタブとして開きます。

管理ダッシュボードのヘッダーにある「KDS」リンクをクリックするか、ブラウザで直接 `/kds/` にアクセスします。

## 7.1 初回起動

### 起動手順

1. KDS画面にアクセスすると、認証チェックが行われます
2. 複数店舗に所属している場合、店舗選択オーバーレイが表示されます
   - 初回のみ表示。選択した店舗はブラウザに記憶されます（localStorage キー：`mt_kds_store`）
3. **音声起動オーバーレイ**が表示されます
   - 「タップして開始」ボタンが表示されます
   - このタップでブラウザの音声再生が許可されます（Web Audio APIのunlock）
   - タップしないと効果音やビープ音が鳴りません
4. KDS画面が表示され、注文のポーリング（3秒間隔）が開始されます

::: warning 音声起動について
ブラウザのセキュリティ制限により、ユーザーの操作（タップ/クリック）なしに音声を再生することはできません。KDS起動時の「タップして開始」は音声機能を有効にするために必要です。この手順をスキップすると、新規注文の通知音やアラート音が鳴りません。
:::

## 7.2 画面構成

### 3カラムのカンバンボード

KDS画面は3つのカラム（列）で構成されています。注文カードは左から右へ進みます。

| カラム | 名称 | 内容 |
|--------|------|------|
| 左 | 受付 | 新しく受け付けた注文（pending） |
| 中央 | 調理中 | 調理を開始した注文（preparing） |
| 右 | 提供待ち | 調理が完了して提供を待っている注文（ready） |

各カラムが空の場合は「なし」と表示されます。

### ステーションバー

画面上部にステーション選択バーが表示されます。

| ボタン | 説明 |
|--------|------|
| 全品目 | 全てのカテゴリの注文を表示（デフォルト） |
| [ステーション名] | そのステーションに割り当てられたカテゴリの品目のみ表示 |

**ステーションの例：**
- 「厨房」ステーション → メイン料理・前菜カテゴリを担当
- 「ドリンク」ステーション → ドリンクカテゴリを担当
- 「デザート」ステーション → デザートカテゴリを担当

選択中のステーションはブラウザに記憶されます（localStorage キー：`mt_kds_station`）。

::: tip ステーションの設定
KDSステーションは管理ダッシュボードの「店舗設定」→「KDSステーション」サブタブで設定します。ステーションごとに担当カテゴリを割り当てることで、キッチンの各セクションが自分の担当品目だけを見られるようになります。詳しくは「12. 店舗設定」の章を参照してください。
:::

### コースフェーズバー

コース料理の注文がある場合、ステーションバーの下にコースフェーズバーが表示されます。

| 表示要素 | 説明 |
|---------|------|
| テーブルコード | コース注文のテーブル名 |
| コース名 | コース料理の名前 |
| 現在フェーズ | 現在のフェーズ番号 |
| 「次フェーズ」ボタン | 次のフェーズに進める |

コース注文がない場合、このバーは表示されません。

## 7.3 注文カードの表示

### カードの構成

各注文は以下の要素で構成されたカードとして表示されます。

**ヘッダー部分：**

| 要素 | 説明 |
|------|------|
| テーブルコード | 「5番」「A-1」など。テイクアウトの場合は顧客名 |
| テイクアウトバッジ | テイクアウト注文の場合「テイクアウト」バッジ（青色） |
| 受取時間 | テイクアウトの場合、受取予定時刻（「18:30」） |
| コースバッジ | コース注文の場合「コース P1:前菜」形式（フェーズ番号＋フェーズ名） |
| 経過時間 | 注文からの経過時間（「12m 34s」形式） |

**本文部分：**

| 要素 | 説明 |
|------|------|
| メモ | 注文メモがある場合に表示（「ネギ抜き」等） |
| アレルゲン警告 | 「⚠️ アレルギー: 卵, 乳, 小麦」形式で表示 |
| 品目リスト | 品名 × 数量。オプションがある場合は小文字で表示 |

**フッター部分：**

| 要素 | 説明 |
|------|------|
| 合計金額 | 注文合計（コース注文の場合は非表示） |
| アクションボタン | 注文全体のステータス変更ボタン |
| 取消ボタン | 注文をキャンセル |

### 経過時間と色の変化

注文カードは経過時間に応じて色が変わります。

| 経過時間 | カードの状態 | 意味 |
|---------|------------|------|
| 10分未満 | 通常表示 | 正常範囲 |
| 10分以上 | 黄色背景（warning） | やや遅延 |
| 20分以上 | 赤色背景（danger） | 大幅遅延 |

### 品目ごとのステータス表示

各品目には個別のステータスと操作ボタンが表示されます。

| 品目ステータス | 表示 | アクションボタン |
|-------------|------|---------------|
| pending | 品名と数量 | 「調理開始」ボタン |
| preparing | 品名と数量 | 「完成」ボタン |
| ready | 品名と数量 | 「提供済」ボタン |
| served | 取り消し線 + 「✓提供済」バッジ | — |
| cancelled | 取り消し線 + 「❌」バッジ | — |

## 7.4 注文の操作

### 注文全体の操作（カードレベル）

カードのフッターにあるボタンで、注文の全品目を一括操作できます。

| ボタン | 動作 | 表示条件 |
|--------|------|---------|
| 全品調理開始 | 全品目を pending → preparing に変更 | 注文ステータスが pending |
| 全品完成 | 全品目を preparing → ready に変更 | 注文ステータスが preparing |
| 全品提供済み | 全品目を ready → served に変更 | 注文ステータスが ready |
| 取消 | 注文をキャンセル | 常時表示 |

**注文キャンセル：**
1. 「取消」ボタンをクリックします
2. 確認ダイアログが表示されます
3. 「OK」をクリックすると注文ステータスが「cancelled」に変更されます
4. カードがKDSから消えます

### 品目個別の操作（アイテムレベル）

各品目の横にあるボタンで、品目ごとに個別操作できます。

| ボタン | 動作 |
|--------|------|
| 調理開始 | その品目だけを pending → preparing に変更 |
| 完成 | その品目だけを preparing → ready に変更 |
| 提供済 | その品目だけを ready → served に変更 |

::: tip 個別操作と全体操作の使い分け
- 1つの注文に複数品目がある場合、品目ごとに異なるタイミングで調理が進むことがあります
- 例：ドリンクは先に完成、メイン料理は後から完成
- このような場合は品目個別の操作ボタンを使います
- 全品目が同時に進む場合はカードレベルのボタンが効率的です
:::

### 楽観的更新（Optimistic Update）

KDSではUI更新を高速化するため、楽観的更新パターンを使用しています。

1. ボタンをクリックすると、サーバーからのレスポンスを待たずにUIが即座に更新されます
2. バックグラウンドでAPIリクエストが送信されます
3. APIリクエストが成功すれば、そのまま表示が維持されます
4. APIリクエストが失敗した場合、次のポーリング（3秒後）で正しい状態に修正されます

## 7.5 コース料理の操作

### コースフェーズの進行

コース料理の注文は、フェーズ（段階）ごとに料理が提供されます。

**フェーズの進め方：**

1. コースフェーズバーの対象テーブルの「次フェーズ」ボタンをクリックします
2. 確認メッセージが表示されます：「現フェーズの未完了品目は全て提供済みになります」
3. 「OK」をクリックすると以下が実行されます
   - 現在のフェーズの未完了品目が全て「served」に変更されます
   - フェーズ番号が1つ進みます（例：フェーズ1 → フェーズ2）
   - 次のフェーズの品目が自動的にKDSに表示されます

**コース注文の表示上の特徴：**
- カードの左ボーダーが緑色（3px）
- コースバッジ（「コース P1:前菜」形式）がヘッダーに表示
- 合計金額は表示されません（コース料理は一括料金のため）

## 7.6 リアルタイム更新

### ポーリング

KDSは定期的にサーバーから最新の注文データを取得します。

| 状態 | ポーリング間隔 |
|------|-------------|
| 通常時 | 3秒 |
| オフライン時 | 30秒 |
| 接続復旧時 | 即座に3秒に復帰＋強制リフレッシュ |

::: info ポーリング方式について
POSLAはWebSocketやServer-Sent Eventsではなく、HTTPポーリング方式を採用しています。3秒間隔で十分なリアルタイム性を実現しつつ、サーバー構成をシンプルに保っています。
:::

### オフライン検知

インターネット接続が切断された場合：
- ポーリング間隔が30秒に延長されます
- 画面上にオフライン警告が表示されます
- 接続が復旧すると自動的に通常のポーリングに戻ります

## 7.7 スタッフ呼び出しアラート

お客さんがセルフオーダー画面からスタッフを呼び出すと、KDS画面にアラートが表示されます。

### アラートの表示

- 画面上部に赤色のバナーとして表示されます
- 5秒ごとにチェックされます
- アラート1件ごとに以下が表示されます

| 要素 | 説明 |
|------|------|
| 🔔アイコン | ベルアイコン |
| テーブルコード | 呼び出し元のテーブル |
| 呼び出し理由 | お客さんが入力した理由（「お水ください」等） |
| 経過時間 | 呼び出しからの経過時間 |
| 「対応済み」ボタン | アラートを消す |

### 効果音

新しいアラートが検知されると効果音が鳴ります。

- **ビープ音パターン：** 880Hz → 1100Hz（2音）
- **音量：** 30%
- **使用API：** Web Audio API（サイン波）

::: info KDSでのアラートフィルタリング
KDS画面では「staff_call」（スタッフ呼び出し）タイプのアラートのみが表示されます。「product_ready」（料理完成通知）や「kitchen_call」（キッチン内連絡）はKDSには表示されません（これらはハンディPOS画面に表示されます）。
:::

## 7.8 AI音声コマンド

KDSはWeb Speech APIを使用した音声コマンドに対応しています（Chrome推奨）。

### 音声コマンドの起動

1. KDS起動時の「タップして開始」で音声機能が有効化されます
2. ブラウザのマイクアクセス許可が求められる場合があります
3. 音声認識は日本語モードで動作します

### 対応コマンド

音声で注文のステータスを変更できます。具体的なコマンド体系は音声コマンダーモジュール（voice-commander.js）で定義されており、テーブル番号や品目名の音声認識による操作が可能です。

::: warning Chrome推奨
Web Speech APIの continuous モード（連続認識）はGoogle Chromeでのみ安定動作します。Firefox、Safari、Edgeでは音声コマンド機能が使えない場合があります。
:::

## 7.9 AIキッチンダッシュボード

KDS画面にはAIによる調理優先度の提案機能があります。

### パネルの表示

- 画面下部からスライドアップするパネル（画面の40%の高さ）
- パネルを開くとAI分析が開始されます
- **60秒ごとに自動更新**されます
- 手動リフレッシュボタンもあります

### AIの分析内容

AIは現在のpending（受付）とpreparing（調理中）の注文を分析し、3つのカテゴリに分類します。

| カテゴリ | 色 | 意味 |
|---------|-----|------|
| ⚡ 今すぐ着手 | 赤色 | 最優先で調理すべき品目 |
| 🔜 次に着手 | オレンジ色 | 次に調理を開始すべき品目 |
| ⏸ 待機中 | グレー | まだ着手不要な品目 |

### AIからの追加情報

| 情報 | 説明 |
|------|------|
| ペースステータス | 良好 / 通常 / 混雑 |
| サマリーコメント | AIによる状況の要約（日本語） |

::: info AI分析のAPI
AIキッチンダッシュボードはGemini APIを使用して分析を行います。APIキーはPOSLA運営が一括管理しているため、テナント側での設定は不要です。
:::

## 7.10 KDSステーションの設定

KDSステーションは管理ダッシュボードの「店舗設定」→「KDSステーション」サブタブで管理します。

### ステーションの追加

1. 「KDSステーション」サブタブを開きます
2. 「ステーション追加」ボタンをクリックします
3. 以下を入力します

| フィールド | 説明 |
|-----------|------|
| ステーション名 | 日本語名（例：「厨房」「ドリンクバー」） |
| ステーション名（英語） | 英語名（例：「Kitchen」「Drink Bar」） |
| 担当カテゴリ | チェックボックスで複数選択。そのステーションが担当するメニューカテゴリ |

4. 「保存」ボタンをクリックします

### ステーションの編集・削除

- 編集：ステーション名やカテゴリ割り当てを変更
- 有効/無効の切替：無効にするとKDSのステーションバーに表示されなくなります
- 削除：ステーションを完全に削除

### ステーション運用の例

**大規模キッチンの場合：**

| ステーション名 | 担当カテゴリ | KDSタブレット配置場所 |
|--------------|------------|-------------------|
| 厨房（メイン） | メイン料理、前菜、サラダ | メイン調理場 |
| 揚げ場 | 揚げ物 | フライヤー横 |
| ドリンク | ドリンク、デザート | ドリンクバー |

各タブレットでKDSを開き、該当するステーションを選択すると、そのセクション担当の品目だけが表示されます。

**小規模キッチンの場合：**

ステーション設定をせず「全品目」のまま使用すれば、全ての注文が1つの画面に表示されます。
