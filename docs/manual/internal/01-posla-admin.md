# 1. POSLA管理画面

POSLA管理画面（`/public/posla-admin/`）は、プラスビリーフ社内のPOSLA運営スタッフが使用する管理コンソールです。テナント管理・APIキー設定・ダッシュボード統計を提供します。テナント向けの管理ダッシュボードとは完全に分離された画面です。

## 1.1 初期セットアップ

### 最初の管理者アカウント作成

POSLAを新規デプロイした直後、管理者アカウントが存在しない状態でセットアップAPIを呼び出します。

**エンドポイント：** `POST /api/posla/setup.php`

**リクエスト：**
```json
{
  "email": "admin@plusbelief.co.jp",
  "password": "初期パスワード",
  "display_name": "管理者名"
}
```

**条件：** `posla_admins`テーブルが空の場合のみ動作します。既に管理者が存在する場合は403エラーが返されます。

::: danger 本番環境での注意
初期セットアップ完了後、`setup.php`ファイルは本番サーバーから削除することを推奨します。
:::

### ログイン

1. ブラウザで `/public/posla-admin/` にアクセスします
2. ログイン画面が表示されます
3. メールアドレスとパスワードを入力します
4. ログインボタンをクリックします

**セッション設定：**
- セッション有効時間：8時間
- Cookie：HttpOnly、SameSite=Lax
- ログイン成功時にセッションIDが再生成されます
- 最終ログイン日時が記録されます

## 1.2 ダッシュボード

ログイン後、ダッシュボード画面が表示されます。

### タブ一覧

| タブ | 説明 |
|------|------|
| ダッシュボード | 統計情報と最近のテナント |
| テナント管理 | テナント一覧と編集 |
| テナント作成 | 新規テナントの登録 |
| API設定 | グローバルAPIキーの管理 |

### 統計情報

| 指標 | 説明 |
|------|------|
| テナント総数 | 有効なテナント数 |
| 店舗総数 | 全テナントの有効な店舗合計 |
| ユーザー総数 | 全テナントの有効なユーザー合計 |
| プラン分布 | standard / pro / enterprise の内訳 |
| 最近のテナント | 直近5件の新規テナント |

---

## 1.3 テナント管理

### テナント一覧

全テナントが一覧表示されます。各テナントの情報：

| 項目 | 説明 |
|------|------|
| テナント名 | — |
| スラッグ | URL用識別子 |
| プラン | standard / pro / enterprise |
| 有効/無効 | — |
| サブスクリプション状態 | none / active / past_due / canceled / trialing |
| 契約期間終了日 | Stripe Billingの現在の期間の終了日 |
| Stripe Connect | 接続状態とアカウントID |
| 店舗数 | — |
| ユーザー数 | — |

### テナントの編集

テナント名をクリックすると編集モーダルが開きます。

| フィールド | 編集可否 | 説明 |
|-----------|---------|------|
| テナント名 | ✅ | 変更可能 |
| スラッグ | ❌ | 読み取り専用 |
| プラン | ❌ | Stripe Billingで制御（読み取り専用） |
| 有効/無効 | ✅ | テナント全体の有効/無効切替 |

::: warning テナント無効化
テナントを無効にすると、そのテナントの全ユーザーがログインできなくなります。「このテナントは無効化されています」エラーが表示されます。
:::

### テナントの作成

「テナント作成」タブで新規テナントを登録します。

| フィールド | 必須 | 説明 |
|-----------|------|------|
| スラッグ | ✅ | 英小文字・数字・ハイフンのみ。一意である必要があります |
| テナント名 | ✅ | 企業/ブランド名 |
| プラン | — | standard（デフォルト）/ pro / enterprise |

## 1.4 API設定

「API設定」タブで、POSLA全体で共通利用するAPIキーを管理します。

### 管理対象のキー

**AI・地図API：**

| キー名 | 説明 |
|--------|------|
| gemini_api_key | Google Gemini APIキー（AI機能全般に使用） |
| google_places_api_key | Google Places APIキー（競合分析・店舗情報に使用） |

**Stripe Billing（サブスクリプション）：**

| キー名 | 説明 |
|--------|------|
| stripe_secret_key | Stripeのシークレットキー |
| stripe_publishable_key | Stripeの公開キー |
| stripe_webhook_secret | Webhookの署名検証シークレット（whsec_で始まる） |
| stripe_price_standard | Standardプランの価格ID（price_で始まる） |
| stripe_price_pro | Proプランの価格ID |
| stripe_price_enterprise | Enterpriseプランの価格ID |

**Stripe Connect（決済手数料）：**

| キー名 | 説明 |
|--------|------|
| connect_application_fee_percent | テナント決済に対するPOSLAの手数料率（%）。デフォルト1.0% |

**スマレジ連携：**

| キー名 | 説明 |
|--------|------|
| smaregi_client_id | スマレジOAuthクライアントID |
| smaregi_client_secret | スマレジOAuthクライアントシークレット |

### キーの表示とマスキング

APIキーはセキュリティのためマスキングされて表示されます。

**マスキング形式：** 先頭4文字 + ●●●●●●●●

例：`sk_test_1234567890` → `sk_t●●●●●●●●`

- 「表示」ボタンで完全な値を確認できます
- 「削除」ボタンで全キーを一括削除できます

### 手数料計算の仕組み

Stripe Connectの手数料は以下の計算式で算出されます。

```
手数料 = ceil(決済金額 × 手数料率 / 100)
最低手数料 = ¥1
```

例：決済金額 ¥3,000、手数料率 1.0% → 手数料 = ceil(30) = ¥30
