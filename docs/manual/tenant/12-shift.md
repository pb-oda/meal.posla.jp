# 12. シフト管理

シフト管理では、スタッフへの希望提出依頼、スタッフの希望提出、週間シフト作成、公開前チェック、勤務確認、交代・欠勤申請、当日の不足確認、勤怠打刻、実績差分、店舗間ヘルプ、人件費の目安確認を行います。

POSLA の前提は **1 人 1 アカウント**です。device アカウントは KDS、レジ、ハンディなどの備品端末用であり、シフト割当、勤怠、人件費、スタッフ評価の対象には入りません。

## 12-1. 画面構成

| 画面 | 使う人 | 目的 |
|---|---|---|
| スタッフ画面 | staff | 今日の勤務、勤怠打刻、希望提出依頼、未対応、責任者返信、交代候補承諾、勤怠履歴、打刻修正申請を確認 |
| 週間カレンダー | manager 以上 | 週単位でシフトを作成、希望提出依頼、公開前チェック、確定 |
| マイシフト | staff / manager | 自分のシフト確認、確認済み、交代依頼、欠勤連絡 |
| 希望提出 | staff | 勤務できる日、希望時間、勤務不可を提出 |
| 勤怠一覧 | manager 以上 | 出勤、退勤、休憩、打刻修正、スタッフからの打刻修正申請の承認/却下 |
| テンプレート | manager 以上 | 曜日・時間帯・必要人数・持ち場の基本形を登録 |
| シフト設定 | manager 以上 | 休憩、残業判定、GPS、時給、目標人件費率、持ち場を設定 |
| 集計サマリー | manager 以上 | 週次・月次の労働時間、実績人件費、人件費率、労務警告を確認 |
| 当日・ヘルプ | manager 以上 | 今日の未打刻、欠勤、時間帯不足、空きシフト募集、担当作業、休憩不足、スキル不足、呼べる候補、店舗間ヘルプ、交代/欠勤申請を確認 |

## 12-2. 標準フロー

1. マネージャーが **週間カレンダー** の **希望提出依頼** から、対象期間、提出期限、本文を指定してスタッフ画面へ依頼を出す
2. スタッフがスマホの **スタッフ画面** で依頼を確認し、**希望提出** で勤務可否を出す
3. マネージャーが **テンプレート** または **週間カレンダー** でシフトを作る
4. 必要な時間帯に持ち場を入れる
5. 6 時間超、8 時間超の勤務は休憩時間を確認する
6. 週間カレンダー上部の **公開前チェック** を確認する
7. 人数不足、持ち場不足、休憩不足、希望未提出、予定人件費率、労務リスク、ヘルプ候補を見て修正する
8. 問題がなければ **確定する** を押す
9. スタッフは **マイシフト** で自分の確定シフトを確認し、**確認** を押す
10. 交代や欠勤が必要な場合、スタッフはマイシフトから **交代依頼** または **欠勤連絡** を送る
11. 店長は **当日・ヘルプ** で未対応申請、未打刻、当日の時間帯不足を確認する

希望提出依頼は POSLA のスタッフ画面内に表示されます。LINE、SMS、メールの外部通知はこの操作だけでは送られません。急ぎの場合は、公開前チェックの **リマインド文コピー** や店舗の通常連絡手段も使ってください。

## 12-2-b. スタッフ画面

スタッフがログインすると、最初にスタッフ画面が表示されます。

表示される内容:

| 項目 | 内容 |
|---|---|
| 今日の勤務 | 今日または次回のシフト、時間、持ち場、休憩、確認状態 |
| 勤怠打刻 | 出勤、退勤、勤務中の経過時間。GPS 必須店舗では位置情報が必要 |
| 未対応 | 希望提出依頼、変更未確認シフト、担当作業、募集シフト、交代/欠勤申請、責任者返信、交代候補承諾、打刻修正申請 |
| 勤怠履歴・修正申請 | 今日の打刻履歴と、打刻忘れ・時刻誤りの店長向け申請 |

スタッフ画面は、アルバイト・スタッフ本人のスマホ利用を前提にしています。勤怠打刻、未対応、マイシフト、希望提出へ進むボタンはスマホ幅では 1 列で表示され、タップしやすい大きさになります。ハンディ、KDS、POSレジの端末操作画面ではありません。人の勤怠・シフト・担当作業を扱う画面です。

## 12-3. 持ち場

持ち場は、シフトで「何の仕事に入るか」を表す店舗別のマスタです。

初期値は以下です。

| code | 表示名 |
|---|---|
| `hall` | ホール |
| `kitchen` | キッチン |

店舗に合わせて、シフト設定から追加できます。

例:

| code | 表示名 | 用途 |
|---|---|---|
| `drink` | ドリンク | バー、ドリンク場 |
| `wash` | 洗い場 | 洗い場専任 |
| `cashier` | レジ | レジ横担当 |
| `prep` | 仕込み | 仕込み担当 |

運用ルール:

- `code` は英数字、`_`、`-` の 20 文字以内
- 表示名は店長・スタッフが画面で見る名前
- 非表示にした持ち場は新規選択に出なくなる
- 非表示にしても過去のシフト履歴は残る
- 公開前チェックの持ち場不足は、テンプレートに指定した持ち場を基準に判定する

## 12-4. 公開前チェック

週間カレンダーを開くと、カレンダー上部に公開前チェックが表示されます。これは AI ではなく、POSLA 内の実データを集計して判定します。

### 希望提出依頼

週間カレンダー上部の **希望提出依頼** から、対象開始日、対象終了日、提出期限、件名、本文を入力して一斉依頼できます。

依頼後の動き:

- staff ロールのスタッフ画面の **未対応** に **希望提出依頼** として表示される
- スタッフが **確認** を押すと、対象期間の週の **希望提出** 画面へ移動する
- 依頼文、対象期間、提出期限が希望提出画面の上部に表示される
- 対象期間内の日付すべてに希望を提出すると、スタッフ画面の未対応から消える
- 既読は POSLA 内で記録されるが、外部メッセージの既読とは連動しない

現時点の画面操作は、依頼作成とスタッフ画面での表示・提出導線です。LINE、SMS、メールの自動送信は含みません。

### 表示される内容

| 項目 | 内容 |
|---|---|
| 時間帯ごとの人数不足 | テンプレート必要人数、過去 8 週間の同曜日・同時間帯売上、対象日の予約人数から必要人数を出し、予定人数と比較 |
| 持ち場不足 | テンプレートで持ち場が指定されている時間帯について、予定シフトの持ち場数と比較 |
| 休憩未設定/不足 | 6 時間超は 45 分、8 時間超は 60 分を目安に不足を警告 |
| 希望未提出スタッフ | 対象週で希望提出がない、または一部の日だけ未提出のスタッフを一覧表示 |
| リマインド文 | 希望未提出者向けの連絡文をまとめてコピー |
| 予約・売上予測 | 時間帯別に、過去売上平均、平均注文数、予約組数、予約人数を表示 |
| 予定人件費率 | 予定シフトの人件費目安を、過去売上平均と比較。店舗設定の目標人件費率を超えると警告 |
| 労務リスク | 予定上の週 40 時間超、1 日 8 時間超、6 日以上連続勤務を表示 |
| 未確認シフト | 確定済みだが、スタッフがマイシフトでまだ確認していないシフトを表示 |
| 実績差分 | 予定と勤怠打刻を比較し、遅刻、早退、残業、打刻なしを表示 |
| ヘルプ候補 | 人数不足の時間帯について、他店舗スタッフの候補を表示 |

### 赤・黄・情報の考え方

| 表示 | 意味 | 推奨対応 |
|---|---|---|
| 赤 | 人数不足が大きい、休憩 0 分の長時間勤務、予定人件費率が大きく超過など、公開前に確認すべき状態 | シフト追加、休憩入力、時間変更、売上見込みの確認 |
| 黄 | 人数不足 1 人、持ち場不足、希望未提出、勤怠差分、予定人件費率の軽い超過など | 店長判断で修正またはそのまま確定 |
| 情報 | 未確認シフト、軽微な労務リスクなど、見ておくとよい内容 | 必要に応じて確認 |

公開前チェックに警告が残っていても、最終判断として **確定する** ことはできます。POSLA は現場判断を止めず、見落としを減らすために警告を表示します。

### 予定人件費率の見方

予定人件費率は次の式です。

`予定人件費 ÷ 過去売上平均 × 100`

予定人件費は、スタッフ個別時給があれば個別時給、なければ店舗デフォルト時給で計算します。どちらも未設定のスタッフがいる場合、その人の人件費は計算できません。

予約人数は必要人数の判定に使いますが、予約を売上金額へ自動換算して人件費率の分母には入れません。宴会、コース単価、貸切などは店舗側の判断で見てください。

## 12-5. 必要人数提案の考え方

必要人数は以下のうち一番大きい値を採用します。

| 入力データ | 判定 |
|---|---|
| シフトテンプレート | 曜日・時間帯ごとに登録した必要人数 |
| 過去売上 | 過去 8 週間の同曜日・同時間帯の平均売上と平均注文数 |
| 予約 | 対象日の予約組数と予約人数 |

目安ルール:

- 平均注文数が多い時間帯は必要人数を上げる
- 売上が大きい時間帯は必要人数を上げる
- 予約人数が多い時間帯は必要人数を上げる
- テンプレートの必要人数がある場合は、それを最低ラインとして扱う

これは給与計算や労務判断の最終決定ではありません。予約の内容、宴会、研修、新人比率、料理提供の難易度など、POSLA に入っていない事情は店長が調整してください。

## 12-6. マイシフト確認

スタッフは、確定した自分のシフトを **マイシフト** で確認します。

操作:

1. スタッフが管理画面にログイン
2. **シフト管理** → **マイシフト** を開く
3. 自分のシフトだけを見る
4. 確定シフトに問題がなければ **確認** を押す
5. ステータスが **確認済** になる

店長側では、公開前チェックに未確認件数と未確認者が表示されます。外部メッセージの既読確認ではなく、POSLA のマイシフト上で確認されたかを表します。

### 12-6-b. 変更された人だけ再確認

スタッフが一度 **確認済** にした後で、店長がそのシフトを変更した場合、POSLA はそのスタッフだけを自動で **変更あり** に戻します。

対象になる変更:

| 変更 | 再確認 |
|---|---|
| 開始時刻の変更 | 必要 |
| 終了時刻の変更 | 必要 |
| 休憩時間の変更 | 必要 |
| 持ち場の変更 | 必要 |
| メモの変更 | 必要 |

店長が追加で「再確認してください」と入力する必要はありません。変更されたシフトだけ、スタッフのマイシフトに **変更あり** と表示されます。

スタッフ側の動き:

1. スタッフが **マイシフト** を開く
2. 変更されたシフトに **変更あり** と表示される
3. 変更内容（例: `10:00 → 10:30`、`休憩 0分 → 45分`）を確認する
4. 問題なければ **確認** を押す
5. 再び **確認済** になる

店長側の考え方:

- 変更していないスタッフには何も出ません
- 変更したスタッフだけ再確認対象になります
- 変更前と変更後は、開始、終了、休憩、持ち場、メモの単位で表示されます
- 外部メッセージ通知は自動送信しません。急ぎの変更は店舗の通常連絡手段も使ってください
- 「言った/聞いていない」を減らすための POSLA 内確認です

## 12-7. 交代依頼と欠勤連絡

スタッフは、未来日または当日の確定済みシフトについて、マイシフトから交代依頼または欠勤連絡を送れます。

### 交代依頼

1. スタッフが **マイシフト** を開く
2. 対象シフトの **交代依頼** を押す
3. 候補スタッフが表示される
4. 候補を選ぶ、または候補を指定せずに送る
5. 責任者へのメモを入力して送信

候補スタッフを指定した場合、候補スタッフ本人のマイシフトに **交代候補** として表示されます。候補スタッフが **引き受ける** と回答するまで、責任者はその候補へ差し替える承認はできません。辞退された場合は、責任者が申請スレッドで確認するか、別の穴埋め導線を使います。

候補は以下を見て出ます。

- 同じ店舗の staff アカウント
- 対象シフトと同じ時間帯に別シフトが重なっていない
- 希望提出が `希望` または `出勤可` なら優先
- 希望が `不可` の人は除外
- 対象時間と希望時間が重ならない人は除外
- 過去 90 日のシフト履歴から、同じ持ち場経験があれば優先

### 欠勤連絡

1. スタッフが **マイシフト** を開く
2. 対象シフトの **欠勤連絡** を押す
3. 理由や責任者へのメモを入力する
4. 送信する

欠勤連絡を送っただけでは、シフトは自動削除されません。責任者が当日・ヘルプ画面で承認または却下します。

### 責任者の承認

店長・副店長などの manager 権限を持つ責任者は **当日・ヘルプ** の **交代・欠勤申請** で対応します。

| 操作 | 結果 |
|---|---|
| 本人承諾済みの交代スタッフを選んで承認 | 元のシフト担当者が交代スタッフに差し替わる |
| 交代なしで承認 | 欠勤だけ承認される。シフトの不足は当日不足に残る |
| 却下 | 申請は却下され、元のシフトは変わらない |
| キャンセル | 申請を取り消す |

申請ごとに **やり取り** ボタンがあり、スタッフと責任者が確認メッセージを残せます。責任者からの返信、承認、却下、候補承諾依頼はスタッフ画面の未対応に未読として表示されます。対応済み申請には、承認・却下・キャンセルを行った責任者名が表示されます。これは POSLA 内の画面通知であり、LINE やメールの送信とは別です。

一度対応済みになった申請は、未対応一覧には戻りません。誤って承認した場合は、週間カレンダーでシフトを直接修正してください。

### 12-7-b. 欠勤承認後の穴埋め導線

欠勤を **交代なし** で承認すると、当日・ヘルプに穴埋め導線が出ます。

できること:

| 操作 | 内容 |
|---|---|
| 候補を見る | 同店舗で同時間帯に重複しないスタッフ候補を見る |
| 空きシフト募集を作成 | 欠勤シフトと同じ日付・時間・持ち場で募集を作る |
| 他店舗ヘルプ要請 | 欠勤シフトと同じ日付・時間・持ち場を入れた状態でヘルプ要請を開く |

店長は、候補がいれば通常連絡手段で本人確認し、必要なら週間カレンダーで直接シフトを変更します。候補がいない場合は **空きシフト募集** または **他店舗ヘルプ要請** に進みます。

## 12-8. 当日・ヘルプ

**当日・ヘルプ** は、店長・副店長などの責任者が「今日まず何を見るべきか」を確認する画面です。

### 12-8-a. 今日の責任者チェック

画面の最上部には **今日の責任者チェック** が表示されます。これは、責任者が画面内を探し回らなくても、今日先に見るべきものだけを並べる集約欄です。

表示される内容:

| 表示 | 内容 | 主な対応 |
|---|---|---|
| 遅刻/未打刻 | 予定開始から 15 分超過しても出勤打刻がないスタッフ | 本人確認、打刻漏れ修正、欠勤判断 |
| 交代・欠勤未対応 | まだ承認・却下していない申請 | 承認または却下 |
| 欠勤穴埋め | 欠勤だけ承認され、交代スタッフが入っていないシフト | 穴埋めボタンから候補、空きシフト募集、他店舗ヘルプへ進む |
| 時間帯不足 | テンプレート必要人数に対して予定人数が不足 | シフト追加、ヘルプ要請、店長フォロー |
| 休憩不足 | 法定目安に対して休憩時間が不足 | 休憩時間の入力、休憩に出せる時間の調整 |
| 未完了作業 | 今日の担当作業が未完了 | 担当者へ声かけ、別担当へ変更 |
| 変更未確認 | 変更済みシフトをスタッフがまだ再確認していない | 通常連絡手段で声かけ |

遅刻/未打刻が出ている場合は、未打刻行からそのまま次の操作ができます。

| ボタン | 内容 |
|---|---|
| 連絡済み | 本人へ連絡したことを記録する。未打刻としては残るが、店長が対応済みだと分かる |
| 遅刻連絡あり | 本人から遅刻連絡があったことを記録する。出勤打刻が入るまでは未打刻として残る |
| 欠勤にする | 未打刻シフトを承認済み欠勤として処理する。以降、未打刻からは外れ、承認済み欠勤に移る |
| 穴埋め | 欠勤処理を行ったうえで、同店舗候補、空きシフト募集、他店舗ヘルプ要請へ進む |

チェック項目は、別の承認フローを増やすものではありません。下にある **今日の不足**、**現場シフト運用**、**交代・欠勤申請** から、今日の重要項目だけを先頭に持ち上げています。

### 12-8-b. 今日の不足

今日の不足では、当日の勤怠・欠勤・時間帯不足を確認します。

表示される内容:

| 項目 | 内容 |
|---|---|
| 開始済み未打刻 | 予定開始から 15 分超過しても出勤打刻がないスタッフ |
| 未対応の交代/欠勤 | まだ承認・却下されていない申請 |
| 承認済み欠勤 | 欠勤だけ承認され、交代スタッフが入っていないシフト |
| 時間帯不足 | テンプレート必要人数に対して、当日の予定人数が不足している時間帯 |
| 今日呼べる候補 | 今日シフトが入っていない同店舗スタッフ |
| 店舗間ヘルプ | 他店舗への送信要請、他店舗からの受信要請 |

承認済み欠勤は、未打刻とは別に表示されます。欠勤が承認済みなのに未打刻として残ると現場が迷うためです。

## 12-8-c. 現場シフト運用

当日・ヘルプには **現場シフト運用** が追加されています。これは、店長が「人が足りない」「誰が何をやるか分からない」「休憩が回っていない」を一画面で見つけるためのボードです。

### 12-8-c-1. 対応候補

画面上部に、POSLA が実データから作った対応候補が表示されます。AI ではなく、シフト、希望提出、勤怠、募集、担当作業、時給設定を集計した固定ロジックです。

| 表示 | 何を見ているか | 店長の対応 |
|---|---|---|
| 募集に応募なし | 空きシフト募集に応募がない | 直接声かけ、時間変更、店舗間ヘルプ |
| 休憩不足 | 6 時間超で 45 分未満、8 時間超で 60 分未満 | 休憩時間を入力、担当者を入れ替える |
| スキル不足 | 持ち場に必要なスキル人数を満たしていない | スキル保有者を入れる、条件を見直す |
| 未完了作業あり | 担当作業が未完了 | 担当者へ声かけ、別担当へ変更 |
| 人件費率超過 | 今日の予定人件費率が目標を超過 | 人数・時間・売上見込みを確認 |
| 経験者なし | 過去 90 日で勤務実績が少ない人だけの時間帯 | 店長または経験者を入れる |
| 希望未提出 | 直近 7 日の希望提出がない | 通常連絡手段で提出依頼 |

### 12-8-c-2. 空きシフト募集

空きシフト募集は、シフト表に直接人を入れられない時に使います。「この日時に入れる人いますか」を POSLA 上に出し、スタッフがマイシフトから応募します。

店長の作成手順:

1. **当日・ヘルプ** を開く
2. **現場シフト運用** の **募集作成** を押す
3. 日付、開始、終了、休憩、持ち場、必要スキル、メモを入力する
4. **作成** を押す
5. 募集一覧に表示される

スタッフの応募手順:

1. スタッフが **マイシフト** を開く
2. **募集シフト** を見る
3. 入れる募集の **応募** を押す
4. 責任者へのメモがあれば入力する
5. 応募済みになる

店長の承認手順:

1. **当日・ヘルプ** の募集一覧を見る
2. 応募者の **承認** を押す
3. POSLA がそのスタッフのシフトを `published` として作成する
4. 募集は `filled` になる
5. 他の応募者は自動的に却下される

注意:

- 応募しただけではシフトは作成されません。店長の承認が必要です。
- スタッフに同時間帯の既存シフトがある場合は応募または承認できません。
- 必要スキルを指定した募集は、そのスキルを持つスタッフだけが応募できます。
- 承認後のシフトは、通常の週間カレンダーで確認・修正できます。

### 12-8-c-3. 今日の担当作業

担当作業は、開店、営業中、閉店で「誰が何をやったか」を迷わないようにする仕組みです。

初期テンプレート:

| コード | 表示名 | タイミング |
|---|---|---|
| `open_register` | レジ開け | 開店 |
| `open_prep` | 開店前チェック | 開店 |
| `mid_shift_clean` | 中間清掃 | 営業中 |
| `close_register` | レジ締め | 閉店 |
| `close_clean` | 閉店清掃 | 閉店 |

店長の追加手順:

1. **当日・ヘルプ** を開く
2. **今日の担当作業** の **作業追加** を押す
3. 日付、作業、担当スタッフ、メモを入力する
4. **追加** を押す

スタッフの完了手順:

1. スタッフが **マイシフト** を開く
2. **担当作業** を見る
3. 終わった作業の **完了** を押す
4. 間違えた場合は **戻す** で未完了へ戻す

運用の考え方:

- 「レジ開け」「レジ締め」のように毎日迷いやすい作業を優先して登録します。
- 完了ボタンは 1 タップで終わるため、現場で説明を読まなくても使えます。
- 完了者と完了時刻は DB に残ります。ただし、ログインしている個人スタッフアカウントを基準に記録します。ハンディ/KDS/レジの device アカウントでは人の作業実績にはしません。

### 12-8-c-4. スキル・資格

スキルは、「誰でもよい人数」ではなく「この作業ができる人がいるか」を見るためのタグです。

初期スキル:

| コード | 表示名 |
|---|---|
| `cashier_close` | レジ締め可 |
| `trainer` | 新人指導可 |
| `key_holder` | 鍵管理可 |

できること:

- スキルを追加する
- スタッフにスキルを付与する
- 持ち場ごとに必要スキルと必要人数を設定する
- 当日のシフトが、そのスキル人数を満たしているか確認する

例:

| 持ち場 | 必要スキル | 必要人数 | 意味 |
|---|---|---:|---|
| `hall` | `cashier_close` | 1 | ホールにレジ締めできる人が 1 人必要 |
| `hall` | `trainer` | 1 | 新人が多い日は指導できる人が 1 人必要 |
| `kitchen` | `key_holder` | 1 | 閉店時に鍵管理できる人が必要 |

スキル不足が出た場合、POSLA は自動でシフトを変更しません。店長がスタッフを入れ替える、時間をずらす、またはその日は店長が担当するなど、現場判断で対応してください。

### 12-8-c-5. 休憩・人件費・経験者

現場シフト運用では、今日の見落としを小さく表示します。

| 表示 | 内容 |
|---|---|
| 売上 | 今日の `paid` 注文売上 |
| 人件費 | 今日の予定シフト × 時給の目安 |
| 率 | 予定人件費 ÷ 今日の売上 |
| 休憩不足 | 6 時間超/8 時間超の休憩不足 |
| 経験者なし | 過去 90 日の勤務実績が少ない人だけの時間帯 |

時給未設定のスタッフがいる場合、人件費は未計算または参考値になります。正確に見るには、シフト設定の店舗デフォルト時給、またはスタッフ個別時給を設定してください。

### 12-8-c-6. 希望未提出・変更未確認

希望未提出・変更未確認では、直近 7 日の希望未提出と、今日のシフト変更をスタッフがまだ再確認していないものを見ます。交代申請やスキル付与などの監査ログはここには表示されません。

| 表示 | 意味 |
|---|---|
| 未提出 N 人 | その日の希望シフトが未提出のスタッフ数と主なスタッフ名 |
| 変更未確認 | 時間、休憩、持ち場、メモなどが変更され、スタッフの再確認が必要な今日のシフト |

変更未確認は、スタッフに再確認してもらうための現場用表示です。給与確定や監査の詳細確認は、監査ログ本体や勤怠一覧で確認してください。

## 12-9. ヘルプ募集

人手が足りない時間帯は、公開前チェックと当日・ヘルプ画面の両方で候補を確認できます。

### 公開前チェックで見る

人数不足がある時間帯に、他店舗スタッフ候補が表示されます。

候補は以下を見て並びます。

- 同じテナント内の他店舗スタッフである
- 対象日・対象時間帯に既存シフトが重なっていない
- 希望提出が `希望` または `出勤可` なら優先
- 希望未提出でも、重複シフトがなければ「要確認」の候補として表示
- 過去 90 日のシフト履歴から、指定持ち場の経験があれば優先

### 当日・ヘルプ画面から送る

1. **当日・ヘルプ** タブを開く
2. **新規要請** を押す
3. 送信先店舗、日付、時間、必要人数、持ち場を入力する
4. 入力欄の下に候補スタッフが表示される
5. 候補を見て、送信先店舗が妥当か確認する
6. **送信** を押す

承認するのは要請先店舗のマネージャーです。承認時に派遣スタッフを選択すると、要請元店舗のシフトにヘルプとして登録されます。

## 12-10. テンプレートの作り方

テンプレートは、公開前チェックの最低必要人数にも使われます。

例:

| 名前 | 曜日 | 時間 | 必要人数 | 持ち場 |
|---|---|---|---|---|
| 平日ランチ ホール | 月〜金 | 10:00-14:00 | 2 | ホール |
| 平日ランチ キッチン | 月〜金 | 10:00-14:00 | 2 | キッチン |
| 週末ディナー | 土日 | 17:00-22:00 | 5 | 指定なし |

持ち場不足まで正確に見たい場合は、ホール用、キッチン用、ドリンク用などに分けてテンプレートを作ってください。持ち場を「指定なし」にすると、人数不足は確認できますが、持ち場別の不足までは判定しません。

## 12-11. 勤怠打刻と実績差分

スタッフは出勤時に **出勤**、退勤時に **退勤** を打刻します。GPS 打刻を有効にしている店舗では、店舗付近での打刻が必要です。

実績差分は公開前チェックと集計サマリーで確認します。

| 差分 | 表示条件 |
|---|---|
| 打刻なし | 確定シフトが過去日で、対応する勤怠打刻がない |
| 遅刻 | 予定開始より 5 分超遅れて出勤 |
| 早退 | 予定終了より 5 分超早く退勤 |
| 残業 | 予定労働時間より 15 分超長く勤務 |

打刻漏れや誤りがある場合は、**勤怠一覧** から該当行を修正します。修正理由は監査ログに残るため、理由欄は空にしないでください。

## 12-12. 人件費確認

人件費は以下を使って目安計算します。

- 店舗デフォルト時給
- スタッフ個別時給
- シフト時間
- 休憩時間
- 深夜時間帯の割増目安
- 勤怠実績
- 実際の売上

週間カレンダーの公開前チェックは **予定人件費率**、集計サマリーは **実績人件費率** を見ます。

| 場所 | 使うデータ | 目的 |
|---|---|---|
| 公開前チェック | 予定シフト、時給、過去売上平均 | 公開前に組みすぎを見つける |
| 集計サマリー | 勤怠実績、時給、期間内の実売上 | 実際の人件費と売上のバランスを見る |

給与計算の最終確定は、店舗または本部の給与運用に従ってください。POSLA の表示は現場確認用の目安です。

## 12-13. シフト設定

シフト設定では以下を設定します。

| 項目 | 内容 |
|---|---|
| 希望提出締切日 | 希望提出の目安日 |
| デフォルト休憩 | 新規シフト作成時の休憩目安 |
| 残業判定分 | 勤怠差分の残業判定に使う目安 |
| 早出許容分 | 早めに出勤打刻した場合の許容目安 |
| 自動退勤時間 | 打刻忘れ対策の自動退勤閾値 |
| GPS 設定 | 店舗緯度、経度、許容半径、GPS 必須 |
| 店舗デフォルト時給 | 人件費目安の計算に使う |
| 目標人件費率 | 公開前チェックと集計サマリーの警告基準 |
| 持ち場 | 店舗ごとの役割名を登録・非表示 |

## 12-14. 困ったとき

| 症状 | 確認すること |
|---|---|
| 公開前チェックで人数不足が出る | テンプレート必要人数、予約人数、売上予測、実際の割当人数を確認 |
| 持ち場不足が出ない | テンプレートの持ち場が指定されているか確認 |
| 希望未提出が多い | リマインド文をコピーして通常連絡手段で送る |
| スタッフが確認済みにならない | スタッフがマイシフトで **確認** を押しているか確認 |
| 交代候補が出ない | 同時間帯シフト重複、希望不可、希望時間外、同店舗スタッフ登録を確認 |
| 欠勤承認後に不足が残る | 交代スタッフなしで承認した状態。必要なら当日・ヘルプから呼べる候補、または店舗間ヘルプを使う |
| ヘルプ候補が出ない | 他店舗スタッフ、希望提出、同時間帯シフト重複、店舗所属を確認 |
| 打刻できない | ログイン中のアカウント、位置情報許可、店舗付近かを確認 |
| 人件費が想定と違う | 時給設定、休憩、深夜時間、勤怠修正、期間内売上の有無を確認 |
