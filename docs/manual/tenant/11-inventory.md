# 11. 在庫・レシピ管理

在庫・レシピ管理では、原材料、在庫数、使用量、棚卸し、AI需要予測を管理します。品切れ防止、原価確認、仕込み量の判断、売り切れ管理に役立ちます。

POSLAの在庫は「商品単位」ではなく、商品に紐づく「原材料単位」で管理します。たとえば、唐揚げ定食に「鶏肉 180g」「米 200g」「タレ 30ml」を登録しておくと、会計完了時に注文数に応じて原材料在庫が減ります。

## 画面構成

管理画面の **在庫・レシピ** には、主に2つのサブタブがあります。

| サブタブ | できること | 主な用途 |
|---|---|---|
| 原材料一覧 | 原材料の追加、編集、削除、棚卸し、CSVインポート/エクスポート | 在庫数、単位、原価、低在庫アラートの管理 |
| レシピ設定 | メニューごとに使用する原材料と数量を登録、CSVインポート/エクスポート | 会計時の自動在庫減算、原価計算、品切れ連動の土台 |

::: warning 最初にメニュー登録が必要です
レシピ設定は、登録済みのメニューに原材料を紐づけます。メニューがない場合、レシピ設定に対象商品が表示されません。先にカテゴリ、商品、店舗メニューの設定を済ませてください。
:::

## 在庫の自動更新

POSLAでは、以下のタイミングで在庫が自動的に更新されます。

- **決済完了時**: 会計が完了した時点で、実在庫が正式に減算されます。
- **セルフ会計の確定時**: お客様側で会計が確定した場合も、同じレシピ設定を使って在庫が減算されます。
- **棚卸し保存時**: 店舗で数えた実在庫を入力すると、原材料の現在庫が更新されます。

::: info 会計を優先する設計です
一時的な通信不良が起きても、お客様のお会計を止めにくいようにしています。在庫反映にずれが出た場合は、営業後に棚卸しや履歴を確認して調整してください。
:::

### 在庫消費の正確性
通信エラーなどで同じ会計が再実行されても、在庫が二重に減らないようにしています。
- 各会計に対して、在庫を一度だけ反映する仕組みがあります。
- どの注文で、いつ、どの原材料が使われたかの履歴を残しているため、後から確認できます。
- レシピや原材料が未設定の商品は、自動減算の対象になりません。
- 会計済み注文の取消や返金を行った場合は、在庫を自動で戻す前提にせず、店舗の棚卸し・調整で整えてください。

## 原材料を登録する

1. 管理画面で **在庫・レシピ** を開きます。
2. **原材料一覧** タブを開きます。
3. **原材料追加** を押します。
4. 原材料名、単位、初期在庫数、原価、在庫少アラート閾値を入力します。
5. 保存します。

| 項目 | 内容 | 入力例 |
|---|---|---|
| 原材料名 | 店舗で扱う食材や備品の名前 | 鶏肉、米、卵、ソース |
| 単位 | 在庫を数える単位 | g、kg、ml、L、個、枚 |
| 在庫数 | 現在の在庫量 | 5000 |
| 原価 | 単位あたりの原価 | 1gあたり2円、1個あたり40円 |
| 在庫少アラート閾値 | この数値以下で注意表示・品切れ連動の対象 | 1000 |

原材料一覧では、在庫数がアラート閾値以下になると目立つ表示になります。原材料を無効にした場合、その原材料は在庫連動の対象から外れます。

## 原材料CSV

原材料一覧では、CSVエクスポートとCSVインポートができます。大量登録や棚卸し前の整理に使います。

| カラム | 内容 | 必須 |
|---|---|---|
| `id` | 既存原材料を更新する場合のID。空欄なら新規作成 | 任意 |
| `name` | 原材料名 | 必須 |
| `unit` | 単位。空欄の場合は「個」扱い | 任意 |
| `stock_quantity` | 現在庫 | 任意 |
| `cost_price` | 単位あたり原価 | 任意 |
| `low_stock_threshold` | 低在庫アラート閾値 | 任意 |
| `sort_order` | 表示順 | 任意 |
| `is_active` | 有効/無効。通常は `1` | 任意 |

CSVはUTF-8で保存してください。Excelで開いて文字化けする場合は、いったんCSVをエクスポートして、その列構成をサンプルとして編集すると安全です。

## レシピを設定する

商品ごとに使用する原材料と数量を登録します。注文数に応じて在庫を確認しやすくなり、会計完了時の在庫減算、原価確認、品切れ連動の土台になります。

1. **レシピ設定** タブを開きます。
2. 対象商品の **レシピ編集** を押します。
3. 使用する原材料を選びます。
4. 1食あたりの使用量を入力します。
5. 必要な原材料を複数行追加します。

例:

| 商品 | 原材料 | 1食あたり使用量 |
|---|---|---:|
| 唐揚げ定食 | 鶏肉 | 180g |
| 唐揚げ定食 | 米 | 200g |
| 唐揚げ定食 | タレ | 30ml |

同じ商品に同じ原材料を重複登録することはできません。数量を変えたい場合は、既存行を編集します。

## レシピCSV

レシピ設定でもCSVエクスポートとCSVインポートができます。CSVでは商品名と原材料名で紐づけます。

| カラム | 内容 | 必須 |
|---|---|---|
| `recipe_id` | 既存レシピのID。エクスポート時に出力されます | 任意 |
| `menu_name` | メニュー名 | 必須 |
| `ingredient_name` | 原材料名 | 必須 |
| `ingredient_unit` | 原材料の単位。確認用 | 任意 |
| `quantity` | 1食あたり使用量 | 必須 |

インポート時に、`menu_name` に一致するメニュー、`ingredient_name` に一致する原材料が見つからない行は取り込まれません。先にメニューと原材料を登録してからCSVを読み込んでください。

## 品切れ連動

レシピに紐づく原材料の在庫がアラート閾値以下になると、その原材料を使うメニューを店舗メニュー側で品切れ扱いにできます。

- 会計完了後、在庫が減ったタイミングで品切れ同期が実行されます。
- 棚卸しで在庫が回復した場合は、在庫状態に応じて品切れ解除も行われます。
- 店舗メニューで手動の品切れ設定をしている場合は、在庫回復後に店舗側の表示も確認してください。

::: warning レシピ未設定の商品は自動判定されません
商品にレシピがない場合、POSLAはその商品がどの原材料を使っているか判断できません。売れ筋商品、限定商品、仕込み量が厳しい商品から優先してレシピを設定してください。
:::

## 棚卸し

営業終了後や月末など、店舗の運用に合わせて棚卸しを行います。

1. **原材料一覧** を開きます。
2. **棚卸し開始** を押します。
3. 各原材料の実在庫を入力します。
4. **棚卸し保存** を押します。
5. 保存後、低在庫アラートや品切れ表示を確認します。

実在庫とシステム上の在庫がずれている場合は、以下を確認してください。

- レシピの使用量が実際の仕込み量と合っているか
- レシピ未設定の商品が売れていないか
- 廃棄、まかない、試作、ロスを手入力で調整していないか
- 返金や注文取消後の在庫戻しを棚卸しで補正したか
- 複数店舗で同じ原材料を扱う場合、店舗単位の運用ルールが決まっているか

## AI需要予測

過去の販売傾向、原材料在庫、レシピ設定をもとに、仕込みや発注の参考値を確認できます。AIアシスタントでは、在庫が少ない食材や、売れ筋メニューに必要な原材料を踏まえた相談ができます。

ただし、天候、周辺イベント、臨時休業、団体予約、仕入れ遅延など、システムに反映されない事情は店舗判断で調整してください。

## 日次運用のおすすめ

| タイミング | やること |
|---|---|
| 開店前 | 主要原材料の在庫数と品切れ表示を確認 |
| ランチ後 | 売れ筋商品の原材料が夜営業分まで足りるか確認 |
| 夕方 | 限定メニューや品切れ候補を店舗メニューで確認 |
| 閉店後 | 棚卸し、廃棄・ロスの確認、翌日の仕込み判断 |
| 月末 | 原材料原価、レシピ設定、棚卸し差分を見直し |

## 困ったとき

| 症状 | 対処 |
|---|---|
| 在庫数が合わない | 棚卸し、廃棄、仕入れ、手入力の履歴を確認してください。 |
| 品切れ表示が合わない | 商品とレシピの紐づけ、低在庫の目安、店舗メニュー側の手動品切れ設定を確認してください。 |
| 需要予測が極端 | 休業日、イベント、異常値が含まれていないか確認してください。 |
| CSVで文字化けする | UTF-8形式で保存してから取り込んでください。 |
| レシピCSVが取り込まれない | `menu_name` と `ingredient_name` が登録済みの名称と一致しているか確認してください。 |
| 会計しても在庫が減らない | レシピが設定されているか、商品がメニューテンプレートに紐づいているか確認してください。 |
