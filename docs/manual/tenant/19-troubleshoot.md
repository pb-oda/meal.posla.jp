# 19. トラブルシューティング

問題が起きたときは、画面名、操作内容、表示されたメッセージ、発生時刻を確認してください。POSLAサポートへ連絡する際にも、この情報があると確認が早くなります。

## まず確認すること

1. インターネット接続を確認します。
2. ブラウザを再読み込みします。
3. 別の端末またはブラウザで試します。
4. ログイン中のユーザーと権限を確認します。
5. 店舗選択が正しいか確認します。

## よくある症状

| 症状 | 確認すること |
|---|---|
| ログインできない | ユーザー名、パスワード、アカウント状態 |
| 注文が届かない | Wi-Fi、KDS画面、店舗・テーブル設定 |
| 会計できない | 注文状態、支払い方法、PIN、決済端末 |
| メニューが出ない | 商品の販売状態、カテゴリ、店舗表示設定 |
| QRコードが読めない | 印刷状態、貼り間違い、テーブル開放状態 |
| レポートが合わない | 期間、営業日区切り、会計済み状態 |
| 画面に黄色いバナーで「古いデータを表示中」と出る | 通信が一時的に切れています。**操作は通信復帰後に行ってください**。詳細は『KDS デバイスガイド』「通信が一時的に切れた場合の挙動」 |

## POSLAサポートへ連絡する前に

以下を控えてください。

- 店舗名
- 操作した画面
- 操作したユーザー名
- 発生日時
- 表示されたメッセージ
- 可能であればスクリーンショット

## 緊急時の考え方

営業中に注文や会計が止まる場合は、まず店舗の代替運用に切り替えてください。紙伝票、手書き会計、手動レジ締めなどの緊急対応を行い、営業後にPOSLAサポートへ状況をご共有ください。

## 困ったとき

管理画面右下のAIヘルプデスクも利用できます。質問時は「どの画面で」「何をしたら」「どのメッセージが出たか」を入力してください。
