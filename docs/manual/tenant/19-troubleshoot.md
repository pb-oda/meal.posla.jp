# 19. トラブルシューティング

よくある問題と対処法をまとめています。

## 19.1 ログイン関連

### ログインできない

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 「ユーザー名またはパスワードが正しくありません」 | 入力ミス | 大文字/小文字を確認。ユーザー名は全て小文字の場合が多い |
| 「このアカウントは無効化されています」 | アカウント無効化 | オーナーまたはマネージャーに連絡してアカウントを有効化してもらう |
| 「このテナントは無効化されています」 | テナント自体が無効 | POSLA運営に連絡 |
| 「通信エラーが発生しました」 | インターネット未接続 | Wi-Fi/モバイル通信を確認して再試行 |
| 「サーバーが空のレスポンスを返しました」 | サーバー障害 | しばらく待って再試行。繰り返す場合はPOSLA運営に連絡 |

### セッションが頻繁に切れる

- 25分間操作がないとタイムアウト警告が表示されます
- 30分で自動ログアウトされます
- ブラウザのクッキーを削除した場合もログアウトされます
- 解決策：操作を継続するか、警告が出たら「操作を続ける」ボタンを押す

## 19.2 注文関連

### セルフオーダーで注文できない

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 「注文する」ボタンが無効 | ラストオーダー中 | マネージャーがラストオーダーを解除する（フロアマップの「LO解除」） |
| QRコードを読み取れない | QRコードが古い | 管理ダッシュボードで新しいQRコードを生成・印刷 |
| 「品切れの品目が含まれています」 | カート内品目が品切れ | 該当品目をカートから削除して再送信 |
| 画面が真っ白 | セッショントークン期限切れ | テーブルのセッショントークンが更新されている。新しいQRコードを使用 |

### ハンディPOSで注文が送信されない

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 「カートが空です」 | カートに品目がない | メニューから品目を選択してカートに追加 |
| 「テーブルを選択してください」 | テーブル未選択 | ドロップダウンからテーブルを選択 |
| 通信エラー | ネットワーク問題 | Wi-Fi接続を確認。安定した接続を確保 |

---

## 19.3 KDS関連

### KDSに注文が表示されない

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 全く注文が表示されない | ステーション選択の問題 | 「全品目」ボタンを選択して確認 |
| 一部の品目だけ表示されない | ステーションのカテゴリ割当 | 店舗設定でステーションの担当カテゴリを確認 |
| 更新が遅い | ポーリング | 3秒間隔で自動更新されます。手動更新はありません |
| 音が鳴らない | 音声未許可 | KDS起動時の「タップして開始」を確実に行う |

## 19.4 会計・レジ関連

### レジで会計できない

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 会計待ちテーブルが表示されない | 未会計の注文がない | 注文が入っているか確認 |
| 「会計する」ボタンが押せない | 処理中 | 二重押し防止。しばらく待つ |
| カードリーダーが接続できない | Stripe Terminal問題 | 「再検出」ボタンで再接続。端末の電源を確認 |

### レジの過不足が大きい

- レジレポートでキャッシュログを確認し、入金・出金の記録漏れがないか確認
- 出金を記録せずに現金を取り出していないか確認
- 閾値（デフォルト¥500）を超える過不足は自動で警告表示されます

## 19.5 ネットワーク関連

### オフライン警告が表示される

POSLAはクラウド型システムのため、常時インターネット接続が必要です。

- 画面上部に赤いバナーで「オフライン」警告が表示されます
- オフライン中は新規注文の送信やデータ取得ができません
- 接続が復旧すると自動的に緑のバナーが表示され、通常動作に戻ります

**対処法：**
1. Wi-Fiルーターの状態を確認
2. モバイル通信に切り替えて試す
3. 有線LAN接続を検討（安定性向上）
4. バックアップ用のモバイルルーターを準備

### セルフオーダーの注文がオフライン時に保留される

お客さんのスマートフォンがオフラインの場合、注文はブラウザに保留されます。接続が復旧すると自動的に送信されます。24時間以上経過した保留注文は自動破棄されます。

## 19.6 メニュー関連

### メニュー変更が反映されない

管理ダッシュボードでメニューを変更した後、各画面への反映タイミングは以下のとおりです。

| 画面 | 反映タイミング |
|------|-------------|
| セルフオーダー | メニューバージョンチェック（3分間隔）またはページリロード |
| ハンディPOS | メニューバージョンチェック（3分間隔） |
| KDS | 次の注文受信時 |

急ぎの場合は、各画面でブラウザをリロード（F5）してください。

---

## 19.7 シフト関連

### GPS打刻でエラーが出る

| 症状 | 原因 | 対処法 |
|------|------|--------|
| 位置情報の許可エラー | ブラウザの位置情報が無効 | ブラウザの設定で位置情報を許可する |
| 距離エラー | 店舗から離れすぎている | 店舗の許容半径内（シフト設定で確認）に移動してから打刻 |
| 「既に出勤中です」 | 二重打刻 | 前回の退勤打刻ができていない。マネージャーに確認 |

---

## 19.8 推奨環境の確認

問題が解決しない場合、以下の環境を確認してください。

| 項目 | 推奨 |
|------|------|
| ブラウザ | Google Chrome（最新版）推奨 |
| 管理ダッシュボード | PC/タブレット（10インチ以上） |
| KDS | タブレット（10インチ以上、横置き） |
| ハンディPOS | スマートフォン（5〜8インチ） |
| インターネット | 安定した接続（Wi-Fi推奨、有線LANがベスト） |

---

## 19.9 POSLA運営への問い合わせ

上記で解決しない場合は、POSLA運営（プラスビリーフ）にお問い合わせください。

問い合わせ時に以下の情報をお伝えいただくと、迅速な対応が可能です。

- テナント名・店舗名
- 問題が発生した日時
- 問題の症状（エラーメッセージがあれば正確に）
- 使用しているブラウザとOS
- 問題が発生する画面（管理ダッシュボード/KDS/ハンディPOS/セルフオーダー等）
