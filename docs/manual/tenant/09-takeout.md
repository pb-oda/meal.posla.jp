# 9. テイクアウト管理

POSLA のテイクアウトは、店頭の注文管理とは別に「お客様の事前注文」「オンライン決済」「受取時間枠」「厨房キャパ」「梱包確認」「準備完了通知」「受取遅れ・キャンセル・返金対応」をまとめて扱うための機能です。

この章では、お店で実際に使う画面と運用手順を説明します。

## 利用前に確認する設定

管理画面の設定で、テイクアウトを有効にします。

| 設定 | 内容 |
|---|---|
| テイクアウト有効 | お客様向けテイクアウト注文を受け付けるかどうか |
| 最短準備時間 | 注文時点から最短で何分後の受取を許可するか |
| 受付開始・受付終了 | お客様が選べる受取時間帯 |
| 時間枠あたり最大注文数 | 同じ受取時間に受けられる注文数 |
| 時間枠あたり最大品数 | 同じ受取時間に受けられる合計商品数。0 の場合は品数上限なし |
| ピーク時間帯 | 混雑時間だけ別の注文数・品数上限を使う場合に設定 |
| 混雑時追加待ち時間 | 忙しい時間帯の受取可能時刻を遅らせるための追加時間 |
| SLA警告 | 受取予定時刻の何分前から「遅れそう」と表示するか |
| オンライン決済 | お客様が事前決済できるようにする設定 |

お客様向けテイクアウト注文は、現在の実装ではオンライン決済を前提にしています。Stripe の店舗別決済設定が未完了の場合、注文画面では決済へ進めません。

## お客様の注文手順

1. お客様がテイクアウト注文画面を開きます。
2. 商品を選びます。
3. 名前・電話番号・受取時刻・必要に応じてメモを入力します。
4. 受取時間を選択します。
5. オンライン決済に進みます。
6. 決済完了後、注文詳細画面で注文状況を確認できます。

受取時間は、店舗の受付時間・最短準備時間・時間枠上限・ピーク時間設定・既存注文数をもとに自動判定されます。上限に達している時間帯は、お客様が選べない状態になります。

## 再注文候補

お客様が同じ端末で名前・電話番号を入力すると、過去のテイクアウト注文から再注文候補を表示できます。

再注文候補は、次の条件を満たすものだけが対象です。

| 条件 | 内容 |
|---|---|
| 同じ店舗 | 別店舗の注文は表示されません |
| 電話番号一致 | 入力された電話番号で過去注文を検索します |
| 完了済み注文 | 準備完了・受渡済み・会計済みの注文が候補になります |
| 現在販売中の商品 | 現在のメニューに存在し、売り切れではない商品だけを再追加します |

過去に注文した商品でも、現在メニューにない商品や売り切れの商品は再注文に含まれません。

## 受取前の状況確認

お客様は注文詳細画面で、次の内容を確認できます。

| 表示 | 内容 |
|---|---|
| 注文状態 | 支払待ち、受付済み、調理中、準備完了、受渡済みなど |
| 受取予定時刻 | 注文時に選んだ受取時刻 |
| 注文内容 | 商品名、数量、金額 |
| 受取到着連絡 | 店頭到着または車内待機を店舗へ知らせる操作 |
| LINE連携案内 | 店舗側の LINE 連携が有効な場合に表示 |

受取到着連絡では「店頭に着きました」「車で待っています」を選べます。メモも入力できます。キャンセル済み・受渡済み・会計済みの注文では、到着連絡はできません。

## 店舗側のテイクアウト管理

店舗側では、テイクアウト管理画面で当日の注文を確認します。

上部には、次の集計が表示されます。

| 集計 | 内容 |
|---|---|
| 本日の注文数 | 表示対象日のテイクアウト注文数 |
| SLA警告 | 受取予定時刻が近く、遅れそうな注文 |
| 遅延中 | 受取予定時刻を過ぎても完了していない注文 |
| 梱包未完了 | 梱包チェックが完了していない注文 |
| 通知失敗 | 準備完了通知の送信に失敗した注文 |
| 返金対応 | キャンセル・返金対応が必要な注文 |
| 到着待機 | お客様から到着連絡が入っている注文 |

管理画面の **運営管理 → 運営コックピット** にも、テイクアウト遅延リスクが表示されます。受取時刻が近い未完了注文がある場合は、テイクアウト管理を開いて、調理、梱包、通知、受け渡しを確認してください。

## 店舗側の表示とフィルター

テイクアウト管理では、注文状態や運用状態を見ながら、受取時刻順に対応します。

主な見方:

| 見る場所 | 内容 |
|---|---|
| 上部KPI | 本日の注文、SLA警告、遅延中、梱包未完了、通知失敗、返金対応、到着待機 |
| 注文カード | 受取時刻、注文者、電話番号、商品、合計、支払い、メモ |
| ステータス操作 | 受付済み、調理中、準備完了、受渡済みへ進める |
| 梱包チェック | 商品、箸、袋、ソース、領収書の確認 |
| 通知状態 | 未送信、送信済み、送信失敗、スキップ |
| 運用ステータス | 遅延リスク、お客様遅れ、キャンセル依頼、返金待ちなど |

準備完了にした注文は、お客様へ受け渡すまで **準備完了** のまま残ります。受け渡し後は **受渡済み** にしてください。準備完了のまま閉店すると、閉店前チェックで確認対象になります。

## 注文ステータス

店舗側の主なステータス遷移は次の通りです。

| 現在 | 次に進める状態 | 主な操作 |
|---|---|---|
| 支払待ち | 受付済み、キャンセル | 決済後の受付、またはキャンセル |
| 受付済み | 調理中 | 調理開始 |
| 調理中 | 準備完了 | 商品完成 |
| 準備完了 | 受渡済み | お客様へ受け渡し |

準備完了にすると、店舗の LINE 連携設定とお客様の LINE 紐付け状態に応じて、準備完了通知が送信されます。送信できなかった場合は、管理画面に失敗として表示されます。

## 梱包チェックリスト

注文ごとに、次の梱包チェックを確認します。

| チェック | 内容 |
|---|---|
| 商品 | 注文商品が揃っているか |
| 箸 | 必要な箸・カトラリーが入っているか |
| 袋 | 袋に入っているか |
| ソース | ソース・調味料が揃っているか |
| 領収書 | 領収書が必要な場合に入っているか |

すべてにチェックが入ると、梱包完了として扱われます。

## 運用ステータス

注文ステータスとは別に、店舗内部の対応状況を管理できます。

| ステータス | 用途 |
|---|---|
| 通常 | 特別対応なし |
| 遅延リスク | 遅れそうな注文 |
| 遅延中 | 受取予定時刻を過ぎている注文 |
| お客様遅れ | お客様がまだ来店していない注文 |
| キャンセル依頼 | キャンセル相談・依頼中 |
| キャンセル済み | キャンセル処理済み |
| 返金待ち | 返金確認が必要 |
| 返金済み | 返金対応済み |

キャンセル済みにすると、受渡済み・会計済みでない注文はキャンセル状態になります。返金そのものは決済事業者側の処理も確認してください。

## 準備完了通知

準備完了通知は、注文を「準備完了」にしたタイミングで送信されます。店舗側から再送操作もできます。

通知状態は次のいずれかで表示されます。

| 表示 | 意味 |
|---|---|
| 未送信 | まだ通知していません |
| 送信済み | 通知送信に成功しました |
| 送信失敗 | 通知送信に失敗しました |
| スキップ | LINE連携やお客様の紐付けがなく、送信対象外です |

現在のテイクアウト準備完了通知は LINE 連携を前提にしています。SMS通知はテイクアウトの標準機能としては実装されていません。

LINE連携が未設定、またはお客様が注文とLINEを紐付けていない場合、通知状態はスキップまたは失敗として表示されます。この場合は、電話など店舗の通常連絡手段で連絡してください。

## 誤会計の取消とオンライン決済返金

POS レジでゲートウェイを経由せずに会計したテイクアウトは、POS レジ側の取引ジャーナルから会計取消できる場合があります。現金、手動 card、手動 qr など、ゲートウェイ非経由の会計が対象です。

オンライン決済済みのテイクアウト注文で誤会計やキャンセルが発生した場合は、会計取消ではなく返金対応になります。POSLA の運用ステータスで「返金待ち」「返金済み」を管理し、決済事業者側の返金状態も確認してください。

## 受取遅れ・キャンセル・返金対応

遅れそうな注文は SLA警告として表示されます。受取予定時刻を過ぎても準備完了または受渡済みになっていない場合は、遅延中として扱われます。

キャンセルや返金が必要な場合は、運用ステータスを「キャンセル依頼」「キャンセル済み」「返金待ち」「返金済み」に変更して、対応状況を残します。オンライン決済の返金は、POSLA 上の表示だけで完了するものではありません。決済管理画面または決済連携側の返金状態も確認してください。

## お客様画面でのエラー表示

Stripe 決済が完了したのに、POSLA 側で注文の記録に失敗したごく稀なケースでは、お客様の画面に次の内容が表示されます。

```text
決済確認に失敗しました
決済は Stripe 側で完了している可能性があります。
お手数ですが、下記の注文番号を添えて店舗まで直接お問い合わせください。
注文番号: XXXXXXXX
```

この場合は、注文番号と発生時刻を POSLA サポートへ共有し、店舗側でも同じ時間帯の注文一覧と決済状況を確認してください。

## 閉店前チェック

営業終了前に、次を確認します。

- 準備完了のまま受渡済みになっていない注文が残っていないか
- 梱包未完了の注文がないか
- 通知失敗の注文に電話などで連絡済みか
- キャンセル依頼・返金待ちの注文が残っていないか
- 到着連絡がある注文を見落としていないか

## よくあるトラブル

| 困ったこと | 確認すること |
|---|---|
| お客様が注文できない | テイクアウト有効、受付時間、オンライン決済設定、売り切れ商品を確認 |
| 受取時間が表示されない | 最短準備時間、時間枠上限、ピーク上限、受付時間を確認 |
| 準備完了通知が届かない | 店舗の LINE 設定、お客様の LINE 紐付け、通知失敗表示を確認 |
| 再注文に商品が出ない | 現在のメニューに商品があるか、売り切れでないかを確認 |
| 返金済みか分からない | POSLA の運用ステータスと決済事業者側の返金状態を両方確認 |
| 注文を停止したい | テイクアウト設定を無効にし、案内QRの掲示も確認 |
