# 13. レポート

レポートでは、売上、会計、商品、客席、スタッフ、在庫、予約、満足度、カート離脱などの状況を確認できます。日々の振り返り、問い合わせ対応、月次改善に使います。

## 主なレポート

| レポート | 確認できること |
|---|---|
| 週次サマリー | 直近1週間の売上、客数、客単価、前週比、人気商品Top3 |
| 売上レポート | 日別、時間帯別の売上と客数 |
| 売上ドリルダウン | 日別 → 商品 → 注文 → 会計履歴までの追跡 |
| レジ分析 | 現金、入金、出金、差額、外部決済控え、返金、締め前チェック |
| 注文履歴 | 過去注文、注文内容、取消、問い合わせ時の確認 |
| 回転率・客層分析 | テーブル回転、滞在時間、調理時間、人数傾向 |
| スタッフ評価・監査 | 勤怠、担当別実績、操作履歴、打刻精度 |
| 満足度分析 | 評価件数、平均評価、低評価の理由・品目・時間帯 |
| 併売分析 | 一緒に注文されやすい品目の組み合わせ |
| カート離脱・キャンセル分析 | カートに入れた後で外された商品、注文内キャンセル |
| 緊急会計台帳 | 緊急レジで記録した会計の確認 |
| 予約レポート | 予約件数、来店数、キャンセル、no-show、時間帯傾向 |

## 週次サマリー

売上レポートの先頭には、直近7日間の週次サマリーが表示されます。

確認できる項目:

- 売上合計
- 客数
- 客単価
- 前週比
- 人気商品Top3

週次サマリーは、日々の細かい伝票確認ではなく、店長が「今週の流れが良いか悪いか」を早く見るためのものです。詳細な注文や会計は、注文履歴、レジ分析、売上ドリルダウンで確認してください。

## 売上ドリルダウン

管理画面の **運営管理 → 売上ドリル** では、売上を次の順で追跡できます。

1. 期間内の日別売上
2. 指定日の商品別売上
3. 指定商品を含む注文一覧
4. 注文詳細と会計履歴

使いどころ:

- 「この商品の売上がなぜ伸びたか」を注文単位で確認する
- 問い合わせ時に、注文IDから会計履歴まで確認する
- 返金や取消の可能性がある注文を、レジ分析と突き合わせる
- 人気商品や売上上位商品の提供時間・時間帯を確認する

売上ドリルで表示される商品別売上は、注文内の品目情報から集計します。会計済み注文を中心に見るため、未会計や取消が混ざる場合は注文履歴・レジ分析も確認してください。

## 満足度分析

管理画面 → レポート → 満足度分析 から開きます。お客様のセルフ注文画面で送信された評価（1〜5）を集計し、店舗改善に活用できます。

- **サマリー**: 評価件数、平均評価、低評価件数（1〜2）、低評価率、高評価件数（4〜5）
- **低評価の理由ランキング**: 提供が遅かった／味／温度／量／注文内容の相違／接客／清潔感／価格／その他
- **低評価が出た品目ランキング**: どの品目に低評価が集中しているか
- **時間帯別 低評価**: ピーク時に提供品質が落ちていないか確認
- **直近の低評価一覧**: 日時・品目・評価・理由・自由コメント・テーブル・注文ID

::: tip 低評価は改善のヒントとして活用してください
低評価はスタッフ個人の責任ではなく、「品目・時間帯・店舗運営の改善点」として活用する前提で設計しています。顧客セルフ注文ではスタッフ担当者が特定されないケースも多いため、まずは品目・時間帯・オペレーションの見直しをお勧めします。
:::

オーナー画面の「全店舗横断レポート」にも、全店舗合算の満足度サマリー・店舗別平均評価・低評価が多い品目・理由ランキングが表示されます。

## カート離脱・キャンセル分析

管理画面の **運営管理 → カート離脱** では、お客様がカートへ入れた後に外した商品を確認できます。注文履歴に残る注文内キャンセルも、記録がある環境では同じ画面に表示されます。

確認できる項目:

| 項目 | 内容 |
|---|---|
| カート投入 | 対象期間に商品がカートへ入った回数 |
| カート削除 | カートから外された回数 |
| 削除率 | カート投入に対する削除割合 |
| 最終 | 最後に投入または削除された時刻 |
| 注文内キャンセル | 注文後に外された商品、数量、注文ID |

削除率が高い商品は、次を確認してください。

- 写真が実物と合っているか
- 価格が分かりにくくないか
- 必須オプションが複雑すぎないか
- 説明文やアレルゲンが不足していないか
- 売り切れや代替提案の表示が適切か

カート離脱は「売れなかった理由」を断定するものではありません。改善候補を見つけるための参考データとして使います。

## レジ分析

レジ分析では、閉店時の現金差額や外部決済控えを確認します。

主な確認項目:

- 開局金額、入金、出金、締め金額
- 過不足額
- 支払い方法別売上
- 外部カード、外部QR、その他外部決済の控え番号
- 返金、取消、領収書再発行
- 閉店引き継ぎメモ

カード決済やQR決済は、POSLAが決済そのものを実行するのではなく、店舗が契約している外部端末の完了結果を記録します。差額や問い合わせがある場合は、外部端末の控えとPOSLAの会計記録を突き合わせてください。

## 注文履歴

注文履歴は、過去注文の検索と問い合わせ対応に使います。

見るポイント:

- 注文ID
- 注文日時
- テーブル、注文者情報
- 注文内容、数量、金額
- 注文ステータス
- 取消や返金に関係するメモ

お客様から「何を頼んだか」「会計済みか」「キャンセルしたか」を聞かれた場合は、注文履歴とレジ分析を合わせて確認します。

## 回転率・客層分析

回転率・客層分析では、テーブルの滞在時間、回転数、人数傾向、調理時間を確認します。

活用例:

- 予約枠の標準滞在時間を見直す
- ピーク時間のシフト人数を見直す
- 提供が遅れやすい時間帯を見つける
- テーブル配置や席種を見直す

回転率は、テーブル開放、着席、会計、退店処理が正しく行われているほど精度が上がります。ハンディやレジでセッション終了を忘れると、滞在時間が長く見えることがあります。

## スタッフ評価・監査

スタッフ評価・監査では、スタッフごとの操作や勤怠に関係する情報を確認します。

確認できる内容:

- 担当した注文数や操作履歴
- 打刻の有無、遅刻、早退、残業
- 交代/欠勤申請や承認記録
- 監査ログに残る重要操作

スタッフ評価は、共有アカウントでは正しく集計できません。POSLAでは、全スタッフが個別アカウントでログインする運用を前提にしています。

## 併売分析

併売分析では、一緒に注文されやすい品目の組み合わせを確認します。

活用例:

- セットメニューを作る
- おすすめ表示を見直す
- ドリンクやサイドメニューの提案を強化する
- KDSの提供順や仕込み量を調整する

併売は、単品の売上だけでは見えない組み合わせを見るための参考です。期間や曜日によって傾向が変わるため、ランチ、ディナー、週末などに分けて見ると判断しやすくなります。

## 期間の選び方

日報は営業終了後、週報は週明け、月報は月末または翌月初に確認する運用を推奨します。深夜営業の店舗では、営業日区切り時刻に注意してください。

深夜営業の店舗では、店舗設定の「営業日区切り時刻」により、暦日と営業日の集計が異なる場合があります。売上が想定と違う場合は、期間だけでなく営業日区切りも確認してください。

## CSV出力

必要に応じてCSVを出力し、会計ソフトや表計算ソフトで集計できます。出力後は個人情報や売上情報の取り扱いに注意してください。

## 活用例

- 売れ筋商品を確認して仕入れ量を調整する
- 回転率の低い時間帯を見つけて人員配置を見直す
- 返金やレジ差額を確認して運用改善につなげる
- スタッフ別の実績を評価や教育に活用する

## 困ったとき

| 症状 | 対処 |
|---|---|
| 数値が合わない | 期間、営業日区切り、会計済み状態を確認してください。 |
| CSVが開けない | UTF-8対応のソフトで開いてください。 |
| 特定店舗が出ない | 店舗選択と閲覧権限を確認してください。 |
| レジ差額がある | 入金、出金、返金、現金残高を確認してください。 |
| 商品別売上が注文履歴と違う | 会計済み注文、取消、返金、対象期間、営業日区切りを確認してください。 |
| カート離脱が多い | 写真、価格、説明、必須オプション、売り切れ表示を確認してください。 |
