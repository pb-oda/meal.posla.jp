# 10. プラン・コース管理

管理ダッシュボード（`dashboard.html`）の「プラン・コース」タブでは、**食べ放題プラン（時間制限付きプラン）** と **コース料理テンプレート** を管理します。マネージャー以上のロールで利用できます。

::: warning 用語の混同に注意
本章で扱う「プラン」は **飲食店のメニュープラン**（食べ放題・飲み放題など）です。POSLA の **サブスクリプション契約プラン**（月額 ¥20,000 等、α-1 構成）とは別概念です。サブスクリプション側は 21 章を参照してください。
:::

## 10.0 はじめにお読みください

### 10.0.1 この章の読み方

| あなたの店舗 | 重点的に読む節 |
|------------|-------------|
| 食べ放題・飲み放題を提供している | 10.1（食べ放題プラン）+ 10.4（運用） |
| コース料理（懐石・会席・コース盛り合わせ）を提供する | 10.2（コース料理）+ 10.5（運用） |
| 予約サイトからコース指定で来店受付したい | 10.5.3（コース予約連携）+ 24 章 |
| 単品（アラカルト）のみで運用 | 本章は読み飛ばし可。04 章（メニュー）のみで完結 |

### 10.0.2 「プラン」と「コース」の使い分け早見表

| 観点 | プラン（time_limit_plans） | コース（course_templates） |
|---|---|---|
| 課金モデル | **時間内 食べ放題 / 飲み放題**（定額） | **決まった料理セット**（定額） |
| お客様の自由度 | 対象品目から自由に何度でも注文 | 順番・品目はお店が決める |
| 時間制限 | あり（90分等。LO 連動） | 基本なし（推奨は 120 分以内） |
| KDS 表示 | 通常注文と同じ | フェーズごとに自動 / 手動投入 |
| 想定店舗 | 居酒屋、しゃぶしゃぶ、焼肉 | 懐石、会席、フレンチ、コース寿司 |
| 排他 | 同一テーブルでコースと同時不可 | 同一テーブルでプランと同時不可 |

### 10.0.3 操作前のチェックリスト

| # | 確認項目 | 確認方法 | NG 時の対応 |
|---|---------|--------|-----------|
| 1 | あなたのロールが manager 以上 | 画面右上 → ロール表示 | owner に依頼 |
| 2 | 対象店舗が選択されている | dashboard 上部の店舗セレクタ | 切替 |
| 3 | カテゴリが事前に作成済み | 04 章「カテゴリ管理」 | 先に 04 章でカテゴリ追加 |
| 4 | KDS が起動している（運用テストする場合） | レジ・キッチンの KDS 画面 | 17 章「KDS」 |
| 5 | テーブルが登録済み（ハンディから割当る場合） | dashboard「テーブル」タブ | 先に「+ テーブル追加」 |

## 10.1 食べ放題プラン

### 10.1.1 プラン一覧画面

「プラン・コース」タブ → **「食べ放題プラン」** サブタブを選択すると、登録済みプランの一覧が表示されます。

#### 表示要素一覧

| 表示要素 | 説明 |
|---------|------|
| プラン名 | 日本語名（英語名があればカッコ内に表示） |
| 制限時間 | 「90分」形式 |
| ラストオーダー | 「LO 15分前」形式 |
| 価格 | 「¥2,980」形式（税込） |
| ステータスバッジ | 緑色「有効」 / 赤色「無効」 |
| 並び順 | 表示順の数値（小さいほど上） |
| 「メニュー管理」ボタン | プラン専用メニューの展開（10.1.4 へ） |
| 「編集」ボタン | プランの編集モーダル |
| 「削除」ボタン | プランの削除（DB から完全削除） |

#### 画面イメージ

```
┌──────────────────────────────────────────┐
│  プラン・コース  >  食べ放題プラン         │
│                                          │
│  [+ プラン追加]                            │
│                                          │
│  ┌──────────────────────────────┐       │
│  │ 90分食べ放題（90min All-You）│       │
│  │ 90分 / LO 15分前 [有効]      │ ¥2,980│
│  │  [メニュー管理] [編集] [削除] │       │
```

### 10.1.2 プランの追加

#### 操作前チェックリスト

| # | 確認項目 |
|---|---------|
| 1 | 提供時間帯（ランチ食べ放題かディナーか）を決めている |
| 2 | 対象品目のリストアップ済み（後で 10.1.4 で登録） |
| 3 | 価格・LO（ラストオーダー）何分前かが決まっている |

#### クリック単位の操作手順

1. dashboard ログイン
2. 上部タブから **「プラン・コース」** をクリック
3. サブタブ **「食べ放題プラン」** を選択
4. 画面右上の **「+ プラン追加」** ボタンをクリック
5. モーダルダイアログが開きます
6. 各フィールドに入力（10.1.3 表参照）
7. 画面下部 **「追加」** ボタンをクリック
8. 「プランを追加しました」とトースト通知
9. 一覧に新規プランが追加される

#### 10.1.3 入力フィールドの詳細

| # | フィールド | 種類 | 必須 | 初期値 | 文字種・範囲 | 説明 |
|---|-----------|------|-----|------|-----------|------|
| 1 | プラン名 | テキスト | ✅ | — | 全角OK、最大100文字 | 例：「90分食べ放題」 |
| 2 | プラン名（英語） | テキスト | — | — | ASCII 推奨 | 例：「90min All-You-Can-Eat」 |
| 3 | 制限時間（分） | 数値 | ✅ | 90 | 1〜480（8時間） | プランの時間制限 |
| 4 | LO（終了N分前） | 数値 | — | 15 | 0〜120 | 終了の何分前にラストオーダー扱いとするか |
| 5 | 料金（税込） | 数値 | — | 0 | 0〜999,999 | プラン料金（人数分一括または1人分） |
| 6 | 表示順 | 数値 | — | 0 | -32,768〜32,767 | 小さいほど上 |
| 7 | 説明 | テキストエリア | — | — | 2行 | プランの説明文 |

#### 入力例ブロック

```
プラン名: しゃぶしゃぶ食べ放題 90分
プラン名（英語）: Shabu-Shabu 90min All You Can Eat
制限時間（分）: 90
LO（終了N分前）: 15
料金（税込）: 3800
表示順: 10
説明: 国産黒毛和牛のしゃぶしゃぶ食べ放題。野菜・締めの麺類も食べ放題対象。
```

### 10.1.4 プランの編集

既存プランの編集では、10.1.3 の項目に加えて以下が変更可能です。

| フィールド | 種類 | 説明 |
|-----------|------|------|
| 有効 / 無効 | セレクト | 無効にすると着席時のプラン選択肢から消える（既に適用中のセッションは維持） |

#### 操作手順

1. プラン一覧で対象プランの **「編集」** ボタンをクリック
2. モーダルに既存値が表示される
3. 変更したいフィールドを修正
4. **「更新」** ボタンをクリック
5. 「プランを更新しました」とトースト

::: tip プラン無効化の使いどき
- 季節限定プラン（夏季限定など）の終了
- メニュー仕入れ都合で一時停止
- 料金改定の準備期間（旧プラン無効 → 新プラン作成）
:::

### 10.1.5 プラン専用メニュー（plan_menu_items）

食べ放題プランには **プラン専用の品目** を登録できます。これらは通常メニュー（menu_templates）とは別管理で、プラン契約者だけが注文できます。

#### 専用メニューの表示・追加

1. プラン一覧で対象プランの **「メニュー管理」** ボタンをクリック
2. ボタンが「閉じる」に変わり、プラン直下に専用メニュー一覧が展開
3. **「+ 品目追加」** ボタンをクリック → モーダル

#### 品目追加フォーム入力フィールド

| # | フィールド | 種類 | 必須 | 説明 |
|---|-----------|------|-----|------|
| 1 | 品目名 | テキスト | ✅ | 日本語名（例：「ビール」「ハイボール」） |
| 2 | 品目名（英語） | テキスト | — | 英語名 |
| 3 | カテゴリ | セレクト | — | 04 章のカテゴリから選択 |
| 4 | 説明 | テキストエリア | — | 2行 |
| 5 | 表示順 | 数値 | — | 0 |

::: tip 入力例
- ビール（カテゴリ：ドリンク）
- 生中ジョッキ（カテゴリ：ドリンク）
- ハイボール（カテゴリ：ドリンク）
- レモンサワー（カテゴリ：ドリンク）
:::

#### 専用メニューの編集・削除

各品目行右側の **「編集」** / **「削除」** ボタンから操作。編集時は「表示」セレクトで「無効」にもできます（プラン適用中の品切れ運用に便利）。

### 10.2.2 コースの追加

#### 操作前チェックリスト

| # | 確認項目 |
|---|---------|
| 1 | コースの全フェーズと提供順が決まっている |
| 2 | 各フェーズの品目と数量がリストアップ済み |
| 3 | フェーズ間の自動進行 / 手動進行ポリシーを決めている |

#### 操作手順

1. dashboard > プラン・コース > **「コース管理」** サブタブ

4. 入力フィールドに記入
5. **「追加」** ボタン
6. 「コースを追加しました」とトースト
7. 一覧の末尾にコース行が追加される
8. 続けて **「+ フェーズ追加」** で各フェーズを登録（10.2.3）

#### 入力フィールドの詳細

| # | フィールド | 種類 | 必須 | 初期値 | 説明 |
|---|-----------|------|-----|------|------|
| 1 | コース名 | テキスト | ✅ | — | 例：「松コース」 |
| 2 | コース名（英語） | テキスト | — | — | 例：「Matsu Course」 |
| 3 | 料金（税込） | 数値 | — | 0 | コース料金（人数分一括または1人分） |
| 4 | 表示順 | 数値 | — | 0 | 一覧上の順番 |
| 5 | 説明 | テキストエリア | — | — | 2行 |

### 10.2.3 フェーズの追加

コース料理は **フェーズ（段階）** で構成されます。各フェーズは順番に提供される料理のグループです。

#### 操作手順

1. コース一覧の対象コースの **「+ フェーズ追加」** ボタン
2. モーダルが開く
3. 入力フィールドに記入
4. **「追加」** ボタン
5. フェーズ番号は自動的に **連番**（既存最大 +1）で付与される

#### 入力フィールドの詳細

| # | フィールド | 種類 | 必須 | 説明 |
|---|-----------|------|-----|------|
| 1 | フェーズ名 | テキスト | ✅ | 例：「前菜」「メイン」「デザート」 |
| 2 | フェーズ名（英語） | テキスト | — | 例：「Appetizer」 |
| 3 | 自動発火（前フェーズ完了後N分） | 数値 | — | 空欄＝手動。値あり＝N分後に自動投入 |
| 4 | 品目 | テキストエリア | ✅ | 1行に1品目（10.2.4 参照） |

#### 10.2.4 品目の入力形式

```
サラダ
スープ ×2
刺身盛り合わせ ×1
```

##### パース規則

- **数量を省略した場合は 1** として扱われます
- **`×`**（全角ばつ）と **`x`**（半角エックス）どちらも使用可能
- 1行が空白のみなら無視
- 品目名と数量の間にスペース必須（`サラダ×2` ではなく `サラダ ×2`）
- 数量は半角整数のみ（`サラダ ×二` は弾かれる）

##### 入力例ブロック

```
先付け盛り合わせ
お椀（季節の汁物）
お造り 三種盛り ×1
焼物 季節魚 ×1
天ぷら 五種 ×1
ご飯
香の物
水菓子
```

### 10.2.5 フェーズの発火方式

| 方式 | 表示 | 動作 |
|------|------|------|
| 手動 | バッジ「手動」 | KDS 画面の「次フェーズ」ボタンで進める |
| 自動 | バッジ「自動 N分後」 | 前のフェーズ開始からN分後に自動で次のフェーズの料理が KDS に表示される |

::: tip 発火方式の選び方
- **回転率重視（居酒屋・大衆向け）**: 自動進行（10〜15 分間隔）
- **高級店・お客様のペース重視**: 手動進行
- **混合**: フェーズ1（前菜）は手動、フェーズ2以降は自動 など
:::

### 10.2.6 フェーズの編集

各フェーズの **「編集」** ボタンから 10.2.3 と同じモーダルで変更可能。

::: warning フェーズ番号の変更は要注意
モーダル内では `phase_number` の直接編集 UI は非表示（API は対応）。フェーズの並び替えは現状「削除して再追加」が推奨です。
:::

### 10.2.7 フェーズの削除

各フェーズの **「削除」** ボタン → 確認ダイアログ → **「OK」**。

削除後、`course_templates.phase_count` が自動で再カウント（残フェーズ数）されます。

::: warning コース削除時の注意
コースを削除すると、そのコースに紐づく **全てのフェーズも削除されます**（カスケード削除）。
:::

## 10.3 着席時のプラン・コース選択

### 10.3.1 ハンディからの割当

#### 前提
- ハンディ（`/public/handy/index.html`）に device アカウント or staff/manager でログイン
- 対象店舗の GPS 圏内（device は強制 GPS チェック）

#### 操作手順

1. ハンディトップ → **「テーブル状況」** タブ
2. 着席させたいテーブルをタップ
3. **「着席」** ボタンをタップ
4. 着席モーダルが開く
5. **人数入力** + **プラン or コース選択**（プルダウン）
6. **「着席を確定」** をタップ

#### 着席モーダル ASCII

```
┌─────────────────────────┐
│  T-12 着席登録           │
│                         │
│  人数: [ 4 ] 人           │
│                         │
│  プラン:                  │
│  ┌──────────────────┐  │
│  │ なし（アラカルト）   ▼│  │
│  │ 90分食べ放題          │  │
│  │ ランチ食べ放題         │  │
│  └──────────────────┘  │
│                         │
│  コース:                  │
│  ┌──────────────────┐  │
│  │ なし                  ▼│  │
│  │ 松コース ¥8,000      │  │
│  │ 竹コース ¥5,000      │  │
│  └──────────────────┘  │
│                         │
│  [キャンセル] [着席を確定]│
└─────────────────────────┘
```

### 10.3.2 セルフメニュー側の表示

お客様が QR スキャンで開く `menu.html` 側では、着席時に設定されたプラン・コース情報が以下のように反映されます。

| 種類 | お客様画面での表示 |
|------|-----------------|
| プラン適用 | プラン対象品目は **¥0** で表示。プラン外品目は通常価格 |
| コース適用 | フェーズ進行を上部バナーで表示。「現在 1/3 フェーズ：前菜」 |
| LO アラート | プラン LO 時刻に達すると「ラストオーダーまで残り N 分」バナー |
| プラン時間タイマー | 残り時間の常時表示（緑→黄→赤） |

### 10.3.3 排他制御

::: warning プランとコースは同テーブル同時不可
1 つのテーブルセッションに **プランとコースの両方** を設定することはできません。同じグループ内でプランの人とコースの人を分けたい場合は以下のいずれか：

- **別テーブル** に分かれて着席
- **サブセッション**（F-QR1 / sub-sessions）で 1 テーブルを論理分割
:::

## 10.4 食べ放題プランの運用

### 10.4.1 想定運用例

| 項目 | 値 |

| 価格 | ¥1,500 |

#### ケース 2：しゃぶしゃぶ食べ放題

| 項目 | 値 |
|------|------|
| プラン名 | 「しゃぶしゃぶ食べ放題 90 分 ¥3,800」 |
| 制限時間 | 90 分 |
| LO | 15 分前 |
| 営業時間帯 | 16:00 〜 21:00 |
| 対象品目 | 肉・野菜・締め麺類カテゴリ |
| 価格 | ¥3,800（大人）|

#### ケース 3：ランチ食べ放題

| 項目 | 値 |
|------|------|
| プラン名 | 「ランチビュッフェ 60 分 ¥1,200」 |
| 制限時間 | 60 分 |
| LO | 10 分前 |
| 営業時間帯 | 11:30 〜 14:30 |
| 対象品目 | サラダ、スープ、メイン、ドリンク（アルコール除く） |

### 10.4.2 プラン適用のタイミング

```
[着席]
   ↓ ハンディから「90分食べ放題」を選択
[セッション開始 18:00]
   ↓ time_limit_plans + table_sessions.plan_id 紐付け
[お客様セルフメニュー]
   ↓ プラン対象品目は ¥0 表示、プラン外は通常価格
[注文 → KDS → 提供] (回数無制限)
   ↓
[19:30 LO アラート発火]（90 - 15 = 75 分経過時点）
   ↓ セルフメニュー上部に「ラストオーダー」バナー
[19:45 プラン終了 = 注文不可]
   ↓ 新規カート追加ボタンが disabled
[20:00 退店]
   ↓ POS レジで会計
[請求] = プラン料金 × 人数 + プラン外注文（通常価格）
```

### 10.4.3 プラン時間管理

- 開始時刻：テーブルセッション開始時刻（着席ボタン押下時刻）
- 終了時刻：開始 + プラン時間（`duration_min`）
- ハンディ「テーブル状況」で **残り時間が常時表示**（色変化）
  - 残り 30 分以上：緑
  - 残り 10〜30 分：黄
  - 残り 10 分以下：赤
- LO 時刻に達するとセルフメニューで **ラストオーダーアラート**
- LO 後の追加注文は API 側で `LAST_ORDER_PASSED` エラー返却

### 10.4.4 アラカルト追加（プラン外注文）

プラン中でもプラン対象外の品目（特別メニュー・追加ドリンク等）は通常価格で注文可能。

- セルフメニューでプラン外品目を表示
- 通常カートに追加 → 通常価格で会計
- KDS 側ではプラン注文と同じレーンに表示

## 10.5 コース料理の運用

### 10.5.1 想定運用例：懐石コース ¥8,000

| フェーズ | 名称 | 自動発火 | 主な品目 |
|---------|------|---------|---------|
| 1 | 先付け（前菜） | 手動 | 季節の前菜盛り合わせ、お椀 |
| 2 | お造り | 自動 10 分後 | 三種盛り、あしらい |
| 3 | 焼物 | 自動 15 分後 | 季節魚塩焼き |
| 4 | 揚物 | 自動 15 分後 | 天ぷら五種 |
| 5 | 〆（食事） | 自動 15 分後 | 季節炊き込みご飯、汁物、香の物 |
| 6 | 水菓子 | 自動 10 分後 | 季節フルーツ、抹茶アイス |

### 10.5.2 自動進行 vs 手動進行

| モード | 動作 | 想定店舗 |
|------|------|--------|
| **自動進行** | 現フェーズ全品 served → `auto_fire_min` 経過 → 次フェーズ自動投入 | 標準コース、回転率重視 |
| **手動進行** | KDS の「次フェーズ」ボタン押下まで待機 | 高級店、お客様のペースに合わせる |

### 10.5.3 コース予約との連携

予約 LP（24 章）でお客様がコースを選んで予約 → 来店時に自動でテーブルにコース割当 → セッション開始でフェーズ 1 自動投入。

詳細フロー：

```
[予約 LP] お客様: 松コース ×4名 / 18:00
   ↓
[POSLA reservations テーブル: course_id 紐付け]
   ↓
[来店日 17:50 リマインド SMS / LINE 自動送信]
   ↓
[18:00 来店 → ホールスタッフがハンディで「予約から着席」]
   ↓
[table_sessions.course_id 自動セット]
   ↓
[フェーズ 1 自動 KDS 投入]
   ↓
[KDS でフェーズ 1 全品 served → auto_fire_min 経過 → フェーズ 2 投入]
```

### 10.5.4 KDS 側の操作

詳細は 17 章「KDS」参照。要点のみ：

- 各フェーズの料理が **同じテーブル番号 + フェーズ番号** で表示
- 全品 `served` ボタン → 自動進行モードならカウントダウン開始
- 手動モードなら **「次フェーズ」** ボタンで明示的に進行
- 途中の特定品目を **「取消」** すると `cancelled` 扱い、フェーズ進行には影響なし（アレルギー対応等）

## 10.X よくある質問（FAQ）

### Q1〜Q5: 概念・違い

**Q1. プランとコースの違い**

A. **プラン** は時間制（食べ放題 90 分等）、お客様が好きな品目を何度でも選べる。**コース** は料理セット（前菜→メイン→デザート）、決まった順番で料理が出る。両者は table_sessions レベルで排他。

**Q2. 1 つのテーブルにプランとコース両方は使える？**

A. **不可**。排他選択です。同じグループでもプランの人とコースの人を同席させたい場合は **別テーブル** にするか、**サブセッション**（F-QR1）で分割。

**Q3. 飲み放題と食べ放題を別プランにすべき？**

A. ケースバイケース。同価格・同時間なら 1 プランで対象品目に両方含める。価格や時間が異なる場合は 2 プラン作成して、お客様が選択。

**Q4. プランの時間は人数で延長される？**

A. 自動延長はされません。プラン時間は 1 セッション一律。人数で時間が変わる運用なら、人数別プランを複数登録（例：「2 名様 90 分」「4 名様 120 分」）。

**Q5. コースのフェーズ数に上限はある？**

A. DB 上は無制限ですが、運用上は 6〜8 フェーズが現実的です。10 を超えると KDS の表示が窮屈になります。

### Q6〜Q10: プラン運用

**Q6. プラン時間延長したい**

A. ハンディの該当テーブル → **【メモ編集】** → プラン情報修正（DB 直接更新）。または管理画面でセッションメモ修正。次バージョンで「+15 分延長」専用ボタン予定。

**Q7. プラン時間が過ぎたら自動で会計？**

A. 自動会計はしない。スタッフが手動でラストオーダーを促す or 退店時に POS レジで会計。

**Q8. プラン料金はどう計算される？**

A. プラン料金は固定（例：大人 ¥3,800）。テーブルにプラン設定 + 人数指定で `プラン料金 × 人数` が会計に追加される。

**Q9. プラン途中で追加注文（アラカルト）**

A. プラン外として別カート行で追加可。プラン対象外なので通常価格で会計。

**Q10. プラン対象品目の指定方法は？**

A. プラン編集画面の **「メニュー管理」** ボタン → プラン専用メニュー（plan_menu_items）に登録。ここに登録された品目だけがプラン中に ¥0 表示される。

### Q11〜Q15: コース運用

**Q11. コースフェーズの自動進行条件**

A. 各フェーズ全品目が `served` になり、`auto_fire_min`（例：15 分）経過したら自動的に次フェーズへ。手動進行モードならボタンを押すまで進まない。

**Q12. コースの途中で 1 品だけ抜きたい（アレルギー対応）**

A. KDS で該当品目を **【取消】** ボタン → `cancelled` 扱い。次フェーズには進む。

**Q13. コース途中でフェーズ順を入れ替えたい（緊急対応）**

A. KDS の「次フェーズ」ボタンで強制進行可能。`auto_fire_min` を待たずに進めれる。

**Q14. 複数テーブルで同じコースを並行進行できる？**

A. 可能。各テーブルセッションが独立して進行する。KDS は店舗全体のフェーズ状況を一覧表示。

**Q15. コース価格は人数 × 1 名分？それとも一卓？**

A. POSLA の標準は **人数 × 1 名分**。一卓固定にしたいなら、フェーズ品目側で「人数分まとめ」として登録するワークアラウンドあり。

### Q16〜Q20: レポート・売上

**Q16. プランやコースの売上はレポートに出る？**

A. 出る。13 章 13.2 商品別ランキングで「プラン売上」「コース売上」として集計。

**Q17. プラン適用時、注文された ¥0 品目はレポートでカウントされる？**

A. カウントされる（数量のみ。金額は ¥0）。プラン消費量の分析に使える。

**Q18. コースの全フェーズが完了したかどうかは記録される？**

A. `table_sessions.current_phase_number` で現フェーズが記録される。退店時に `current_phase_number == phase_count` なら全完了。

**Q19. 食べ放題で人気の品目を分析したい**

A. 13 章「商品別ランキング」で `plan_id` 絞り込み機能を使う（次バージョン予定。現状は CSV エクスポート → Excel 集計）。

**Q20. プラン売上 / コース売上の月次比較**

A. 13 章「期間比較」で前月との比較可能。コース別・プラン別の絞り込みは Q19 と同様、CSV ベース。

### Q21〜Q25: 連携・予約

**Q21. 予約サイト経由でコース予約を受けたい**

A. 24 章「予約管理」のコース連携を使う。予約時にコース選択 → 来店時に自動セット。

**Q22. 食べ放題の人数を後から変更したい**

A. ハンディから着席セッション編集で人数変更可。プラン料金は人数 × 単価で再計算される。

**Q23. プラン中の途中追加（19:30 着席が 20:00 に 1 名追加）**

A. ハンディからセッションの人数変更。追加分のプラン料金が会計に上乗せ（プラン時間は元の終了時刻のまま、追加メンバーは短い時間しか使えない）。

**Q24. ハッピーアワー（時間帯別プラン価格）**

A. 現状は 1 プラン = 1 価格固定。ハッピーアワー対応は別プラン（「ハッピーアワー食べ放題 16-18 時 ¥1,000」）を作成して使い分ける。

**Q25. コース予約でフェーズの自動投入を遅らせたい（料理人準備時間）**

A. フェーズ 1 の `auto_fire_min` を 5 分等に設定し、着席時にすぐ投入されないようにする（現状は仕様上 0 分扱いなのでカスタマイズ要相談）。

### Q26〜Q28: 設計・将来

**Q26. プランやコースの英語名はいつ表示される？**

A. セルフメニューの言語切替で英語に切り替えると表示。ハンディ・KDS は基本日本語。

**Q27. 将来追加予定の機能**

A.
- プラン延長専用ボタン（+15 分 / +30 分）
- コースフェーズの並び替え UI（現状は削除→再追加が必要）
- プラン売上の専用ダッシュボード
- 多言語対応（英語以外：中国語・韓国語）

**Q28. プランやコースを廃止したい**

A. 削除ボタンで物理削除可能。ただし過去のセッションで使われた `plan_id` / `course_id` は履歴として残るので、レポート上は引き続き集計される。

## 10.Y トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| プランが選択肢に出ない | プランが `is_active=0` | プラン編集で「有効」トグル ON |
| プラン時間が表示されない | テーブルにプラン未設定 | ハンディからプラン設定 |
| コース料理がフェーズ進まない | 全品 `served` にしてない / `auto_fire_min` 未経過 | KDS で全品 `served` にする or 「次フェーズ」ボタン |
| 時間超過しても料金変わらない | 仕様 | 別途精算（手動でアラカルト追加） |
| プランで¥0の品目が金額計上された | プラン専用品目（plan_menu_items）に未登録 | プラン編集 → 「メニュー管理」で品目追加 |
| コース内の特定品目が出ない | 品切れ | 在庫確認 + 代替品目の設定 |
| プラン専用品目がセルフメニューに出ない | プラン未適用のテーブル | 着席時にプラン選択 |
| コース追加で「migration_required」エラー | DB マイグレーション未適用 | `sql/migration-*.sql` を運営に依頼 |
| フェーズ品目入力で ×3 が認識されない | 全角/半角混在、スペース欠落 | `品名 ×3` の形式（前後半角スペース） |
| プラン料金が会計に乗らない | 人数未入力 | 着席モーダルで人数入力 |
| コース予約から自動着席にならない | 予約と着席店舗の不一致 | 予約時の店舗 ID を確認 |
| LO アラートが出ない | LO 時刻計算ロジックの異常 | サーバー時刻と店舗 timezone の整合性確認 |
