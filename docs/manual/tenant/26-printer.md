# 26. レシートプリンタ (Star / Epson)

POSLA は **ドライバ不要** で Star Micronics / Epson のネットワークプリンタに対応しています。購入したプリンタを Wi-Fi または Ethernet で店舗 LAN に接続するだけで動作します。

---

## 26.0 はじめにお読みください

::: tip 章の読み方ガイド
- **プリンタをこれから選ぶ方**: 26.1 (機種選定) → 26.2 (購入ガイド) 必読
- **既に Star / Epson 所有**: 26.3 (初期設定) から読めば OK
- **トラブル対応**: 26.10 (トラブルシューティング) が最重要
- **ブラウザ印刷のまま運用する方**: 26.0 のみで OK (プリンタ購入不要)
:::

### プリンタを導入するか?の判定フロー

```
Q1: レシートが必要 (税務 / 顧客要望)?
  NO  → プリンタ不要 (ブラウザ印刷で代用)
  YES → Q2 へ

Q2: 1日あたり何枚印刷?
  10枚未満 → ブラウザ印刷で十分 (タブレット接続のプリンタ流用)
  10-100枚 → Star mC-Print3 推奨 (¥40,000 前後)
  100枚以上 → Epson TM-m30III 推奨 (耐久性高、¥45,000 前後)

Q3: キッチン伝票も必要?
  YES → 2台目追加購入 (カウンター用 + キッチン用)
  NO  → 1台で OK
```

### 必要なもの

- **ネットワークプリンタ本体** (Star / Epson)
- **店舗 LAN** (Wi-Fi or Ethernet)
- **感熱ロール紙** (80mm 幅推奨)
- **管理画面へのオーナー / マネージャー権限**

### 所要時間

| 作業 | 目安 |
|---|---|
| プリンタの開梱 + 電源接続 | 5 分 |
| Wi-Fi / LAN 接続設定 | 10 分 |
| POSLA 側の設定 + テスト印刷 | 5 分 |
| **初期導入合計** | **約 20 分** |
| 用紙補充 | 30 秒 |

### 26.2.2 セット商品の落とし穴

「POS セット」で販売されているものは、多くの場合:
- 専用ソフトウェアライセンス付属 (POSLA では不要)
- ドライバディスク (POSLA では不要)
- スタンド (必要に応じて)

**POSLA は HTTP プロトコルで直接通信するため、本体だけでOK**。余計なソフト類は付属していても気にしないでください。

- プリンタと同じ LAN に接続できるもの

## 26.3 Star mC-Print3 の初期設定

### 26.3.1 開梱と電源接続

1. 箱を開ける (内容物):
   - プリンタ本体
   - 電源ケーブル + AC アダプタ
   - 感熱ロール紙 (テスト用 1 本入り)
   - 取扱説明書
2. AC アダプタを本体背面の DC 入力に接続
3. AC コンセントに電源ケーブル差し込み
4. プリンタ本体背面の電源スイッチを ON
5. 電源 LED が緑色に点灯したら OK

### 26.3.2 用紙セット

1. プリンタ上部のカバーを開ける (中央のボタンを押す)
2. 付属の感熱ロール紙を取り出す
3. **紙の表裏に注意**: 感熱面 (ツルツルした面) が上を向くように
4. 紙をローラーにセット
5. 紙を 10cm ほど引き出してカバーを閉める
6. カット部分から紙が出ていることを確認

### 26.3.3 テスト印刷 (プリンタ本体のセルフテスト)

1. **FEED ボタン** (前面中央) を長押し (3 秒)
2. セルフテストレシートが自動で印刷される
3. レシートに以下が印字されているか確認:
   - 機種名 (Star mC-Print3)
   - ファームウェアバージョン
   - **IP アドレス** (例: `192.168.1.50`) ← これが重要
   - MAC アドレス

IP アドレスをメモしてください。POSLA 設定で使います。

### 26.3.4 Wi-Fi 接続設定 (初めての場合)

セルフテストで表示された IP アドレスで Web 管理画面にアクセス:

1. 同じ LAN の PC / タブレットでブラウザを開く
2. アドレスバーに `http://{IP アドレス}` (例: `http://192.168.1.50`) を入力
3. Star の Web 管理画面が開く
4. ユーザー名 `admin` / パスワード `public` (初期値) でログイン
5. **Wi-Fi 設定** メニュー
6. SSID / パスワード入力 → **保存**
7. プリンタ再起動 → 新しい Wi-Fi IP で起動

### 26.3.5 固定 IP の推奨設定

DHCP のままだと IP が変わる可能性あり。固定 IP を推奨:

1. Web 管理画面 → **ネットワーク設定**
2. **Static IP** 選択
3. IP アドレス: 例 `192.168.1.50`
4. サブネットマスク: `255.255.255.0`
5. ゲートウェイ: 例 `192.168.1.1`
6. **保存** → 再起動

## 26.4 Epson TM-m30III の初期設定

### 26.4.1 開梱と電源接続

Star と同様。本体 + 電源ケーブル + 用紙 1 本。

### 26.4.2 用紙セット

1. 本体上部のボタンを押してカバーを開く
2. 感熱紙をセット (感熱面が上)
3. カバーを閉じる

### 26.4.3 IP アドレス確認 (セルフテスト)

1. **FEED ボタン** (前面) を長押し
2. レシートに IP アドレスが印字
3. メモ

### 26.4.4 Web 管理画面 (EpsonNet Config)

1. 同じ LAN の PC でブラウザ開く
2. `http://{IP}/` にアクセス
3. 初期ログイン: ユーザー名 `epson` / パスワード: 本体 MAC アドレス下 6 桁
4. **ネットワーク設定** → Wi-Fi SSID / パスワード設定
5. **ePOS-Print** が **有効** になっていることを確認 (デフォルト有効)
6. 固定 IP 設定 (推奨)

::: warning ePOS-Print が無効化されていると動かない
まれに工場出荷時に ePOS-Print が無効の個体あり。Web 管理画面で **必ず有効化** してください。
:::

## 26.5 POSLA 側の設定手順

### 26.5.1 設定画面を開く

1. 管理画面にオーナー or マネージャーでログイン
2. 画面上部のメインタブ **【店舗設定】** をクリック
3. サブタブ **【店舗設定】** をクリック (他のサブタブではない)
4. 画面を下にスクロール
5. 以下の見出しを探す:

```
🖨 プリンタ設定 (U-2)
Star mC-Print3 / Epson TM-m30 (Wi-Fi/Ethernet 接続) 対応。
ドライバインストール不要。詳細は プリンタガイド 参照。
```

### 26.5.2 入力フィールド

| 項目 | 設定内容 | 例 |
|---|---|---|
| **プリンタ種別** | ドロップダウンから選択 | `Star mC-Print3 (WebPRNT)` |
| **プリンタ IP** | プリンタ側で確認した IP | `192.168.1.50` |
| **ポート** | 通常 80 | `80` |
| **用紙幅** | 58 または 80 mm | `80mm` |
| **注文確定時にキッチン伝票を自動印刷** | チェック ON/OFF | ☑ (ON) |
| **会計完了時にレシートを自動印刷** | チェック ON/OFF | ☑ (ON) |

### 26.5.3 プリンタ種別の選び方

| 選択肢 | 用途 |
|---|---|
| **ブラウザ標準 (初期)** | プリンタなし、ブラウザ印刷ダイアログで対応 |
| **Star mC-Print3 (WebPRNT)** | Star 製プリンタ全般 (mC-Print2/3 共通) |
| **Epson TM-m30 (ePOS-Print)** | Epson 製 TM-m30 / TM-m30III 系 |
| **無効** | 印刷機能を完全に隠す |

### 26.5.4 【設定を保存】ボタン

画面下部の **【設定を保存】** (青ボタン) をクリックして反映。

::: warning 保存せずに画面遷移すると入力が消える
プリンタ設定は保存ボタンを押すまで DB に反映されません。他のタブに移動する前に必ず保存してください。
:::

## 26.6 テスト印刷

### 26.6.1 テスト印刷の実行

1. プリンタ設定の入力 + 保存後
2. **【🖨 テスト印刷】** ボタン (緑枠) をクリック
3. プリンタから以下が印刷される:

```
┌──────────────────────┐
│   POSLA プリンタテスト │
│ 2026-04-18 17:30:00  │
│ ─────────             │
│ プリンタ種別: star    │
│ IP: 192.168.1.50     │
│ 用紙幅: 80mm          │
│ ─────────             │
│ 正常に印刷されました   │
└──────────────────────┘
```

### 26.6.2 成功時の表示

画面右下にトースト通知:
```
テスト印刷を送信しました
```

### 26.6.3 失敗時の表示と対処

| トーストメッセージ | 原因 | 対処 |
|---|---|---|
| `network_error` | プリンタと通信不可 | IP / Wi-Fi 再確認 |
| `timeout` | プリンタ応答なし | 電源確認 |
| `http_404` | API が無効 | Web 管理画面で WebPRNT/ePOS 有効化 |
| `http_401` | 認証要求 | プリンタ側で認証を無効にする |
| `印刷失敗: popup_blocked` | ブラウザ標準モードでポップアップブロック | ブラウザのポップアップ許可 |

## 26.7 自動印刷の動作

### 26.7.1 注文確定時にキッチン伝票を自動印刷 (ON)

**タイミング**: お客様がセルフメニュー or ハンディから注文を送信した瞬間

**印刷内容**:
```
★ キッチン伝票 ★
テーブル: T03
時刻: 19:32

─────
鶏の唐揚げ  x2
生ビール    x1

─────
全体メモ: アレルギー:小麦
```

**用途**:
- キッチンが印刷物を見て調理開始
- KDS と併用 or KDS 代替

### 26.7.2 会計完了時にレシートを自動印刷 (ON)

**タイミング**: レジ画面で会計完了ボタンを押した瞬間

**印刷内容**:
```
    松の屋 渋谷店
    TEL: 03-XXXX-XXXX
    東京都渋谷区...

─────
日時: 2026-04-19 19:55
テーブル: T03
注文番号: abc123

─────
鶏の唐揚げ x2  ¥1,200
生ビール   x1    ¥600

─────
       合計 ¥1,800
       支払: カード

─────
ありがとうございました
```

### 26.7.3 OFF 時の動作

- **キッチン伝票 OFF**: 注文時に自動印刷されない (KDS でのみ把握)
- **レシート OFF**: 会計時に自動印刷されない (お客様から要望があれば手動ボタンで印刷)

手動印刷は以下から可能:
- レジ画面 → 取引履歴 → 該当行 → **「領収書」** ボタン

### 26.7.4 手動キッチン伝票印刷 (Phase 2 機能)

現状は「注文確定時の自動印刷のみ」。個別注文の再印刷機能は Phase 2 で追加予定。

## 26.8 用紙の補充

### 26.8.1 交換タイミング

- **赤線が見えた時**: ロール紙の残り約 10m (1 日分)
- **印刷が薄くなった**: ヘッドクリーニング or 用紙変更

### 26.8.2 Star mC-Print3 での交換手順

1. カバー中央の**ボタン**を押してカバー開放
2. 使用済みロールの芯を取り出す
3. 新しいロール紙を取り出し、**開封**
4. 紙の**先端を切り揃える** (ハサミで真っ直ぐ)
5. ロールをローラーにセット (**感熱面が上**)
6. 紙を 10cm ほど引き出す
7. カバーを閉じる
8. **FEED ボタン** で紙送り確認

### 26.8.3 感熱紙の表裏の見分け方

| 面 | 見分け方 |
|---|---|
| **感熱面 (表)** | ツルツル、爪で軽くこすると黒く跡が付く |
| **裏** | ザラザラ、爪でこすっても色が変わらない |

表裏を間違えると**白紙で出てくる**のですぐ分かります。

### 26.8.4 用紙の保管

- 直射日光を避ける (感熱紙は光で劣化)
- 湿度 60% 以下の乾燥した場所
- 熱源 (冷蔵庫の排気口等) から離す
- 開封後は 1 ヶ月以内に使い切る

## 26.9 ブラウザ標準印刷 (デフォルト)

プリンタ未設定の店舗 or プリンタなし運用時のデフォルト。

### 26.9.1 動作

- レジ画面で「会計完了」「領収書」ボタン → 新しいタブで印刷用プレビューが開く
- ブラウザの印刷ダイアログ (Cmd+P / Ctrl+P) が自動で出る
- タブレットに接続された AirPrint プリンタ or 家庭用プリンタで印刷可

### 26.9.2 メリット

- ハードウェア購入不要
- 設定不要 (初期状態)
- どのプリンタでも印刷可

### 26.9.3 デメリット

- 印刷ダイアログが毎回出る (自動印刷にならない)
- 感熱紙専用機ほどの速度は出ない
- キッチン伝票の即時印刷には不向き

### 26.9.4 推奨シナリオ

- **小規模店舗 (1日 20 組以下)**: ブラウザ印刷で十分
- **レシート不要な店**: プリンタ完全不要
- **将来の拡張用**: 今はブラウザ、売上伸びたら Star/Epson に移行

## 26.10 よくある質問 20 件

### A. 購入前の質問

**Q1.** プリンタなしでも営業できる?
**A1.** はい。ブラウザ印刷で代用可能。ただし自動印刷や高速印刷はできません。

**Q2.** iPad に USB で直接繋げる?
**A2.** 不可。POSLA は Wi-Fi / Ethernet 経由の HTTP API のみサポート。

**Q3.** どの機種を選べばいい?
**A3.** 予算が許すなら **Star mC-Print3** がコスパ最強。毎日大量印刷なら **Epson TM-m30III**。

**Q4.** 中古品でも動く?
**A4.** ファームウェアが新しければ OK。ただし保証なし。メルカリ等で購入する時は出品者に「ePOS-Print 有効」を確認。

**Q5.** キッチン用と カウンター用 の 2 台は必要?
**A5.** 規模次第。1 日 50 組以上なら分けた方が効率的。小規模なら 1 台で両方兼用可。

### B. 設定に関する質問

**Q6.** プリンタの IP が変わってしまった
**A6.** 固定 IP 設定を推奨。DHCP のままだと月 1 回程度 IP が変わる可能性あり。

**Q7.** 複数店舗で 1 台のプリンタを共有できる?
**A7.** 非推奨。店舗ごとに物理プリンタを用意してください (LAN が別の場合は通信不可)。

**Q8.** プリンタ設定変更を保存したのに印刷しない
**A8.** ブラウザをリロード (Cmd+Shift+R) してください。キャッシュが残っている場合あり。

**Q9.** テスト印刷は成功するが自動印刷が出ない
**A9.** 「注文確定時に自動印刷」または「会計時に自動印刷」のチェックが外れている可能性。設定を見直してください。

**Q10.** 用紙幅を 58mm にしたら内容がはみ出す
**A10.** 58mm は伝票 (キッチン) 向け。レシートは 80mm 推奨。

### C. 運用に関する質問

**Q11.** 用紙がすぐ切れる
**A11.** 1 日 100 枚印刷で 80m ロール 1 本 (= 約 10 日)。多い店は 20 本 / 月 購入。

**Q12.** 印刷が薄くなってきた
**A12.** ヘッドクリーニング (専用キット) or 用紙を別ロットに変更。

**Q13.** 毎回カット部分で引っかかる
**A13.** 用紙セットがずれている可能性。**紙の端が左右均等** になるようセットし直す。

**Q14.** 電源を切ったらプリンタ設定が消えた?
**A14.** 設定は DB に保存されているので消えません。プリンタ本体側の設定 (Wi-Fi パスワード等) は本体メモリに保存。

**Q15.** QR コードは印刷できる?
**A15.** Star / Epson どちらも可能 (printer-service.js の lines に `{type:'qr', data:'...'}` を入れる)。ただし現状 POSLA UI で呼び出す機能はまだ少ない (Phase 2 で拡張予定)。

### D. トラブル系

**Q16.** プリンタが全く反応しない
**A16.** 電源 → LED → Wi-Fi → IP の順で確認。セルフテスト (FEED 長押し) が出ないなら本体故障。

**Q17.** 「network_error」と出る
**A17.** PC / タブレットがプリンタと別の LAN に繋がっている可能性。同じ Wi-Fi に接続してください。

**Q18.** 文字化けする
**A18.** 古いファームウェアの可能性。メーカーサイトから最新ファームに更新。

**Q19.** カッターが動かない
**A19.** 紙詰まりが残っている可能性。カバーを開けて紙片を除去。

**Q20.** 故障した場合の問い合わせ先は?
**A20.** 本体はメーカー保証 (1 年)。POSLA 側の問題は `info@posla.jp` へ。

## 26.11 トラブルシューティング表

### 26.11.1 接続関連

| 症状 | 原因 | 対処 |
|---|---|---|
| `network_error` | IP 違い / 別 VLAN | PC とプリンタが同じ LAN か確認 |
| `timeout` | プリンタ電源 OFF | 電源 ON 確認 |
| `http_404` | HTTP API 未起動 | Web 管理画面で WebPRNT/ePOS 有効化 |
| `http_401` | 認証要求 | プリンタの認証を無効化 or 初期値に戻す |
| IP が毎日変わる | DHCP | 固定 IP に設定 |

### 26.11.2 印刷品質

| 症状 | 原因 | 対処 |
|---|---|---|
| 文字化け | 古いファーム / 文字コード | メーカーサイトから最新ファーム適用 |
| 白紙で出る | 用紙の表裏逆 | 感熱面を上にしてセット |
| かすれ | ヘッド汚れ | アルコール綿棒でヘッド清掃 |
| 印刷が遅い | Wi-Fi 干渉 | Ethernet 接続に変更 |
| 一部欠け | 用紙ズレ | カバー再セット + FEED |

### 26.11.3 用紙関連

| 症状 | 原因 | 対処 |
|---|---|---|
| 紙詰まり | 紙が斜め | カバー開けて真っ直ぐセット |
| カッターが動かない | 紙片が残ってる | カバー開けて紙片を除去 |
| FEED で紙出ない | 用紙切れ | 新しいロール補充 |
| 赤線が出る | 残量少 | 早めに補充準備 |

### 26.11.4 設定関連

| 症状 | 原因 | 対処 |
|---|---|---|
| 設定保存しても反映されない | ブラウザキャッシュ | Cmd+Shift+R でリロード |
| テスト印刷ボタンがない | printer-service.js 未ロード | dashboard.html リロード |
| 用紙幅設定変更しても印字されず同じ | プリンタ側設定と不一致 | プリンタ本体の DIP スイッチ確認 (機種による) |

## 26.12 データベース構造 (POSLA 運営向け)

::: tip 通常テナントは読まなくて OK

### 26.12.1 store_settings テーブル拡張 (migration-u2-printer.sql)

```sql
ALTER TABLE store_settings
  ADD COLUMN printer_type ENUM('none','browser','star','epson') NOT NULL DEFAULT 'browser'
    AFTER receipt_footer,
  ADD COLUMN printer_ip VARCHAR(45) DEFAULT NULL
    AFTER printer_type,
  ADD COLUMN printer_port SMALLINT UNSIGNED NOT NULL DEFAULT 80
    AFTER printer_ip,
  ADD COLUMN printer_paper_width SMALLINT UNSIGNED NOT NULL DEFAULT 80
    AFTER printer_port,
  ADD COLUMN printer_auto_kitchen TINYINT(1) NOT NULL DEFAULT 0
    AFTER printer_paper_width,
  ADD COLUMN printer_auto_receipt TINYINT(1) NOT NULL DEFAULT 1
    AFTER printer_auto_kitchen;
```

### 26.12.2 printer-service.js の API

#### window.PrinterService.print(config, jobType, lines)

config:
- `printer_type`: 'none' | 'browser' | 'star' | 'epson'
- `printer_ip`: IP 文字列
- `printer_port`: ポート番号
- `printer_paper_width`: 58 or 80

jobType: 'receipt' | 'kitchen' | 'refund' | 'test'

lines: [
  { type: 'text', text: '...', align: 'left|center|right', size: 'large' },
  { type: 'line' },
  { type: 'qr', data: '...' },
  { type: 'barcode', data: '...' }
]

#### ビルトインテンプレート

- `PrinterService.buildReceiptLines(store, order)` → レシート lines 配列
- `PrinterService.buildKitchenLines(store, order)` → キッチン伝票 lines 配列
