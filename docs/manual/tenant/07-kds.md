# 7. KDS（キッチンディスプレイ）

KDSは、注文を厨房に表示する画面です。紙伝票の代わりに、調理状況、提供状況、注文の優先度を確認できます。

## 開店時に確認すること

1. KDS用端末でKDS画面を開きます。
2. 店舗、表示するステーション、音量、画面維持の状態を確認します。
3. 黒い起動画面が表示されたら **KDSを開始** をタップします。通知音、音声コマンド、画面維持の準備が始まります。
4. ヘッダーの **開店前チェック** を押し、KDS通信、自動更新、通知音、マイク入力、画面維持、ステーションを確認します。
5. 必要に応じて **着手時間** を確認します。テイクアウト受取時刻や予約時刻の何分前から厨房に出すかを、テイクアウト・予約・カテゴリ別に設定できます。

::: tip 着手時間の例
- 寿司・コース料理など準備に時間がかかるカテゴリ: 30分前
- 通常料理: 20分前
- ドリンク・簡単なデザート: 5分前

設定はKDS端末ごとに保存されます。厨房端末を複数台使う場合は、各端末で同じ設定になっているか開店前に確認してください。
:::

## KDSの基本操作

1. 注文が入るとカードとして表示されます。
2. 調理を始めたらステータスを更新します。
3. 料理が完成したら「完成」にします。ホールのハンディへ料理完成通知が出ます。
4. 提供が完了したら「提供済み」にし、画面から消します。
5. 間違って押した場合は10秒取り消しまたは配膳取消で戻します。

## ステータスの意味と連動

KDSのステータスは、注文全体と品目単位の両方で管理されます。

| ステータス | 厨房での意味 | 連動 |
|---|---|---|
| 受付 | 注文直後。まだ調理に入っていない | KDSに新着カードとして表示 |
| 調理中 | 厨房が着手した | 調理開始時刻が残る |
| 提供待ち | 料理が完成し、ホール配膳待ち | ハンディの運用タブに料理完成通知を作成 |
| 提供済み | お客様へ提供済み | 品目が完了扱いになり、全品提供済みなら注文も提供済みへ進む |
| キャンセル | 作らない、または取り消した品目 | 完了判定から除外 |

品目を **完成** にすると、ハンディ側に `product_ready` の呼び出しが出ます。ホールが配膳対応済みにすると、該当品目は提供済みへ連動します。厨房側で全品をまとめて提供済みにした場合も、注文全体が完了方向へ進みます。

混在状態の注文では、誤って全品を戻さないよう、一括操作が抑制される場面があります。品目単位で状態が違う場合は、対象の品目を個別に操作してください。

### Expeditor（まとめ表示）モード
Expeditorモードは、提供順に特化した単一カラムのリスト表示です。
- **メリット**: 調理の優先順位が明確になり、提供ミスを防げます。
- **活用シーン**: 注文数が多いピーク時や、デリバリー・テイクアウトのパッキング作業に最適です。

まとめ表示では、提供待ち、遅れ、テイクアウト受取時刻、予約時刻などを見ながら、厨房とホールが同じ優先順位で動けます。通常の3列表示に戻したい場合は、画面ボタンまたは音声コマンドの **通常表示** を使います。

## 画面の見方

- **受付**: 注文直後で、まだ調理を始めていない品目です。
- **調理中**: 調理開始済みの品目です。
- **提供待ち**: 料理完成済みで、ホールの配膳待ちです。
- **ステーション状況帯**: ステーション別の注文数、未完了品目数、受付、調理中、提供待ち、最上位の注意理由を表示します。
- **端末状態帯**: 最終更新、通信遅延、stale/live、音声成功時刻、通信異常を常時表示します。

ステーション状況帯は、担当場所ごとの負荷を見るための帯です。未完了数が多い、提供待ちが溜まっている、遅れが出ているステーションは強調されます。ピーク中は、まず状況帯を見て「どこが詰まっているか」を確認してください。

予約から着席した卓の注文カードには、予約者名、人数、予約時刻、コース、予約メモ、好み、注意点、来店回数などの予約文脈が表示されます。予約顧客にアレルギー情報がある場合は、通常の注文メモとは別に「予約アレルギー」として強調されます。

端末状態帯が赤の場合は、KDS端末が止まっている、通信が不安定、または古いデータを表示している可能性があります。厨房作業を進める前に、Wi-Fi、端末のスリープ、ブラウザの再読み込みを確認してください。

## ステーション

ステーションは「厨房」「ドリンク」「デザート」など、担当場所ごとの表示分けです。商品ごとに適切なステーションを設定すると、担当者が必要な注文だけを確認できます。

ステーションを選ぶと、そのステーションで必要な品目だけが表示されます。注文が見当たらない場合は、まず **全品目** に戻して確認してください。

ステーション設定は、管理画面の **店舗設定 → KDSステーション** で行います。カテゴリやメニュー品目とステーションの紐づけがずれていると、注文が想定のKDSに表示されません。

## テイクアウト・予約の表示

テイクアウト注文は、受取時刻とお客様名を見ながら準備します。着手時間設定により、受取時刻の何分前からKDSに出すかを調整できます。

予約から着席した注文では、予約者名、予約時刻、人数、コース、メモ、好み、注意点、アレルギーが表示されます。アレルギーは通常メモよりも見落としやすいため、カード内の予約アレルギー表示を必ず確認してください。

## 取り消し・戻し操作

誤って状態を進めた場合は、直後であれば画面上の取り消し操作、または音声の **戻して** を使います。すでにホール側で配膳対応済みになっている場合は、ハンディ側の通知や注文状態も確認してください。

調理済み・提供済みを戻す操作は、現場混乱につながりやすいため、対象の卓・品目・数量を確認してから行ってください。会計済みの商品を厨房側だけで戻しても、会計は自動で取り消されません。

## 推奨端末

KDSはAndroidタブレット（10インチ以上）+ Google Chrome最新版を **標準環境** としています。iPad/Safariでも画面は表示できますが、音声コマンドや通知音が安定しないためサポート対象外です。詳細は「KDSデバイス端末ガイド」をご覧ください。

## 音声コマンド

音声コマンドは Android の Google Chrome 最新版での利用を標準としています。Safari / iPad では認識が安定しにくいため、音声操作を使う場合は Android 端末をご利用ください。

周囲の騒音が大きい場合は認識しにくくなるため、必要に応じて手操作で対応してください。騒がしい厨房や広い店内では、**マイク付き片耳Bluetoothヘッドセット**（任意の推奨備品）の併用もご検討ください。詳しくは「23. KDSデバイス端末ガイド」をご覧ください。

主な音声操作:

| 言葉 | 動作 |
|---|---|
| `T01 調理開始` | T01の注文を調理中へ進める |
| `T01 完成` | T01の注文を提供待ちへ進める |
| `T01 提供済み` | T01の注文を提供済みにする |
| `T01 残り` | その卓の未完成品を確認する |
| `まとめ表示` / `通常表示` | Expeditor表示と通常3列表示を切り替える |
| `品切れ一覧` / `品切れ管理` | 品切れ確認または品切れ管理画面へ移動する |
| `音声再起動` | 音声認識だけを再起動する |
| `音声診断` | 端末の音声認識状態を確認する |

厨房でよく使う操作はAIを使わないファストパスで処理します。ヘッダーの **AIシェフON/OFF** は、AIシェフパネルと、ファストパスで処理できない曖昧な言い回しの補助解析をまとめて切り替える設定です。通常運用ではOFF推奨です。

## 営業中の確認ポイント

- KDS端末が常に充電または給電されていること
- Wi-Fiが安定していること
- 画面がスリープしない設定になっていること
- 端末状態帯の「最終更新」「遅延」「stale/live」を見ること
- 注文カードの更新が止まっていないこと
- 音声操作を使う場合は、端末状態帯の「音声成功」とヘッダーの「音声診断」を確認すること

## 困ったとき

| 症状 | 対処 |
|---|---|
| 注文が表示されない | 店舗、ステーション、ネットワークを確認してください。ステーションを **全品目** に戻すと切り分けしやすくなります。 |
| 端末状態帯が赤い | 通信停止、stale、または最終更新20秒超の可能性があります。Wi-Fi確認、画面スリープ解除、ブラウザ再読み込みを行ってください。 |
| 音が鳴らない | 端末音量、ブラウザの音声許可、ミュート設定、起動時の **KDSを開始** タップを確認してください。Safariでは通知音が安定しないため、Android Chromeをお試しください。 |
| 音声操作が反応しない | AndroidのGoogle Chrome最新版を利用し、マイク許可、音声ON、音声診断を確認してください。Safari/iPadは音声コマンド非対応です。 |
| 画面更新が遅い | 端末状態帯の遅延秒数とWi-Fi環境を確認し、必要に応じて再読み込みしてください。 |
| テイクアウトや予約が早く/遅く表示される | ヘッダーの **着手時間** を確認してください。カテゴリ別設定が優先されます。 |
