# 21. 決済設定

決済設定では、POSLAの月額契約と、セルフ会計・テイクアウトで使う **POSLA決済** を確認します。

通常の店頭レジでは、店舗がすでに契約しているカード端末・QR決済・現金をそのまま使います。POSLAは店頭決済を実行せず、支払い方法と会計完了の記録だけを残します。

## 決済の種類

| 種類 | 内容 |
|---|---|
| POSLA利用料 | POSLAの月額利用料のお支払い |
| 通常レジ会計 | 現金 / 店舗カード端末 / 店舗契約QRで支払いを受け、POSLAには記録だけ残す |
| POSLA決済 | セルフ会計・テイクアウトで Stripe Connect 経由のオンライン決済を行う |
| 取消・返金 | 通常レジ会計は記録取消、POSLA決済は決済サービス側の返金処理を伴う |

## 店舗側で確認すること

1. 契約プランと請求先を確認します。
2. 店頭で使う決済端末・QR決済契約を確認します。
3. セルフ会計・テイクアウトを使う場合は、Stripe Connect のオンボーディングを完了します。
4. 返金時の承認ルールを店舗内で決めます。
5. レジ締め時に、外部決済端末の集計とPOSLA上の売上記録を突き合わせます。

## 通常レジ会計

通常レジでは、POSLAは決済を実行しません。

| 支払い方法 | POSLAで行うこと |
|---|---|
| 現金 | 預かり金・お釣り・会計完了を記録 |
| カード | 店舗の外部カード端末で決済完了後、カード会計として記録 |
| QR | 店舗契約のQR決済で支払い完了後、QR会計として記録 |

カード端末やQRアプリ側で支払いが完了していない状態で、POSLAの会計を確定しないでください。

## POSLA決済

POSLA決済は、セルフ会計・テイクアウトなど、お客様が画面上で支払いまで完了する機能で使います。

- Stripe Connect を使います。
- 店頭レジのカード/QRボタンとは別の機能です。
- Stripe設定が未完了、またはセルフ決済がOFFの場合、お客様画面のお会計ボタンは表示されません。

## 返金・取消

- 通常レジの現金・カード・QRは、POSLA上では会計記録の取消として扱います。実際の返金は、現金手渡しまたは外部決済端末・QRアプリ側で行ってください。
- POSLA決済で完了した支払いは、Stripe側の返金処理が必要です。

## 困ったとき

| 症状 | 対処 |
|---|---|
| 店頭カード決済が完了しない | POSLAではなく店舗契約のカード端末・決済会社の状態を確認してください。 |
| QR決済が完了しない | QR決済アプリ・端末側の入金確認画面を確認してください。 |
| セルフ会計のお会計ボタンが出ない | Stripe Connect設定とセルフ決済ON/OFFを確認してください。 |
| 返金できない | 通常レジ会計かPOSLA決済かを確認し、通常レジ会計は外部決済側で返金してください。 |
