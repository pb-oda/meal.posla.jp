# 16. 店舗設定

店舗設定では、営業時間、税率、支払い方法、レシート、テイクアウト、予約ページ、デザイン、通知などを管理します。

## 基本設定

| 項目 | 内容 |
|---|---|
| 店舗名 | 管理画面やお客様画面に表示される店舗名 |
| 住所 | レポート、予約、Google連携などで利用 |
| 営業時間 | 受付時間や営業日集計に影響 |
| 税率 | 会計金額に影響 |
| 支払い方法 | 現金、カード、QR決済など |

## 営業時間と営業日

深夜営業がある店舗では、営業日区切り時刻を確認してください。日報や売上集計の基準になります。

## レシート・領収書

店舗名、住所、登録番号、但し書き、ロゴなどを確認します。営業開始前にテスト印刷を行ってください。

## テイクアウト・予約ページ

テイクアウトや予約を利用する場合は、受付時間、受取時間、案内URL、QRコードを確認してください。

## スタッフ・端末管理

スタッフアカウント、端末専用アカウント、表示する業務端末を管理します。退職や端末入替時は不要なアカウントを無効化してください。

## 外部連携

「オーナーダッシュボード」の **「外部連携」タブ** から、以下のサービスと接続できます。

### LINE公式アカウント連携
- **用途**: 予約確定、24時間前/2時間前リマインド、テイクアウト準備完了の通知（LINE v1）を自動送信します。
- **未提供機能**: メッセージの自由配信UI、リッチメニューの管理、LIFF 連携などは LINE v2 として計画中のバックログであり、現在は利用できません。
- **設定**: LINE Developers で発行したトークンを登録します。

### スマレジ連携
- **用途**: POSLA の注文データを自動的にスマレジへ送信し、売上を統合します。
- **連携手順**:
    1. 「スマレジ連携」画面から **OAuth 認証** を行い、スマレジアカウントにログインして接続を承認します。
    2. 連携成功後、**店舗マッピング** 画面で POSLA の各店舗とスマレジの各店舗を紐付けます。
    3. 必要に応じて「メニューインポート」を実行し、スマレジ側の商品情報を POSLA へ取り込みます。

## 困ったとき

| 症状 | 対処 |
|---|---|
| 設定が保存できない | 必須項目と入力形式を確認してください。 |
| 税率が違う | 店舗設定と商品ごとの税区分を確認してください。 |
| QRコードが表示されない | 対象機能が有効になっているか確認してください。 |
| レシート内容が違う | 店舗情報、領収書設定、プリンタ設定を確認してください。 |
