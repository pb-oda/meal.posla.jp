# 24. 予約管理

POSLA の予約管理は、Web予約・電話予約・当日受付・待ち客・着席中・キャンセル待ち・予約金・リマインド通知を、店舗の運用画面でまとめて確認するための機能です。

この章では、お客様向け予約と店舗側の予約ボードの使い方を説明します。

## 予約管理でできること

| 機能 | 内容 |
|---|---|
| Web予約受付 | お客様が日時・人数・連絡先を入力して予約できます |
| 電話・店頭予約登録 | 店舗側で予約を手入力できます |
| 当日受付ボード | 予約、飛び込み、待ち、着席中を同じ画面で確認できます |
| 空席判定 | 営業時間、席、既存予約、待ち客の仮押さえを考慮して空き枠を判定します |
| キャンセル待ち | 満席時にキャンセル待ち登録し、空きが出たら候補へ案内できます |
| 予約金 | 条件に応じて予約金を求める運用ができます |
| リマインド通知 | 24時間前・2時間前の予約リマインドを送れます |
| 常連情報 | 前回来店、好み、注意点、アレルギー、VIP、ブラックリストを確認できます |
| 変更履歴 | 日時・人数・席・メモ・ステータスなどの変更前後を確認できます |

## 利用前に確認する設定

予約設定では、次の内容を設定できます。

| 設定 | 内容 |
|---|---|
| Web予約有効 | お客様向け予約受付を有効にするか |
| 受付リードタイム | 何時間前まで予約を受け付けるか |
| 受付可能日数 | 何日先まで予約を受け付けるか |
| 標準滞在時間 | 予約1件あたりの席利用時間 |
| 予約枠間隔 | 15分、30分など予約枠の間隔 |
| 最小・最大人数 | Web予約で受け付ける人数範囲 |
| 営業時間 | Web予約で表示する開始・終了時刻 |
| ラストオーダー前締切 | 閉店前の予約受付をどこで止めるか |
| 定休日 | Web予約を受け付けない曜日 |
| 前後バッファ | 席の入れ替え時間を確保するための前後余白 |
| 電話番号・メール必須 | お客様入力で必須にする項目 |
| キャンセル期限 | お客様が自分でキャンセルできる期限 |
| Web予約枠ルール | 大人数は電話のみ、ピーク時間帯の組数・人数制限、席エリア制限 |
| 予約金 | 予約金の有効化、1人あたり金額、対象人数 |
| 高リスク予約金 | no-show履歴、大人数、ブラックリスト近似時の自動予約金 |
| リマインド | 24時間前・2時間前通知、再送間隔、再送上限 |
| キャンセル待ち仮押さえ | 空席通知後、候補者向けに枠を保持する時間 |
| SMS Webhook | 外部SMSサービスへ通知を連携するURL |
| AI予約 | お客様の自然文入力から予約内容を読み取る機能 |

## お客様の Web予約手順

1. お客様が予約ページを開きます。
2. 日付・人数を選びます。
3. 空いている時間枠を選びます。
4. 名前・電話番号・メールアドレス・メモを入力します。
5. 内容を確認して予約します。
6. 予約完了後、予約詳細ページのURLが発行されます。

予約詳細ページでは、予約内容の確認、予約金の支払い、予定のカレンダー追加、予約変更、キャンセルができます。予約変更やキャンセルは、店舗のキャンセル期限設定と予約状態によって制限されます。

## 空席判定の考え方

POSLA は予約枠を表示するときに、次の条件を確認します。

| 判定項目 | 内容 |
|---|---|
| 営業日・営業時間 | 定休日、開始時刻、終了時刻、ラストオーダー前締切 |
| リードタイム | 直前すぎる予約を受け付けない |
| 最大受付日数 | 遠すぎる日付を受け付けない |
| 人数制限 | 最小・最大人数の範囲内か |
| 既存予約 | 確定済み・保留中など有効な予約と重ならないか |
| 席の空き | 1卓、2卓結合、3卓結合までの範囲で割り当て可能か |
| 待ち客の仮押さえ | キャンセル待ち候補に保持している枠を重複して出さない |
| 前後バッファ | 前後の予約と入れ替え時間を確保できるか |
| Web予約枠ルール | 大人数の電話誘導、ピーク時間帯の組数・人数制限、席エリア制限 |

同じ時間・同じ席に過剰に予約が入らないよう、予約作成時にもサーバー側で再確認します。画面に表示された後に別のお客様が先に予約した場合は、作成時に満席として扱われます。

## 満席時のキャンセル待ち

空き枠がない場合、条件によってキャンセル待ち登録ができます。

キャンセル待ちに登録されると、店舗側の予約ボードに候補として表示されます。キャンセルや no-show が出た場合、条件に合う候補へ空席案内を送れます。空席案内時には、設定された時間だけ候補者向けに枠を仮押さえします。

現在の空席案内は、メールアドレスがある候補者へのメール送信を前提にしています。

## 店舗側の予約ボード

予約ボードでは、当日の予約状況を確認します。

| 表示 | 内容 |
|---|---|
| 予約一覧 | 時刻、人数、名前、連絡先、メモ、ステータス |
| タイムライン | 予約時間と席の利用状況 |
| 待ち客・キャンセル待ち | 待機中、通知済み、呼び出し状況 |
| 着席状況 | 予約から着席へ進めたお客様 |
| 要対応リスト | 遅刻、no-show履歴、ブラックリスト、大人数、通知失敗など |
| 常連情報 | 前回来店、好み、注意点、アレルギー、タグ |
| 変更履歴 | 誰が、いつ、何を、変更前後どう変えたか |

遅刻中、no-show履歴あり、ブラックリスト、大人数などはリスクとして表示され、電話確認や予約金検討などの対応候補が分かるようになっています。

## 予約ステータス

| ステータス | 意味 |
|---|---|
| 保留 | 予約受付後、確定前の状態 |
| 確定 | 来店予定として確定している状態 |
| 着席中 | お客様を席へ案内した状態 |
| no-show | 無断キャンセルとして処理した状態 |
| キャンセル | キャンセル済み |
| 完了 | 来店完了 |
| キャンセル待ち | 空席待ちの状態 |

no-show 処理をすると、顧客情報の no-show 回数に反映されます。予約金の与信がある予約では、設定と決済状態に応じて予約金の捕捉処理が行われます。

## 変更履歴・変更前後比較

予約の詳細画面では、変更履歴を確認できます。

記録対象には、主に次の項目があります。

- 顧客名、電話番号、メールアドレス
- 人数
- 予約日時
- 滞在時間
- メモ
- タグ
- コース
- 割り当て席
- ステータス
- 来店フォロー状態
- キャンセル待ち呼び出し状態

履歴には、変更者、変更日時、変更前、変更後が残ります。予約トラブル時は、まず詳細画面の変更履歴を確認してください。

## リマインド通知

予約リマインドは、設定により 24時間前・2時間前に送信できます。

通知経路は、店舗設定とお客様情報に応じて使われます。

| 経路 | 条件 |
|---|---|
| メール | お客様メールアドレスがあり、送信設定がある場合 |
| LINE | 店舗の LINE 連携が有効で、お客様のLINE紐付けがある場合 |
| SMS Webhook | SMS連携URLが設定され、電話番号がある場合 |

送信に失敗したリマインドは、設定された再送間隔と再送上限に従って再送対象になります。失敗が上限に達すると、店長が確認すべき通知として扱われます。

## 予約金と高リスク予約

予約金を有効にすると、設定した人数以上の予約に予約金を求めることができます。予約金の決済には店舗の決済設定が必要です。

高リスク予約金を有効にした場合、次のような条件で予約金必須にできます。

| 条件 | 内容 |
|---|---|
| no-show履歴 | 指定回数以上の no-show があるお客様 |
| 大人数 | 指定人数以上の予約 |
| ブラックリスト近似 | 顧客情報の注意対象に近いと判定される場合 |

店舗側で電話予約を作成する場合は、必要に応じて予約金をスキップする運用もできます。

## 常連情報

予約詳細では、顧客情報として次の内容を確認できます。

- 前回来店日
- 来店回数
- no-show 回数
- キャンセル回数
- 累計利用金額
- 好み
- 注意点
- アレルギー
- VIP
- ブラックリスト
- タグ

アレルギーや注意点は、来店前の共有事項として必ず確認してください。

## AI予約

AI予約を有効にしている場合、お客様が自然文で希望内容を入力し、日時・人数・名前・連絡先などの読み取り補助を受けられます。

AIが読み取った内容は、予約作成前にお客様が確認します。空席判定、予約金、高リスク判定は通常のWeb予約と同じサーバー側処理で確認されます。

## 当日の基本運用

1. 営業前に当日の予約ボードを開きます。
2. 要対応リストで、遅刻リスク・no-show履歴・大人数・通知失敗を確認します。
3. 必要に応じて電話確認、予約金確認、席調整を行います。
4. 来店したお客様を着席に進めます。
5. 遅刻している予約は来店フォロー状態を更新します。
6. no-show が確定した場合は no-show 処理を行います。
7. キャンセルや no-show で空きが出たら、キャンセル待ち候補を確認します。

## よくあるトラブル

| 困ったこと | 確認すること |
|---|---|
| Web予約枠が出ない | Web予約有効、営業時間、定休日、リードタイム、最大受付日数、席設定を確認 |
| 大人数だけ予約できない | Web予約枠ルールで電話のみ対象になっていないか確認 |
| ピーク時間だけ満席になる | ピーク最大組数・最大人数の設定を確認 |
| リマインドが届かない | メール、LINE紐付け、SMS Webhook、通知失敗・再送状態を確認 |
| 予約金が求められない | 決済設定、予約金有効、対象人数、高リスク予約金設定を確認 |
| キャンセル待ち通知が届かない | 候補者のメールアドレス、空席条件、仮押さえ期限を確認 |
| 何が変更されたか分からない | 予約詳細の変更履歴・変更前後比較を確認 |
