# 1. はじめに

POSLAは、飲食店の注文受付、キッチン連携、会計、スタッフ管理、レポート確認をまとめて行うクラウド型システムです。お客様はテーブルのQRコードから注文し、店舗スタッフは管理画面、ハンディ、KDS、レジ画面を使って営業を進めます。

## POSLAでできること

- セルフオーダー、ハンディPOS、テイクアウト、予約の受付
- KDSによる厨房への注文共有
- POSレジでの会計、領収書、返金対応
- メニュー、テーブル、フロアマップ、在庫、シフトの管理
- 売上、回転率、スタッフ評価、ABC分析などのレポート確認
- AIアシスタントによる投稿文作成、売上分析、需要予測

## 推奨環境

POSLAの店舗業務端末（KDS・レジ・ハンディ）は、**Android端末のGoogle Chrome最新版** でのご利用を **標準環境** としています。管理画面はPCまたはAndroid端末のChrome/Edgeで快適に操作できます。

| 用途 | 標準環境 | 推奨環境 | サポート対象外 |
|---|---|---|---|
| 管理画面 | PC + Chrome最新版 | PC + Edge最新版 / Androidタブレット + Chrome | Internet Explorer・古いブラウザ |
| KDS | Androidタブレット（10インチ以上）+ Chrome最新版 | — | iPad/Safari、Fireタブレット、独自ブラウザ |
| POSレジ | Androidタブレット + Chrome最新版 | PC + Chrome最新版 | iPad/Safari、Fireタブレット |
| ハンディPOS | Androidスマホ/タブレット + Chrome最新版 | — | iPhone/Safari、Fireタブレット |
| お客様のセルフオーダー | お客様のスマートフォン（iPhone/Android問わず） | 主要ブラウザ全般（Safari/Chrome等）| — |

::: tip 顧客セルフメニューはブラウザ自由
お客様がQRから利用するセルフメニュー、予約、テイクアウト画面はブラウザを限定しません。お手持ちのスマートフォンでそのままご利用いただけます。
:::

::: warning 業務端末でiPad/Safariを使う場合の制限
iPad/Safariでも一部画面は表示できますが、KDSの音声コマンドや通知音、PWAインストール導線は安定して動作しません。安定運用を希望される場合はAndroid端末のChrome最新版をご利用ください。
:::

店舗のWi-Fiが不安定な場合、注文送信や画面更新が遅れることがあります。

## 業務端末をホーム画面アプリとして設置する（PWA）

POSLAの**管理画面 / ハンディ / KDS / POSレジ**は、スマホやタブレットの**ホーム画面にアイコンを追加して、1タップでアプリのように起動**できます（PWA: Progressive Web App）。ブラウザのタブやアドレスバーが消え、業務に集中しやすくなります。

::: tip 対象は業務端末のみ
ホーム画面追加の案内は **管理画面・ハンディ・KDS・POSレジ** で表示されます。お客様がQRから使うセルフメニュー／予約／テイクアウト画面はその都度ブラウザで開く想定なので、PWA化していません（意図的な設計）。
:::

### Android（標準環境・Google Chrome）

1. 対象端末のChromeで対象画面にログインする。管理画面は個人スタッフアカウント、ハンディ・KDS・POSレジは端末専用アカウントを使う。
2. 画面下部に **「📱 POSLAをホーム画面に追加して1タップで起動できます。インストールしますか？」** というバナーが自動で表示される。
3. **［インストール］** を押すとホーム画面にアイコンが追加される。
4. 以降はホーム画面のアイコンから直接起動できる（ブラウザのタブ・アドレスバーは非表示）。

バナーが出ない場合：Chromeのメニュー（右上︙）→「**ホーム画面に追加**」または「**アプリをインストール**」から手動で追加できます。

### iPhone / iPad（Safari・サポート対象外）

iPhone / iPad の Safari では、自動のインストール案内が表示されないことがあります。その場合は、以下の手順で手動追加してください。

1. Safariで対象画面を開いてログインする。
2. 画面下の**共有ボタン（□に↑の四角いアイコン）**をタップ。
3. メニューから **「ホーム画面に追加」** を選ぶ。
4. タイトルを確認して **「追加」** をタップ。

::: warning iPhone / iPad は業務端末としてはサポート対象外
ホーム画面追加自体は可能ですが、KDSの音声コマンド・通知音・自動インストールバナーなど一部機能が安定しません。**業務端末はAndroid＋Chromeのご利用を強く推奨**します（お客様側のセルフメニューはiPhone Safariでも問題なくご利用いただけます）。
:::

### アイコンを消したいとき

ホーム画面のPOSLAアイコンを長押し → 削除（iOS：「ブックマークを削除」／Android：「アンインストール」相当の表示）で外せます。再度追加したい場合は、ブラウザで対象URLを開き直してから上記手順をやり直してください。

詳しい設定や不明点がある場合は、POSLAサポートへお問い合わせください。

## 業務端末で受け取れる通知

ホーム画面に追加したPOSLA（管理画面・ハンディ・KDS・POSレジ）は、**画面を閉じていても業務通知を受け取れます**（Web Push / Android Chrome は全面対応）。

### 受け取れる通知（5種類）

| # | 通知 | 届くタイミング | 届く相手 | タップで開く画面 |
|---|---|---|---|---|
| 1 | **お客様呼び出し** | お客様がセルフメニューの呼び出しボタンを押したとき | 店舗の全端末 | ハンディ画面 |
| 2 | **キッチンから呼び出し** | KDSから「提供お願いします」等の呼び出しが入ったとき | 店舗の全端末 | ハンディ画面 |
| 3 | **低評価アラート** | お客様の満足度評価が★2以下で送信されたとき | マネージャー／オーナー | 管理画面ダッシュボード |
| 4 | **重要エラー通知** | 監視エージェントが障害や異常を検知したとき | マネージャー／オーナー | 管理画面ダッシュボード |
| 5 | **テスト送信** | POSLA管理画面「PWA / Push」タブの「テスト送信」から | 自分または店舗全員 | 指定した画面 |

::: tip 通知タップでアプリが自動で開きます
ホーム画面アイコン経由でPOSLAを開いていた場合、通知をタップすると対応する画面（ハンディ／管理画面など）が直接立ち上がります。ブラウザのタブを毎回開き直す手間がなくなります。
:::

### 通知を有効にする手順

1. 対象端末でPOSLAにログインする。管理画面は個人スタッフアカウント、ハンディ・KDS・POSレジは端末専用アカウントを使う。
2. 初回ログイン時、**「通知を許可しますか？」** のブラウザダイアログが表示されます。
3. **「許可」** を選ぶと、そのアカウント＋端末の組み合わせが通知購読リストに登録されます。
4. ホーム画面にアプリを追加していれば、画面を閉じてもロック画面に通知が表示されます。

「ブロック」を選んでしまった場合は、Chromeの「設定 → サイトの設定 → 通知」から該当URLを「許可」に戻してください。

### 通知が届かないときの確認

| 症状 | 原因 / 対処 |
|---|---|
| 通知が全く届かない | ブラウザで通知がブロックされている。Chrome「設定 → 通知」を確認 |
| Android Chrome で届くが iPhone Safari で届かない | **iOSのWeb Push は対応条件が厳しい**（ホーム画面追加後 + iOS 16.4以降）。業務端末はAndroid推奨 |
| ハンディにだけ届かない | 該当デバイスアカウント端末で一度ログアウト → 再ログイン（購読が期限切れの可能性） |
| マネージャー画面に低評価通知が来ない | ログイン中のユーザーのロールが `manager` / `owner` になっているか確認 |
| 何度も同じ通知が鳴り続ける | 端末の通知設定でPOSLA通知の「バナー重複」をオフに |

::: warning 通知の内容には個人情報を含めません
通知本文には「テーブル番号」「商品名」「評価点」などの業務情報のみが含まれます。クレジットカード情報・お客様の氏名・電話番号は通知payloadに一切載せません（Web Push の盗聴対策）。
:::

通知設定で迷った場合は、POSLAサポートへお問い合わせください。

## 音と通知の仕組み（ポーリング音 と PWA通知 の組み合わせ）

POSLAの業務端末では、**新規イベント（注文・呼び出し・低評価など）を気付かせる仕組みが2系統**動いています。両者は独立しており、状況に応じてどちらか、または両方が作動します。

### 2つの系統

**系統A: 画面を開いているときの「アプリ内ポーリング + 音」**
画面が開いている間、POSLAはサーバーへ定期的に問い合わせて新着を取りに行きます。新着があれば**画面内のチャイム音＋視覚アラート＋バイブ**でスタッフに気付かせます。**PWA導入なしでもブラウザだけで動作**します。

| 画面 | ポーリング間隔 | 検知対象 |
|---|---|---|
| KDS（注文一覧） | 約2秒 | 新規注文、ステータス変更 |
| KDS / ハンディ（呼び出しアラート） | 約5秒 | スタッフ呼び出し |
| POSレジ（注文ポーリング） | 約5秒 | 会計対象の新規注文 |
| ハンディ（テーブル状態） | 約5秒 | テーブル着席・会計状況 |
| 売上サマリー・品切れ情報 | 約30秒 | 集計更新 |

::: tip 初回の音出し許可（ハンディ）
ハンディ端末はモバイルブラウザの自動再生規制があるため、**起動直後に画面を1回タップ**してAudioContextを解放する必要があります。「タップして音声を有効化」バナーが出たら、画面のどこかを1回タップしてください。一度有効化すれば、その画面セッション中は以後のイベントで自動的に鳴ります。
:::

**系統B: PWA導入 ＋ 通知許可による「OS通知（Web Push）」**
ホーム画面追加＋通知許可をセットで設定すると、**画面やアプリを閉じていても**OSのロック画面・通知センターに通知が届きます。前節「業務端末で受け取れる通知」の5種類がこれに該当します。

### 両方が動くときの動作（4パターン）

| 画面の状態 | 通知許可 | 届き方 | 備考 |
|---|---|---|---|
| 画面を開いている | 許可あり | **系統A の画面内音** + **系統B のOS通知** の**両方**が動く | 見逃し防止には有効。「音が二重」と感じる場合は下記の対処を参照 |
| 画面を開いている | 許可なし | 系統A の画面内音のみ | PWA導入前と同じ挙動。最小構成 |
| 画面を閉じている／別アプリ中 | 許可あり | 系統B のOS通知のみ | ポーリングは止まっているため画面内音は鳴らない。画面を開き直すと未対応イベントがまとめて表示される |
| 画面を閉じている／別アプリ中 | 許可なし | 何も届かない | 次にPOSLAを開いたときに溜まったイベントを確認する運用になる |

::: warning Android Chrome は画面を開いていてもOS通知が出ます
Android Chromeの仕様で、PWAがフォアグラウンド（画面を見ている状態）でもWeb Push通知はOS通知として表示されます。そのため上記「画面を開いている + 許可あり」では系統Aのアプリ内音と系統BのOS通知が**実質同時に発火**することがあります。POSLA側では自動的な重複排除は行っていません（見逃し防止を優先）。
:::

### 「音が二重に鳴るのが気になる」場合

| 対処 | 方法 | 影響 |
|---|---|---|
| A: OS通知の音だけ無音にする | Android「設定 → 通知 → POSLA → サウンド: なし」（バッジとバナーは残す） | 画面内音のみ鳴る。画面を閉じた時は静かなバナーだけ届く |
| B: PWA側の通知許可を切る | Chrome「設定 → サイトの設定 → 通知 → POSLAのURL: ブロック」 | 画面を開いている間だけ音が鳴る（PWA導入前と同じ状態に戻す） |
| C: 業務端末では両方オン | （標準推奨） | 画面を開いていれば両方、閉じていればOS通知のみ。見逃しが一番少ない |

### PWAを導入すると何が増えるか（通知以外）

- **ホーム画面アイコンから1タップ起動**：ブラウザのタブを探さずに済む
- **OS通知（画面を閉じていても届く）**：前節の5種類
- **アプリバッジ**：ホーム画面アイコンに未対応件数の赤い丸数字（KDS・ハンディの呼び出し件数に連動、Android対応ブラウザのみ）
- **全画面表示**：ブラウザのアドレスバー・タブが消え、業務情報の表示領域が広がる
- **起動が少し速い**：ブラウザ分のUIを省けるため

オフライン機能（通信が切れても直近の画面を表示し続ける）は段階的に拡張中です。現在は**KDSと緊急レジが限定的に対応**しています（詳細は Part 8: トラブルシューティング）。

## ロールの違い

| ロール | 主な利用者 | できること |
|---|---|---|
| オーナー | 経営者・本部担当者 | 全店舗の管理、ユーザー管理、複数店舗レポート、契約・決済設定 |
| マネージャー | 店長・責任者 | 担当店舗のメニュー、注文、会計、在庫、シフト、レポート管理 |
| スタッフ | ホール・キッチン担当 | 勤怠打刻、シフト確認、希望シフト提出など |
| 端末専用アカウント | KDS・レジ・ハンディ端末 | 店舗備品端末での業務利用 |

POSLAでは、全スタッフに個別アカウントを作成してください。共有アカウントを使うと、勤怠、監査ログ、スタッフ評価で「誰が操作したか」を正しく記録できません。

## ご契約プラン

POSLA は **単一プラン + 任意アドオン** の構成です。すべての機能（セルフオーダー、ハンディPOS、KDS、POSレジ、テイクアウト、予約、在庫、シフト、AIアシスタント、多言語、レポート、決済連携など）はすべての契約者に標準提供されます。

| 項目 | 月額 |
|---|---|
| 基本料金（1店舗目） | ¥20,000 |
| 追加店舗（2店舗目以降） | ¥17,000 / 店舗（15%引き） |
| 本部一括メニュー配信アドオン（任意） | +¥3,000 / 店舗 |

**お試し利用：** 初回ご契約から **30日間無料**。期間中に解約された場合は課金されません。31日目から自動で月額課金が開始されます（カード登録は契約時に必要）。

ご契約内容の確認・店舗追加・アドオンのオンオフは、オーナーダッシュボードの「契約・お支払い」から操作できます。詳しい手順は [Part 7: 契約・お支払い](/operations/part7-billing) をご覧ください。

## 導入時の基本順序

1. オーナーアカウントでログインします。
2. 店舗情報、営業時間、税率、支払い方法を設定します。
3. スタッフアカウントと端末専用アカウントを作成します。
4. メニュー、テーブル、フロアマップを登録します。
5. KDS、レジ、プリンタ、決済端末を確認します。
6. テスト注文とテスト会計を行います。
7. 問題がなければ、お客様用QRコードを掲示します。

## 困ったとき

まずは画面に表示されたメッセージ、操作した画面名、発生時刻を控えてください。管理画面右下の **「📚 サポートガイド」** から FAQ / 症状別の案内 / エラー番号検索を利用できます。解決しない場合は POSLAサポートへご連絡ください。上記の情報を添えていただくと確認がスムーズです。
