# 1. はじめに

## 1.1 POSLAとは

POSLAは、飲食店向けのクラウド型セルフオーダー＆店舗管理システムです。お客さんがQRコードからスマホで注文し、注文がリアルタイムにキッチンのKDS（Kitchen Display System）へ表示されます。スタッフのハンディ端末やPOSレジとも連携し、メニュー管理・在庫管理・シフト管理・売上レポートまで、店舗運営に必要な機能を一つのシステムでカバーします。

1つの企業アカウントで複数の店舗を管理できます。本部メニューの一元管理と、店舗ごとの価格・品揃えカスタマイズを両立させているため、チェーン展開や多店舗運営にも対応しています。

| Safari（最新版） | 対応 | iOS端末のデフォルトブラウザ |
| Microsoft Edge（最新版） | 対応 | |
| Firefox（最新版） | 対応 | 音声コマンド非対応 |

### 推奨端末

| 用途 | 推奨端末 | 画面サイズ |
|------|---------|-----------|
| 管理ダッシュボード | PC / タブレット | 10インチ以上推奨 |
| KDS（キッチンディスプレイ） | タブレット（10インチ以上） | 横置き推奨 |
| ハンディPOS | スマートフォン / 小型タブレット | 5〜8インチ |

| セルフオーダー（お客さん） | お客さんのスマートフォン | 制限なし |

### インターネット接続

POSLAはクラウド型のシステムです。常時インターネット接続が必要です。ただし、一時的な接続切断時には「オフライン検知機能」が動作し、画面上部に赤いバナーで警告が表示されます。接続が復旧すると自動的に緑のバナーが表示され、通常動作に戻ります。

::: warning 注意
オフライン中は新規注文の送信やデータの取得ができません。Wi-Fi環境が不安定な場合は、有線LAN接続またはモバイルルーターのバックアップを推奨します。
:::

## ロールの違い

POSLAには3つのユーザーロールがあります。ロールによって操作できる機能が異なります。

### オーナー（owner）

テナント全体の管理者です。全店舗に暗黙的にアクセスでき、以下の操作が可能です。

- 店舗の追加・編集・削除
- 全店舗のユーザーアカウント管理（マネージャー・スタッフの作成）
- クロス店舗レポート（複数店舗の売上を横並び比較）
- ABC分析（売上貢献度によるメニュー分類）
- 契約・プラン管理（サブスクリプション）
- 決済設定（Stripe Connect）
- 統合シフトビュー（全店舗のシフト横断表示、全契約者に標準提供）
- 外部POS連携（スマレジ）
- AIアシスタント（店舗を選択して分析）

ログイン後は **オーナーダッシュボード**（owner-dashboard.html）に自動遷移します。

### マネージャー（manager）

割り当てられた店舗の管理者です。以下の操作が可能です。

- メニュー管理（カテゴリ・テンプレート・オーバーライド・限定メニュー・オプション）
- テーブル・フロアマップ管理
- 在庫管理（食材・レシピ・棚卸し）
- 注文管理（ハンディPOS・テイクアウト）
- KDS操作（キッチンディスプレイ）
- レジ操作（会計・精算）
- レポート閲覧（売上・レジ・回転率・スタッフ評価・併売分析）
- 店舗設定（営業時間・税率・支払い方法・ブランディング等）
- スタッフ管理（自店舗のスタッフアカウント作成）
- シフト管理（全機能：テンプレート・カレンダー・勤怠一覧・サマリー・AI提案・ヘルプ要請）
- AIアシスタント（SNS投稿生成・売上分析・競合調査・需要予測）
- 監査ログ閲覧
- 領収書設定

ログイン後は **管理ダッシュボード**（dashboard.html）に自動遷移します。

### スタッフ（staff）

割り当てられた店舗の操作担当者です。以下の操作が可能です。

- ハンディPOS（注文入力）
- KDS操作（キッチンディスプレイ）
- レジ操作（会計）
- 勤怠打刻（出勤・退勤）
- マイシフト確認（自分のシフト閲覧）
- 希望シフト提出

ログイン後は **管理ダッシュボード**（dashboard.html）に自動遷移します。マネージャー専用のタブ（メニュー管理、レポート等）は非表示になります。

::: tip 1人1アカウントは必須です
POSLAでは全スタッフに個別のユーザーアカウントを作成してください。「kitchen」「hall」などの共有アカウントは禁止です。監査ログ・勤怠打刻・スタッフ評価・シフト管理の全てが「誰が操作したか」を前提に設計されています。アカウント未作成のスタッフはシフト管理・勤怠の対象外になります。
:::

## 1.X POSLA の哲学

### 業務フローで売る

機能別プランを廃止。飲食店の業務は **予約 → 着席 → 注文 → 厨房 → 提供 → 会計 → 在庫 → 分析 → 次回予約** と全部つながっているので、一部だけ使う運用は本質的にない。お客さん（テナント）視点で「何を選べばいいか」迷わせない。

### 単一プラン

- 1店舗目 ¥20,000 / 月
- 2店舗目以降 ¥17,000 / 月 (15% 引き)
- 本部一括配信アドオン +¥3,000 / 月 / 店舗

### 30日無料トライアル

カード登録必須。期間内キャンセルで請求なし。

### 全機能を全契約者に標準提供

POSLA の機能差別化は **本部一括メニュー配信** (`hq_menu_broadcast`) の 1 つだけ。それ以外（KDS、AI、シフト、予約、テイクアウト、多言語、ABC 分析、監査ログ、決済ゲートウェイ等）はすべての契約者に標準提供。

### 業務端末は店舗備品が前提

ハンディ／KDS／POSレジは店舗が用意する備品端末で、device アカウントで運用。スタッフ個人のスマホを業務に使う BYOD 運用は推奨しない（労務・セキュリティ・補償の問題回避）。

## 1.B 4 つのロール（owner / manager / staff / device）の使い分け

POSLA には **4 ロール** あります。1.X までで紹介した owner / manager / staff の他に、**device**（端末専用ログイン）が 2026-04-08 P1a で追加されました。

| ロール | レベル | ログイン後の遷移先 | 主な用途 | シフト/勤怠対象 |
|---|:---:|---|---|:---:|
| owner | 3 | `owner-dashboard.html` | 本部運営・契約・複店舗集計 | 対象 |
| manager | 2 | `dashboard.html` | 店舗運営全般 | 対象 |
| staff | 1 | `dashboard.html` (シフトタブのみ) | 自分のシフト確認・希望提出・勤怠打刻 | 対象 |
| **device** | 0 | `handy/index.html` または `kds/cashier.html` または `kds/index.html` | KDS / レジ / ハンディの **端末ログイン** (1 端末 1 アカウント) | **対象外** |

### 1.B.1 device が「人」と分けて管理される理由

- **端末は備品である** という運用前提（CLAUDE.md / プロジェクトメモリ「業務端末は全て店舗備品」）
- スタッフ個人のスマホで業務するとセキュリティ・労務・補償の問題が発生する
- レジ・KDS・ハンディは店舗が用意した端末に device アカウントでログインしておき、出勤したスタッフは PIN で「誰の操作か」を識別する設計
- そのため `users.role='device'` のユーザーは:
  - シフト割当の対象に **ならない** (`apply-ai-suggestions.php` が `DEVICE_NOT_ASSIGNABLE` で拒否)
  - 人件費・労働時間集計の対象に **ならない**
  - 監査ログの「スタッフ別操作履歴」レポートの対象に **ならない**
  - owner-dashboard の「ユーザー管理」一覧には **表示されない** (`role != 'device'` で絞り込み)

### 1.B.2 device 作成は dashboard 「スタッフ管理」タブ

- `dashboard.html` → 店舗設定タブ → スタッフ管理サブタブ → **「+ デバイス追加」**
- 必要権限: manager 以上
- `visible_tools`: `kds` / `register` / `handy` のいずれか、または複数（カンマ区切り）
- 詳細は [2 章 2.5 デバイス管理](./02-login.md)

## 1.C 推奨環境（くわしく）

### 1.C.1 ブラウザ別の機能差

| ブラウザ | 推奨度 | 理由・制約 |
|---|---|---|
| Google Chrome (PC / Android) | ★★★ | Web Speech API (continuous モード) 完全対応、PWA / 通知対応 |
| Safari (iOS) | ★★ | Web Speech API は continuous 非対応 → **KDS 音声コマンドは Chrome 必須** |
| Microsoft Edge | ★★★ | Chromium ベースなので Chrome と同等 |
| Firefox | ★★ | 音声コマンド非対応。それ以外は問題なし |
| iOS / Android 古いバージョン | ✗ | Android 8 以下、iOS 13 以下は非対応 |

### 1.C.2 端末別の推奨スペック

| 用途 | 端末例 | 必須スペック |
|---|---|---|
| 管理ダッシュボード (PC) | Mac / Windows ノート | RAM 8GB 以上、画面 13 インチ以上 |
| 管理ダッシュボード (タブレット) | iPad Air / Pro | RAM 4GB 以上、画面 10 インチ以上 |
| KDS (キッチン) | iPad Pro 12.9 / Lenovo Tab P12 | 画面 10 インチ以上、横置き、防水カバー推奨 |
| ハンディ POS | iPhone 12 以降 / Pixel 7 / 業務 Android | 画面 5〜7 インチ、片手で持てる重量 |
| POS レジ | iPad Pro / Windows ノート | カードリーダー (Stripe Reader S700) と同 Wi-Fi 必須 |
| お客様セルフオーダー | お客様私物のスマホ | 制限なし (QR を読めればよい) |

### 1.C.3 Wi-Fi 環境

POSLA はクラウド型のため、Wi-Fi が安定していないと「注文が送れない」「KDS の画面が更新されない」「会計時にカードリーダーが応答しない」などの症状が出ます。以下を推奨します。

| 項目 | 推奨 | NG例 |
|---|---|---|
| 回線 | 光 100Mbps 以上 | モバイル WiFi のみ (帯域制限あり) |
| ルーター | 業務用 (BUFFALO WAPM など) / メッシュ Wi-Fi | 家庭用 IEEE 802.11n 単発 |
| 同時接続数 | 想定端末数 + お客様スマホ 50 台 / 100 台 | 同時 10 台で詰まるルーター |
| バックアップ回線 | LTE モバイルルーター 1 台 | なし (店長の個人テザリングは NG) |

::: warning Wi-Fi 障害が POSLA トラブルの 7 割
店舗から POSLA 運営に来る「動かない」連絡の **約 7 割は Wi-Fi 起因**です。導入前に必ず Wi-Fi 強度を実測してください ([19 章 19.5 ネットワーク関連](./19-troubleshoot.md#19-5-ネットワーク関連) 参照)。
:::

## 1.F よくある質問 (FAQ)

**Q1. POSLA は無料で試せますか？**

A. はい、**30 日間無料トライアル**があります (Stripe にカード登録は必要)。期間内に解約すれば 1 円も請求されません。詳細は [21.2](./21-stripe-full.md#21-2-stripe-billing-posla月額料金) 参照。

**Q2. 何店舗まで運用できますか？**

A. 上限はありません。実績ベースでは 30 店舗を超える本部運営にも対応しています。1 店舗目 ¥20,000 / 2 店舗目以降 ¥17,000/店舗の従量課金です。

**Q3. 既存の POS を残したまま POSLA を試せますか？**

A. はい。**スマレジ連携** ([22 章](./22-smaregi-full.md)) を使えば既存 POS とデータ同期できます。完全乗り換え前のパラレルラン期間を設けられます。

**Q4. オフライン対応はありますか？**

A. **限定的**です。お客様セルフオーダーはオフライン保留 → 復旧時自動送信に対応しますが、KDS / レジの完全オフライン運用は **非対応**です。Wi-Fi 障害時は手書き伝票運用 ([19.10.3](./19-troubleshoot.md#19-10-3-営業継続のためのバックアップ手順)) を準備してください。

**Q5. データはどこに保存されますか？**

A. さくらインターネット (国内) の MySQL 5.7 サーバーに保存されます。テナントごとに DB は論理分離 (物理 1 DB / `tenant_id` で分離)。

**Q6. 解約したらデータはどうなりますか？**

A. 解約後 30 日間はレポート閲覧・CSV エクスポート可能。30 日経過後は POSLA 運営の判断で物理削除されます (事前通知あり)。詳細は [21.2.8 Q14](./21-stripe-full.md)。

**Q7. お客様の個人情報は POSLA に保存されますか？**

A. **基本的に保存しません**。セルフオーダーは匿名 (テーブルセッション単位) です。カード決済情報は Stripe 側に保存され、POSLA は last4 桁のみ保持します。予約時のみ氏名・電話番号を保存 ([24 章](./24-reservations.md))。

**Q8. 多店舗で同じメニューを一括配信したい**

A. **本部一括メニュー配信アドオン** (¥3,000/月/店舗) を契約してください。owner-dashboard の「本部メニュー」タブから全店舗に配信できます ([4 章](./04-menu.md))。

**Q9. AI 機能 (SNS 投稿生成 / 売上分析 / AI ウェイター) は別料金？**

A. **不要です**。基本料金にすべて含まれます。Gemini API 利用料は POSLA 運営が一括負担します ([14 章](./14-ai.md))。

**Q10. 言語は何カ国語に対応していますか？**

A. **5 言語**: 日本語 / 英語 / 中国語 (簡体・繁体) / 韓国語。お客様セルフオーダーは自動翻訳されます ([4 章 翻訳機能](./04-menu.md))。

**Q11. 個人スマホをハンディ代わりに使えますか？**

A. **推奨しません**。労務・セキュリティ・補償の問題回避のため、ハンディは店舗備品 (device アカウント) で運用してください。技術的には可能ですが、`personal_phone_handy=0` がデフォルトで個人ログインからのハンディ利用は制限されています。

**Q12. レビュー誘導機能はありますか？**

A. はい。満足度評価 4-5 星のお客様に Google マップへのレビュー投稿リンクを表示します ([20 章](./20-google-review.md))。

**Q13. プリンタ連携はどのメーカーが使えますか？**

A. **Star (mC-Print3 / TSP100IIILAN) と Epson (TM-m30II)** の IP 印刷モデル。USB / Bluetooth プリンタは非対応。詳細は [26 章](./26-printer.md)。

**Q14. KDS は専用アプリですか？**

A. いいえ、**ブラウザだけ**です。タブレットの Chrome で `kds/index.html` を開けば動作します。専用アプリのインストール不要 ([23 章](./23-kds-devices.md))。

**Q15. 予約管理はありますか？**

A. はい。**ガント台帳・自由文 AI 予約・予約金・コース・VIP/ブラックリスト** に対応 ([24 章](./24-reservations.md))。

**Q16. 帳簿 (会計ソフト) と連携できますか？**

A. CSV エクスポートに対応しています。日次売上 / 注文明細 / レジ明細を CSV ダウンロードできるので、freee / マネーフォワード等にインポートしてください ([13 章](./13-reports.md))。

**Q17. 営業中にメニューを変えたら即反映されますか？**

A. 各画面でキャッシュタイミングがあります。セルフ・ハンディは 3 分のメニューバージョンチェック、KDS は次の注文受信時。即反映したい場合はブラウザリロード ([19.6](./19-troubleshoot.md#19-6-メニュー関連) 参照)。

**Q18. 予約金 / 食べ放題 / コース料理は別料金？**

A. **不要です**。すべて基本料金内で利用できます。

**Q19. AI ヘルプデスクはどこから使えますか？**

A. 全管理画面の **右下 オレンジ色の ? ボタン** (FAB)。1 タップでチャットパネルが開き、本マニュアル全章 + エラーカタログを知識ベースに回答します ([14.6](./14-ai.md))。

**Q20. 料金プランは将来変わる可能性がありますか？**

A. POSLA 運営の判断で変更される可能性はありますが、契約済みテナントには事前通知のうえ移行措置を取ります。2026-04-09 の α-1 移行時も既存テナントは自動マイグレーションされました。

## 1.G ホーム画面追加でアプリ化できます (PWA)

POSLA の **スタッフ向け業務画面 3 種** はホーム画面に追加すると、独立した「アプリ」として 1 タップで全画面起動できます。Chrome / Safari の標準機能 (PWA = Progressive Web App) で実現しているため、専用アプリのインストール (App Store / Google Play) は **不要**です。

### 1.G.1 対応スコープ

| スコープ | 用途 | URL 例 | アイコン色 |
|---|---|---|---|
| 管理ダッシュボード | owner / manager / staff の管理操作 | `https://meal.posla.jp/public/admin/dashboard.html` | 青 (#1976d2) / アイコン「管」 |
| KDS / レジ | キッチンディスプレイ・POSレジ | `https://meal.posla.jp/public/kds/index.html` `https://meal.posla.jp/public/kds/cashier.html` | 緑 (#2e7d32) / アイコン「厨」 |
| ハンディ POS | ホールスタッフの注文入力 | `https://meal.posla.jp/public/handy/index.html` | 橙 (#ff6f00) / アイコン「配」 |

3 つともホーム画面に追加するとそれぞれ別アプリとして起動し、用途別にアイコン色と「管 / 厨 / 配」の文字で識別できます。1 台のタブレットに 3 つとも追加することも可能です。

### 1.G.2 顧客のスマホには追加しません

セルフオーダー (お客様が QR から開くメニュー画面 `public/customer/menu.html` 等) は **意図的に PWA 化していません**。理由:

- 1 回限りの来店フロー (QR スキャン → 注文 → 会計) のため、ホーム画面に残しても再利用しない
- セッション ID / テーブル ID / トークンが URL に含まれるため、過去のキャッシュが他人 (次の来客) に流用されると事故になる
- 来店ごとにメニュー差分があるため、古い情報がキャッシュに残ると間違った注文を生む

お客様には「ブラウザでそのまま開いてください」と案内してください。

### 1.G.3 詳しい追加手順

OS 別の具体的な手順 (iPhone Safari / Android Chrome) は次のページを参照:

- ステップごとの追加手順 (写真なしテキスト版): [Part 0 「準備・ログイン」 0-13 ホーム画面追加](../operations/part0-getting-started.md#0-13-ホーム画面追加でアプリ化する-pwa)
- KDS タブレットを「常時オン化 + 他アプリ切替防止」する運用: [23 章 23.12 KDS タブレットの常時オン化 (PWA + キオスク)](./23-kds-devices.md#23-12-kds-タブレットの常時オン化-pwa-キオスク)

::: tip 古い画面が出続けるとき
ホーム画面追加後に古い画面が表示される場合は、**一度アイコンを削除して再追加** すれば直ります。それでも直らない場合は POSLA 運営にご連絡ください ([19 章](./19-troubleshoot.md))。
:::
