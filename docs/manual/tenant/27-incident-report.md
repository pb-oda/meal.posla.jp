# 27. 障害報告窓口とお問い合わせコード

POSLAで注文、会計、予約、KDS、ログインなどに問題が出た場合は、画面に表示された **お問い合わせコード** と発生状況を POSLA 運営へ共有してください。

::: tip 最初に伝えるもの
- 店舗名
- 画面名
- 表示されたお問い合わせコード
- 発生時刻
- 何をしようとしたか
- 可能であればスクリーンショット
:::

## 27.1 障害報告窓口の役割

障害報告窓口は、POSLAの操作中に起きた不具合や業務停止に近い問題を報告するための窓口です。

対象になるもの:

| 対象 | 例 |
|---|---|
| レジ / 会計 | 会計できない、決済が途中で止まる、レシートが出ない |
| 注文 | 注文がKDSへ出ない、注文履歴に残らない |
| KDS | 厨房画面が更新されない、調理ステータス変更ができない |
| 予約 | 予約台帳に入らない、予約通知が届かない |
| ログイン | 正しいID/パスワードでも入れない |
| 画面エラー | `E3035` のようなお問い合わせコードが出る |

対象外のもの:

| 対象外 | 連絡方法 |
|---|---|
| 使い方の質問 | サポートガイド、通常のお問い合わせ |
| 新機能の要望 | POSLA運営への要望連絡 |
| 契約・請求の相談 | 契約窓口 |
| 店舗内のWi-Fiや端末故障 | 店舗側のネットワーク・端末確認 |

## 27.2 お問い合わせコードとは

POSLAのエラーには、原因調査を早くするための番号が付くことがあります。

例:

```text
処理に失敗しました
お問い合わせコード: E3035
発生時刻: 2026-04-27 10:15
```

この番号があると、POSLA運営側で該当エラー、関連API、直近のログ、対象店舗、対象cellを確認しやすくなります。

## 27.3 報告時に必要な情報

障害報告では、できるだけ次を入力してください。

| 項目 | 例 | 理由 |
|---|---|---|
| お問い合わせコード | `E3035` | 原因候補を絞るため |
| 画面名 | POSレジ、KDS、予約台帳 | 関連機能を特定するため |
| 店舗名 | 渋谷店 | 複数店舗運用で対象を間違えないため |
| 発生時刻 | 2026-04-27 18:30頃 | ログ確認範囲を絞るため |
| 操作内容 | 現金会計を確定しようとした | 再現手順を確認するため |
| 支払い方法 | 現金、カード、QR | 決済系の切り分けに必要 |
| スクリーンショット | 画面全体 | 表示文言の取り違えを防ぐため |

::: warning コードと画面が合わない場合があります
お問い合わせコードだけで判断できないことがあります。
たとえば「レジで会計できない」と申告されても、コードが予約AIのエラーを示している場合は、まず実際の画面とコードの取り違えを確認します。
:::

## 27.4 緊急度の目安

| 緊急度 | 状況 | 店舗側の初動 |
|---|---|---|
| 高 | 会計できない、注文が通らない、レジが止まる | 代替運用へ切替し、すぐ報告 |
| 中 | 一部機能だけ使えない、特定店舗だけ発生 | 店長が状況を整理して報告 |
| 低 | 表示が遅い、再読み込みで直る、単発エラー | 再発するか確認してから報告 |

営業中に会計や注文が止まる場合は、まず店舗の営業継続を優先します。

- 紙伝票で注文を控える
- 手書きで会計を控える
- 決済端末やレジが使えない場合は店舗ルールに従う
- 復旧後にPOSLAへ入力し直す必要がある内容を残す

## 27.5 POSLA運営側で行うこと

障害報告が送信されると、POSLA運営側では次を確認します。

1. 対象の顧客名、cell、店舗を特定
2. お問い合わせコードの意味を確認
3. 直近のエラーログ、監視イベント、デプロイ履歴を確認
4. 必要に応じてAI運用支援基盤で調査
5. コード修正、設定修正、顧客への確認依頼のどれが必要か判断

店舗側は、調査のために追加情報を求められる場合があります。

## 27.6 送ってはいけない情報

障害報告には、次の情報を入力しないでください。

- クレジットカード番号
- セキュリティコード
- お客様の不要な個人情報
- POSLA以外のサービスのパスワード
- 店舗Wi-Fiのパスワード

必要な場合でも、POSLA運営から案内された安全な方法で共有してください。

## 27.7 よくある報告例

### 良い例

```text
店舗: 渋谷店
画面: POSレジ
お問い合わせコード: E3035
発生時刻: 2026-04-27 18:32
内容: 現金会計で確定ボタンを押したところ、エラーが出て会計完了になりません。
補足: 同じ端末で2回発生。別端末では未確認。スクリーンショットあり。
```

### 情報が不足している例

```text
レジが動きません。
```

この場合、画面、コード、時刻、支払い方法がないため、原因調査に時間がかかります。

## 27.8 復旧後に確認すること

POSLA運営から復旧連絡を受けたら、店舗側で次を確認してください。

1. 同じ操作をもう一度実行できるか
2. 注文・会計・予約などのデータが重複していないか
3. レシート、レポート、売上が想定通りか
4. 代替運用で控えた内容をPOSLAに反映する必要があるか

復旧後も同じお問い合わせコードが出る場合は、同じ報告に追記するか、再度障害報告を送信してください。
