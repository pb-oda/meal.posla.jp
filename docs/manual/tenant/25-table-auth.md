# 25. テーブル開放認証

テーブル開放認証は、お客様がテーブルQRから注文する前に、スタッフがその卓を利用可能にする仕組みです。POSLAでは、QRコードを印刷してテーブルに置くだけでは注文できません。スタッフがハンディで `QR開放` した後、一定時間内にお客様がQRを読むことで注文セッションが開始します。

この仕組みは、誤注文、いたずら注文、前のお客様のURL再利用、店外からの遠隔注文を防ぐために使います。

## 25-1. 用語

| 用語 | 意味 |
|---|---|
| テーブルQR | 卓に置く紙のQRコード |
| QR開放 | スタッフがその卓を注文可能にする操作 |
| 開放トークン | QR開放時に発行される一時的な許可情報 |
| セッション | そのお客様グループの着席から会計終了までの利用単位 |
| セッション終了 | 会計済み、清掃完了、着席キャンセルなどで利用を終えること |
| QR取消 | 誤って開放したQR許可を取り消すこと |
| QR再発行 | 古い開放を取り消し、新しい開放を作ること |

## 25-2. 基本フロー

1. お客様が来店
2. スタッフが空席へ案内
3. ハンディの `テーブル状況` または `運用` を開く
4. 対象卓が `空き` であることを確認
5. `QR開放` をタップ
6. POSLAが短時間だけ有効な開放トークンを作成
7. お客様がテーブルQRを読み込む
8. 開放トークンが有効なら、テーブルセッションを開始
9. お客様はセルフメニューから注文
10. 会計完了後、セッションは終了
11. 同じURLを再度開いても、前のお客様の注文画面としては使えない

## 25-3. QR開放の手順

1. ハンディを開く
2. `テーブル状況` タブを開く
3. 案内する卓を探す
4. 卓カードの状態が `空き` であることを確認
5. `QR開放` をタップ
6. 確認ダイアログを読む
7. 問題なければOK
8. 卓カードが `QR開放中` になる
9. お客様にQRを読んでもらう

運用タブからも同じ操作ができます。壁掛け端末でQR開放中の卓を確認したい場合は、運用タブの `QR開放中` カードと卓クイック操作を見ます。

## 25-4. QR開放中に表示される情報

| 表示 | 内容 |
|---|---|
| QR読込待ち | お客様がまだQRを読んでいない |
| 残り○分 | 開放の有効期限までの残り時間 |
| 開放: 端末名 | どのデバイスアカウントで開放したか |
| QR取消 | 誤開放を取り消す |
| QR再発行 | 新しい開放に作り直す |

`開放: 端末名` は、複数のハンディや壁掛け端末で営業しているときに、どの端末で開放したか確認するために使います。ハンディはデバイスアカウント前提のため、人の担当者選択は行いません。

## 25-5. QR取消

### 25-5-1. 使う場面

| 場面 | 理由 |
|---|---|
| 間違った卓を開放した | 別のお客様が読んでしまう前に止める |
| お客様が席を変えた | 移動前の卓の開放を残さない |
| お客様が入店を取りやめた | 開放だけが残るのを防ぐ |
| 予約席を誤って開放した | 予約客以外が使わないようにする |

### 25-5-2. 手順

1. `テーブル状況` または `運用` を開く
2. `QR開放中` の卓を探す
3. `QR取消` をタップ
4. 確認してOK
5. 卓が空き状態に戻る

取消後は、まだQRを読み込んでいないお客様は注文画面へ進めません。すでに読み込み済みでセッションが開始している場合は、QR取消ではなく着席キャンセルや会計/清掃の流れで対応します。

## 25-6. QR再発行

### 25-6-1. 使う場面

| 場面 | 操作 |
|---|---|
| 残り時間が少ない | QR再発行 |
| スマホの読み込みに失敗した | QR再発行 |
| 古い開放を使わせたくない | QR再発行 |
| スタッフが開放し直したい | QR再発行 |

### 25-6-2. 手順

1. `QR開放中` の卓を探す
2. `QR再発行` をタップ
3. POSLAが古い開放を取り消す
4. 新しい開放を作る
5. 残り時間が新しくなる
6. お客様に再度QRを読んでもらう

## 25-7. 有効期限

QR開放は短時間だけ有効です。読み込み待ちのまま時間切れになった場合、再度 `QR開放` または `QR再発行` を行ってください。

| 理由 | 対応 |
|---|---|
| お客様がスマホ操作に時間がかかった | 再発行して案内 |
| 読み込み前に席移動した | 移動元を取消、移動先を開放 |
| 開放後にスタッフが別対応へ行った | 運用タブでQR開放中を確認 |

## 25-8. セッション開始後の扱い

お客様がQRを読み込んでセッションが開始すると、その卓は `着席` または `食事中` として扱われます。

| 操作 | 説明 |
|---|---|
| 注文 | スタッフがハンディで追加注文 |
| QR表示 | お客様向け/個別QRを再表示 |
| メモ | アレルギーや接客メモを保存 |
| 人数変更 | 来店人数の修正 |
| 席移動 | セッションと注文を別卓へ移動 |
| 会計呼び出し | 会計待ちにする |

## 25-9. 会計完了後のURL再利用防止

会計が完了すると、対象セッションは終了します。前のお客様がスマホの履歴や同じURLから再度アクセスしても、そのセッションの注文・履歴・会計情報へ進めないように制御されます。

店舗側で必ず行うこと:

1. レジで会計を完了する
2. テーブル状態が会計済み/清掃待ち/終了済みに進んだことを確認
3. 清掃完了後、次のお客様を案内する
4. 次のお客様には必ず新しくQR開放する

::: warning 前のお客様の画面が残っていると言われた場合
まず会計が完了しているか、テーブルセッションが終了しているかを確認してください。会計完了前にお客様がブラウザ履歴を開いた場合は、まだ同じセッションとして扱われます。会計完了後も見える場合は、店舗・卓番号・発生時刻を控えてPOSLAサポートへ連絡してください。
:::

## 25-10. 席移動とQR開放

席移動では、セッション、未会計注文、QR情報、メモ、呼び出し、サブセッションをまとめて移します。

| 状況 | 操作 |
|---|---|
| 移動元が着席中 | `席移動` で空席へ移す |
| 移動先がQR開放中 | 先に移動先のQR取消、または別卓を選ぶ |
| 移動元だけQR開放中 | QR取消してから移動先を開放 |
| 移動後にセルフ注文を続ける | 移動先でQR表示または必要に応じて再案内 |

## 25-11. 予約来店との関係

予約客が来店した場合は、予約タブまたはテーブル状況から操作します。

| 操作 | 使う場面 |
|---|---|
| 予約着席 | スタッフが予約客を直接着席済みにする |
| QR開放 | 予約客にセルフ注文してもらう |
| no-show | 予約客が来なかった |

予約席を誤って別のお客様にQR開放しないよう、テーブルカードの予約表示とメモを確認してください。

## 25-12. トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| お客様がQRを読んでも進めない | QR未開放、期限切れ、通信不安定 | ハンディでQR開放中か確認し、必要なら再発行 |
| 別卓の画面が開く | 紙QRの貼り間違い、卓選択ミス | QR紙と卓番号を確認し、誤開放なら取消 |
| QR開放中が残っている | お客様が読まずに時間が経過 | 読む予定がなければ取消 |
| 前のお客様の情報が見える | 会計/終了処理漏れの可能性 | レジ会計完了、清掃完了、セッション終了を確認 |
| 予約客が注文できない | 予約卓が未開放、別卓に案内 | 予約タブで対象予約と卓を確認し、QR開放 |
| 席移動後に注文が旧卓に見える | 画面が古い | 運用タブまたはテーブル状況を更新 |
| QR再発行後も進めない | 端末側キャッシュ、通信不安定 | ブラウザを閉じて再読込、Wi-Fi確認 |

## 25-13. 店舗ルールとして決めておくこと

| ルール | 推奨 |
|---|---|
| QR開放端末 | 案内時に近いハンディ、または壁掛け端末で開放する |
| QR開放後の声かけ | 2分程度で読めているか確認 |
| 誤開放時 | 気づいたスタッフが即取消 |
| 席移動時 | 移動前に卓番号を復唱し、移動後に運用タブで確認 |
| 会計後 | レジ担当が終了状態を確認し、ホールが清掃完了 |
| 共有端末 | 壁掛け端末は運用タブ、音ONで使う |
