# 6. 注文管理（ハンディPOS）

この章では、ホールスタッフが使う **ハンディPOS** の操作を、来店から注文、配膳、呼び出し、会計待ち、席移動、取消、未送信復旧までまとめて説明します。

ハンディは「注文を入れる画面」だけではありません。現在のPOSLAでは、ホールが見るべき **今やること** を集約する運用ボードも兼ねています。壁掛け端末として置く場合も、同じハンディ画面を使います。

::: tip 先に結論
通常運用でスタッフの手数を増やさないため、ハンディの基本操作は次の考え方で使います。

1. お客様が来店したら、テーブルを確認してQRを開放する
2. セルフ注文なら、お客様がQRから注文する
3. スタッフが注文を取る場合だけ、ハンディの店内注文から入力する
4. KDSで料理完成になったら、運用タブで配膳する
5. 呼び出し、会計待ち、清掃待ち、時間超過は運用タブで上から処理する
6. 間違いが起きたときだけ、取消・戻し・席移動・未送信復旧を使う
:::

## 6-1. ハンディで扱う業務範囲

| 業務 | ハンディで行うこと | 主なタブ |
|---|---|---|
| 来店受付 | 空席確認、着席、QR開放、予約着席 | テーブル状況、予約、運用 |
| セルフ注文開始 | QR開放、開放取消、再発行、残り時間確認 | テーブル状況、運用 |
| スタッフ注文 | テーブル選択、商品選択、数量・オプション・メモ入力、送信 | 店内注文 |
| テイクアウト | 顧客名、電話番号、受取時間、商品入力、送信 | テイクアウト |
| 配膳 | KDSの料理完成通知を確認し、配膳完了にする | 運用 |
| 呼び出し対応 | 客席呼び出し、厨房呼び出し、対応済み | 運用 |
| 会計待ち | 会計待ちにする、間違い時に食事中へ戻す | テーブル状況、運用 |
| 清掃待ち | 会計後の清掃待ち確認、清掃完了 | 運用、テーブル状況 |
| 席移動 | 卓Aから卓Bへセッション・注文・QR・メモ・呼び出しを移す | テーブル状況、運用 |
| 注文修正 | 注文取消、品目取消、数量変更、配膳取消 | 注文履歴、運用 |
| 通信失敗復旧 | 未送信注文の中身確認、再送、破棄 | ヘッダー、運用 |

## 6-2. 画面構成

### 6-2-1. ヘッダー

| 表示 | 説明 |
|---|---|
| ハンディ | 画面名 |
| 店舗名 | 現在操作している店舗 |
| 店舗切替 | 複数店舗に所属している場合に表示 |
| ハンディ / 壁掛け / レジ横 | 端末モード切替 |
| 音ON / 音OFF | 運用通知音のON/OFF |
| 未送信 | 通信失敗で端末内に残っている注文件数 |
| ユーザー名 | ログイン中スタッフ |
| ログアウト | 退勤時・端末交代時に使う |

### 6-2-2. 端末モード

端末モードは、同じハンディ画面をどこに置くかで表示密度を変えるための切替です。

| モード | 用途 | 表示の特徴 |
|---|---|---|
| ハンディ | スタッフが持ち歩くスマホ/小型端末 | 通常密度。注文入力、卓操作、履歴確認をすべて使う |
| 壁掛け | ホール共有のタブレット/モニタ | 文字大きめ。呼び出し、料理完成、会計待ち、清掃待ちを優先 |
| レジ横 | レジ付近で会計待ちや清掃待ちを見る端末 | 会計・清掃に寄せた表示。卓の補助操作は少なめ |

端末モードは端末ごとに保存されます。別のスマホで切り替えても、今使っている端末の表示設定だけが変わります。

### 6-2-3. 通知音

`音ON` にすると、運用タブで未対応の通知があるときに音が鳴ります。

| 項目 | 仕様 |
|---|---|
| 初期値 | OFF |
| ONにする場所 | ヘッダーの `音OFF` ボタン |
| 対象 | 呼び出し、料理完成、会計待ち放置、清掃待ち放置、時間超過 |
| 再通知 | 未対応のまま残ると約3分ごと |
| 強調通知 | 呼び出し3分超、料理完成5分超などは強めの音 |
| 保存単位 | 端末ごと |

::: warning 通知音が初期OFFの理由
ChromeやSafariなどのブラウザでは、ユーザーが画面を一度操作するまで自動で音を鳴らせません。そのため、壁掛け端末では開店時に一度だけ `音OFF` を押して `音ON` にしてください。
:::

## 6-3. タブごとの役割

### 6-3-1. 店内注文

店内席のお客様からスタッフが注文を取るときに使います。

| 表示 | 説明 |
|---|---|
| テーブル選択 | 注文を入れる卓を選ぶ |
| 選択卓パネル | 卓の状態、経過時間、注文数、会計待ち、QR開放中などを表示 |
| 卓クイック操作 | 注文、会計呼び出し、会計解除、メモ、人数、席移動など |
| よく出る | 最近よく出ている商品をワンタップで追加 |
| この卓の直近 | 同じ卓の直近注文を再注文 |
| カテゴリ | 定食、ドリンクなどの商品分類 |
| 商品グリッド | 商品をタップしてカートへ入れる |
| カート | 数量変更、メモ、送信 |

### 6-3-2. テイクアウト

店頭や電話で受けたテイクアウト注文をスタッフが入力します。

| 項目 | 説明 |
|---|---|
| 顧客名 | 受け渡し時に確認する名前 |
| 電話番号 | 連絡が必要な場合に使う |
| 受取時間 | KDSとレジで確認する受取予定時刻 |
| 商品 | テイクアウトする商品 |

### 6-3-3. テーブル状況

全卓の状態を確認し、来店受付や卓操作を行います。

| 状態 | 意味 | 主な操作 |
|---|---|---|
| 空き | まだ使用していない卓 | 着席、QR開放 |
| QR開放中 | スタッフがQRを開放し、お客様の読込待ち | QR取消、QR再発行 |
| 着席 | セッションはあるが、注文がまだ少ない状態 | 注文、QR表示、メモ、人数、席移動 |
| 未注文 | 着席後まだ注文が入っていない | 注文確認、声かけ |
| 食事中 | 注文があり、通常利用中 | 注文、会計呼び出し、QR表示 |
| 会計待ち | お客様が会計へ進む状態 | 会計解除、レジ案内 |
| 時間超過 | 時間制限プランの残り時間を超過 | 声かけ、延長確認 |
| 清掃待ち | 会計後、次の案内前に片付けが必要 | 清掃完了 |

### 6-3-4. 運用

運用タブは、ホールが見るべき **今やること** を集約します。上から順に処理すれば、重要な未対応が残りにくい構成です。

| 表示 | 説明 |
|---|---|
| サマリーカード | 未送信、QR開放中、料理完成、呼び出し、会計待ち、清掃待ち、時間超過、未注文の件数 |
| KDS通知 | 客席呼び出し、厨房呼び出し、料理完成 |
| 次にやること | 放置・未対応の卓を優先表示 |
| 端末ログ | どの端末で配膳完了、呼び出し対応済み、清掃完了などを行ったか |
| 卓クイック操作 | 各卓の現在状態と操作ボタン |

### 6-3-5. 予約

当日の予約を確認し、予約客のQR開放や即着席を行います。

| 操作 | 説明 |
|---|---|
| 空席確認 | 日付、時刻、人数から予約可能か確認 |
| 電話予約追加 | 名前、電話番号、人数、時刻、テーブル、メモを入力 |
| QR開放 | 予約客が来店した卓をQR開放 |
| 即着席 | 予約客をスタッフ操作で着席済みにする |
| no-show | 無断キャンセルとして記録 |

### 6-3-6. 注文履歴

本日の注文を確認し、必要に応じて修正します。

| 操作 | 説明 |
|---|---|
| 更新 | 最新の注文履歴を取得 |
| 修正 | 注文内容をカートへ戻して更新 |
| 注文取消 | 注文全体を取消 |
| 品目取消 | 1商品だけ取消 |
| 数量変更 | 品目単位で数量を増減 |
| 配膳取消 | 間違って配膳完了した品目を提供待ちへ戻す |

## 6-4. 標準フロー1: 来店後、お客様がセルフ注文する

### 6-4-1. 流れ

1. お客様が来店
2. スタッフが空席を確認
3. 卓へ案内
4. ハンディでQRを開放
5. お客様が卓のQRを読み込む
6. セルフメニューが開く
7. お客様が自分のスマホで注文
8. KDSに注文が表示
9. 調理完了後、運用タブに料理完成通知
10. ホールが配膳
11. お客様が会計へ進む
12. 会計完了後、セッションが終了し、同じURLでは注文できなくなる

### 6-4-2. QRを開放する手順

1. ハンディを開く
2. `テーブル状況` タブをタップ
3. 案内する卓を探す
4. 卓が `空き` であることを確認
5. `QR開放` をタップ
6. 確認ダイアログで内容を確認
7. 問題なければOK
8. 卓の状態が `QR開放中` になる
9. 運用タブにも `QR開放中` として表示される

QR開放中に見える情報:

| 表示 | 意味 |
|---|---|
| QR読込待ち | まだお客様がQRを読み込んでいない |
| 残り○分 | QR開放の有効期限までの残り時間 |
| 開放: 端末名 | どのデバイスアカウントでQRを開放したか |

### 6-4-3. お客様への案内文

スタッフは次のように案内します。

> こちらのQRコードをカメラで読み込んでください。メニューが開いたら、そのままご注文いただけます。

高齢のお客様やスマホ操作に不慣れなお客様には、次のように補足します。

> カメラを起動して、このQRに向けてください。画面上に出たリンクを押すとメニューが開きます。

### 6-4-4. QR開放を間違えた場合

1. `運用` または `テーブル状況` を開く
2. `QR開放中` の卓を探す
3. `QR取消` をタップ
4. 確認してOK
5. その卓は空き状態に戻る

取消後、すでにお客様がそのQRを読み込んでいなければ、注文画面には進めません。

### 6-4-5. QRを再発行する場合

| 状況 | 操作 |
|---|---|
| 開放したが時間切れになりそう | QR再発行 |
| 間違って古いQRを見せてしまった | QR再発行 |
| お客様のスマホで読み込みがうまくいかない | QR再発行 |

`QR再発行` は、古い開放を取り消して新しい開放を作り直します。運用タブの残り時間も新しくなります。

### 6-4-6. 会計後に同じURLを開いた場合

会計が完了すると、対象テーブルのセッションは終了します。

終了後は、前のお客様が持っていたURLやブラウザ履歴から同じページを開いても、注文・履歴・会計情報へ進めないように制御されます。次のお客様には、スタッフがあらためてQRを開放してから使ってもらいます。

::: warning 会計後に必ず確認すること
会計が終わったら、レジ側の会計完了だけでなく、テーブル状態が `清掃待ち` または終了済みになっていることを確認してください。前のお客様のセッションが残っていると、次のお客様の注文と混ざる原因になります。
:::

## 6-5. 標準フロー2: スタッフがハンディで注文を取る

### 6-5-1. 注文入力の手順

1. `店内注文` タブを開く
2. `テーブル` から対象卓を選ぶ
3. 選択卓パネルで卓番号と状態を確認
4. カテゴリを選ぶ
5. 商品をタップ
6. オプションがある商品は、オプションを選んで `カートに追加`
7. 数量はカート内の `+` / `-` で調整
8. 必要ならメモ欄に「ネギ抜き」「先出し」などを入力
9. `注文送信` をタップ
10. 成功メッセージを確認
11. KDSに注文が出たことを確認

### 6-5-2. テーブル選択時に確認すること

| 確認項目 | 理由 |
|---|---|
| 卓番号 | 別卓に注文を入れないため |
| 状態 | 会計待ち、清掃待ち、QR開放中などでは通常注文しないため |
| 経過時間 | 時間制限プランや長時間滞在の確認 |
| 注文数・合計金額 | すでに注文済みか確認 |
| メモ | アレルギー、VIP、子連れ、記念日などの注意を確認 |
| 会計待ち表示 | 追加注文を受ける前に会計解除が必要な場合がある |

### 6-5-3. よく出る・この卓の直近

| 表示 | 使い方 |
|---|---|
| よく出る | 店舗で最近よく出ている商品をワンタップ追加 |
| この卓の直近 | 同じ卓の直近注文を再度カートに追加 |
| 直近再注文 | テーブル未選択時やテイクアウトで直近注文を再利用 |

再注文後も、必ずカート内の商品・数量を確認してください。品切れ商品は追加できません。

### 6-5-4. 注文メモ

| 入力例 | 用途 |
|---|---|
| ネギ抜き | 調理指示 |
| 先にドリンク | 提供順 |
| アレルギー確認済み | 安全確認 |
| 子ども用皿希望 | サービス |
| 誕生日 | 接客メモ |

メモは短く、厨房がすぐ読める内容にしてください。長文の接客メモはテーブルメモへ入れます。

### 6-5-5. 会計待ち中の追加注文

テーブルが `会計待ち` の場合、追加注文は通常受けません。POSLAでは、会計待ちの卓に対して **セルフ追加注文は停止中** と表示されます。

間違って会計待ちにした場合:

1. `運用` または `テーブル状況` を開く
2. 対象卓の `会計解除` をタップ
3. 確認してOK
4. 卓が `食事中` に戻る
5. 必要なら追加注文を入力

会計待ちのまま追加注文を受けると、レジ側の合計確認とお客様の認識がずれる可能性があります。追加注文を受ける場合は、先に会計解除してから入力します。

## 6-6. 標準フロー3: KDSから料理完成、配膳する

### 6-6-1. KDS側の流れ

1. 注文がKDSの `受付` に入る
2. 厨房スタッフが `調理開始`
3. 調理が終わったら `完成`
4. 完成した商品がハンディの運用タブへ `料理完成` として表示される

### 6-6-2. ホール側の配膳手順

1. ハンディの `運用` タブを開く
2. `料理完成` の件数を確認
3. KDS通知欄で卓番号と商品名を見る
4. 商品を取りに行く
5. 卓へ配膳
6. ハンディで `配膳完了` をタップ
7. 端末ログに、どの端末で配膳完了したか残る

### 6-6-3. 1タップ運用の考え方

ハンディはデバイスアカウントでログインする前提です。人の担当者を都度選ばせると、現場の操作が増えてしまいます。そのため、通常業務では `対応開始` を挟まず、料理を運んだ後に `配膳完了` を1回押して完了します。

| 操作 | 通常の扱い |
|---|---|
| 料理完成 | 配膳後に `配膳完了` |
| 客席呼び出し | 用件対応後に `対応済み` |
| 厨房呼び出し | 用件確認後に `対応済み` |
| 清掃待ち | 清掃後に `清掃完了` |

誰が行ったかではなく、**未対応が残っていないか** を優先します。人の確認が必要な運用は、口頭や店舗ルールで補い、ハンディでは作業消込を最短にします。

### 6-6-4. 配膳完了を間違えた場合

方法1: 運用タブの端末ログから戻す

1. `運用` タブを開く
2. `端末ログ` を見る
3. 直近の `配膳完了` を探す
4. `配膳取消` をタップ
5. 対象商品が `提供待ち` に戻る

方法2: 注文履歴から戻す

1. `注文履歴` タブを開く
2. 対象注文を探す
3. 対象品目の `配膳取消` をタップ
4. 対象商品が `提供待ち` に戻る

## 6-7. 標準フロー4: お客様から呼び出し

### 6-7-1. 呼び出しの種類

| 種類 | 発生元 | 運用タブの表示 |
|---|---|---|
| 客席呼び出し | お客様のセルフメニュー | 客席呼出 |
| 厨房呼び出し | KDS | 厨房呼出 |
| 料理完成 | KDSの完成操作 | 料理完成 |

### 6-7-2. 呼び出し対応手順

1. `運用` タブを開く
2. `呼び出し` 件数を見る
3. KDS通知欄で卓番号と内容を見る
4. 卓へ向かう
5. 用件を確認
6. 対応が終わったら `対応済み` をタップ
7. 端末ログに、どの端末で対応済みにしたか残る

### 6-7-3. 放置時の強調

| 対象 | 強調される目安 |
|---|---|
| 客席呼び出し | 約3分 |
| 厨房呼び出し | 約3分 |
| 料理完成 | 約5分 |
| 会計待ち | 約5分 |
| 清掃待ち | 約10分 |
| 時間超過 | 表示された時点で要確認 |

壁掛け端末で `音ON` にしている場合、未対応が残ると約3分ごとに再通知されます。

## 6-8. 運用タブの詳しい見方

### 6-8-1. サマリーカード

| カード | 意味 | タップ時 |
|---|---|---|
| 未送信 | 端末内に残っている未送信注文 | 未送信注文モーダルを開く |
| QR開放中 | QRを開放したが、まだ読まれていない卓 | 卓カードで取消/再発行 |
| 料理完成 | KDSで完成した料理 | KDS通知欄で配膳 |
| 呼び出し | 客席/厨房からの呼び出し | KDS通知欄で対応 |
| 会計待ち | 会計へ案内する卓 | 卓操作へ移動 |
| 清掃待ち | 会計後、片付け待ち | 清掃完了 |
| 時間超過 | 時間制限を超えた卓 | 声かけ |
| 未注文 | 着席後まだ注文がない卓 | 注文確認、声かけ |

### 6-8-2. 次にやること

| 表示 | 対応 |
|---|---|
| 未送信注文を再送してください | 未送信モーダルを開いて中身確認、再送または破棄 |
| QR開放後の読込待ち | お客様がQRを読めているか確認。間違いならQR取消 |
| 会計待ちの卓をレジへ案内 | レジへ案内。間違いなら会計解除 |
| 会計済みの卓を清掃 | 卓を片付け、清掃完了 |
| 時間超過の卓を確認 | 延長・ラストオーダー・退店案内を判断 |
| 着席後まだ注文がない卓 | 注文に困っていないか声かけ |

### 6-8-3. 端末ログ

| 操作 | ログの意味 |
|---|---|
| 配膳完了 | どの端末で料理を提供済みにしたか |
| 呼び出し対応済み | どの端末で呼び出しを完了したか |
| QR開放 | どの端末でQRを開放したか |
| QR取消 | どの端末でQR開放を取り消したか |
| 会計待ち | どの端末で会計待ちにしたか |
| 食事中へ戻し | どの端末で会計待ちを解除したか |
| 席移動 | どの卓からどの卓へ移動したか |
| 注文取消 | どの端末で注文を取り消したか |
| 品目取消 | どの端末で品目を取り消したか |
| 数量変更 | どの端末で数量を変更したか |
| 配膳取消 | どの端末で配膳完了を戻したか |

::: tip 監査の考え方
ハンディはデバイスアカウントで使うため、人の担当者までは判別しません。ハンディでは「どの端末で操作したか」と「未対応が残っていないか」を重視します。会計や現金移動など人の責任を明確にする必要がある操作は、レジ側のPIN入力で扱います。
:::

## 6-10. 席移動

### 6-10-1. 席移動が必要な場面

| 場面 | 例 |
|---|---|
| 予約席を空けたい | 予約客が来るため、別卓へ移動 |
| 人数が増えた | 2名席から4名席へ移動 |
| 眺めや設備の都合 | お客様希望で席を変更 |
| 相席調整 | 大きな卓を空けたい |
| トラブル対応 | 空調、照明、騒音などで移動 |

### 6-10-2. 席移動の手順

1. `テーブル状況` または `運用` を開く
2. 移動元の卓を探す
3. `席移動` をタップ
4. 移動先候補から空席を選ぶ
5. `移動する` をタップ
6. 成功メッセージを確認
7. 移動元が空き、移動先にセッションが移ることを確認

### 6-10-3. 席移動で一緒に移るもの

| 対象 | 説明 |
|---|---|
| テーブルセッション | 着席状態、開始時刻、人数、プラン、コース |
| 未会計注文 | 会計前の注文 |
| 注文履歴のテーブル表示 | 注文が移動先卓として見える |
| QR関連 | 現在のセッションQR、次回開放中の情報 |
| 卓メモ | アレルギー、接客メモなど |
| 呼び出し | 客席呼び出し、料理完成通知など |
| サブセッション | 相席/個別QRのグループ |

### 6-10-4. 席移動できない場合

| 理由 | 対処 |
|---|---|
| 移動先が使用中 | 別の空席を選ぶ |
| 移動先がQR開放中 | QR取消してから移動、または別卓を選ぶ |
| 対象セッションがない | 最新状態へ更新 |
| 通信エラー | Wi-Fi確認後に再操作 |

## 6-11. 注文取消・数量変更

### 6-11-1. 注文全体を取り消す

1. `注文履歴` を開く
2. 対象注文を探す
3. `注文取消` をタップ
4. 取消理由を入力
5. OK
6. 注文ステータスが取消になる
7. 端末ログに残る

| 理由 | 使う場面 |
|---|---|
| 卓間違い | 別卓に入れてしまった |
| お客様キャンセル | 注文直後に不要になった |
| 重複送信 | 同じ注文を二重に入れた |
| 品切れ | 注文後に在庫切れが分かった |
| 厨房判断 | 調理不可、提供不可 |

### 6-11-2. 品目単位で取り消す

1. `注文履歴` を開く
2. 対象注文を開く
3. 対象商品の `品目取消` をタップ
4. 理由を入力
5. OK
6. その品目だけ取消になる
7. 注文全体は残る

### 6-11-3. 数量を変更する

1. `注文履歴` を開く
2. 対象注文を探す
3. 対象商品の `数量+` または `数量-` をタップ
4. 調理中以降の場合は理由を入力
5. OK
6. 数量が変更される

::: warning KDS送信後の変更
KDS送信後に取消・数量変更を行う場合は、厨房に口頭でも伝えてください。システム上は記録されますが、すでに調理に入っている商品は厨房の手元作業とズレる可能性があります。
:::

### 6-11-4. 取消・変更で残る情報

| 情報 | 用途 |
|---|---|
| どの端末で操作したか | 端末単位の確認、問い合わせ対応 |
| いつ操作したか | 問い合わせ、会計差異確認 |
| 何を変更したか | 注文内容の追跡 |
| 理由 | 厨房・会計・店長確認 |

## 6-12. 未送信注文の復旧

### 6-12-1. 未送信注文とは

未送信注文は、ハンディで注文送信しようとしたが、通信障害や一時的なAPIエラーで送れなかった注文です。POSLAは注文内容を端末内に一時保存し、後で再送できるようにします。

未送信があると、ヘッダーに `未送信 1` のように件数が表示されます。運用タブのサマリーにも表示されます。

### 6-12-2. 未送信注文の確認

1. ヘッダーの `未送信` をタップ
2. 未送信注文一覧が開く
3. 卓番号またはテイクアウト名を確認
4. 商品・数量・合計金額を確認
5. 失敗理由と再送回数を確認

### 6-12-3. 再送する

1. 未送信注文の中身を確認
2. まだ必要な注文なら `再送信` をタップ
3. ボタンが `送信中` になる
4. 成功すると一覧から消える
5. KDSに注文が出たことを確認

### 6-12-4. 破棄する

1. お客様がキャンセルした、または別端末で入れ直したことを確認
2. `破棄` をタップ
3. 確認してOK
4. 未送信一覧から消える

::: warning 二重送信に注意
未送信を再送する前に、KDSや注文履歴に同じ注文がすでに入っていないか確認してください。POSLAは二重送信防止用のキーを持っていますが、スタッフが別端末で同じ注文を手入力した場合は別注文として扱われます。
:::

## 6-13. 壁掛け端末として使う

### 6-13-1. 設置場所

| 場所 | 理由 |
|---|---|
| ホール中央の壁 | 全員が見やすい |
| レジ横 | 会計待ち、清掃待ちに気づきやすい |
| 厨房入口付近 | 料理完成と配膳の連携に使いやすい |

避ける場所:

| 場所 | 理由 |
|---|---|
| 客席から丸見えの位置 | 内部メモや端末ログが見える可能性 |
| 水がかかる場所 | 端末故障の原因 |
| 強い逆光 | 視認性が下がる |
| Wi-Fiが弱い場所 | 更新遅延が起きる |

### 6-13-2. 開店時の設定

1. ハンディを開く
2. ヘッダーの端末モードで `壁掛け` を選ぶ
3. `運用` タブを開く
4. 必要なら `音OFF` をタップして `音ON` にする
5. 画面が見やすい角度に固定する
6. 端末の省電力設定で画面スリープ時間を長めにする

### 6-13-3. 壁掛けで優先して見るもの

| 優先度 | 表示 | 理由 |
|---|---|---|
| 1 | 呼び出し | お客様を待たせない |
| 2 | 料理完成 | 提供遅れを防ぐ |
| 3 | 会計待ち | 会計滞留を防ぐ |
| 4 | 清掃待ち | 次の案内を早くする |
| 5 | 時間超過 | 声かけ漏れを防ぐ |
| 6 | 未注文 | 注文に困っている可能性 |
| 7 | QR開放中 | 読込待ち・誤開放を確認 |

## 6-14. レジ横端末として使う

| 場面 | 使い方 |
|---|---|
| 会計待ち確認 | レジ担当がどの卓を会計するか見る |
| 清掃待ち確認 | 会計後、ホールへ片付けを依頼 |
| 未送信確認 | レジ横で通信状況を確認 |
| 呼び出し補助 | レジ担当がホールへ声かけ |

レジ横モードでは、壁掛けと同じく細かい卓操作は減らし、会計・清掃に必要な情報を見やすくします。

## 6-15. トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| QRを開放したのにお客様が注文画面に進めない | QRの有効期限切れ、別卓QR、通信不安定 | 運用タブでQR開放中か確認し、必要ならQR再発行 |
| 間違った卓をQR開放した | 卓選択ミス | すぐにQR取消 |
| 会計待ちにした後に追加注文したい | 会計待ち中は追加注文停止 | 会計解除して食事中へ戻す |
| 料理完成が放置される | ホールが気づいていない | 壁掛け端末を運用タブ、音ONで使う |
| 通知音が鳴らない | 音OFF、ブラウザの自動再生制限、端末音量ゼロ | 音ONを押す、端末音量、マナーモードを確認 |
| 未送信が残る | 通信障害、APIエラー | 中身を確認し、KDSに重複がなければ再送 |
| 注文が二重に入った | 再送前に別端末で入れ直した | 注文履歴で片方を取消し、理由を残す |
| 席移動できない | 移動先が使用中、QR開放中 | 空席を選ぶ。必要ならQR取消 |
| 配膳完了を押し間違えた | 操作ミス | 運用ログまたは注文履歴から配膳取消 |
| 前のお客様のURLで画面が見える | セッション終了漏れの可能性 | 会計完了と清掃/終了状態を確認。次回QRは必ず再開放 |

## 6-16. 現場運用ルールの推奨

### 6-16-1. 通常操作は増やさない

| 操作 | 理由 |
|---|---|
| QR開放 | 来店時に毎回必要 |
| 注文入力 | 通常業務 |
| 呼び出し対応済み | 回転を止めない |
| 配膳完了 | 通常業務 |
| 会計待ち | 通常業務 |
| 清掃完了 | 通常業務 |

### 6-16-2. 理由を残す操作

| 操作 | 理由が必要な理由 |
|---|---|
| 注文取消 | 売上・厨房・会計に影響する |
| 品目取消 | 商品単位で会計額が変わる |
| 数量変更 | 会計額と厨房作業に影響する |
| 配膳取消 | KDSとホールの状態を戻す |

### 6-16-3. 店長に確認する操作

| 操作 | 理由 |
|---|---|
| 調理後の取消 | 原価・廃棄・クレーム対応に影響 |
| 会計後の修正 | 売上とレジ締めに影響 |
| 大幅な数量変更 | 不正・入力ミスの可能性 |
| 席移動後の会計差異 | 注文の紐付け確認が必要 |

## 6-17. システムが内部で行うこと

| 処理 | 内容 |
|---|---|
| 店舗境界 | ログインユーザーがアクセスできる店舗だけ操作可能 |
| QR開放端末 | どのデバイスアカウントでQRを開放したか保存 |
| QR残り時間 | 有効期限までの秒数を表示 |
| 会計待ち解除 | 誤って会計待ちにした卓を食事中へ戻す |
| 席移動 | セッション、注文、QR、メモ、呼び出し、サブセッションを移動 |
| 取消ログ | 注文取消、品目取消、数量変更、理由、操作端末を保存 |
| 配膳ログ | 配膳完了、配膳取消、操作端末を保存 |
| 端末設定 | 端末ごとの表示モード、通知音、再通知間隔を保存 |
| 未送信復旧 | 通信失敗時、端末内に注文を保持 |
| 二重送信抑止 | 注文送信時に再送用キーを付与 |
| 会計後セッション終了 | 会計済み/終了済みセッションのURL再利用を防止 |
