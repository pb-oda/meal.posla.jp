# 4. メニュー管理

メニュー管理では、カテゴリ、商品、価格、表示順、売り切れ、オプション、多言語表示を設定します。お客様のセルフオーダー画面、ハンディPOS、KDSに反映される重要な設定です。

## 基本の流れ

1. カテゴリを作成します。
2. 商品を登録します。
3. 価格、税区分、説明文、画像、表示順を設定します。
4. セルフメニュー用の属性を設定します。
5. 必要に応じてオプションを設定します。
6. 品切れ時の代替提案を設定します。
7. メニュー整備チェックで不足項目を確認します。
8. セルフオーダー画面で表示を確認します。
9. テスト注文を行い、KDSと会計まで確認します。

## カテゴリ管理

カテゴリは「定食」「一品料理」「ドリンク」など、お客様が商品を探しやすい単位で作成してください。表示順を整えると、セルフオーダー画面でも見やすくなります。

## 商品登録

商品名、価格、説明文、画像、販売状態を登録します。写真や説明文は注文率に影響するため、実際の提供内容とずれないようにしてください。

## セルフメニュー用の属性

お客様画面の検索・絞り込みでは、商品名だけでなく店舗が登録した属性を使います。以下を登録しておくと、お客様が迷わず選びやすくなります。

- **辛さ**: なし、控えめ、辛い、激辛から選びます。
- **早く出る**: ドリンク、前菜、提供が早い品目に付けます。
- **ベジ**: 店舗としてベジ対応と案内できる品目に付けます。
- **子ども向け**: 子どもにすすめやすい品目に付けます。
- **提供目安分**: 店舗が明示したい場合だけ入力します。未入力の場合は、過去の提供実績が十分にある品目だけ自動の目安時間を表示します。

属性が未設定の品目でも、セルフメニュー側は品名・説明文・過去実績から補助判定します。ただし、誤判定を避けたい品目は店舗側で明示設定してください。

## 売り切れ設定

一時的に販売できない商品は、売り切れに設定します。売り切れにすると、お客様画面で注文できない状態になります。営業再開時には解除を忘れないようにしてください。

## オプション管理

トッピング、サイズ、辛さ、麺量などをオプションとして設定できます。追加料金がある場合は、会計金額に反映されることをテスト注文で確認してください。

セルフオーダー画面では、オプション選択時に以下を明確に表示します。

- **必須/任意**: 必須オプションは未選択のままカートに入れられません。
- **選択上限**: 複数選択オプションは、選択済み数と上限を表示します。上限に達すると未選択の候補は選べなくなります。
- **価格差**: 追加料金、値引き、価格差なしを選択肢ごとに表示します。
- **売り切れオプション**: 店舗別オプション上書きで提供不可にした選択肢は、画面に残したまま「売り切れ」と表示し、選択できない状態にします。

必須オプションの全候補が提供不可になっている場合、その商品はお客様がカートに入れられません。営業中にこの状態が起きた場合は、対象商品の売り切れ設定、またはオプション候補の提供可否を見直してください。

## 品切れ時の代替提案

店舗メニューの「品切れ時の代替提案」では、「この商品が売り切れたら、この候補を表示する」という手動ルールを設定できます。

例:

- 生ビールが売り切れ → 瓶ビール、ハイボール、レモンサワー
- 人気定食が売り切れ → 同じ主菜の単品、近い価格帯の定食
- デザートAが売り切れ → デザートB、ドリンクセット

この設定は、セルフオーダー画面でカート内の商品が途中で品切れになった場合に優先されます。手動設定がない場合は、同じカテゴリ、今日のおすすめ、人気、早く出る品目から自動候補を出します。

売り切れ商品そのものは、お客様のメニュー一覧には表示しません。お客様には「売り切れ一覧」を見せるのではなく、今注文できる候補だけを見せる設計です。

## メニュー整備チェック

店舗メニューの「メニュー整備チェック」では、セルフメニューの品質に影響する不足項目を一覧で確認できます。

チェック対象:

- **写真なし**: 商品画像が未設定です。
- **説明なし**: 商品説明が未設定です。多言語翻訳がある場合でも、元の説明を整えておくと翻訳品質が安定します。
- **アレルゲン未設定**: アレルゲン情報が未設定、または未確認です。該当なしの場合も、店舗内で確認してから運用してください。
- **タグ未設定**: 辛さ、早く出る、ベジ、子ども向け、提供目安分など、セルフメニューの絞り込みに使う属性が未設定です。
- **価格未設定**: 価格が0円以下です。食べ放題/コース専用ではない通常メニューで出ている場合は確認してください。

この画面は、店舗スタッフが日々修正するためのものというより、メニュー公開前、季節メニュー追加後、CSV取込後の確認用です。全項目を必ずゼロにするという意味ではなく、「お客様が迷う・呼び出す・注文をやめる原因」を事前に見つけるために使います。

## CSV入出力

メニュー点数が多い場合は、CSVで一括登録・更新できます。取り込み前に、文字化け、価格、カテゴリ名、販売状態を確認してください。

POSLA のメニューCSVは、対象によって役割が分かれています。

| 画面 | CSVの役割 | 主な用途 |
|---|---|---|
| オーナー画面の本部メニュー | 本部マスタを登録・更新 | 複数店舗で共通の商品名、カテゴリ、基本価格を管理 |
| 管理画面の店舗メニュー | 本部マスタに対する店舗別上書き | 価格、非表示、品切れ、画像URLを店舗ごとに変更 |
| 管理画面の限定メニュー | その店舗だけの商品を登録・更新 | 日替わり、季節商品、店舗限定品 |
| プラン・コース | 食べ放題プラン、プラン専用メニュー、コースを登録・更新 | 大量登録、季節変更、既存表からの移行 |
| 在庫・レシピ | 原材料、レシピを登録・更新 | 原材料数が多い店舗、レシピ初期登録 |

### 店舗メニューCSV

管理画面の **メニュー管理 → 店舗メニュー** には、CSV一括更新パネルがあります。最初にCSVをエクスポートし、編集したCSVをインポートしてください。

店舗メニューCSVは、本部マスタの商品に対して店舗別の上書きを行います。商品そのものを新規作成するCSVではありません。

使用する列:

```csv
template_id,name,category_name,base_price,price,is_hidden,is_sold_out,image_url
```

| 列 | 意味 | 注意 |
|---|---|---|
| `template_id` | 本部マスタ商品のID | 変更しない |
| `name` | 商品名 | 確認用。商品名変更は本部メニュー側で行う |
| `category_name` | カテゴリ名 | 確認用 |
| `base_price` | 本部マスタの基本価格 | 確認用 |
| `price` | 店舗別価格 | 店舗だけ価格を変える場合に入力 |
| `is_hidden` | 非表示 | `1` でお客様画面に出さない |
| `is_sold_out` | 売り切れ | `1` で注文不可 |
| `image_url` | 店舗別画像URL | 店舗独自写真へ差し替える場合に入力 |

### 限定メニューCSV

日替わりや季節商品など、本部マスタにない商品は **限定メニュー** タブで登録します。

使用する列:

```csv
id,category_name,name,name_en,description,price,image_url,is_sold_out
```

| 列 | 意味 | 注意 |
|---|---|---|
| `id` | 既存限定メニューID | 新規作成時は空欄でもよい |
| `category_name` | 所属カテゴリ | 既存カテゴリ名と一致させる |
| `name` | 商品名 | お客様画面、ハンディ、KDSに表示 |
| `name_en` | 英語名 | 未入力でも運用可能 |
| `description` | 説明文 | 多言語翻訳の元にもなる |
| `price` | 価格 | 数字で入力 |
| `image_url` | 商品画像URL | 画像未設定でも登録可能 |
| `is_sold_out` | 売り切れ | `1` で注文不可 |

CSVを取り込んだ後は、必ず次を確認してください。

- 店舗メニュー一覧で価格、非表示、品切れが意図通りか
- 限定メニューが正しいカテゴリに入っているか
- セルフオーダー画面で表示されるか
- ハンディ、KDS、レジで注文から会計まで通るか
- メニュー整備チェックで写真なし、説明なし、アレルゲン未設定が増えていないか

::: warning CSVは「表の置き換え」ではなく「対象機能の一括更新」です
店舗メニューCSVは本部マスタに対する店舗別上書き、限定メニューCSVは店舗限定品の登録です。CSVの種類を間違えると、思った画面に反映されません。まず対象画面からエクスポートし、その列構成を保ったまま編集してください。
:::

## 多言語表示

多言語設定を利用すると、外国語メニューを表示できます。自動翻訳後は、店舗で実際の表現を確認し、不自然な訳があれば調整してください。

メニュー名、カテゴリ名、説明文、オプション名はメニュー翻訳データで表示します。セルフメニューのボタン、検索、絞り込み、代替提案などの画面文言は `ui_translations` で管理します。2026-04-29時点のセルフメニュー画面では、日本語、英語、簡体中文、韓国語を選択できます。

## 困ったとき

| 症状 | 対処 |
|---|---|
| 商品が表示されない | カテゴリ、販売状態、店舗表示設定を確認してください。 |
| 価格が違う | 商品価格とオプション価格を確認してください。 |
| 売り切れが戻らない | 売り切れ設定を解除し、画面を再読み込みしてください。 |
| 品切れ時の候補が意図と違う | 「品切れ時の代替提案」で手動候補を設定してください。未設定時は自動候補になります。 |
| 必須オプションが選べない | オプション候補が全て提供不可になっていないか確認してください。 |
| CSVが取り込めない | 文字コード、必須項目、価格の形式を確認してください。 |
