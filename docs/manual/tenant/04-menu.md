# 4. メニュー管理

メニュー管理では、お客さんに表示するメニューの作成・編集・品切れ管理を行います。POSLAのメニューは**3層構造** (本部マスタ → 店舗オーバーライド → 店舗限定品) になっており、本部で統一メニューを管理しつつ、店舗ごとに価格や表示をカスタマイズできます。

---

::: tip 章の読み方ガイド
- **単店舗**: 4.1 カテゴリ + 4.2 メニューテンプレート + 4.4 限定メニュー が中心
- **多店舗 / チェーン**: 4.2 本部マスタ + 4.3 店舗オーバーライド の両軸を使う
- **一括入力**: 4.10 CSV インポート / 4.11 スマレジ連携
- **多言語対応**: 4.9 AI 翻訳機能（Gemini）
- **困ったとき**: 4.Y トラブルシューティング → 4.X FAQ
:::

### 4.0.1 操作前のチェックリスト

| チェック項目 | 確認方法 |
|---|---|
| ログインしているロールが `owner` または `manager` であること | 画面右上のユーザーバッジに `owner` / `manager` 表示があるか |
| 操作対象店舗を正しく選択していること（店舗単位の画面のみ） | ヘッダーの店舗セレクターで対象店舗を選択 |
| 本部一括配信アドオンの契約有無 | owner-dashboard に「本部メニュー」タブが出るか |
| 編集対象が本部マスタか店舗かを区別していること | 本章 4.0.2 の表で確認 |
| カテゴリが先に作成済みであること | カテゴリ未作成だとメニュー追加時にカテゴリ選択ができない |

::: warning 権限について
- `staff` ロールはメニュー編集ができません（読取は可能）。意図的な制約です（誤操作・退職スタッフによる価格改竄を防ぐため）
- `device` ロール（KDS / レジ端末）はメニュー編集 UI に到達できません
- 本部マスタ（`menu_templates`）の編集は `owner` 専用（hq_menu_broadcast=1 のテナント）
:::

### 4.0.2 POSLA メニュー 3 層構造

```
┌─────────────────────────────────────────────────────────────┐
│ menu_templates        (本部マスタ)                           │
│   tenant 単位、全店共通                                      │
│   name / name_en / category / base_price / image_url         │
│   description / calories / allergens                         │
└──────────┬──────────────────────────────────────────────────┘
           │ 店舗ごとに上書き / 非表示 / 品切れ
           ▼
┌─────────────────────────────────────────────────────────────┐
│ store_menu_overrides  (店舗オーバーライド)                   │
│   store 単位                                                 │
│   price=NULL → 本部 base_price 継承                          │
│   is_hidden=1 → この店舗で非表示                             │
│   is_sold_out=1 → この店舗で品切れ                           │
│   override_image_url → 店舗固有画像                          │
└──────────┬──────────────────────────────────────────────────┘
           │ お客さん画面では統合表示
           ▼
┌─────────────────────────────────────────────────────────────┐
│ store_local_items     (店舗限定品)                           │
│   store 単位、本部マスタに紐づかない独自メニュー             │
│   その店舗でしか出ない                                       │
└─────────────────────────────────────────────────────────────┘
                          ↓
            api/lib/menu-resolver.php の resolve_store_menu()
            が 3 層を統合してセルフオーダー / KDS に返す
```

### 4.0.3 本部一括配信アドオン

`tenants.hq_menu_broadcast = 1`（基本料金 + アドオン ¥3,000/店舗 契約）の tenant では、編集権限が階層化されます。

| 編集対象 | アドオン契約あり | アドオン契約なし |
|---|---|---|
| 本部マスタ（`menu_templates`） | **owner-dashboard** からのみ（owner 専用） | dashboard「店舗メニュー」から事実上編集可 |
| 店舗オーバーライド（価格・非表示・品切れ） | 店舗 dashboard から（manager 以上） | 同左 |
| 店舗限定品（`store_local_items`） | 店舗 dashboard「限定メニュー」から | 同左 |
| 本部マスタの readonly フィールド（店舗側） | name / name_en / category_id / calories / allergens / base_price / image_url が**ロック** | ロックなし |

実装フラグ: `MenuOverrideEditor.init(container, features)` の `features.hq_menu_broadcast` で `_isHqLocked` が決まります（P1-29）。

管理ダッシュボードの**「メニュー管理」タブ**をクリックし、以下のサブタブを切り替えて操作します：

- **カテゴリ管理**（カテゴリ + テンプレートメニュー一覧）
- **店舗メニュー**（店舗オーバーライド + おすすめ）
- **限定メニュー**（店舗独自品）
- **オプション管理**（オプショングループ + 選択肢）

## 4.1 カテゴリ管理

カテゴリは、メニューをグループ分けする単位です。セルフオーダー画面やハンディPOS画面でカテゴリごとにメニューが表示されます。

**サブタブ：「カテゴリ管理」**

### 4.1.1 画面の見方

```
[カテゴリ管理] サブタブ
┌──────────────────────────────────────────────────────┐
│  カテゴリ (3)                       [+ カテゴリ追加] │
├──────────────────────────────────────────────────────┤
│  1  前菜          (Appetizers)         [編集] [削除] │
│  2  メイン        (Main)               [編集] [削除] │
│  3  デザート      (Dessert)            [編集] [削除] │
└──────────────────────────────────────────────────────┘
```

カテゴリは表示順に並び、横に通し番号が表示されます。

### 4.1.2 カテゴリの作成

1. 「カテゴリ管理」サブタブを開く
2. **「カテゴリ追加」ボタン**を押す
3. モーダルが開く（タイトル：「カテゴリを追加」）
4. 以下を入力する：

| 項目 | 必須 | 説明 | 入力例 |
|------|------|------|--------|
| カテゴリ名 | ✓ | 日本語の表示名 | 前菜 |
| カテゴリ名（英語） | — | 英語の表示名。多言語対応時に使用 | Appetizers |

5. **保存**ボタンを押す
6. 「カテゴリを追加しました」トーストが表示され、一覧に追加される

### 4.1.3 カテゴリの編集

1. カテゴリ一覧で、編集したいカテゴリの**編集ボタン**を押す
2. モーダルが開き、現在の値が表示される
3. 以下を変更できる：

| 項目 | 説明 |
|---|---|
| カテゴリ名 | 日本語名 |
| カテゴリ名（英語） | 英語名 |
| 表示順 | 数値。小さいほど上に表示 |

4. **保存**ボタンを押す

### 4.1.4 カテゴリの削除

1. 削除したいカテゴリの**削除ボタン**を押す
2. 確認ダイアログが表示される
3. **OK**を押すと削除される

::: warning 注意
カテゴリを削除すると、そのカテゴリに属するメニューは「カテゴリなし」の状態になります。削除前に、メニューを別のカテゴリに移動させることを推奨します。
:::

### 4.1.5 並び順のルール

- カテゴリの表示順序は「表示順」の数値（`sort_order`）で決まります
- 数値が小さいカテゴリが上（左）に表示されます
- セルフオーダー画面・ハンディPOS画面でも同じ順序で表示されます
- **実例（matsunoya）**: 1=朝定食、2=昼定食、3=夜定食、10=サイドメニュー、99=その他、のように 10 飛ばし運用にすると、後から間に追加しやすい

## 4.2 メニューテンプレート（本部マスタ）

メニューテンプレートは、全店舗で共通のマスターメニューです。ここで作成したメニューが全店舗の基本メニューになります。店舗ごとの価格変更や非表示設定は「店舗メニューオーバーライド」で行います。

**サブタブ：「カテゴリ管理」**（カテゴリ一覧の下にメニュー一覧が並ぶ）

::: tip アドオン契約時の制約
- アドオン契約 = `hq_menu_broadcast=1` の tenant では、このタブからのテンプレート追加・編集は **owner-dashboard** に集約されます（P1-28）
- 店舗 dashboard 側はマスタ項目が readonly（P1-29）。店舗側で本部マスタを編集する必要があるなら、owner にエスカレーションしてください
:::

### 4.2.1 メニューの作成

1. **「メニュー追加」ボタン**を押す
2. モーダルが開く
3. 以下を入力する：

| 項目 | 必須 | 説明 | 入力例 |
|------|------|------|--------|
| メニュー名 | ✓ | 日本語の表示名 | 特製唐揚げ |
| メニュー名（英語） | — | 英語の表示名 | Special Karaage |
| カテゴリ | ✓ | ドロップダウンからカテゴリを選択 | メイン料理 |
| 価格（税込） | ✓ | 税込価格（円）。0円も可（後述） | 880 |
| 説明文 | — | メニューの詳細説明 | 自家製タレで味付けした鶏もも肉... |
| 説明文（英語） | — | 英語の説明文 | Chicken thigh marinated in... |
| カロリー | — | kcal 単位の数値 | 450 |
| アレルゲン | — | 25 項目のチェックボックスから選択 | 卵 / 小麦 / 大豆 |
| 画像 | — | メニュー写真。ドラッグ＆ドロップまたはクリックでアップロード | — |

4. **保存**ボタンを押す
5. メニュー一覧に追加される

::: tip 0円メニューの使い所
コース料理のフェーズ内料理（前菜→メイン→デザート）はコース定義側で価格を持つため、テンプレート側は 0 円で作成して「コース専用」として運用することがあります。
:::

::: tip 画像について
- 対応形式：JPEG、PNG、WebP、GIF
- 最大ファイルサイズ：5MB
- アップロード時に自動的に 800×800 ピクセル以下にリサイズされます（品質 0.8 の JPEG）
- 画像なしでも作成できます。画像なしの場合は🍽️アイコンが表示されます
- 画像 URL は HTTPS で配信される必要があります。HTTP は混在コンテンツとしてブラウザにブロックされます
:::

### 4.2.2 アレルゲン 25 項目（実装一覧）

`MenuOverrideEditor._allergenList` / `LocalItemEditor._allergenList` に定義されている公式 25 項目：

```
卵, 乳, 小麦, えび, かに, そば, 落花生, アーモンド, カシューナッツ,
くるみ, ごま, さけ, さば, いか, あわび, いくら, 大豆, まつたけ,
もも, りんご, キウイ, バナナ, オレンジ, ゼラチン,
鶏肉, 牛肉, 豚肉
```

うち、消費者庁「特定原材料 8 品目」（卵 / 乳 / 小麦 / えび / かに / そば / 落花生 / くるみ）は法律上の表示義務があります。残りは「特定原材料に準ずる 20 品目」相当 + POSLA 独自追加（鶏肉・牛肉・豚肉）。

### 4.2.3 メニューの編集

1. メニュー一覧で、編集したいメニューの**編集ボタン**を押す
2. モーダルが開き、現在の値が表示される
3. 各項目を変更する
4. **品切れ**チェックボックス：チェックを入れると**全店舗で**品切れ表示になります（店舗側で解除不可）
5. **保存**ボタンを押す

### 4.2.4 メニューの削除

1. 削除したいメニューの**削除ボタン**を押す
2. 確認ダイアログが表示される：「このメニューを削除すると、全店舗のオーバーライド設定も削除されます」
3. **OK**を押すと削除される

::: danger カスケード削除に注意
メニューテンプレートを削除すると、そのメニューに対する全店舗のオーバーライド設定（価格変更、品切れ状態等）も一緒に削除されます。元に戻せません。

過去注文（`orders.items` JSON カラム）は名称・価格スナップショットを保持しているため、過去レポートには影響しません。
:::

### 4.2.5 品切れのトグル

メニュー一覧の各メニューに**品切れボタン**があります。

- **グレー**（通常）→ クリックすると**赤**（品切れ）に変わる
- **赤**（品切れ）→ クリックすると**グレー**（通常）に戻る
- 品切れにすると、セルフオーダー画面・ハンディPOS画面で即座に「品切れ」表示になり、お客さんは注文できなくなります
- メニューバージョンが自動更新され、お客さんの画面は 2 秒以内に品切れ状態が反映されます

### 4.2.6 カテゴリでフィルター

メニュー一覧の上部にカテゴリフィルターのドロップダウンがあります。カテゴリを選択すると、そのカテゴリのメニューだけが表示されます。「全て」を選択すると全メニューが表示されます。

## 4.3 店舗メニューオーバーライド

店舗メニューオーバーライドでは、本部テンプレートのメニューを店舗ごとにカスタマイズします。価格の変更、メニューの非表示、店舗固有の画像設定が可能です。

**サブタブ：「店舗メニュー」**

::: warning 店舗を選択してください
この画面は店舗単位で操作します。ヘッダーの店舗セレクターで、操作したい店舗を選択してから開いてください。未選択時は「店舗を選択してください」と表示されます。
:::

### 4.3.1 画面の見方（テーブル形式）

テーブル形式で一覧表示されます。各行に以下の情報があります：

```
┌──────┬─────────────────┬───────┬──────┬──────┬──────┬────┐
│ 画像 │ メニュー名      │カテゴリ│ 価格 │非表示│品切れ│操作│
├──────┼─────────────────┼───────┼──────┼──────┼──────┼────┤
│ 📷  │ 特製唐揚げ      │ メイン │¥880  │ —   │ —   │編集│
│ 📷  │ ステーキ重 [非表示]│ メイン │¥1480 │ ✓  │ —   │編集│
│ 📷  │ 季節のサラダ    │ 前菜   │¥620  │ —   │[品切]│編集│
└──────┴─────────────────┴───────┴──────┴──────┴──────┴────┘
```

| 列 | 説明 |
|----|------|
| 画像 | 店舗固有の画像（`override_image_url`）。設定がなければ本部テンプレートの画像（`image_url`） |
| メニュー名 | 名前（日本語/英語） |
| カテゴリ | 所属カテゴリ名 |
| 価格 | 店舗価格（上書きされている場合はその価格、なければ本部価格） |
| 非表示 | この店舗でメニューを非表示にするかどうか。非表示の行は半透明（opacity:0.5）で表示 |
| 品切れ | この店舗での品切れ状態 |
| 操作 | 編集・削除ボタン |

「**本部品切れ**」バッジが表示されている場合、本部テンプレート側で品切れに設定されているため、店舗側で解除できません（4.2.5 でテンプレ側を解除する必要があります）。

### 4.3.2 メニューの価格を変更する

1. 変更したいメニューの**編集ボタン**を押す
2. モーダルが開く
3. **価格**欄に新しい価格を入力する（**空欄にすると本部価格に戻ります** = `price=NULL`）
4. **保存**ボタンを押す

::: tip 実例: matsunoya / torimaru
- matsunoya 渋谷店：本部 base_price ¥880 → 渋谷店 ¥980（家賃調整）
- matsunoya 横浜店：本部 base_price ¥880 → 価格上書きなし（本部価格を継承）
- torimaru：本部マスタを直接編集（hq_menu_broadcast=0 のため）
:::

### 4.3.3 メニューを非表示にする

この店舗では提供しないメニューを非表示にできます（例：ランチ専門店は夜定食を非表示）。

1. メニュー一覧で、非表示にしたいメニューの**非表示トグル**を押す
2. メニューが半透明表示になり、セルフオーダー画面に表示されなくなります
3. もう一度押すと表示に戻ります

### 4.3.4 品切れにする

1. メニュー一覧で、品切れにしたいメニューの**品切れトグル**を押す
2. 赤いバッジが表示され、セルフオーダー画面で品切れ表示になります
3. もう一度押すと品切れを解除できます

### 4.3.5 カロリー・アレルゲン情報を設定する

メニューの編集モーダルで、以下を設定できます：

| 項目 | 説明 | 入力方法 |
|------|------|---------|
| カロリー | kcal 単位の数値 | 数値入力（例：450） |
| アレルゲン | 含まれるアレルゲン | チェックボックスから選択（25 項目から複数選択可） |

設定したカロリーとアレルゲンはセルフオーダー画面のメニュー詳細に表示されます。

::: tip アレルゲン情報の運用
- 法律上の表示義務（特定原材料 8 品目）は調理工程で混入する可能性も含めて表示する必要があります
- 「コンタミ」（製造ライン共通による微量混入）が想定される場合は、説明文に「※製造工程上、卵を含む製品と共通の設備で製造しています」のように記載
- アレルゲン情報を更新したら必ず**多言語翻訳の再実行**を推奨（4.9 参照）
:::

### 4.3.6 オプショングループを紐付ける

メニューの編集モーダルの下部に、オプショングループの紐付け設定があります。

1. 編集モーダルを開く
2. 下部の「オプション」セクションを確認する
3. 紐付けたいオプショングループに**チェック**を入れる
4. 「必須」チェックボックス：チェックを入れると、お客さんは注文時にこのオプションを必ず選択する必要があります
5. **保存**を押す

### 4.3.7 店舗固有の画像を設定する

1. 編集モーダルを開く
2. **画像アップロード**エリアに画像をドラッグ＆ドロップ、またはクリックして選択
3. プレビューが表示される
4. **保存**を押す
5. この店舗では、本部テンプレートの画像の代わりにこの画像が表示されます

### 4.3.8 アドオン契約時の readonly 挙動

`hq_menu_broadcast=1` の tenant では、店舗 dashboard 「店舗メニュー」タブから編集モーダルを開いても以下の項目はグレーアウトされて編集できません（P1-29）：

- メニュー名（日本語 / 英語）
- カテゴリ
- カロリー
- アレルゲン
- 本部マスタ価格（base_price）
- 本部マスタ画像（image_url）

編集できるのは：

- 店舗価格（`price`、空欄で本部継承）
- 店舗画像（`override_image_url`）
- 非表示フラグ（`is_hidden`）
- 品切れフラグ（`is_sold_out`）
- 紐付けオプショングループ + 必須フラグ

これらマスタ項目を変更したい場合は owner-dashboard「本部メニュー」タブから操作してください。

## 4.4 店舗限定メニュー

店舗限定メニューは、特定の店舗でのみ提供するオリジナルメニューです。本部テンプレートには存在しない、その店舗だけのメニューを作成できます。

**サブタブ：「限定メニュー」**

::: warning 「限定メニュー」と「店舗メニュー」の違い
- 「店舗メニュー」 = 本部マスタ + 店舗オーバーライドの統合表示（既存品の調整）
- 「限定メニュー」 = その店舗だけの独自品（新規追加）

新規メニューを「店舗メニュー」タブから追加すると本部マスタに入ってしまいます（アドオン未契約時）。**店舗独自品は必ず「限定メニュー」タブから追加してください。**
:::

### 4.4.1 限定メニューの作成

1. **「メニュー追加」ボタン**を押す
2. モーダルが開く
3. 以下を入力する：

| 項目 | 必須 | 説明 |
|------|------|------|
| メニュー名 | ✓ | 日本語の表示名 |
| メニュー名（英語） | — | 英語の表示名 |
| カテゴリ | ✓ | ドロップダウンから選択 |
| 価格（税込） | ✓ | 税込価格（円） |
| 説明文 | — | メニューの詳細説明 |
| 画像 | — | メニュー写真 |
| カロリー | — | kcal（数値） |
| アレルゲン | — | 25 項目のチェックボックスから選択 |
| オプショングループ | — | 紐付けるオプション |

4. **保存**ボタンを押す
5. メニュー一覧に「**限定**」バッジ付きで追加される

限定メニューはセルフオーダー画面で通常のメニューと同様に表示されます（「限定」バッジは管理画面のみの表示で、お客さん側には表示されません）。

### 4.4.2 限定メニューの一覧表示

販売中 / 品切れトグルは灰色（販売中）/ 赤色（品切れ）で切り替わります。

### 4.4.3 品切れ・編集・削除

- **販売中 / 品切れ** ボタンで品切れ切替
- **編集** ボタンでモーダル再オープン
- **削除** ボタンで削除（確認ダイアログあり）

## 4.5 オプション・トッピング

オプショングループは、メニューに付加する選択肢（トッピング、サイズ、焼き加減など）を管理する機能です。

**サブタブ：「オプション管理」**（owner 専用）

### 4.5.1 オプショングループの作成

1. **「グループ追加」ボタン**を押す
2. モーダルが開く
3. 以下を入力する：

| 項目 | 必須 | 説明 | 入力例 |
|------|------|------|--------|
| グループ名 | ✓ | 日本語名 | ごはんの量 |
| グループ名（English） | — | 英語名 | Rice Size |
| 選択タイプ | ✓ | `single`（単一選択 / ラジオ）または `multi`（複数選択 / チェック） | single |
| 最小選択数 | — | 必須選択の最小数 | 1 |
| 最大選択数 | — | 選択できる最大数 | 1 |

4. **保存**ボタンを押す

### 4.5.2 選択タイプの違い

| タイプ | 内部値 | UI 表示 | 自動設定 | 例 |
|---|---|---|---|---|
| 単一選択 | `single` | ラジオボタン形式。1 つだけ選択 | 最小=1、最大=1 | サイズ（S / M / L） |

### 4.5.3 選択肢の追加

オプショングループを作成したら、そのグループに選択肢を追加します。

1. グループのカード内にある**「+ 選択肢」ボタン**を押す
2. モーダルが開く
3. 以下を入力する：

| 項目 | 必須 | 説明 | 入力例 |
|------|------|------|--------|
| 選択肢名 | ✓ | 日本語名 | Lサイズ |
| 選択肢名（英語） | — | 英語名 | Large |
| 価格差（円） | — | 追加料金。マイナスも可 | +200 / -50 |
| デフォルト | — | チェックすると初期選択される | — |

4. **保存**を押す

### 4.5.4 選択肢の表示例

```
┌─ ごはんの量 (単一選択) (3メニュー)              [+ 選択肢] [編集] [削除] ─┐
│  選択: 1〜1                                                                │
├──────────────────────────────────────────────────────────────────────────┤
│  小盛り (Small)         ±0      [既定]   [編集] [削除]                     │
│  並盛り (Regular)       ±0                [編集] [削除]                     │
│  大盛り (Large)         +100円           [編集] [削除]                     │
└──────────────────────────────────────────────────────────────────────────┘
```

### 4.5.5 グループの有効/無効

グループの編集モーダルに「有効」チェックボックスがあります。無効にすると、そのグループが紐付けられたメニューでもオプション選択が表示されなくなります。

::: tip 使いどころ
季節限定のトッピングを一時的に無効にしたい場合や、在庫切れのオプションを停止したい場合に使います。紐付け自体を削除する必要はありません。
:::

### 4.5.6 メニューへの紐付け

オプショングループは、テンプレートメニュー側または店舗オーバーライド側 / 限定メニュー側の編集モーダル下部から紐付けます（4.3.6 / 4.4.1 参照）。「必須」チェックを入れると注文時に必ず選択させる動作になります。

## 4.6 今日のおすすめ

「今日のおすすめ」は、日替わりで特定のメニューをセルフオーダー画面の上部に目立つように表示する機能です。

**サブタブ：「店舗メニュー」内のおすすめセクション**

### 4.6.1 おすすめの追加

1. 「店舗メニュー」サブタブを開く
2. おすすめセクションのフォームが表示されている
3. 以下を設定する：

| 項目 | 必須 | 説明 |
|------|------|------|

| バッジ種別 | ✓ | `おすすめ` / `人気` / `新着` / `限定` / `本日限り` の 5 種類から選択 |
| コメント | — | 最大 200 文字のひとこと説明（例：「本日入荷の新鮮素材で作りました」） |

定常運用で「前日分が残ったまま」を防ぐには、シフト開始時の業務チェックリストに「前日のおすすめ削除 → 当日分追加」を入れてください。
:::

## 4.7 品切れ管理

品切れの設定は以下の 3 箇所から行えます：

| 場所 | 範囲 | 操作者 | 即時反映 |
|------|------|--------|---|
| メニューテンプレート（カテゴリ管理タブ） | **全店舗**で品切れ | manager 以上 | ✓（2 秒） |
| 店舗メニューオーバーライド（店舗メニュータブ） | **その店舗のみ**品切れ | manager 以上 | ✓（2 秒） |
| KDS / レジ画面（売り切れトグル） | **その店舗のみ**品切れ | KDS / レジ操作者（device 含む） | ✓（2 秒） |

品切れ設定の反映は即時です。セルフオーダー画面は 2 秒ごとにメニューバージョン（`menu_version`）をチェックしており、品切れ設定が変更されるとお客さんの画面に自動反映されます。

::: tip 在庫管理との連動
在庫管理機能（11 章、全契約者に標準提供）を使用している場合、食材の在庫が 0 になると、その食材を使用するメニューが自動的に品切れ表示になります（レシピ BOM 連動）。

品切れを「在庫補充して解除」する場合は在庫側もリセットしないと再び品切れに戻ります。
:::

### 4.7.1 品切れ運用フロー（実例）

```
ピークタイム
  ├─ 仕込み分が残り少ない → ハンディPOS で「品切れ」トグル
  ├─ お客さん画面 2 秒以内に「売切」表示
  ├─ 仕込み追加完了 → 品切れ解除トグル
  └─ お客さん画面 2 秒以内に再表示
```

## 4.9 メニュー多言語対応（AI 翻訳）

全契約者に標準提供されます（2026-04-09 以降、旧 pro プラン制限は撤廃）。メニューを指定した言語に一括翻訳する機能です。

### 4.9.1 操作手順

1. 「店舗メニュー」サブタブを開く
2. **「翻訳」ボタン**を押す
3. 翻訳先の言語を選択する（複数同時選択可）
4. 全カテゴリ・全メニュー名・全説明文・全オプション名が一括で AI 翻訳されます
5. 翻訳結果はキャッシュ（`menu_translations`）に保存され、セルフオーダー画面でお客さんの端末の言語設定に応じて自動表示されます

### 4.9.2 対応言語（実装一覧）

`api/store/translate-menu.php` の `$allowedLangs` で許可された言語のみ：

| 言語コード | 言語 | 備考 |
|---|---|---|
| `en` | 英語 | デフォルト ON（P1-31 で追加） |
| `zh-Hans` | 中国語簡体字 | デフォルト ON |
| `ko` | 韓国語 | デフォルト ON |

::: warning zh-Hant（繁體中文）は廃止
P1-31 で運用判断により無効化（DB 既存データは残置）。台湾・香港対応を再開する場合は posla 運営に相談してください。
:::

### 4.9.3 強制再翻訳（force フラグ）

メニュー名や説明文を変更した場合は、再翻訳が必要です。

| ボタン | 動作 |
|---|---|
| 「翻訳」 | 未翻訳項目のみ Gemini に投げる（既存キャッシュ維持） |
| 「強制再翻訳」 | 全項目を強制再生成（`force=true` で API 呼び出し） |

リクエスト例（フロント側内部処理）:
```
POST /api/store/translate-menu.php
{ "store_id": "xxx", "langs": ["en","zh-Hans","ko"], "force": false }
```

### 4.9.4 翻訳の仕組み

- モデル: Gemini 2.0 Flash（POSLA サーバー側）
- 30 件ごとにバッチ処理
- 翻訳結果は `menu_translations` テーブルに保存（キャッシュ）
- 同一文字列の再翻訳はキャッシュを使用
- API キーは POSLA 運営が一括負担（`posla_settings.gemini_api_key`）

### 4.9.5 Gemini の癖と注意点

| 現象 | 原因 | 対処 |
|---|---|---|
| カタカナ語が中国語になりにくい（「ハンバーグ」が `汉堡` のままなど） | Gemini はカタカナ外来語の元語推定が弱い | 強制再翻訳で改善 / 手動修正（P1-31b で改善） |
| 同じ料理名が日によって違う訳になる | LLM の確率的揺らぎ | キャッシュされるので 1 度確定したら維持される |
| 「特製」「自家製」が「Special」「Homemade」固定にならず文脈で変わる | Gemini の意訳挙動 | 強制再翻訳で安定 |
| 韓国語の助詞が違和感ある | 機械翻訳の限界 | 説明文を短く区切ると改善 |

### 4.9.6 翻訳の手動修正

機械翻訳が不適切な場合、`menu_translations` テーブルを直接 SQL で書き換えるか、メニュー名側に英語別名（`name_en`）を入れることで対処します。`name_en` が入っていれば翻訳時にそれが優先される設計です。

## 4.10 CSV 一括インポート・エクスポート

メニュー関連のデータを CSV ファイルで一括管理できます。Excel や Google Sheets で大量編集してから取り込めます。

### 4.10.1 エクスポート（書き出し）

1. 対象のサブタブを開く（店舗メニュー、限定メニュー等）
2. **「CSV エクスポート」ボタン**を押す
3. CSV ファイルがダウンロードされます
4. ファイルは UTF-8（BOM 付き）エンコーディングで、Excel でそのまま開けます

### 4.10.2 インポート（読み込み）

1. 対象のサブタブを開く
2. **「CSV インポート」ボタン**を押す
3. CSV ファイルを選択する
4. インポートが実行される
5. 結果が表示される：作成された件数、更新された件数、エラーがあればエラー内容

::: warning CSV インポートの注意点
- インポートは UPSERT 方式です（既存のデータがあれば更新、なければ新規作成）
- トランザクション内で実行されるため、途中でエラーがあった場合は**全件ロールバック**されます
- エラーがあった場合、「行 2: カテゴリが見つかりません」のように行番号とエラー内容が表示されます
- 大量行数（>1000 行）はタイムアウトに注意。100〜500 行ずつに分割推奨
:::

### 4.10.3 CSV インポート/エクスポート対象

| データ | エクスポート | インポート | API |
|--------|-----------|-----------|---|
| 店舗限定メニュー | ✓ | ✓ | `/api/store/local-items-csv.php` |
| メニューオーバーライド | ✓ | ✓ | `/api/store/menu-overrides-csv.php` |
| 本部メニューテンプレート | ✓ | ✓ | `/api/owner/menu-csv.php` |
| 食べ放題プラン | ✓ | ✓ | — |
| プラン専用メニュー | ✓ | ✓ | — |
| コース＋フェーズ | ✓ | ✓ | — |
| 食材マスタ | ✓ | ✓ | 11 章参照 |
| レシピ（BOM） | ✓ | ✓ | 11 章参照 |

### 4.10.4 CSV カラム例（店舗オーバーライド）

```
template_id, name, category, base_price, override_price, is_hidden, is_sold_out, override_image_url
abc-001, 特製唐揚げ, メイン, 880, 980, 0, 0, /uploads/karaage_shibuya.jpg
abc-002, ステーキ重, メイン, 1480, , 1, 0,
```

- `override_price` を空欄にすると本部 base_price 継承
- `is_hidden` / `is_sold_out` は 0 / 1
- `override_image_url` を空欄にすると本部 image_url 継承

## 4.X よくある質問（FAQ 30 件）

### A. 3 層構造・本部マスタ

**Q1. 本部マスタと店舗メニューの違いは?**
A. マスタ（`menu_templates`） = 全店共通の基礎情報。店舗（`store_menu_overrides`） = 店舗ごとの価格・表示カスタム。アドオン契約時のみ本部マスタ編集が owner 専用。

**Q2. アドオン契約と未契約では何が違う?**
A. 契約時は本部マスタが owner-dashboard 専用、店舗側は readonly。未契約時は店舗 dashboard から本部マスタを直接編集できる（事実上）。

**Q3. 店舗独自メニューを本部に追加されたくない**
A. 「限定メニュー」タブから追加。本部マスタには入りません。「店舗メニュー」タブから新規追加すると本部マスタに入る点に注意。

**Q4. 価格を店舗ごとに変えたい**
A. 「店舗メニュー」の編集モーダルで `価格` 欄に値を入れて保存。本部マスタは維持されます。空欄に戻すと本部価格に戻る。

**Q5. 本部マスタの価格を変えたら全店舗に反映される?**
A. はい。ただし店舗オーバーライドで `price` を上書きしている店舗は、その店舗の上書き値が優先されます。

### B. カテゴリ・並び順

**Q6. カテゴリの並び順は?**
A. カテゴリ管理で表示順を数値指定。小さいほど上。10 飛ばし運用を推奨。

**Q7. カテゴリを削除したらメニューはどうなる?**
A. 「カテゴリなし」状態になります。事前にメニューを別カテゴリに移動推奨。

**Q8. カテゴリ名を多言語化したい**
A. 「カテゴリ名（英語）」欄に英語名を入れる + 4.9 の一括翻訳で zh-Hans / ko も生成。

### C. 品切れ・在庫連動

**Q9. 品切れはどこから設定?**
A. 3 箇所（テンプレ / オーバーライド / KDS）。範囲が異なる（4.7 表参照）。

**Q10. 品切れ解除しても客側で買えない**
A. 在庫管理機能で食材在庫が 0 のままだと自動で品切れに戻る。在庫側もリセット。

**Q11. 本部側で品切れにすると店舗で解除できない?**
A. はい。本部品切れ（テンプレ側 `is_sold_out=1`）は全店共通で、店舗側で解除不可。

**Q12. 売切表示はどれくらいで反映する?**
A. 約 2 秒。お客さん画面が 2 秒ごとに `menu_version` をポーリング。

### D. オプション・トッピング

**Q13. オプショングループの使い所**
A. トッピング / 辛さ / 麺の硬さ / ごはんの量 など、同一品目の選択肢。

**Q14. 必須オプションって何?**
A. 注文時に必ず選ばないとカートに入らないオプション。「ごはんの量」など。

**Q15. オプションの一時無効化は?**
A. グループ編集で「有効」チェックを外す。紐付けを削除する必要なし。

**Q16. オプションに価格差マイナスを入れられる?**
A. 可能。例: 「ごはん少なめ -50 円」。

### E. 多言語翻訳

**Q17. 多言語翻訳は無料?**
A. POSLA 運営が Gemini API 費を負担。テナントは無料。

**Q18. 一括翻訳と強制再翻訳の違い**
A. 一括 = 未翻訳項目のみ。強制再翻訳 = キャッシュ無視で全件再生成。

**Q19. カテゴリ名の中国語化が変**
A. カタカナ外来語は翻訳されにくい（P1-31b で改善）。手動修正 or `name_en` を充実させる。

**Q20. 繁體中文（zh-Hant）が選べない**
A. P1-31 で廃止。再開希望なら posla 運営に相談。

**Q21. 翻訳が反映されない**
A. ブラウザキャッシュ。Ctrl+F5 で強制リロード。

### F. CSV / スマレジ

**Q22. CSV インポートの形式**
A. テンプレートダウンロード可。カラム: template_id / name / category / base_price / override_price / is_hidden / is_sold_out / image_url 等。

**Q23. CSV 文字化け**
A. UTF-8 BOM 付きで保存。Excel は「名前を付けて保存」→「CSV UTF-8」を選択。

**Q24. スマレジインポートで既存メニューが消える?**
A. 消えません（非破壊インポート、P1-20）。既存は維持し、新商品のみ追加。

**Q25. スマレジ連携エラー (OAuth 切れ)**
A. owner-dashboard「スマレジ連携」タブから再認証。

### G. 画像・運用

**Q26. 画像が表示されない**
A. 画像 URL が HTTPS か確認。HTTP だとブラウザでブロックされる（混在コンテンツ）。

**Q27. 画像のサイズは?**
A. アップロード時に 800×800 以下に自動リサイズ（品質 0.8 JPEG）。元画像 5MB まで。

**Q28. メニュー編集権限を staff に渡したい**
A. 不可。`manager` 以上のみ（セキュリティ上）。staff の業務範囲外。

**Q29. 削除したメニューを復元したい**
A. 不可。事前バックアップ（CSV エクスポート）推奨。過去注文データは `orders.items` JSON に名称・価格スナップショットが残っているのでレポートには影響なし。

**Q30. メニュー数の上限は?**
A. ソフト上限なし。実用上 1000 件超えるとセルフオーダー画面の描画が遅くなるためカテゴリ整理推奨。

## 4.Y トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| メニューが表示されない | カテゴリ未設定 / `is_active=0` | カテゴリ設定 + メニュー有効化 |
| 価格変更が反映されない | キャッシュ（最大 3 分） / `menu_version` 未更新 | 3 分待つ or リロード or 該当メニューを再保存 |
| 翻訳ボタンを押しても反応なし | Gemini API キー未設定 | POSLA 運営に連絡（`posla_settings.gemini_api_key`） |
| 翻訳結果が前と同じ | キャッシュヒット | 「強制再翻訳」を使う |
| CSV 文字化け | UTF-8 でない / BOM なし | UTF-8 BOM 付きで保存 |
| CSV インポートで全件ロールバック | どこかの行でエラー | エラー行番号を見て修正 → 再投入 |
| スマレジ連携で「未認証」 | OAuth トークン切れ | 再認証 |
| 画像アップロードで失敗 | ファイル 5MB 超 / 拡張子非対応 | 縮小 / JPG・PNG・WebP・GIF のみ |
| 画像が表示されない | HTTP 配信 / 404 | HTTPS 配信を確認 |
| 限定メニューが本部にも出る | 「店舗メニュー」タブから追加してしまった | 削除して「限定メニュー」タブから再追加 |
| アドオン契約後に編集ロックされた | hq_menu_broadcast=1 で readonly | owner-dashboard「本部メニュー」から編集 |
| 品切れ解除しても客側に出ない | 在庫連動で再度品切れ | 在庫もリセット |
| カテゴリの並び順が変 | 表示順の数値が同じ | 一意な数値を入れる |
| オプション「必須」なのに選択スキップされる | min_select=0 の設定漏れ | min_select≥1 に修正 |
| お客さん画面で旧メニュー名 | menu_translations キャッシュ古い | 強制再翻訳 |

## 4.Z データベース構造（運営向け）

::: tip 通常テナントは読まなくて OK

### 主要テーブル

| テーブル | 役割 | 主キー | 主な FK |
|---|---|---|---|
| `menu_templates` | 本部マスタ | `id` (UUID) | `tenant_id`, `category_id` |
| `store_menu_overrides` | 店舗オーバーライド | `id` | `store_id`, `template_id` |
| `store_local_items` | 店舗限定品 | `id` | `store_id`, `category_id` |
| `categories` | カテゴリ | `id` | `tenant_id` |
| `option_groups` | オプショングループ | `id` | `tenant_id` |
| `option_items` / `option_choices` | 選択肢 | `id` | `group_id` |
| `menu_option_groups` | メニュー × オプション紐付け | (template_id, group_id) | — |
| `menu_translations` | 翻訳キャッシュ | (item_type, item_id, lang) | — |
| `time_limit_plans` | 食べ放題プラン | `id` | `store_id` |
| `course_templates` / `course_phases` | コース料理 | `id` | `store_id` |
| `ingredients` / `menu_ingredients` | 食材・レシピ（C-5 / 11 章） | `id` | `store_id`, `template_id` |

### 主要 API

- `GET/POST/PATCH/DELETE /api/owner/menu-templates.php` — 本部メニュー（owner 専用）
- `GET/POST /api/owner/menu-csv.php` — 本部メニュー CSV
- `GET/POST/PATCH/DELETE /api/owner/categories.php` — カテゴリ管理
- `GET/POST/PATCH/DELETE /api/owner/option-groups.php` — オプショングループ
- `GET/PATCH /api/store/menu-overrides.php` — 店舗オーバーライド
- `GET/POST /api/store/menu-overrides-csv.php` — オーバーライド CSV
- `GET/POST/PATCH/DELETE /api/store/local-items.php` — 店舗限定メニュー
- `GET/POST /api/store/local-items-csv.php` — 限定メニュー CSV
- `POST /api/store/translate-menu.php` — Gemini 経由の多言語翻訳
- `POST /api/smaregi/import-menu.php` — スマレジから一括インポート（owner 専用）

### フロント JS

- `public/admin/js/category-editor.js` — カテゴリ管理
- `public/admin/js/menu-template-editor.js` — 本部マスタ管理
- `public/admin/js/menu-override-editor.js` — 店舗オーバーライド（`_isHqLocked` 制御）
- `public/admin/js/local-item-editor.js` — 限定メニュー
- `public/admin/js/option-group-editor.js` — オプショングループ + 選択肢
- `public/admin/js/time-limit-plan-editor.js` — 食べ放題プラン
- `public/admin/js/course-editor.js` — コース料理

### メニューリゾルバ

`api/lib/menu-resolver.php` の `resolve_store_menu($pdo, $storeId)` が 3 層を統合してセルフオーダー / KDS に返します：

```
1. menu_templates から tenant_id 配下の全件取得
2. store_menu_overrides で is_hidden=1 のものを除外、price/image を上書き
3. store_local_items を merge
4. オプショングループ（menu_option_groups）を join
5. menu_translations から該当言語の訳を join
→ お客さん画面に渡す
```
