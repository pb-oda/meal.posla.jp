# 23. KDSデバイス端末ガイド

KDS、レジ、ハンディで利用する端末の選び方と設置時の注意点を説明します。

## 標準環境のご案内

POSLAの店舗業務端末（KDS・レジ・ハンディ）は、**Android端末 + Google Chrome最新版** を標準環境としています。これは、音声コマンド（Web Speech API）、通知音の安定再生、PWAインストール、常時ポーリングなどが、Android Chromeで最も安定して動作するためです。

iPad/Safariでも一部画面は表示できますが、音声コマンドや通知音の動作が安定せず、運用負荷が大きくなるため、業務端末としてはサポート対象外とさせていただいています。

## 端末選定（標準/推奨/サポート対象外）

| 用途 | 標準環境 | 推奨環境 | サポート対象外 |
|---|---|---|---|
| KDS | Androidタブレット 10インチ以上、横置き、給電運用、Chrome最新版 | — | iPad/Safari、Fireタブレット、Chromeを更新できない端末、メーカー独自ブラウザ、アプリ内ブラウザ |
| レジ | Androidタブレット + Chrome最新版、決済端末（Stripe Reader）と安定接続できるもの | PC（Windows/Mac）+ Chrome最新版 | iPad/Safari、Fireタブレット |
| ハンディ | Androidスマホ/小型Androidタブレット + Chrome最新版 | — | iPhone/Safari、Fireタブレット、古いAndroid端末 |
| プリンタ連携 | 店舗ネットワークに常時接続できる端末 | — | — |

::: warning Fireタブレット・古いAndroidについて
- Fireタブレット（Amazon）の標準ブラウザ「Silk」はサポート対象外です。
- Android 9以前の古い端末はChromeの自動更新が止まり、音声コマンドや通知音が安定しないためサポート対象外です。
- メーカー独自のブラウザ（Samsung Internet、Huawei Browser、UC Browser など）、アプリ内ブラウザ（LINE等で開いた画面）もサポート対象外です。
:::

## 設置時の注意

- 厨房では水、油、熱、衝撃に注意してください。
- 端末は営業中に電池切れにならないよう給電してください。
- Wi-Fiの電波が弱い場所には設置しないでください。
- 端末ごとに用途を決め、不要な私用アプリを入れない運用を推奨します。
- KDS常時運用では **画面ロックOFF・省電力モードOFF** を設定してください（自動でスリープすると注文受信が止まります）。

## 画面が暗くならないようにする補助機能（Wake Lock）

KDSでは、対応端末で **画面が自動的に暗くなりにくくする補助機能（Wake Lock）** を有効にしています。「KDSを開始」ボタンを押した時点で有効化を試みます。KDSヘッダー右側に状態が表示されます。

| 表示 | 意味 |
|---|---|
| 画面維持: 有効 | 機能が動作中。画面表示中は暗くなりにくくなります |
| 画面維持: 端末設定を確認 | 端末の省電力設定・画面ロック設定により無効化されています |
| 画面維持: 未対応 | 端末・ブラウザがこの機能に対応していません |

::: warning Wake Lock は補助機能で、保証機能ではありません
端末側の省電力設定・画面ロック設定・OS のバッテリーセーバーによっては、このWake Lock機能が無効化されることがあります。KDS常時運用では、端末側で以下も設定してください。

- 画面ロックを「なし」に設定
- 省電力モード OFF
- バッテリーセーバー OFF
- 画面消灯時間を最長または「なし」に設定（Android: 設定 → ディスプレイ → 画面消灯）
:::

## 端末専用アカウント

KDS、レジ、ハンディなどの店舗備品には、端末専用アカウントでログインします。人の操作が必要な場面では、担当スタッフのPINで誰が操作したかを記録します。

## 音声コマンド

音声コマンドはAndroidのGoogle Chrome最新版を標準としています。Safari/iPadでは音声コマンドは利用できません。マイク許可、周囲の騒音を確認してください。厨房が騒がしい場合は、手操作と併用してください。

## 音声まわりの推奨備品（任意）: マイク付き片耳Bluetoothヘッドセット

KDSを騒がしい厨房や広い店内で利用する場合は、**マイク付き片耳Bluetoothヘッドセット**（飲食店の現場では「インカム」とも呼ばれます）の併用をおすすめします。通知音や音声案内を耳元で直接聞き取りやすくなり、音声コマンドを使う場合のマイク入力も安定しやすくなります。POSLAの必須機器ではなく、店舗環境に応じた推奨備品としてご検討ください。

### 目的

- 厨房やホールの騒音の中でも、KDSの通知音や音声案内を聞き取りやすくする
- 音声コマンドを使う場合、マイク入力を安定させやすくする
- 片耳タイプであれば、周囲の声・来客対応・安全確認も残しやすくする

### 選び方の目安

- マイク付きの片耳Bluetoothヘッドセットをお選びください
- 長時間装着しても疲れにくい形状・重さのもの
- 営業時間中に必要な連続使用時間を確保できるもの（予備バッテリーや交互充電も検討）
- ノイズ低減マイク付きだと、音声コマンドの認識がさらに安定しやすくなります
- 厨房で使う場合は、防水・防塵・清掃しやすさもあわせて確認してください
- 完全ワイヤレスの両耳イヤホンは、周囲の音をふさぎやすく紛失もしやすいため、業務利用の標準推奨にはしていません

### 運用上の注意

- 営業前に **Bluetooth接続・音量・マイク許可** を確認してください
- **充電残量** を確認し、必要に応じて予備バッテリーや充電ケーブルを準備してください
- 1台のKDS端末に接続するヘッドセットを決めておくと、接続切替の混乱を避けられます
- 共有して使う場合は衛生管理を行ってください。できれば **個人別または端末別** に管理することをおすすめします
- Bluetooth接続が不安定な場合に備えて、KDS画面のボタンによる **手操作にいつでも戻せる** 状態にしておいてください

::: tip 動作保証について
お使いの端末・ヘッドセット・店舗のWi-Fi/Bluetooth環境の組み合わせによって接続性は変わるため、POSLAでは特定のBluetooth機器の動作を保証していません。導入前に、実際の営業環境で連続利用テストを行ってください。
:::

## 通知を受け取る（補助機能）

KDS / レジ / ハンディの画面ヘッダ右側に「通知: 未設定 (タップで許可)」と表示されている場合、タップすると通知許可ダイアログが出ます。**Android Chrome 端末で PWA をインストール済み** の場合、お客様の呼び出しなど重要な業務通知をホーム画面の通知欄から受け取れます。

::: warning 通知は補助機能です
- 通知が届かなかった場合に備えて、KDS の画面表示やハンディの呼び出しバナーなど、画面側の確認は引き続き行ってください
- 通知が来ない場合の確認:
  - Android Chrome の最新版か（iPhone / iPad / Safari は業務端末の標準環境外）
  - PWA をホーム画面に追加（インストール）しているか
  - 通知許可が ON か（Android: 設定 → アプリ → POSLA → 通知）
  - 省電力モード / バッテリーセーバーで通知が制限されていないか
  - 端末がオンラインか
:::

ホーム画面のアイコンには未対応の場合もありますが、KDS の呼び出し件数や、ハンディの呼び出し件数が **バッジ (赤い数字)** として表示される場合があります（端末・ブラウザの対応状況によります）。

## 通信が一時的に切れた場合の挙動（補助機能）

KDS / レジ / ハンディでは、Wi-Fi が一時的に不安定になっても **画面が真っ白になることを防ぐ補助機能** が入っています。

- 直前に取得できた注文一覧・呼び出し・テーブル状況・会計対象 などが画面にそのまま残ります
- 画面上部に黄色いバナーで「**古いデータを表示中です。操作は通信復帰後に行ってください。**」と表示されます
- この状態では **調理開始 / 完成 / 提供済 / 取消 / 会計 / 返金 / 領収書発行 / 注文送信** などの操作はできません。ボタンは無効化されるか、押すとエラー表示されます
- **通信が復帰すると緑のバナーで「通信が復帰しました」と表示**され、最新情報が自動で再取得されます

::: warning オフライン中に業務を進めないでください
通信が切れている間は、古い注文情報のまま新たに会計・注文送信・提供済み更新などを行うと **二重処理や記録漏れの原因になります**。POSLA はオフライン中の操作を意図的に受け付けません。通信が復帰してから操作してください。
:::

::: tip 「再取得」ボタン
古いデータ表示中のバナー内に「再取得」ボタンがあります。手動で最新情報を取りに行きたい場合に使います (画面によっては一度リロードが走ります)。
:::

## ホーム画面に追加する（PWAインストール）

業務端末では、POSLAをホーム画面に追加して1タップで起動できるようにすることを推奨します。

- **Android Chrome（推奨）**: アドレスバーに「アプリをインストール」または画面内の「POSLAをインストール」ボタンが表示された場合はタップ。表示されない場合はメニュー（︙）→「ホーム画面に追加」/「アプリをインストール」。
- **iOS Safari**: ブラウザ仕様によりWeb側からのインストールダイアログは表示できないため、Safariの共有メニュー → 「ホーム画面に追加」で追加します（業務端末としてはサポート対象外）。

::: tip 顧客セルフメニューはインストール不要
お客様向けのセルフメニュー、予約、テイクアウト画面は1回限りの利用を想定しているため、PWAインストールは行わずブラウザのまま利用してください。
:::

## 起動前チェックリスト

KDS/レジ/ハンディの開店前チェック項目です。詳しい操作手順は「Part 0: 準備」「Part 1: 毎日の営業」をご覧ください。

- [ ] Android端末か（iPad/Safariはサポート対象外）
- [ ] Google Chrome最新版か（Chromeのバージョン更新を確認）
- [ ] PWAとしてホーム画面に追加済みか（任意。1タップ起動を推奨）
- [ ] マイク権限が許可されているか（KDS音声コマンド利用時）
- [ ] 通知音テストができるか（KDS起動オーバーレイの「タップして開始」を1回押す）
- [ ] Wi-Fi接続があるか（端末右上のWi-Fiアイコンを確認）
- [ ] 画面ロックOFF・省電力モードOFF（KDS常時運用時）
- [ ] KDS画面が常時前面で開かれているか（他アプリへ切り替わっていないか）
- [ ] （任意）マイク付き片耳Bluetoothヘッドセットを使う場合、Bluetooth接続・音量・マイク許可・充電残量を確認

## 困ったとき

| 症状 | 対処 |
|---|---|
| 画面がスリープする | 端末の自動ロック設定を「なし」にしてください。Androidの場合は「設定 → ディスプレイ → 画面消灯」を最大に。 |
| Wi-Fiが切れる | 設置場所とアクセスポイントを確認してください。 |
| 音声が認識されない | AndroidのGoogle Chrome最新版か、マイク許可、周囲の騒音を確認してください。Safari/iPadでは音声コマンドは利用できません。 |
| 通知音が鳴らない | 画面起動時の「タップして開始」オーバーレイを1回タップしてください。Safariは通知音が安定しないためAndroid Chromeをお試しください。 |
| 端末を入れ替えたい | 新しい端末で端末専用アカウントにログインしてください。 |
| 「標準サポート対象外」のバナーが出る | 標準環境（Android端末 + Chrome最新版）への入れ替えをご検討ください。閉じるボタンで一時的に消すことはできます。 |
| Bluetoothヘッドセットの音が出ない / マイクが反応しない | Bluetooth接続を確認し、必要なら再ペアリングしてください。充電残量、端末側の音声出力先（スピーカー/ヘッドセット）、マイク許可もあわせてご確認ください。接続が安定しない場合は、KDS画面の手操作に戻して運用を続けてください。 |
