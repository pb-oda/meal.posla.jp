## 23.1 KDSデバイスの全体像

### 23.1.1 登場人物

KDSを動かすためには、以下の機材が連携して動きます。

```
┌─────────────────┐    Wi-Fi     ┌──────────────┐
│ ハンディPOS        │ ──────────→ │ POSLAサーバー   │
│ (注文を入力)      │              │ (注文を保存)    │
└─────────────────┘              └──────────────┘
                                          ↑ 3秒ごと
                                          │ ポーリング
                                  ┌──────────────┐
                                  │  KDSタブレット   │ ← 壁掛けマウント
                                  │  (Chrome起動)   │ ← 油はねカバー
                                  │   ┌─────┐    │ ← 常時電源
                                  │   │ 受付  │    │
                                  │   │ 調理中 │    │
                                  │   │ 提供待 │    │
                                  │   └─────┘    │
                                  └──────────────┘
                                          ↓
                                    🔊 ビープ音
                                    🎙️ マイク（音声コマンド）
```

### 23.1.2 KDSとは何の画面か

KDS（Kitchen Display System）は、キッチンスタッフが注文を見て調理進行を管理する画面です。

- ハンディPOSで入力された注文が **3秒ごとに自動で表示**される
- 「受付 / 調理中 / 提供待ち」の3カラムで表示
- 注文が来るとビープ音（880Hz → 1100Hz）が鳴る
- 「調理開始」「完成」「提供済み」のボタンで進行管理する
- 経過時間が10分超えると黄色、20分超えると赤色に変わる

詳しい操作方法は **第7章「KDS（キッチンディスプレイ）」** を参照してください。この章では「その画面を映すための機材」の話だけをします。

### 23.1.3 推奨構成パターン

**小規模キッチン（カフェ・小料理屋・ラーメン店 など）:**

```
キッチン
└─ KDSタブレット 1台（壁掛け、13インチ）
    ステーション = 全品目モード
    マイク = 内蔵マイク
```

**中規模キッチン（居酒屋・定食屋 など）:**

```
キッチン
├─ KDSタブレット A（メイン調理場、15インチ）
│   ステーション = 「メイン」
│   マイク = USB外付けマイク
└─ KDSタブレット B（ドリンクバー、10インチ）
    ステーション = 「ドリンク」
    マイク = 内蔵マイク
```

**大規模キッチン（チェーン店・ホテルレストラン など）:**

```
キッチン
├─ KDSタブレット A（メイン調理場、15インチ）
│   ステーション = 「メイン」
│   マイク = ヘッドセット
├─ KDSタブレット B（揚げ場、15インチ）
│   ステーション = 「揚げ物」
│   マイク = ヘッドセット
├─ KDSタブレット C（ドリンクバー、10インチ）
```

## 23.2 KDSタブレット本体の選び方

### 23.2.1 推奨機種

POSLAは Web ベースなので、**Chromeが動くタブレットなら何でも動きます**。ただし、業務用途で安定動作させるには以下の機種が推奨されます。

| OS | 推奨機種 | 画面 | 価格目安 | 備考 |
|---|---|---|---|---|
| **iPadOS** | iPad（無印・第10世代以降） | 10.9インチ | ¥58,000〜 | 最も安定。Chrome iOS版可 |
| iPadOS | iPad Air（第5世代以降） | 10.9インチ | ¥84,800〜 | より高速 |
| **Android** | Lenovo Tab M11 | 11インチ | ¥30,000〜 | コスパ良好 |
| Android | Samsung Galaxy Tab A9+ | 11インチ | ¥40,000〜 | 業務用途で安定 |
| **Windows** | Microsoft Surface Go 4 | 10.5インチ | ¥80,000〜 | キーボード・マウス併用可 |
| Windows | 中華系15インチタブレット | 15インチ | ¥25,000〜 | 安いが品質ばらつき注意 |

::: warning iPad で音声コマンドを使う場合の注意
**iPad（iOS / iPadOS）の Safari と Chrome は Web Speech API の continuous モードに対応していません。** iPadで音声コマンドを使いたい場合は、Android タブレット or Windows タブレットの Chrome を選ぶ必要があります。詳細は **23.6** を参照してください。
:::

::: tip Windowsの15インチ中華タブレットは「あり」
価格 ¥25,000〜の中華系Windowsタブレットは、品質にばらつきはあるものの「壁掛けで動かしっぱなし」のKDS用途には向いています。タッチ精度が低いものでも、KDSのボタンは大きいので問題になりにくいです。複数台買って予備を持つ運用がおすすめです。
:::

### 23.2.2 画面サイズの選び方

| 画面サイズ | 視認距離（読める距離） | 用途 |
|---|---|---|
| 10インチ | 1m以内 | 1人用、ドリンクバー、サブ画面 |
| 11〜13インチ | 1〜2m | 標準、メイン調理場 |
| 15インチ | 2〜3m | 大規模キッチン、複数人で見る |
| 21インチ以上（モニタ＋スティックPC） | 3m以上 | チェーン店本部、特殊用途 |

::: tip 視認距離の目安
キッチンスタッフの目から KDS 画面までの距離が **1m なら 10〜11インチ**、**2m なら 13〜15インチ** あれば文字が読めます。揚げ場のように画面から離れて作業する場合は、必ず大きめの画面を選んでください。
:::

### 23.2.3 防水・防油・耐熱の現実

キッチンは「水・油・熱」の3つの大敵がある環境です。

| 環境 | 対策 |
|---|---|
| **水はね・蒸気** | 防水ケース（IPX4以上）or 透明アクリルカバー |
| **油はね** | 透明アクリルカバー or 拭き取り可能なフィルム |
| **熱（揚げ場の近く）** | 直射熱から1m以上離す。直接置かない |

::: warning タブレット本体の防水性は期待しないでください
「IP68防水のタブレット」もありますが、業務用途で年単位で使うとシール部分が劣化します。**外側にケース or アクリルカバーを必ずつける**のが基本です。素のまま置いて1か月で死んだ事例が多数あります。
:::

#### 推奨：防水ケース（汎用）

- **OtterBox Defender**（iPad用、¥8,000〜¥15,000）
- **iVoler 防水ケース**（汎用、¥3,000〜）
- **業務用透明シリコンスリーブ**（汎用、¥2,000〜）

#### 推奨：透明アクリルカバー（壁掛け用）

- **メニュースタンド型アクリル板**（¥2,000〜¥5,000）
- **ホームセンターのアクリル板を自前カット**（¥1,000〜）

### 23.2.4 メモリ・CPU の最低要件

POSLAのKDSは Chrome 上で動く Web アプリです。**3秒ごとのHTTPポーリング**と **DOMの再描画**しか行わないので、要件は低めです。

| 項目 | 最低 | 推奨 |
|---|---|---|
| RAM | 2GB | 4GB以上 |
| ストレージ | 16GB | 32GB以上 |
| CPU | 4コア / 1.5GHz以上 | 8コア / 2.0GHz以上 |
| 画面解像度 | 1280×800 | 1920×1080以上 |
| Chrome バージョン | 最新版 | 最新版 |

::: tip 中古でも十分動きます
2020年以降に発売された中古タブレットなら大体動きます。メルカリやヤフオクで¥10,000〜¥20,000の中古でも実用になります。ただし**バッテリーは劣化前提**で、常時電源接続することを前提に選んでください。
:::

### 23.2.5 バッテリーは持たない前提

KDSは「常時電源接続」が前提の運用です。営業中ずっと画面を表示しっぱなしなので、バッテリー駆動は実質不可能です。

- **店舗営業時間 ≒ 充電しっぱなし** が基本
- バッテリー劣化を気にせず長く使えるよう、**充電制限機能**（80%で止まる）があるとベター
- iPad は iOS 13 以降「最適化されたバッテリー充電」機能あり
- Windows タブレットは多くの機種で BIOS 設定可能

## 23.3 設置とマウント

### 23.3.1 壁掛けマウント（推奨）

最も推奨される設置方法は **壁掛けマウント** です。カウンターを占有せず、油はねからも遠ざけられます。

#### 必要なもの

| 機材 | 価格目安 | 備考 |
|---|---|---|
| VESA マウント（75mm or 100mm） | ¥3,000〜¥10,000 | アームの可動範囲で選ぶ |
| VESA 変換アダプタ（タブレット用） | ¥2,000〜¥5,000 | iPad など VESA非対応端末用 |
| 電源ケーブル（5m以上） | ¥1,500〜¥3,000 | コンセントから配線 |
| ケーブルカバー / 配線モール | ¥500〜¥2,000 | 見た目と安全のため |

#### 取り付けステップ（推奨手順）

1. 壁にマウント位置を決める。**目線の高さ**（床から140〜160cm）が標準
2. 壁の素材を確認する
   - 石膏ボード → アンカー必須（プラスチック・金属どちらも可）
   - コンクリート壁 → コンクリートビス + 振動ドリル
   - 木材下地 → 普通の木ねじでOK
3. マウント本体を壁にビス止めする
4. VESA アダプタにタブレットを取り付ける
5. アダプタをマウントに装着する
6. 電源ケーブルを配線する
7. ケーブルモールでケーブルを隠す
8. タブレットの傾き・角度を調整する

::: warning 油はねから遠ざける
揚げ場の真上や、コンロの真横には設置しないでください。最低 **1.5m以上** 離してください。煙感知器の位置を参考にすると判断しやすいです。
:::

### 23.3.2 スタンド設置（カウンター置き）

壁掛けが難しい場合、カウンター上のスタンド設置でも運用できます。

| スタンド種類 | 価格 | 用途 |
|---|---|---|
| シンプルスタンド | ¥1,000〜¥3,000 | 一時的・小規模 |
| ヘビーデューティ業務用スタンド | ¥5,000〜¥15,000 | 角度調整・盗難防止ロック付き |
| 卓上アームスタンド | ¥3,000〜¥8,000 | 角度を自在に変えられる |

::: warning カウンター置きの注意点
- カウンターは水・油・粉が落ちやすいので、必ずトレイ or マット の上に置く
- 通行スタッフがぶつかって落下しやすい場所は避ける
- 電源ケーブルが足元を横切らないようにする
:::

### 23.3.3 電源確保

KDSは常時電源接続が前提です。

| 項目 | 推奨 |
|---|---|
| コンセント数 | KDS 1台につき1口 |
| ケーブル長 | 5m以上（コンセントから少し離れた壁掛けに対応） |
| 延長コード | 業務用 4口以上のものを使用 |
| UPS（無停電電源装置） | 大規模店舗では推奨。停電時でも30分は稼働 |

::: tip タップは必ず「業務用」を
家庭用の安いタップは数年で接触不良を起こします。長時間連続稼働するKDSには **業務用OAタップ（雷ガード付き）** を使ってください。
:::

## 23.4 Wi-Fi とネットワーク

### 23.4.1 帯域要件

POSLAのKDSは「3秒に1回、JSONを取得する」だけなので、必要な帯域は非常に少ないです。

| 項目 | 必要帯域 |
|---|---|
| 1台あたりの通信量 | 約 10〜30 KB / 3秒 |
| 1時間あたり | 約 12〜36 MB |
| 1日12時間営業 | 約 144〜432 MB |

つまり、家庭用Wi-Fiでも余裕で動きます。

### 23.4.2 Wi-Fi の選び方

| Wi-Fi | おすすめ度 | 理由 |
|---|---|---|
| **5GHz 帯（11ac / 11ax）** | ◎ | 電子レンジ干渉なし、安定 |
| 2.4GHz 帯（11n） | △ | 電子レンジ・コードレス電話と干渉 |

::: warning 飲食店は2.4GHz が干渉だらけ
飲食店の厨房では、**電子レンジ・IH調理器・コードレスキッチンタイマー** が全部 2.4GHz の電波を出しています。2.4GHz Wi-Fi はすぐ詰まります。**必ず 5GHz 帯 が使える Wi-Fi ルーター** を選んでください。
:::

### 23.4.3 ルーターの推奨機種

| 機種 | 価格 | 用途 |
|---|---|---|
| **Buffalo WSR-1500AX2L** | ¥5,000 | 小規模店舗・1〜3台 |
| **TP-Link Archer AX23** | ¥6,000 | 小規模店舗・コスパ |
| **NEC Aterm WX3600HP** | ¥10,000 | 中規模店舗・5台以上 |
| **メッシュWi-Fi（TP-Link Deco X20 など）** | ¥15,000〜 | 大規模店舗・複数フロア |

### 23.4.4 オフライン時の挙動

POSLAは Wi-Fi が一時的に切れても自動復旧します。

- ポーリング間隔が **3秒 → 30秒** に延長される
- 画面上に「オフライン」警告が表示される
- 接続が復旧すると自動的に **3秒間隔** に戻り、強制リフレッシュが走る

::: tip 完全オフライン運用は対応していません
POSLAはクラウド前提のシステムです。インターネットが完全に切れている状態では、新規注文の受信も会計処理もできません。**1〜2分の瞬断には耐えますが、長時間のオフラインには対応していません**。安定したWi-Fi環境を必ず用意してください。
:::

## 23.5 Chrome 設定（必須）

### 23.5.1 なぜ Chrome 限定か

KDSは Google Chrome での動作を強く推奨しています。理由は以下のとおりです。

| 機能 | Chrome | Safari | Firefox | Edge |
|---|---|---|---|---|
| KDS基本表示 | ✅ | ✅ | ✅ | ✅ |
| 通知音（Web Audio API） | ✅ | ✅ | ✅ | ✅ |
| **音声コマンド（Web Speech API continuous）** | ✅ | ❌ | ❌ | △ |
| **スリープ抑止（Wake Lock API）** | ✅ | ✅ | ❌ | ✅ |
| PWAインストール | ✅ | ✅ | △ | ✅ |

::: warning 音声コマンドを使わないなら他のブラウザでもOK
音声コマンド機能を使わないなら、Safari でも Firefox でも Edge でも動きます。ただし、トラブル発生時のサポートは Chrome のみを基準にしているので、できれば Chrome に統一しておくと診断が早いです。
:::

### 23.5.2 タップして開始 = 音声解禁

KDSを起動すると、最初に **「KDSを開始」ボタン** が出ます。これをタップしないと、ビープ音や音声コマンドが動きません。

```
┌──────────────────────────┐
│           🍳              │
│    KDS キッチンディスプレイ   │
│      渋谷店                │
│                          │
│    ┌──────────────┐      │
│    │  KDSを開始    │      │
│    └──────────────┘      │
│   音声コマンド・通知音が     │
│        有効になります        │
└──────────────────────────┘
```

::: tip ブラウザのセキュリティ仕様
ブラウザは「ユーザーがタップ/クリックする前に勝手に音を鳴らす」ことを禁止しています。これを **autoplay policy** と言います。「KDSを開始」ボタンのタップで、**Web Audio API の AudioContext** がアンロックされ、以降は音が鳴るようになります。
:::

### 23.5.3 スリープ抑止

タブレットを放置していると勝手に画面が消えてしまうと、注文が来てもキッチンに気付きません。スリープ抑止は必須設定です。

#### iPadの場合

1. **設定** アプリを開く
2. **画面表示と明るさ** → **自動ロック** をタップ
3. **「なし」** を選択

#### Androidの場合

1. **設定** アプリを開く
2. **ディスプレイ** → **画面消灯** をタップ
3. **「なし」** または最長時間（30分）を選択

::: tip 開発者向けオプションでさらに強力なスリープ抑止
Androidは「開発者向けオプション」を有効にすると **「充電中はスリープしない」** という設定があります。これを ON にすると、充電ケーブルがつながっている限り絶対にスリープしません。**設定 → タブレット情報 → ビルド番号を7回タップ → 開発者向けオプション → スリープモードにしない（USB接続中）**
:::

#### Windowsの場合

1. **設定** → **システム** → **電源**
2. **画面とスリープ**
3. **電源接続時に画面の電源を切る**：**なし**
4. **電源接続時にスリープにする**：**なし**

### 23.5.4 フルスクリーンモード

KDSはアドレスバーを隠すと画面が広く使えます。

| OS | フルスクリーン方法 |
|---|---|
| Windows / ChromeOS | **F11** キー |
| iPad / Android Chrome | アドレスバーの右の「︙」→ **ホーム画面に追加**（PWA化） |

::: tip PWA化（ホーム画面に追加）が最強
Chrome の「ホーム画面に追加」を実行すると、KDSが独立アプリのように起動するようになります。アドレスバーが完全に消え、誤タップで他サイトに移動する事故もなくなります。**強く推奨**します。
:::

### 23.5.5 キャッシュクリア

POSLAをアップデートしたあとに「画面が古い」と感じたら、キャッシュクリアします。

#### Chrome PC / Android

1. アドレスバーの左の鍵マーク（または ⓘ）をタップ
2. **サイトの設定**
3. **データを消去**
4. ページをリロード（F5 / 引き下げ）

#### スーパーリロード（Windows / ChromeOS）

- **Ctrl + Shift + R** または **Ctrl + F5**

#### iPad Safari

1. **設定** アプリ → **Safari** → **履歴とWebサイトデータを消去**

### 23.5.6 マイク権限の付与

音声コマンドを使うには、Chrome にマイク権限を与える必要があります。

3. **許可** に変更

## 23.6 音声コマンドとマイク

### 23.6.1 音声コマンドの仕組み

POSLAのKDSは **Web Speech API + Gemini AI** を組み合わせて音声コマンドを実現しています。

```
スタッフが話す
   ↓
タブレットのマイクが拾う
   ↓
Chrome の Web Speech API が日本語に変換
   ↓
voice-commander.js がコマンドパターンに一致するか判定
   ↓ (一致しなければ)
Gemini AI がコマンドの意図を解釈
   ↓
KDSが対応するアクションを実行（注文ステータス変更など）
   ↓
完了音 🔊
```

### 23.6.2 内蔵マイクで足りるケース

以下のような環境では、タブレット本体の内蔵マイクで十分動きます。

- 静かなキッチン（カフェ・小料理屋）
- 話者とタブレットの距離が **1m 以内**
- レンジフードが動いていない、または静音タイプ

### 23.6.3 外付けマイクが必要なケース

以下の場合は、外付けマイクが必須です。

- 揚げ場・焼き場でレンジフードが常時動いている
- 話者とタブレットが **1m以上** 離れる
- スタッフが複数で同時に話す
- 食洗機・氷製造機などのモーター音がうるさい

| **USB卓上マイク** | Blue Yeti Nano | ¥10,000 | カウンター置き、1人用 |
| USB卓上マイク | サンワサプライ MM-MCU01BK | ¥3,000 | 安価、エントリー |

::: tip ヘッドセットは「片耳タイプ」を選ぶ
キッチンスタッフは周囲の音（呼びかけ・タイマー・客の話し声）を聞く必要があります。**両耳を塞ぐタイプではなく、片耳のヘッドセット** を選んでください。

:::

### 23.6.5 ノイズキャンセリング対策

レンジフード・食洗機などの定常騒音は、以下の対策で軽減できます。

1. **ハードウェアでの対策**: ノイズキャンセリング搭載マイクを使う
2. **OSでの対策**: Windowsの「マイクのノイズ抑制」を ON にする
3. **設置位置**: 騒音源から離れた壁面に取り付ける
4. **指向性マイク**: 単一指向性マイクを話者の方向に向ける

### 23.6.6 認識精度を上げる話し方

- **ゆっくりはっきり** 話す（早口は精度ダウン）
- **語尾を伸ばさない**（「〜してくださーい」より「〜してください」）
- **専門用語より一般語**（「ライスで」より「ご飯を一つ」）
- **テーブル番号は数字で**（「五番テーブル」より「テーブル5」）

### 23.6.7 対応コマンド一覧

POSLAの音声コマンドで実行できる操作は以下です（V-1 / U-6 / I-7 で実装）。

| カテゴリ | コマンド例 | 動作 |
|---|---|---|
| **注文操作** | 「テーブル5、調理開始」 | テーブル5の全品目を pending → preparing |
| 注文操作 | 「テーブル5、完成」 | 全品目を preparing → ready |
| 注文操作 | 「テーブル5、提供済み」 | 全品目を ready → served |
| **品目単位** | 「ハンバーグ、調理開始」 | 該当品目のみ pending → preparing |
| 品目単位 | 「ライス、提供済み」 | 該当品目のみ ready → served |
| **品切れ** | 「ハンバーグ、品切れ」 | sold-out フラグON |
| 品切れ | 「ハンバーグ、復活」 | sold-out フラグOFF |
| **テーマ切替** | 「ライトモード」 | 白背景に切替 |
| テーマ切替 | 「ダークモード」 | 黒背景に切替 |
| **スタッフ呼び出し** | 「ホールを呼んで」 | キッチン→ホール呼び出しアラート発火 |

::: tip ファストパス vs Gemini
voice-commander.js には2段階の処理があります。
- **ファストパス**: 「テーブル5、調理開始」のような決まった形のコマンドはローカル正規表現で即座に処理（300ms以内）
- **Gemini AI**: ファストパスに合致しない自由な言い回しは Gemini API で意図解析（1〜2秒）

ファストパスで通る言い回しの方が早くて安定します。
:::

### 23.6.8 トラブル：認識しない / 暴発する

#### 認識しない場合

| 症状 | 原因 | 対処 |
|---|---|---|
| ボタンを押しても反応しない | マイク権限なし | 23.5.6 の手順で権限付与 |
| 「許可しますか？」が出ない | 既に拒否されている | サイト設定でリセット |
| 話しても何も起きない | Chromeのバージョン古い | Chrome を最新版に更新 |
| iPadで使えない | iOS Safari 非対応 | Android / Windows に変更 |

#### 暴発する場合（勝手に動く）

| 症状 | 原因 | 対処 |
|---|---|---|
| 客の会話で勝手に動く | マイク感度が高すぎ | OS でマイク入力レベルを下げる |
| BGMを認識する | 環境音を拾っている | 指向性マイクに変更 / 設置位置を調整 |
| 同じコマンドを連続実行 | ハウリング | スピーカーとマイクを離す |

## 23.9 よくある質問30件

### A. 機材選定（Q1〜Q5）

**Q1. KDSタブレットは iPad と Android、どっちがいい？**
A. **音声コマンドを使うなら Android（Chrome）**、使わないなら iPad（Safari か Chrome）でOKです。iPad iOS の Web Speech API は continuous モード非対応のため音声コマンドが動きません。

**Q2. 中古タブレットでも大丈夫？**
A. はい、2020年以降の機種なら問題ありません。バッテリー劣化前提で常時電源接続するなら、メルカリで¥10,000の中古でも実用可能です。

**Q3. 画面は何インチがいい？**
A. **視認距離 1m なら 10〜11インチ、2m なら 13〜15インチ** が目安。揚げ場など離れた場所からも見るなら 15インチ以上を推奨します。

**Q4. Windowsタブレットでも動く？**
A. はい、Chrome がインストールできれば動きます。Windows は **マウス・キーボード併用** ができるので、トラブル時の操作が楽というメリットもあります。

**Q5. 1台で何注文まで処理できる？**
A. POSLAは1台のKDSで **同時表示 100注文以上** を問題なく処理できます。1日の注文数で言えば1000件以上でもOKです。

### B. 設置（Q6〜Q10）

**Q6. 壁掛けとカウンター置き、どっちがいい？**
A. **壁掛けが断然推奨**です。カウンターを占有せず、油はねからも遠ざけられます。賃貸店舗で壁に穴を開けられない場合は、カウンタースタンドで運用してください。

**Q7. 賃貸店舗で壁に穴を開けられない**
A. **突っ張り棒タイプの自立スタンド**（メルカリ・モニタースタンドで検索）か、**強力両面テープ式マウント** を使ってください。完全自立でいけます。

**Q8. 揚げ場の真上に置いても大丈夫？**
A. **NG**。最低 **1.5m 離してください**。油・熱・水蒸気がタブレットを早死にさせます。

**Q9. KDSの推奨設置高さは？**
A. **目線の高さ（床から140〜160cm）** が標準。スタッフの身長に合わせて微調整してください。

**Q10. 壁に取り付ける VESA マウントの推奨サイズは？**
A. **10〜13インチタブレット → 75mm**、**15インチ以上 → 100mm** が一般的です。マウント購入前にタブレットの VESA 対応を必ず確認してください。

### C. ネットワーク（Q11〜Q15）

**Q11. 既存のWi-Fi で動く？**
A. **5GHz 帯（11ac/11ax）が使えるルーター** であれば問題なく動きます。2.4GHz しかない古いルーターは厨房環境では干渉が多いため、買い替えを推奨します。

**Q12. KDSタブレットは何台まで接続できる？**
A. ルーターの性能次第ですが、家庭用ルーターで **5〜10台**、業務用なら **20台以上** OK です。

**Q13. インターネット回線は何Mbps必要？**
A. POSLA自体は **下り 5Mbps、上り 1Mbps** あれば動きます。光回線の最安プランでも余裕です。

**Q14. Wi-Fi が一時的に切れたらどうなる？**
A. KDSは自動的にオフラインモードに切り替わり、復旧後に自動同期します。1〜2分の瞬断には耐えますが、長時間オフラインだと営業に支障が出ます。

**Q15. モバイルWi-Fi（ポケットWi-Fi）でも運用できる？**
A. 一時的にはOKですが、**業務用の固定回線を強く推奨**します。モバイルWi-Fiは回線品質が不安定で、ピーク時に詰まるリスクがあります。

### D. 音声・マイク（Q16〜Q20）

**Q16. 内蔵マイクで足りる？**
A. **静かなカフェなら内蔵でOK**。ですが、揚げ場・焼き場・レンジフード稼働中は必ず外付けマイクが必要です。

**Q17. ヘッドセットの推奨機種は？**
A. **片耳USBヘッドセット（Logicool H540 など、¥4,000）** が定番。両耳塞ぐタイプは周囲の音が聞こえなくなるので不可です。

**Q18. 音声コマンドが動かない**
A. 主な原因と対処は **23.6.8** を参照。最頻原因は **iPadで使っている**（Safari/Chrome iOS 非対応）、**マイク権限未付与**、**Chromeバージョンが古い** の3つです。

**Q19. ノイズで誤認識する**
A. ノイズキャンセリングマイクを使う、設置位置を騒音源から離す、Windowsの「マイクのノイズ抑制」をONにする、の3点で改善します。

**Q20. 音声コマンドの言い方の正解は？**
A. 「テーブル5、調理開始」のように **「テーブル番号 → 動作」** の順で話してください。早口・語尾伸ばし・専門用語は避けてください。

### E. トラブル（Q21〜Q25）

**Q21. 画面が突然消える**
A. **スリープ抑止が解除されています**。**23.5.3** の手順でスリープを「なし」に設定してください。

**Q22. 注文音が鳴らない**
A. 起動時に **「KDSを開始」** ボタンを押していますか？このタップで音声機能がアンロックされます。押さないと一切音が鳴りません。

**Q23. 文字が小さくて読めない**
A. タブレットを大きくする以外に、Chrome の **Ctrl + +** で拡大表示できます（タブレットなら設定 → ディスプレイ → フォントサイズ）。

**Q24. ポーリングが止まる（注文が更新されない）**
A. ブラウザのタブをバックグラウンドに置くと、Chrome が省電力モードに入りポーリングが遅延することがあります。**必ずフォアグラウンドで開きっぱなし**にしてください。

**Q25. アップデート後に動きがおかしい**
A. **キャッシュクリア** を実行してください。**23.5.5** の手順または **Ctrl + Shift + R** でスーパーリロードします。

### F. 運用（Q26〜Q30）

**Q26. KDSタブレットが故障したらどうする？**
A. POSLAはクラウドベースなので、別のタブレットで `/kds/` を開いて再ログインすれば、注文データはそのまま続きから操作できます。**予備タブレットを必ず1台用意**することを推奨します。

**Q27. 営業時間中にタブレットが動かなくなった**
A. 一時的なフォールバックとして **スマホのChromeで `/kds/` にアクセス** してください。画面は小さいですが応急対応できます。

**Q28. KDSステーションの設定はどこから？**
A. 管理ダッシュボード → **店舗設定 → KDSステーション** サブタブから設定します。詳細は **第7章 7.10** 参照。

**Q29. 音声コマンドのカスタマイズはできる？**
A. 現状、コマンドのカスタマイズUIはありません。voice-commander.js のソースを変更することで追加可能ですが、POSLA運営にご相談ください。

**Q30. 1日の終わりにKDSは何かする必要がある？**
A. 特になし。閉店時はそのまま電源を入れたままでもOKですし、節電したいなら **タブレット画面のオフ** または **スリープ** にしてください。再起動は不要です。

## 23.10 トラブルシューティング表

### 起動・接続関連

| 症状 | 原因 | 対処 |
|---|---|---|
| KDSが開かない | URLが間違っている | `/kds/` で開く |
| 「未認証」エラー | ログアウト状態 | 管理画面で再ログインしてから KDS リンク |
| 「店舗を選択してください」が消えない | 複数店舗所属で未選択 | リストから店舗をタップ |
| 起動オーバーレイから進まない | ボタンクリック検知失敗 | ページをリロード（F5） |

### 表示・描画関連

| 症状 | 原因 | 対処 |
|---|---|---|
| 注文が表示されない | ステーションフィルタが効いている | 「全品目」ボタンに切替 |
| カードが古いまま | キャッシュ問題 | Ctrl + Shift + R でスーパーリロード |
| 文字化け | フォント未読込 | リロード or キャッシュクリア |
| 画面が真っ白 | JavaScript エラー | ブラウザの開発者ツール（F12）でログ確認 |

### 音声・マイク関連

| 症状 | 原因 | 対処 |
|---|---|---|
| 音声コマンドが動かない | マイク権限なし | 23.5.6 で権限付与 |
| マイク権限の許可ダイアログが出ない | 過去に拒否済み | サイト設定からリセット |
| 認識しない | iPadで使っている | Android/Windows に変更 |
| 認識精度が低い | 内蔵マイクの感度不足 | 外付けマイクに変更 |
| 暴発する | 環境音を拾う | 指向性マイク or 設置位置調整 |

### 通知音関連

| 症状 | 原因 | 対処 |
|---|---|---|
| 全く音が鳴らない | KDS起動時のタップなし | 「KDSを開始」をタップ |
| 音量が小さい | タブレット本体音量小 | 音量を上げる |
| ハウリングする | スピーカー × マイクの近さ | 物理的に離す |

### ハードウェア関連

| 症状 | 原因 | 対処 |
|---|---|---|
| 画面が消える | スリープ未抑止 | 23.5.3 の手順 |
| 動作が重い・固まる | RAM 不足 / 古い機種 | より新しい機種に変更 |
| バッテリー残量警告 | 電源未接続 | 電源ケーブル接続 |
| Wi-Fi 圏外 | ルーター遠い / 干渉 | 5GHz Wi-Fi に変更 / メッシュ化 |

## 23.11 データベース構造（POSLA運営向け）

::: tip 通常のテナントは読まなくてOK
このセクションは、POSLA運営チームと開発者向けの参考資料です。一般のテナントオーナーや店長が読む必要はありません。
:::

### 関連テーブル

KDSデバイスの動作に直接関わるテーブルは以下です。

| テーブル | 用途 |
|---|---|
| `kds_stations` | KDSステーション設定（ステーション名・担当カテゴリ） |
| `call_alerts` | スタッフ呼び出しアラート（KDSのアラートバナーに表示） |
| `orders` | 注文本体（KDSのカード表示元） |
| `order_items` | 注文の品目（KDS品目単位ステータス管理） |

### `kds_stations` テーブル

```sql
CREATE TABLE kds_stations (
  id varchar(36) NOT NULL,
  tenant_id varchar(36) NOT NULL,
  store_id varchar(36) NOT NULL,
  name varchar(100) NOT NULL,
  name_en varchar(100) DEFAULT NULL,
  category_ids json DEFAULT NULL COMMENT '担当カテゴリIDのJSON配列',
  is_active tinyint(1) NOT NULL DEFAULT 1,
  sort_order int NOT NULL DEFAULT 0,
  created_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  KEY idx_tenant_store (tenant_id, store_id)
);
```

### `call_alerts` テーブル

```sql
CREATE TABLE call_alerts (
  id varchar(36) NOT NULL,
  tenant_id varchar(36) NOT NULL,
  store_id varchar(36) NOT NULL,
  alert_type enum('staff_call','product_ready','kitchen_call') NOT NULL,
  table_id varchar(36) DEFAULT NULL,
  reason varchar(255) DEFAULT NULL,
  status enum('pending','resolved') NOT NULL DEFAULT 'pending',
  created_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
  resolved_at timestamp NULL DEFAULT NULL,
  PRIMARY KEY (id),
  KEY idx_tenant_store_status (tenant_id, store_id, status)
);
```

KDS画面では `alert_type = 'staff_call'` のアラートのみが表示されます。`product_ready` と `kitchen_call` はハンディPOS側で表示されます。

### ポーリング実装メモ

- KDS画面: `polling-data-source.js` が `/api/store/orders.php?include_items=1` を **3秒間隔** で叩く
- アラート: `kds-alert.js` が `/api/store/call-alerts.php` を **5秒間隔** で叩く
- AI シェフ: `ai-kitchen.js` が `/api/store/ai-kitchen.php` を **60秒間隔** で叩く
- オフライン時: ポーリング間隔が 30秒 / 30秒 / 一時停止 に延長
- 復旧時: 即座に通常間隔へ戻し、強制リフレッシュ
