# マニュアル整備ステータス

最終更新: 2026-04-30

## 現在の構成

POSLA のマニュアルは、以下の 3 層で管理します。

| 層 | 目的 | 主な読者 | 現在の規模 |
|---|---|---|---|
| `tenant/` | 機能別ガイド。管理画面、レジ、ハンディ、KDS、シフト、予約、テイクアウトなど画面単位で読む | 店舗スタッフ、店長、オーナー | 6,468 行 |
| `operations/` | 業務フロー別手順書。開店、注文、調理、会計、閉店、レポート確認の流れで読む | 現場スタッフ、店長 | 5,030 行 |
| `internal/` | POSLA運営向け。デプロイ、監視、OP/runner、PWA、Cell配備など | POSLA運営、サーバエンジニア | 16,635 行 |

## メンテナンスルール

新機能や画面変更を入れたら、最低限以下を同時に確認します。

1. 該当する `tenant/` の機能別ページに反映する
2. 現場の一日の流れに影響する場合は `operations/part1-daily-operations.md` に反映する
3. 管理画面や分析に影響する場合は `operations/part4-reports.md` に反映する
4. 索引から探せるか確認する
5. `npm run build:all` で `public/docs-tenant/` と `public/docs-internal/` を更新する
6. ドキュメントを書く前に、対象画面・API・認証ロールの現物を確認する。推測や過去メモだけで「できる」と書かない
7. ハンディ / KDS / POSレジはデバイスアカウント前提。個人スタッフアカウントで使う前提の手順を書かない。人の責任者が必要な操作は担当スタッフ PIN として説明する
