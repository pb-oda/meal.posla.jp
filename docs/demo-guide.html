SECTION 1

お客様機能

お客様のスマホに届く体験。注文から評価まで、すべてがデジタルで完結します。

1

1セルフオーダー目玉

お客様がQRコードをスマホで読み取るだけで、メニュー閲覧・注文が完結します。

お客様メニューURLを開き、カテゴリタブ（定食・丼もの・カレー等）を切り替えながらメニューを眺めてください。好きなメニューの「追加」ボタンをタップしてカートに入れ、画面下部のカートバーから「注文する」を送信します。
「ご注文ありがとうございます」と表示され、注文がキッチンのKDS画面にリアルタイムで届きます。
お客様が「注文する」を押した瞬間にキッチンに届きます。スタッフが注文を取りに行く必要がありません。

2テイクアウト事前注文NEW

来店前にスマホからテイクアウト注文と受取時間の指定ができます。

テイクアウトURLを開き、メニューを選んでカートに追加してください。受取日時を15分刻みのスロットから選択し、お客様情報（名前・電話番号）を入力して注文を送信します。
注文完了画面に注文番号と受取時間が表示されます。管理画面のテイクアウトタブにも注文が即時反映されます。Stripe決済が有効な店舗では、決済画面に遷移して事前支払いも可能です。
セルフオーダー画面のヘッダーにも「テイクアウト」リンクが表示されます（テイクアウト有効店舗のみ）。QRコードを店頭ポスターやSNSに貼れば、そこからテイクアウト注文を受け付けられます。
4

3AIウェイターAI

メニューについてAIにチャットで相談できます。

お客様メニュー画面の右下にある「AIに聞く」ボタンをタップしてください。チャット入力欄が開くので「辛くないおすすめを教えて」と入力して送信します。
AIがこの店のメニュー情報（品名・価格・カロリー・アレルゲン）を理解した上で、条件に合うメニューを提案します。提案メニュー横の「カートに追加」ボタンからそのまま注文できます。
「卵アレルギーでも食べられるものは？」「1000円以下のメニューは？」「一番人気は何？」など自然な日本語で質問できます。
5

4多言語対応NEW

外国人のお客様も母国語でメニューを読めます。5言語に対応しています。

お客様メニュー画面のヘッダー右上にある国旗ドロップダウンをタップし、English / 中文(简) / 中文(繁) / 한국어 のいずれかを選んでください。
メニュー名・カテゴリ名・ボタン・メッセージがすべて選択した言語に切り替わります。AIウェイターや呼び出しボタンの表示も連動して切り替わります。
管理ダッシュボードの「メニュー管理」タブにある「一括翻訳」ボタンを押すだけで、全メニューがAI（Gemini）で自動翻訳されます。飲食の文脈を理解した翻訳なので、直訳にならず自然な表現になります。
翻訳は管理画面から一括実行します。翻訳していない言語を選ぶとメニュー名は日本語のままで表示されます。

5おすすめ・人気ランキング

店長おすすめと、実際の注文データに基づくランキングが自動表示されます。

お客様メニュー画面のカテゴリタブから「おすすめ」をタップしてください。
店長が設定した「本日のおすすめ」メニューがバッジ付きで表示されます（シェフのおすすめ / 季節限定 / 期間限定 / 新メニュー / 本日のおすすめ の5種類）。また、メニュー名の横に金（1位）・銀（2位）・銅（3位）のランキングバッジが付いたメニューがあります。これは過去30日間の注文データから自動計算されています。

6カロリー・アレルギー情報

メニューごとにカロリーとアレルゲン情報が表示されます。

メニューカードの下部にカロリー（例：520 kcal）と、該当するアレルゲン（特定原材料7品目＋準ずる21品目）が表示されます。管理画面で店長が設定した場合のみ表示されます。

7ひとことメモ・アレルギー選択

注文時にメモやアレルギー情報をキッチンに伝えられます。

カート画面を下にスクロールすると「ひとことメモ」入力欄とアレルギー品目のチップ（えび・かに・小麦・そば・卵・乳・落花生）が表示されます。メモに「ネギ抜きでお願いします」と入力し、「卵」チップをタップして注文してください。
KDS画面の注文カードに、金色枠のメモと赤色枠のアレルギー警告が表示されます。キッチンスタッフは調理前にアレルギー情報を確認できます。

8調理進捗リアルタイム表示

注文後、お客様のスマホで調理状況がリアルタイムに確認できます。

注文した各品目の横にプログレスバーとステータスバッジが表示されます。キッチンがステータスを更新するたびに、5秒以内にお客様の画面に自動反映されます。
注文済み
→
調理中
→
まもなく届きます

提供済み
「あとどれくらい？」とスタッフに聞く必要がなくなります。

9満足度評価

料理提供後に、お客様がワンタップで満足度を送信できます。

KDS画面で品目を「提供済み」にした後、お客様メニュー画面に戻ってください。
5段階のフェイスアイコン（とても不満〜とても満足）が自動ポップアップ表示されます。タップで評価を送信、15秒で自動クローズ。複数品目は順番に表示されます。
不満の顔（1〜2）をタップすると、KDS画面に低評価アラートが即座に表示されます。キッチンはリアルタイムで品質フィードバックを受け取れます。

10テーブル呼び出し

お客様がスマホからスタッフを呼べます。

お客様メニュー画面の「呼び出し」ボタンをタップし、理由（注文したい / お会計 / 食器下げて / その他）を選んで送信してください。
KDS・ハンディ画面に赤いバナーとビープ音で通知が届きます。テーブル番号と呼び出し理由が表示されるので、どのテーブルが何を求めているかすぐにわかります。
連続押し防止のため、3分間のクールダウンが設定されています。
6

11セルフレジNEW

スタッフを呼ばずに、スマホからそのままお会計ができます。

注文送信後、画面下部に表示される「お会計」ボタンをタップしてください。注文内容と合計金額を確認し、「カードで支払う」を押すとStripe決済ページに移動します。
カード情報を入力して支払いが完了すると、セルフオーダー画面に「お支払い完了」と表示されます。「領収書を表示」ボタンでお会計明細をスマホに表示できます（スクリーンショットで保存可能）。紙の領収書が必要な場合はスタッフにお申し付けください。
この機能は管理画面の設定タブで「セルフレジ」をONにし、Stripe決済（自前またはPOSLA決済）が設定済みの店舗でのみ利用できます。Square決済のみの店舗では現在未対応です。
注文から支払いまで完全にスマホで完結。スタッフの会計対応を大幅に削減できます。

12ウェルカムメッセージ

QRコードを読み取った直後に、店舗からのご挨拶と調理目安時間を表示します。

初回アクセス時にモーダルで表示されます。多言語対応で、同じセッション中は重複表示されません。

13追加注文・再注文

前回の注文をワンタップで再注文できます。

注文履歴セクションに前回の注文内容が表示されます。品切れチェック付きなので、品切れメニューは自動的に除外されます。

14リアルタイム混雑予報

来店前にスマホで空席状況を確認できるログイン不要の公開ページです。

混雑予報URLを開いてください（ログイン不要）。
テーブル空き状況・混雑度（空いています / やや混雑 / 満席に近い）がリアルタイムで表示されます。本日のおすすめメニューや品切れ情報も自動反映されます。
このURLをGoogleマップ・Instagram・LINE公式アカウントに貼るだけで、お客様が来店前に混雑状況を確認できる集客ツールになります。

15Googleレビュー連携NEW

満足度の高いお客様を自然にGoogleレビューへ誘導します。

満足度評価で4点以上をつけたお客様に、Googleレビューへのリンク付きトースト（6秒間表示）が自動表示されます。多言語対応。
管理画面の設定タブでGoogle Place IDを設定すると有効になります。

キッチン・ホール・レジ。店舗オペレーションのすべてをカバーします。

注文カードにテーブル番号・品目名・数量・経過時間が表示されます。各品目のステータスボタンをタップして進捗を管理します。メモは金色枠、アレルギー警告は赤色枠で強調表示されます。テイクアウト注文は「TO」バッジ＋オレンジ背景＋受取時間で区別されます。

4双方向KDS通知

キッチンとホールがリアルタイムに連携します。

KDS画面で品目を「完成」にした後、すぐにハンディ画面を確認してください。
ハンディ画面の上部に緑色バナーで「テーブル01 — 焼き魚定食 できました」と通知が表示されます。通知音も鳴ります。ホールスタッフが「提供済み」にすると、KDS側のステータスも自動更新されます。全品目が提供完了すると注文が自動的にservedに移行します。
従来は「料理できたよ！」と声で呼ぶか、キッチンカウンターを何度も見に行く必要がありました。POSLAならスタッフがフロアのどこにいても通知が届きます。
ホール

5ハンディ（スタッフ注文端末）

ホールスタッフがスマホやタブレットで注文を入力する端末です。

管理画面ヘッダーの「ハンディ」リンクを新しいタブで開いてください。テーブルを選択し、メニューをタップしてカートに追加、「注文送信」で送信します。
注文がKDS画面にリアルタイムで反映されます。セルフオーダーからの注文と同じ画面に一元表示されるので、キッチンは注文経路を意識する必要がありません。
スマホ操作が苦手なお客様や混雑時に、スタッフが代わりに注文を受けられます。テイクアウト注文にも対応しています。
7

6フロアマップ・テーブル管理

全テーブルの状況をひと目で把握できます。

管理ダッシュボードの「テーブル管理」タブをタップしてください。
全テーブルがカード形式で表示されます。各カードに滞在時間・品目進捗バー（色分け）が表示されます。テーブル枠の色でステータスがわかります: オレンジ枠＝着席済み未注文、青枠＝注文済み、赤枠＝会計要求中。
画面上部のサマリーバーに「未注文テーブル：○卓」「会計要求：○卓」がリアルタイム更新されます。フロア全体の状況を瞬時に把握できます。

7テーブルメモ

スタッフ間の引き継ぎメモをテーブルに紐づけて共有できます。

フロアマップのテーブルカードをタップし、「メモ」ボタンを開いてください。プリセットタグ（アレルギー注意・VIP・誕生日・子連れ・車椅子等）をタップするか、自由入力でメモを保存します。
メモがフロアマップのテーブルカード上に即座に表示されます。ハンディ画面からもリアルタイムで共有されます。会計後に自動リセットされます。
シフト交代時の引き継ぎが劇的にラクになります。口頭伝達に頼らず、確実に次のスタッフに引き継がれます。

8ラストオーダー

全テーブルに一斉にラストオーダーを通知します。

フロアマップの「ラストオーダー」ボタンをタップしてください。
お客様のセルフオーダー画面に黄色いバナーで「ラストオーダーです。ご注文はお早めに！」と表示されます。一定時間後に注文ボタンが無効化されます。
管理ダッシュボードの「設定」タブで定時ラストオーダー（例：毎日21:30）を設定すれば、毎日自動的に発動します。手動ボタンとの併用も可能です。
テスト後は必ず「LO解除」ボタンをタップしてください。解除しないとお客様が注文できない状態が続きます。
レジ・会計

9POSレジ

テーブル選択から会計確定まで、レジ操作を一画面で完結します。

管理画面ヘッダーの「レジ」リンクを新しいタブで開いてください。左カラムからテーブルを選択し、「現金」をタップ、預かり金額を入力して「会計確定」します。
お釣りが自動計算され、レシートプレビューが表示されます。会計確定後、フロアマップ上のテーブルが自動的に「空席」に戻ります。
クイックボタン（1,000円 / 5,000円 / 10,000円）で預かり金をワンタップ入力できます。キーボードショートカットにも対応しています。

10合流会計

複数テーブルのお客様が「一緒に会計したい」場合に使います。

POSレジの「合流会計」ボタンをタップし、合流させたいテーブルを2つ選択してください。品目が統合表示されたら、通常通り支払い処理を行います。
1回の操作で2テーブル分がまとめて精算されます。両テーブルが同時に「空席」に戻ります。
忘年会やグループ来店で「まとめて会計して」というリクエストにワンタップで対応できます。

11分割会計（個別会計）

「自分の分だけ先に払いたい」場合に、品目ごとに個別精算できます。

テーブルを選択し、先に支払いたい品目のみチェックを残して会計確定します。
チェックした品目だけが精算されます。支払済み品目はグレーアウト＋「支払済」ラベルで表示され、二重請求を防止します。全員が支払い終わるとテーブルが自動解放されます。

12領収書発行NEW

お客様から「領収書をください」と言われたら、レジ画面からその場で発行できます。

会計が完了してレシートプレビューが表示されたら、「領収書」ボタンをタップしてください。宛名の入力ダイアログが表示されるので、会社名などを入力します（空欄の場合は「上様」になります）。
領収書が新しいウィンドウで表示され、印刷ダイアログが開きます。レシートと同じプリンターからそのまま印刷できます。発行済みの領収書は管理ダッシュボードの「領収書設定」タブで履歴を確認・再印刷できます。
管理画面の「領収書設定」で適格請求書発行事業者の登録番号（T+13桁）を設定すると、自動的にインボイス制度対応の「適格簡易請求書」として発行されます。税率ごとの内訳（10%対象・8%対象）も自動で記載されます。

13決済端末連携NEW

Stripe決済端末（Stripe Reader S700）と連携して対面カード決済を処理します。自前のStripeアカウントがなくても、POSLA決済を使えばカード決済を始められます。

自前のStripeアカウントがある場合：
オーナーダッシュボードの「決済設定」タブで、StripeのAPIキーを入力してください。

Stripeアカウントがない場合（POSLA決済）：
オーナーダッシュボードの「決済設定」タブで「POSLA決済を利用する」を選択し、画面の案内に従ってStripe Connectのオンボーディング（銀行口座・本人確認の登録）を完了してください。審査完了後、すぐにカード決済が使えるようになります。売上は店舗の銀行口座に直接入金されます。

端末の設置：
Stripe Reader S700（¥52,480）を店舗のWi-Fiに接続し、KDSレジ画面を開いてください。端末が自動検出されます。
POSレジの支払い方法で「カード」を選択すると、S700の画面にカードタッチ/IC挿入の案内が表示されます。決済完了後、結果がPOSLAに自動記録されます。
対応端末はStripe Reader S700（¥52,480）のみです。POSLA決済の場合でも端末の購入は必要です。Stripeアカウントの作成や複雑な設定は不要で、オンボーディングの案内に沿って進めるだけで導入できます。
決済機能を持っていない店舗でも、POSLA決済を使えば初期費用は端末代のみでカード決済・セルフレジ・テイクアウトオンライン決済をすべて導入できます。

11ユーザー管理重要

POSLAを使うスタッフ全員に、1人1つのアカウントを作成します。これはPOSLAの基本ルールです。

「ユーザー管理」タブを開き、「ユーザー追加」ボタンをタップしてください。ユーザー名・パスワード・ロール（owner / manager / staff）・所属店舗を入力して登録します。
スタッフ一覧にアカウントが追加されます。作成したユーザー名とパスワードをスタッフ本人に伝えてください。スタッフは初回ログイン後に自分でパスワードを変更できます。
メールアドレスは不要です。ユーザー名とパスワードだけでログインできます。飲食店スタッフにメールアドレスを聞く必要はありません。ユーザー名は「tanaka」「suzuki」のように、店舗内でわかりやすい名前を付けてください。
アカウント作成権限は階層型です。オーナーはマネージャーとスタッフの両方を作成できます（全店舗対象）。マネージャーは自分の所属店舗のスタッフのみ作成できます。スタッフにはアカウント作成権限はありません。日常運用では、店長（マネージャー）が自店舗のスタッフアカウントを管理する流れになります。
1人1アカウントは必須です。共有アカウント（「kitchen」「hall」など）は使わないでください。監査ログ・勤怠打刻・スタッフ評価・シフト管理の全てが「誰が操作したか」を前提に動いています。アカウント未作成のスタッフはシフト管理・勤怠の対象外になります。

SECTION 4

安心のしくみ — POSLAがお店を守ります

セキュリティや安定性は、意識しなくても常に動いています。

QRコード不正利用防止

注文時にトークンを検証し、有効期限は4時間。店外からの不正注文をブロックします。

自動ログアウト

30分操作がないと自動でログアウト。端末の置き忘れによる不正操作を防ぎます。

同時ログイン制御

同じアカウントが別端末でログインすると警告を表示。管理者がセッションを無効化できます。

全操作の記録
