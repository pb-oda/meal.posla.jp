# KDS 音声コマンド・ハンズフリー運用仕様

**対象画面**: `public/kds/index.html`, `public/kds/sold-out.html`
**主な対象ファイル**: `public/kds/js/voice-commander.js`, `public/kds/js/kds-renderer.js`, `api/kds/orders.php`
**最終更新**: 2026-04-29

---

## 1. 目的

KDSは厨房内で使う画面であり、調理中は衛生面の理由で端末に触れない時間が多い。
そのため、POSLAのKDS音声操作は「AIに任せる」ことではなく、次の順番で人が迷わず操作できる状態を作る。

1. よく使う操作はブラウザ内のファストパスで即時実行する
2. 取り消し可能な操作は10秒取り消しで戻せるようにする
3. 破壊的な操作や一括操作は確認ダイアログを挟む
4. 曖昧な言い回しだけ、端末ごとの「AIシェフON」時にGeminiへ送る
5. 音声認識は長時間稼働で劣化する前提で、自動リフレッシュする

この方針により、KDSはAIシェフOFFでも通常運用できる。

## 2. 音声ON/OFFとAIシェフON/OFF

### 2.1 音声ON/OFF

KDSヘッダーに `音声ON / 音声OFF` ボタンを自動挿入する。

| 項目 | 仕様 |
|---|---|
| 操作場所 | KDSヘッダー |
| 保存先 | `localStorage: mt_kds_voice_active` |
| 対象 | KDS画面と品切れ管理画面で共有 |
| 挙動 | ON時はWeb Speech APIで常時リスニング |
| 復帰 | ページ遷移後も前回ONなら自動復帰 |

### 2.2 AIシェフON/OFF

KDSヘッダーに `AIシェフON / AIシェフOFF` ボタンを表示する。これはAIシェフパネルと、ファストパスで処理できない曖昧な音声コマンドの補助解析をまとめて切り替える唯一のAI入口である。

| 項目 | 仕様 |
|---|---|
| 操作場所 | KDSヘッダー |
| 初期値 | OFF。KDSを開き直した時は必ずOFFに戻す |
| 保存先 | `localStorage: mt_kds_ai_chef_enabled`。互換のため `mt_kds_voice_ai_fallback` にも同期するが、ページ起動時は保存済みONを復元しない |
| 対象 | 端末ごと。壁掛け端末・厨房端末ごとに別設定 |
| OFF時 | AIシェフパネルとGemini補助解析を使わない。未対応の曖昧な言い回しはエラー表示で止める |
| ON時 | AIシェフパネルを開ける。ファストパスで処理できない発話だけGeminiへ送る |
| AI使用不可時 | 利用者画面では赤い障害表示にせず、AIシェフパネル内で「現在使用できません」と中立表示する。通常のKDS操作・音声ファストパスは継続する。裏側では `AI_NOT_CONFIGURED` を監視アラート対象にする |

音声でも切替できる。

| 発話例 | 動作 |
|---|---|
| `AIシェフオン` | AIシェフをON |
| `シェフ補助オン` | 互換ワード。同上 |
| `AI補助オン` | 互換ワード。同上 |
| `AI解析オン` | 同上 |
| `AIフォールバックオン` | 同上 |
| `AIシェフオフ` | AIシェフをOFF |
| `シェフ補助オフ` | 互換ワード。同上 |
| `AI補助オフ` | 互換ワード。同上 |
| `AI解析オフ` | 同上 |
| `AIフォールバックオフ` | 同上 |

旧称の「シェフ補助」「AI補助」は音声互換のため受け付けるが、画面上の表示名と利用者向け説明は `AIシェフ` に統一する。

### 2.3 音声診断

KDSヘッダーに `音声診断` ボタンを表示する。
この画面は個人スタッフではなく、KDS端末そのものの音声状態を確認するためのもの。

| 項目 | 仕様 |
|---|---|
| 操作場所 | KDSヘッダー |
| 音声操作 | `音声診断`, `マイク診断`, `音声状態`, `マイク状態` |
| 保存先 | `localStorage: mt_kds_voice_diag_v1` |
| 保存単位 | 端末単位 |
| 保存しないもの | 個人スタッフID、個人名、サーバー側の監査ログ |

表示する内容:

- 音声状態
- Web Speech API対応状況
- マイク権限
- マイク入力レベル
- AudioContext状態
- AIシェフON/OFF
- AIシェフ状態
- 最後の起動時刻
- 最後のfinal時刻
- 最後の成功/失敗時刻
- 最後の自動リフレッシュ時刻と理由
- 直近聞き取り文字列
- コマンド成功率
- final回数
- AIシェフ使用回数
- AIシェフOFFで止めた回数
- 自動/手動リフレッシュ回数
- no-speech / network / audio-capture 回数

音声診断を開いている間だけ、Web Audio APIでマイク入力レベルを測定する。
これはWeb Speech APIの認識結果ではなく、端末マイクに音が入っているかを切り分けるための表示。
測定用のMediaStreamは診断画面を閉じると停止する。
入力レベル・ピーク値はサーバーには送信しない。

診断画面には `統計リセット` を置く。
端末入替時やテスト後に、端末内の診断値だけを消せる。

### 2.4 開店前KDSチェック

KDSヘッダーに `開店前チェック` ボタンを表示する。
開店前や端末再起動後に、厨房端末がそのまま使える状態かを1画面で確認する。

チェック項目:

| 項目 | 判定内容 |
|---|---|
| KDS通信 | `GET ../../api/kds/orders.php` が成功するか |
| 自動更新 | 直近ポーリングが10秒以内に成功しているか |
| 通知音 | AudioContextを解禁し、テスト音の再生要求が通るか |
| マイク | `getUserMedia({audio:true})` で入力ピークを約2.2秒測定 |
| 画面維持 | Wake Lockが有効化できるか |
| ステーション | 現在の選択ステーションと件数を表示 |

マイクは厨房騒音やマイク位置で結果が変わるため、入力が小さい場合は `確認` 扱いにする。
これは即NGではなく、端末・マイク・イヤホンマイクの位置を確認するための合図。

開店前チェックは個人スタッフを判定しない。
POSLAのKDS端末はdeviceアカウント運用であり、この画面も端末状態の確認に限定する。

## 3. 音声処理の流れ

KDS音声は次の順番で処理する。

1. Web Speech APIが日本語音声を文字列化する
2. ノイズフィルタで短すぎる発話・長すぎる発話・低信頼度のfinalを除外する
3. ファストパスに一致するか確認する
4. ファストパス一致ならGeminiを使わず即時処理する
5. ファストパス不一致かつAIシェフOFFなら止める
6. ファストパス不一致かつAIシェフONならGeminiへJSON解析を依頼する
7. Geminiの返答が低確信度の場合は確認ダイアログを出す

実装上の状態は次の3状態。

| 状態 | 意味 |
|---|---|
| `off` | 音声OFF |
| `standby` | 音声待機中。発話を受け付ける |
| `processing` | API処理中または解析中。二重実行を避けるため発話を無視 |

## 4. ファストパス一覧

ファストパスはGeminiを使わない。厨房の通常操作は原則ここで完結する。

### 4.1 品目単位ステータス変更

品目名と動作語で、該当品目だけを更新する。

| 発話例 | 動作 |
|---|---|
| `ハンバーグ開始` | ハンバーグを調理中へ |
| `チキン南蛮完成` | チキン南蛮を提供待ちへ |
| `トンカツ提供済み` | トンカツを提供済みへ |

動作語:

| ステータス | 認識語 |
|---|---|
| `preparing` | 調理、開始、準備、じゅんび |
| `ready` | 完成、できた、できました、あがり、上がり、アップ、レディ、ready |
| `served` | 提供、済み、すみ、サーブ、サーヴ |

品目名は完全一致だけでなく、`定食` / `セット` / `弁当` / `丼` / `御膳` / `ランチ` / `プレート` などのサフィックスを除いて照合する。

### 4.2 卓単位ステータス変更

卓番号と動作語で、該当卓の未完了注文を更新する。

| 発話例 | 動作 |
|---|---|
| `T01 調理開始` | T01の未完了注文を調理中へ |
| `T01 完成` | T01の未完了注文を提供待ちへ |
| `T01 提供済み` | T01の未完了注文を提供済みへ |

認識する卓番号表現:

- `T01`
- `t1`
- `テーブル1`
- `1番`

### 4.3 卓単位の一括操作

`全品` / `全部` / `まとめて` を含む場合は、卓単位の一括操作として扱う。

| 発話例 | 動作 |
|---|---|
| `T01 全品調理開始` | T01の未完了注文をまとめて調理中へ |
| `T01 全品完成` | T01の未完了注文をまとめて提供待ちへ |
| `T01 全品提供済み` | T01の未完了注文をまとめて提供済みへ |

一括操作は確認ダイアログを出す。
声で `実行` / `やめる` と答えられる。5秒無回答なら自動キャンセルする。

### 4.4 注文取消

| 発話例 | 動作 |
|---|---|
| `T01 キャンセル` | T01の未完了注文を取消 |
| `T01 取消` | 同上 |
| `T01 全品キャンセル` | T01の未完了注文をまとめて取消 |

取消は確認ダイアログを出す。
取消理由には `音声取消: <発話内容>` を付ける。

### 4.5 直前操作の10秒取り消し

KDSのステータス更新・品目更新・一括更新の後、10秒間だけ取り消しできる。

| 発話例 | 動作 |
|---|---|
| `戻して` | 直前操作を戻す |
| `戻す` | 同上 |
| `今のなし` | 同上 |
| `さっきのなし` | 同上 |
| `直前を戻す` | 同上 |

制約:

- 10秒を過ぎると戻せない
- 通信断やstale状態では戻し操作も止める
- 取消済み品目は戻し対象から除外する

### 4.6 配膳完了の取り消し

KDS上部に `直近の配膳完了` を表示する。
直近8件まで、提供済み品目を `提供待ち` に戻せる。

これは画面操作の導線であり、音声の `戻して` と同じ10秒取り消しとは別に表示される。

## 5. Gemini解析

Gemini解析は、ファストパスで処理できない発話だけが対象。
さらに、端末の `AIシェフ` がONの場合だけ実行する。

### 5.1 AIシェフOFF時

未対応の発話はGeminiへ送らない。

表示:

```text
AIシェフOFF: 未対応の音声コマンドです。「コマンド一覧」で確認してください
```

### 5.2 AIシェフON時

`POST ../../api/store/ai-generate.php` に、次の情報を含めて解析を依頼する。

- 発話文字列
- 現在の注文一覧
- メニュー品目一覧
- 操作候補

GeminiにはJSONのみを返すように指示する。

対応する解析種別:

- 注文単位ステータス変更
- 品目単位ステータス変更
- 品切れ設定
- 品切れ解除
- 品切れ一覧
- 品切れ管理画面へ移動

### 5.3 低確信度時の確認

Geminiが `confidence: "low"` を返した場合は確認ダイアログを出す。
声で `実行` または `やめる` と答えられる。
5秒無回答なら自動キャンセルする。

## 6. KDS表示機能

### 6.1 通常3列表示

通常表示は次の3列。

- 受付
- 調理中
- 提供待ち

品目単位でステータスを分けて表示する。
同じ注文内に受付品目と提供待ち品目が混在する場合、該当品目だけが各列に表示される。

### 6.2 Expeditorまとめ表示

`まとめ (提供順)` はExpeditor向けの1カラム表示。

表示内容:

- 提供待ち合計
- 未完成品合計
- 卓ごとの残り数
- 卓ごとの提供待ち・調理中・受付数
- 注文カード内の `提供可能` / `調理中` / `受付` セクション

並び順:

1. 提供待ち品目を持つ注文
2. 赤・黄・事前着手の優先度
3. 経過時間が長い注文

混在状態の注文では、誤操作防止のため一括ボタンを抑止し、品目ごとの操作を促す。

### 6.3 遅延優先表示

KDSは注文ごとに優先度を自動判定する。

| 条件 | 表示レベル |
|---|---|
| 経過10分以上 | warning |
| 経過20分以上 | danger |
| テイクアウト受取時刻まで20分以内 | prep |
| テイクアウト受取時刻まで10分以内 | warning |
| テイクアウト受取時刻を超過 | danger |
| 予約時刻まで20分以内 | prep |
| 予約時刻到達 | warning |
| 予約時刻から10分以上超過 | danger |

表示理由はカード上に表示される。

例:

- `経過12:30`
- `受取18:30まで18:00`
- `受取18:30まで08:00`
- `受取18:30超過`
- `予約19:00まで15:00`
- `予約19:00`
- `予約19:00超過`

予約時刻は `api/kds/orders.php` が、現在の卓セッションに紐づく予約から `reservation_reserved_at` として返す。
予約テーブルが未適用の環境では無視する。

### 6.4 ステーション別状況帯と通知音

KDSステーションを選ぶと、ステーションバー直下に現在の担当範囲だけの状況帯を表示する。
APIは既存のステーションフィルタ済み注文を返すため、ドリンク端末ならドリンク品目、焼き場端末なら焼き場品目だけを集計する。

表示内容:

- 選択中ステーション名
- 注文数
- 未完了品目数
- 受付数
- 調理中数
- 提供待ち数
- 最上位の注意理由

状況帯と選択中ステーションボタンは、最上位優先度に合わせて色を変える。

| 優先度 | 表示 |
|---|---|
| prep | 青系。予約/テイクアウトの着手準備 |
| warning | 黄系。10分超過または受取/予約直前 |
| danger | 赤系。20分超過、受取超過、予約10分超過 |

ステーション別通知音は、選択中ステーションの集計が増えた時だけ鳴る。
初回表示やステーション切替直後は鳴らさないため、画面を開いた瞬間に厨房で音が連発しない。

| 増えた内容 | 音 |
|---|---|
| danger | 3音の強い警告音 |
| warning | 2音の注意音 |
| 提供待ち | 2音の完了音 |
| prep | 短い事前音 |

通知音はKDS起動オーバーレイの `KDSを開始` クリックでAudioContextを解禁する。
ブラウザや端末設定で音声再生が制限されている場合は、画面操作後に鳴り始める。

## 7. 音声認識の安定化

Web Speech APIはブラウザ依存であり、Chromeでも長時間稼働すると認識品質が落ちることがある。
POSLA KDSではこの前提で、自動メンテナンスを行う。

### 7.1 自動リフレッシュ

音声ON中、20秒ごとに状態を監視する。
ただし、20秒ごとに再起動するわけではない。
条件に当たった時だけハードリセットする。

| 条件 | 動作 |
|---|---|
| 12分経過 | 予防リフレッシュ |
| 90秒応答なし | ハードリセット |
| 3分final未確定かつinterim/エラーあり | ハードリセット |
| interimだけ45秒継続 | ハードリセット |
| 3分内にno-speech 4回 | ハードリセット |
| 3分内にnetwork 2回 | ハードリセット |
| 3分内にaudio-capture 2回 | ハードリセット |

ハードリセット手順:

1. interim確定タイマーを止める
2. `SpeechRecognition.abort()` を呼ぶ
3. `recognition = null` にする
4. 350ms待つ
5. `SpeechRecognition` を新規作成する
6. 待機状態へ戻す

### 7.2 自動リフレッシュしない場面

誤操作防止のため、次の場面では自動リフレッシュしない。

- API処理中
- Gemini解析中
- 確認ダイアログ表示中
- 手動停止中
- すでにハードリセット中

### 7.3 手動再起動

自動リフレッシュで復帰しない場合に備え、手動の `音声再起動` は残す。
厨房では端末に触れない前提のため、通常は自動リフレッシュを優先する。

### 7.4 診断ログ

KDSは音声の状態を端末内に保存する。
これは「誰が操作したか」を記録するものではなく、端末とマイクの状態を見える化するためのログ。

保存先:

```text
localStorage: mt_kds_voice_diag_v1
```

保存する主な値:

- 音声起動回数
- Web Speech APIのfinal回数
- コマンド成功回数
- コマンド失敗回数
- AIシェフ使用回数
- AIシェフOFFで停止した回数
- 自動リフレッシュ回数
- 手動リフレッシュ回数
- no-speech / network / audio-capture / other error 回数
- 最後の起動/成功/失敗/final/自動更新の時刻

直近聞き取り文字列は画面表示用の実行時情報であり、端末内診断ログの主目的は集計値。
サーバーDBには送信しない。

### 7.5 KDS端末状態の常時表示

KDSキッチン画面は、ステーション状況帯の下に小さな端末状態帯を常時表示する。
目的は「開きっぱなし端末が止まっていることに早く気づく」こと。

表示項目:

- 最終更新: `GET /api/kds/orders.php` の直近成功時刻からの経過
- 遅延: 直近成功からの秒数
- stale/live: `OfflineStateBanner.isStale()` の判定
- 音声成功: `localStorage: mt_kds_voice_diag_v1.lastSuccessAt` からの経過
- 通信異常: ポーリングエラーがある場合のみ表示

色分け:

- 通常: 直近成功が10秒以内、staleなし
- 黄: 直近成功が10秒超
- 赤: 未更新、20秒超、stale、またはポーリングエラー

この表示は操作ログではない。
ログインしているデバイスアカウントの状態を見える化するもので、人の担当者を識別する用途では使わない。

### 7.6 予約/テイクアウトの着手時間設定

KDSヘッダーの `着手時間` から、予約/テイクアウトの事前表示開始タイミングを端末ごとに設定できる。
保存先は端末内 `localStorage: mt_kds_prep_lead_v1`。

設定項目:

- テイクアウト: テイクアウト受取時刻の基準分
- 予約: 予約時刻の基準分
- カテゴリ別: 寿司30分前、ドリンク5分前のようなカテゴリ単位の上書き。空欄はテイクアウト/予約の設定に従う

判定:

- 同じ注文に複数カテゴリがある場合、未完了品目の中で最も長い着手時間を使う
- テイクアウトは受取時刻超過で赤、着手時間内で事前表示、受取直前は黄色
- 着手時間が10分未満の場合、黄色表示も着手時間より前には出さない
- 予約は予約時刻前に事前表示、予約時刻到達で黄色、10分超過で赤

カテゴリ一覧は `GET /api/kds/sold-out.php` のメニューカテゴリから取得する。
注文API `GET /api/kds/orders.php` は各品目に `category_id` / `category_name` を付与し、KDSレンダラーがカテゴリ別設定を適用する。

現時点では端末ごとの設定。
店舗全体で同一設定に固定したい場合は、将来 `store_settings` へ昇格して管理画面から保存する。

## 8. 通信断・stale時の扱い

KDSが通信断またはstale状態の場合、状態変更系の操作は止める。

対象:

- 調理開始
- 完成
- 提供済み
- 取消
- 10秒取り消し
- 品切れ更新
- 一括操作
- コース次フェーズ
- スタッフ呼び出し

画面上のボタンはdisabled化され、音声操作も成功音を鳴らさない。
通信復帰後、次のポーリングで操作可能に戻る。

## 9. 推奨端末運用

| 項目 | 推奨 |
|---|---|
| ブラウザ | Chrome |
| AIシェフ | 通常OFF。調理優先順位パネルや曖昧な音声補助が必要な店だけON |
| マイク | 可能ならUSB/有線マイク。Bluetoothは省電力・接続切れに注意 |
| 画面 | スリープしない設定 |
| 設置 | 排気音・食洗機・BGMスピーカーの近くを避ける |
| 確認 | 開店前に `音声ON`, `コマンド一覧`, `T01 残り` などで反応確認 |

Web Speech APIの日本語モードは `T01` のような英数字を常に正しく認識するとは限らない。
運用では `テーブル1`, `1番` も併用できるようにしている。

## 10. 現時点の未実装・今後検討

現物確認時点で、次は未実装または運用検証が必要。

| 項目 | 理由 |
|---|---|
| 実機での30分以上の音声耐久テスト | Web Speech APIは端末・マイク・厨房環境依存が大きい |
| 診断ログのサーバー集約 | 複数店舗/複数端末の音声状態を本部で比較したい場合に必要。現時点は端末内localStorageのみ |
| マイク入力レベルの実機閾値調整 | 端末内Web Audio API表示は実装済み。厨房騒音・マイク種類ごとのOK/確認閾値は実機で詰める |
| ステーション別通知音の実機音量検証 | 厨房騒音・端末スピーカー・イヤホン運用で聞こえ方が変わるため |
| コース提供の詳細制御 | フェーズ単位の開始/停止/待機をより明確にする余地がある |
