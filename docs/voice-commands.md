# KDS 音声コマンド一覧

**対象ファイル**: `public/kds/js/voice-commander.js`
**最終更新**: 2026-04-04

---

## 概要

KDS画面で音声コマンドを使用し、手を触れずに操作できる。
コマンドは2段階で処理される:

1. **ファストパス**: ローカルで即座に処理（Gemini API不要、高速）
2. **Gemini解析**: 複雑なコマンドをGemini APIで解析（テーブル番号特定、曖昧な表現の解釈等）

---

## ファストパス（Gemini不使用・即時実行）

### 1. 品目単位ステータス変更

品目名 + 動作語で、その品目だけを個別にステータス変更する。
注文カードは品目ごとに分割され、該当品目のカードのみがカラム間を移動する。

| 発話例 | 動作 |
|--------|------|
| 「ハンバーグ開始」 | ハンバーグを調理中に移動 |
| 「チキン南蛮完成」 | チキン南蛮を提供待ちに移動 |
| 「トンカツ提供済み」 | トンカツを提供済みにする |

**動作語一覧:**

| ステータス | 認識される語 |
|-----------|-------------|
| 調理開始(preparing) | 調理、開始、準備、じゅんび |
| 完成(ready) | 完成、できた、できました、あがり、上がり、アップ、レディ、ready |
| 提供済(served) | 提供、済み、すみ、サーブ、サーヴ |

**品目名マッチング**: 「ハンバーグ定食」→「ハンバーグ」のようにサフィックス（定食/セット/弁当/丼/御膳/ランチ/プレート）を除去して照合する。

## Gemini解析（複雑なコマンド）

ファストパスで処理できないコマンドはGemini APIに送信される。
テーブル番号の特定や曖昧な表現の解釈が必要な場合に使われる。

### 1. 注文全体のステータス変更（テーブル番号指定）

テーブル番号を指定すると、注文全体（全品目）のステータスを一括変更する。

| 発話例 | 動作 |
|--------|------|
| 「T01完成」 | T01の注文全体を提供待ちに移動 |
| 「テーブル1調理開始」 | テーブル1の全品目を調理中に移動 |
| 「1番キャンセル」 | 1番テーブルの注文をキャンセル |

**テーブル番号認識**: T01 / t1 / テーブル1 / 1番 等の表記を認識。

---

### 2. 品目単位のステータス変更（テーブル指定あり）

テーブル番号 + 品目名の組み合わせで、特定テーブルの特定品目を操作。

| 発話例 | 動作 |
|--------|------|
| 「T01のトンカツ完成」 | T01のトンカツを提供待ちに移動 |
| 「テーブル3のハンバーグ調理開始」 | テーブル3のハンバーグを調理中に移動 |

---

### 3. 品切れ設定・解除

| 発話例 | 動作 |
|--------|------|
| 「トンカツ品切れ」 | トンカツを品切れに設定 |
| 「サバ味噌売り切れ」 | サバ味噌を品切れに設定 |
| 「ハンバーグ販売再開」 | ハンバーグの品切れを解除 |
| 「唐揚げ復活」 | 唐揚げの品切れを解除 |

---

### 4. 品切れ管理画面へ移動

| 発話例 | 動作 |
|--------|------|
| 「品切れ管理」 | sold-out.html へ画面遷移 |

## confidence（確信度）

Geminiが返すコマンドには確信度が付与される:
- **high**: 即座に実行
- **low**: 5秒間の確認ダイアログを表示し、ユーザーが「実行する」を選択した場合のみ実行

---

## 技術仕様

- **音声認識**: Web Speech API (continuous mode, Chrome推奨)
- **コマンド解析**: Google Gemini 2.0 Flash (temperature: 0)
- **ノイズフィルタ**: 2文字未満 / 60文字超はノイズとして無視
- **interim確定**: Web Speech API の interim 結果はタイマー(1.2秒)で確定（final未発火対策）
- **待機表示**: 「🎤 音声待機中（例:「T01完成」「スタッフ呼んで」「コマンド一覧」）」
