# 8. 会計・POSレジ

POSレジ画面（`/kds/cashier.html`）は、テーブルの会計処理を行うための画面です。現金・クレジットカード・QRコード決済に対応し、割引・個別会計・レジ開閉・取引履歴・レシート発行の機能を備えています。

管理ダッシュボードのヘッダーにある「レジ」リンクをクリックすると、新しいタブで開きます。

## 8.1 画面構成

### ヘッダー

| 要素 | 説明 |
|------|------|
| 店舗名 | 現在の店舗名 |
| スタッフ名 | ログイン中のスタッフ名 |
| デジタル時計 | リアルタイムの時刻表示（HH:MM:SS） |
| 📋 取引履歴ボタン | 取引履歴パネルを開閉 |
| 💰 レジ開閉ボタン | レジ開け・閉め操作パネルを開閉 |
| 🔊 音声ボタン | 効果音のON/OFF切替（設定はブラウザに保存） |

### 左カラム：会計待ちテーブル一覧

会計を待っているテーブルとテイクアウト注文が一覧で表示されます。

**各カードの表示内容：**

| 要素 | 説明 |
|------|------|
| テーブルコード | 「5番」「A-1」など |
| テイクアウトバッジ | テイクアウト注文の場合「TO」バッジ |
| プラン情報 | プラン/コース適用時：「コース ¥3,500 × 4名 = ¥14,000」 |
| 注文・品目数 | 「2注文 / 5品」 |
| 合計金額 | 注文合計（税込） |
| 「会計」ボタン | このテーブルの会計を開始 |

**合流会計ボタン：**
左カラム上部に「合流会計」トグルボタンがあります。ONにすると複数テーブルの注文をまとめて1回の会計で処理できます。

会計待ちがない場合は「会計待ちの注文はありません」と表示されます。

### 右カラム：POSレジ

テーブルを選択すると、右カラムに注文内容と決済操作が表示されます。

---

## 8.2 会計の流れ

### ステップ1：テーブルを選択する

1. 左カラムの会計待ちテーブル一覧から、会計するテーブルの「会計」ボタンをクリックします
2. 右カラムにそのテーブルの注文内容が表示されます

### ステップ2：注文内容を確認する

右カラムに表示される注文品目を確認します。

**品目の表示形式：**

| 要素 | 説明 |
|------|------|
| チェックボックス | 個別会計用（後述） |
| テーブル名 | 「[テーブル5]」形式 |
| 品名 | メニュー名 |
| 軽減税率ラベル | テイクアウト品目に「軽減」ラベル（8%対象） |
| オプション | 選択されたオプション（小文字） |
| 数量 | 「×2」形式 |
| 価格 | 品目の合計金額 |
| 支払済み | 一部支払い済みの品目は「支払済」表示（グレー） |

### ステップ3：割引を適用する（必要な場合）

割引が必要な場合は、決済前に割引を設定します。

**割引の種類：**

| タイプ | ラジオボタン | 入力例 | 説明 |
|--------|------------|--------|------|
| 金額割引 | 「金額」 | 500 | 固定金額（円）を割引 |
| パーセント割引 | 「%」 | 10 | 合計の○%を割引 |

**割引の操作：**

1. 割引タイプ（金額 or %）を選択します
2. 値を入力します
3. 「適用」ボタンをクリックします
4. 合計金額の下に「割引 -¥500」のように表示されます
5. 割引を取り消す場合は「取消」ボタンをクリックします

### ステップ4：支払方法を選択する

3つの支払方法から選択します。

| 方法 | ボタン | 動作 |
|------|--------|------|
| 現金 | 「現金」 | テンキーで受取金額を入力 |
| クレジットカード | 「クレカ」 | 金額が自動入力される |
| QRコード | 「QR」 | 金額が自動入力される |

### ステップ5：金額を入力する（現金の場合）

現金を選択した場合、テンキーとクイック金額ボタンで受取金額を入力します。

**テンキー配置：**
```
 7   8   9
 4   5   6
 1   2   3
00   0   C
```

- **C**：入力をクリア
- 最大7桁まで入力可能

**クイック金額ボタン：**

| ボタン | 動作 |
|--------|------|
| ¥1,000 | 受取金額に1,000円を設定 |
| ¥5,000 | 受取金額に5,000円を設定 |
| ¥10,000 | 受取金額に10,000円を設定 |
| ぴったり | 合計金額と同額を自動入力 |

**おつりの自動計算：**
受取金額を入力すると、おつりが自動計算されて表示されます。

```
受取金額：¥5,000
おつり　：¥1,360
```

### ステップ6：会計を確定する

1. 「会計する」ボタンをクリックします
2. 会計処理が実行されます
3. レシートプレビューモーダルが表示されます
4. 注文ステータスが「paid」に更新されます
5. テーブルセッションが終了します
6. 左カラムからそのテーブルが消えます

**会計処理中：**
二重送信防止のため、「会計する」ボタンは処理中に無効化されます。

## 8.3 税金の計算

POSLAは日本の消費税制度に対応し、品目ごとに税率を分けて計算します。

### 税率の区分

| 税率 | 対象 | 条件 |
|------|------|------|
| 10%（標準税率） | 店内飲食 | デフォルト |
| 8%（軽減税率） | テイクアウト | テイクアウト注文の場合 |
| 品目別税率 | 個別設定 | 品目に`taxRate`が設定されている場合 |

### 税額の表示

会計画面では税率区分ごとに内訳が表示されます。

```
10%対象  ¥1,100 （税 ¥100）
 8%対象    ¥540 （税  ¥40）
─────────────────────────
合計      ¥1,640
```

**計算方法（税込価格からの逆算）：**
```
税抜額 = 税込価格 ÷ (1 + 税率)  ※小数点以下切り捨て
税額 = 税込価格 − 税抜額
```

## 8.4 個別会計（割り勘）

1つのテーブルの注文を品目単位で分けて、複数回に分けて会計できます。

### 個別会計の操作

1. テーブルの「会計」ボタンをクリックして注文を表示します
2. 品目リストのチェックボックスで、今回支払う品目を選択します
   - 「全選択」：全品目を選択
   - 「全解除」：全品目の選択を解除
   - 個別チェックボックス：品目ごとに選択/解除
3. 一部のみ選択すると「個別会計」バッジが表示されます
4. 選択した品目の合計が再計算されます
5. 「個別会計する」ボタンで選択品目のみを会計します

**個別会計後の状態：**
- 支払い済みの品目は「支払済」表示（グレー）になり、チェックボックスがロックされます
- 残りの品目は次の会計を待ちます
- 全品目が支払い済みになるとテーブルセッションが終了します

---

## 8.5 合流会計

複数テーブルの注文をまとめて1回の会計で処理します。

1. 左カラム上部の「合流会計」ボタンをONにします
2. 複数のテーブルを選択できるようになります
3. 選択したテーブルの全注文が右カラムにまとめて表示されます
4. 通常どおり会計処理を行います

---

## 8.6 レジ開閉操作

POSレジでは、営業開始時の「レジ開け」と営業終了時の「レジ閉め」を行います。

### レジ開け

1. ヘッダーの「💰 レジ開閉」ボタンをクリックしてパネルを開きます
2. 「レジ開け」セクションで初期金額（つり銭準備金）を入力します
3. 実行するとキャッシュログに記録されます

| 記録データ | 値 |
|-----------|-----|
| タイプ | open |
| 金額 | 入力した金額 |
| メモ | 「レジ開け」 |

### 入金・出金

営業中に現金の入出金が発生した場合に記録します。

**入金（Cash In）：**

| フィールド | 説明 |
|-----------|------|
| 金額 | 入金額 |
| メモ | 入金理由（任意） |

**出金（Cash Out）：**

| フィールド | 説明 |
|-----------|------|
| 金額 | 出金額 |
| メモ | 出金理由（任意） |

### レジ閉め

1. 「レジ閉め」セクションで実際の現金残高を入力します
2. 実行すると以下が表示されます

| 項目 | 説明 |
|------|------|
| 実際 | 入力した現金残高 |
| 予想 | システム計算の期待残高（開始金額 + 現金売上 + 入金 − 出金） |
| 差額 | 実際 − 予想（過不足） |

3. 差額が記録され、レジ操作が無効化されます（営業終了）

| 記録データ | 値 |
|-----------|-----|
| タイプ | close |
| 金額 | 入力した金額 |
| メモ | 「レジ閉め（差額: ±XXX）」 |

## 8.7 取引履歴（ジャーナル）

### パネルの開閉

- ヘッダーの「📋 取引履歴」ボタンをクリック
- またはキーボードの **J** キーを押す

### 表示内容

当日の全取引が一覧で表示されます。

| 要素 | 説明 |
|------|------|
| タイムスタンプ | 決済日時 |
| 決済ID | 取引の一意識別子 |
| テーブル/顧客名 | 対象テーブルまたはテイクアウト顧客名 |
| 品目サマリー | 購入品目の概要 |
| 支払方法 | 現金/クレカ/QR |
| 合計金額 | 決済金額 |
| スタッフ名 | 会計を処理したスタッフ名 |

---

## 8.8 レシート

### レシートプレビュー

会計完了後にレシートプレビューモーダルが表示されます。キーボードの **R** キーでも表示できます。

**レシートの内容：**

| セクション | 内容 |
|-----------|------|
| 決済完了ステータス | — |
| テーブルコード | — |
| 品目内訳（10%対象） | 標準税率品目の一覧 |
| 品目内訳（8%対象） | 軽減税率品目の一覧 |
| 合計金額 | — |
| 受取金額 | 現金の場合のみ |
| おつり | 現金の場合のみ |
| 支払方法 | — |

### レシート印刷

1. レシートプレビューモーダルの「レシート印刷」ボタンをクリックします
2. ブラウザの印刷ダイアログが開きます（`window.print()`）
3. 印刷用のCSSが適用されます

::: tip レシート設定
レシートの表示内容は管理ダッシュボードの「店舗設定」→「領収書設定」で設定できます。
:::

## 8.9 Stripe決済端末連携

proプラン以上で、Stripe Connectアカウントが設定されている場合、カードリーダー（Stripe Terminal）での決済に対応します。

### 接続状態

- ヘッダーにカードリーダーの接続状態が表示されます（緑色/赤色のドット）
- 「再検出」ボタンで再接続を試みます
- Stripe Terminalが利用できない場合はステータス表示が自動的に非表示になります

### カード決済の流れ

1. 支払方法で「クレカ」を選択します
2. 「会計する」ボタンをクリックします
3. UIがロックされ、オーバーレイが表示されます
   - 「カードリーダーで決済準備中...」
   - 「カードをリーダーにかざしてください」
   - キャンセルボタン
4. お客さんがカードをリーダーにかざします
5. 決済が完了するとレシートが表示されます

**内部処理の流れ：**
1. Stripe ConnectのPaymentIntentを作成
2. Stripe Terminalでカード情報を収集
3. 決済を処理
4. POSLAに決済結果を記録

---

## 8.10 キーボードショートカット

レジ画面では以下のキーボードショートカットが利用できます。

| キー | 動作 |
|------|------|
| 0〜9 | 受取金額に数字を追加（現金選択時） |
| Backspace | 受取金額の最後の桁を削除 |
| Escape | 金額クリア / モーダル閉じ / パネル閉じ |
| Enter | 会計を確定 |
| F1 | 支払方法を「現金」に切替 |
| F2 | 支払方法を「クレカ」に切替 |
| F3 | 支払方法を「QR」に切替 |
| J | 取引履歴パネルを開閉 |
| R | レシートプレビューを表示 |

::: tip キーボードショートカットの活用
PC接続のレジ端末では、キーボードショートカットを活用することで会計処理を高速化できます。テンキーで金額入力 → Enterで確定、という流れがスムーズです。
:::
