# 5. テーブル・フロアマップ管理

管理ダッシュボードの「テーブル・フロア」タブでは、店舗のテーブル登録とフロアマップ（リアルタイム状況表示）を管理します。このタブはマネージャー以上のロールで利用できます。

## 5.1 テーブル管理

### テーブル管理画面の構成

「テーブル・フロア」タブ → 「テーブル管理」サブタブを選択すると、登録済みテーブルの一覧が表示されます。

画面上部には登録テーブル数がカッコ内に表示されます（例：`(12)`）。テーブルが1つも登録されていない場合は「🪑 テーブルがありません」と表示されます。

各テーブルは以下の情報を持つカードとして表示されます。

| 表示要素 | 説明 |
|---------|------|
| テーブルコード | T01、A-1 などの識別コード |
| ステータスバッジ | 緑色「有効」/ 赤色「無効」 |
| 座席数 | 「4名」のように表示 |
| QRボタン | QRコードを表示 |
| 編集ボタン | テーブル情報を編集 |
| 削除ボタン | テーブルを削除 |

### テーブルの追加

1. 「テーブル追加」ボタンをクリックします
2. モーダルが開きます（タイトル：「テーブルを追加」）
3. 以下のフィールドに入力します

| フィールド | 種類 | 必須 | 初期値 | 説明 |
|-----------|------|------|--------|------|
| テーブルコード | テキスト入力 | ✅ | — | 店舗内で一意のコード（例：T01、A-1、カウンター1）。プレースホルダーは「T01」 |
| 座席数 | 数値入力 | — | 4 | テーブルの定員人数 |

4. 「保存」ボタンをクリックします
5. 成功すると「テーブルを追加しました」トーストが表示され、一覧が更新されます

**バリデーション：**
- テーブルコードが空の場合：「テーブルコードは必須です」エラー
- 同じ店舗に同じコードが既にある場合：「このテーブルコードは既に使用されています」エラー（HTTPステータス409）

**テーブル作成時に自動的に行われること：**
- UUID形式の一意なIDが自動生成されます
- セッショントークン（64文字のランダムな16進数文字列）が自動生成されます
- セッショントークンの有効期限が4時間後に設定されます
- フロアマップ用の初期位置（X=0, Y=0）、サイズ（幅80px, 高さ80px）、形状（四角形）、フロア（1F）が自動設定されます

### テーブルの編集

1. 一覧で対象テーブルの**編集ボタン**をクリックします
2. モーダルが開きます（タイトル：「テーブルを編集」）
3. 以下のフィールドを変更できます

| フィールド | 種類 | 説明 |
|-----------|------|------|
| テーブルコード | テキスト入力 | 新しいコードに変更可能 |
| 座席数 | 数値入力 | 定員の変更 |
| 有効 | チェックボックス | チェックを外すとテーブルが無効になり、フロアマップやセルフオーダーに表示されなくなります |

4. 「保存」ボタンをクリックします
5. 成功すると「テーブルを更新しました」トーストが表示されます

### テーブルの削除

1. 一覧で対象テーブルの**削除ボタン**をクリックします
2. 確認ダイアログが表示されます：「「T01」を削除しますか？」
3. 「OK」をクリックすると削除されます
4. 成功すると「テーブルを削除しました」トーストが表示されます

::: danger 注意
テーブルを削除すると、そのテーブルに紐づく注文履歴やセッション履歴も影響を受けます（外部キー制約によるカスケード削除）。過去データを残したい場合は、削除ではなく「無効」に設定することを推奨します。
:::

### QRコードの表示

各テーブルにはセルフオーダー用のQRコードが生成できます。

1. 一覧で対象テーブルの**QRボタン**をクリックします
2. モーダルが開きます（タイトル：「T01 QRコード」）
3. QRコード画像（200×200ピクセル）が表示されます
4. QRコードの下にURLが表示されます
5. 「閉じる」ボタンでモーダルを閉じます

**QRコードのURL形式：**
```
https://あなたのドメイン/public/customer/menu.html?store_id=店舗ID&table_id=テーブルID
```

お客さんがこのQRコードをスマートフォンのカメラで読み取ると、そのテーブル専用のセルフオーダー画面が開きます。

::: tip QRコードの活用方法
- 印刷してテーブルに貼付する
- アクリルスタンドに入れてテーブルに設置する
- テーブルごとに異なるQRコードが生成されるため、注文がどのテーブルからのものかを自動的に識別できます
:::

**セッショントークンについて：**
QRコードにはテーブルのセッショントークンが紐づいています。トークンは以下のタイミングで自動更新されます。

- テーブル作成時（4時間の有効期限付き）
- お客さんの会計完了時（精算処理でトークンが再生成される）
- 着席キャンセル時（セッション強制終了でトークンが再生成される）

これにより、前のお客さんのQRコードでは次のお客さんが注文できないようになっています。

## 5.2 フロアマップ

### フロアマップ画面の構成

「テーブル・フロア」タブ → 「フロアマップ」サブタブを選択すると、リアルタイムのテーブル状況一覧が表示されます。

**データは5秒ごとに自動更新されます。** 他のタブに切り替えると自動更新（ポーリング）は停止し、フロアマップタブに戻ると再開します。

### サマリーバー

画面上部にはテーブル状況のサマリーが表示されます。

| 項目 | 表示例 | 説明 |
|------|--------|------|
| 全テーブル数 | 「全 12卓」 | 有効なテーブルの総数 |
| 空き | 「空き 3」（緑色） | 使用中でないテーブル数 |
| 使用中 | 「使用中 7」（青色） | セッションがあるテーブル数 |
| 未注文 | 「未注文 2」（オレンジ色） | 着席しているが注文がまだないテーブル数（0の場合は非表示） |
| 合計人数 | 「合計 28名」 | 現在の全テーブルのゲスト合計 |
| 合計売上 | 「¥42,000」（太字） | 現在の全テーブルの未会計注文合計 |
| ラストオーダー状態 | 「ラストオーダー中」（赤色）/ 「LO 22:00」（灰色） | ラストオーダーの状態 |
| ラストオーダーボタン | 「ラストオーダー」/「LO解除」 | ラストオーダーを手動で発動/解除 |

### テーブルカードの表示内容

各テーブルはカード形式で表示されます。テーブルの状態に応じてカードの左ボーダーと背景色が変わります。

#### テーブルステータスと色分け

| ステータス | 色 | バッジ | 条件 |
|-----------|-----|--------|------|
| 空き | 緑（#43a047） | 空き | セッションなし、注文なし |
| 着席 | 青（#1e88e5） | 着席 | セッションあり、ステータス=seated |
| 未注文 | オレンジ（#ff9800） | 未注文 | seated だが注文が0件 |
| 食事中 | 濃い青（#1565c0） | 食事中 | ステータス=eating または注文あり |
| 会計待ち | 赤（#e53935） | 会計待ち | ステータス=bill_requested |
| 時間超過 | 濃いオレンジ（#e65100） | 時間超過 | 経過時間 > 制限時間（食べ放題プラン） |

#### カードに表示される情報

**空きテーブルの場合：**
- テーブルコード
- 「空き」バッジ（緑色）
- 座席数（「4名」）

**使用中テーブルの場合：**
- テーブルコード
- ステータスバッジ（色分け）
- ゲスト数（「4名」）
- 経過時間（「25分経過」）
- 残り時間（食べ放題プラン適用時）：「残り65分」
  - 時間超過の場合は赤文字で「5分超過」
- 注文件数（「2件の注文」）
- 合計金額（「¥4,200」）
- 調理進捗バー（後述）
- メモ（設定されている場合）
- 「メモ」ボタン
- 「取消」ボタン

#### 調理進捗バー

使用中テーブルのカードには、注文品の調理状況を示す横棒グラフが表示されます。4色のセグメントで構成されています。

| セグメント | 色 | 意味 |
|-----------|-----|------|
| 提供済 | — | お客さんに提供完了 |
| 完成 | — | 調理完了、提供待ち |
| 調理中 | — | 現在調理中 |
| 受付 | — | 注文受付、調理未開始 |

各セグメントにマウスを合わせると「提供済 3品」「調理中 1品」などのツールチップが表示されます。

### ラストオーダー機能

ラストオーダーは、セルフオーダー画面での新規注文をブロックする機能です。

**ラストオーダーの発動：**

1. サマリーバーの「ラストオーダー」ボタンをクリックします
2. 確認ダイアログが表示されます：「ラストオーダーを発動しますか？\nセルフオーダー画面で注文がブロックされます。」
3. 「OK」をクリックすると発動します
4. ボタンが赤色の「LO解除」に変わります
5. サマリーバーに「ラストオーダー中」が赤色で表示されます
6. お客さんのセルフオーダー画面で新規注文ができなくなります

**ラストオーダーの解除：**

1. 赤色の「LO解除」ボタンをクリックします
2. 確認ダイアログが表示されます：「ラストオーダーを解除しますか？」
3. 「OK」をクリックすると解除されます
4. ボタンが通常の「ラストオーダー」に戻ります

::: info 自動ラストオーダー
店舗設定でラストオーダー時刻を設定している場合（例：22:00）、設定時刻になると自動的にラストオーダーが発動します。手動ボタンは、設定時刻に関わらずいつでも発動/解除できます。
:::

### メモ機能

テーブルセッションにメモを追加できます。アレルギー情報やVIP対応など、スタッフ間の情報共有に使います。

**メモの追加・編集：**

1. テーブルカードの「メモ」ボタンをクリックします
2. メモモーダルが開きます（タイトル：「T01 メモ」）
3. 6つのプリセットタグが表示されます

| プリセットタグ |
|--------------|
| アレルギー注意 |
| VIP |
| お子様連れ |
| 誕生日 |
| 接待 |
| 車椅子 |

4. プリセットタグをクリックすると、テキストエリアにタグが追記されます（改行区切り）
5. 自由にテキストを入力することもできます
6. 「保存」ボタンをクリックします
7. 保存中はボタンが「保存中...」に変わり無効化されます
8. 成功するとモーダルが閉じ、テーブルカードにメモが表示されます

メモはテーブルカードの下部に灰色のテキストとして表示され、スタッフ全員が確認できます。

### 着席のキャンセル

テーブルの着席（セッション）を強制的に終了させます。

1. テーブルカードの「取消」ボタンをクリックします
2. 確認ダイアログが表示されます：「T01 の着席をキャンセルしますか？」
3. 「OK」をクリックすると以下が実行されます
   - セッションが「closed」ステータスに変更されます
   - そのテーブルの未会計の注文が全て「cancelled」ステータスに変更されます
   - テーブルのセッショントークンが再生成されます（古いQRコードが無効になります）
4. テーブルカードが「空き」状態に戻ります

::: warning 注意
着席キャンセルは未会計の注文を全てキャンセルします。既に調理済みの料理がある場合は、先に会計処理を行ってからキャンセルしてください。
:::

## 5.3 テーブルセッション

テーブルセッションとは、お客さんが着席してから会計が完了するまでの一連の流れを管理するデータです。

### セッションのライフサイクル

```
着席（seated） → 食事中（eating） → 会計待ち（bill_requested） → 支払済（paid） → 終了（closed）
```

各ステータスの意味は以下のとおりです。

| ステータス | 意味 | 遷移タイミング |
|-----------|------|-------------|
| seated | 着席済み | ハンディPOSやフロアマップから着席操作した時 |
| eating | 食事中 | 注文が入った時 |
| bill_requested | 会計待ち | お客さんが会計をリクエストした時 |
| paid | 支払済み | レジで会計が完了した時 |
| closed | 終了 | 支払い後にセッションが閉じられた時、または着席キャンセル時 |

### セッションに記録される情報

| 情報 | 説明 |
|------|------|
| ゲスト数 | 着席人数 |
| 開始日時 | 着席した日時 |
| 終了日時 | 会計完了または強制終了した日時 |
| プラン | 食べ放題プランを選択した場合（プランID、制限時間、ラストオーダー時間、有効期限） |
| コース | コース料理を選択した場合（コースID、現在のフェーズ番号、フェーズ開始日時） |
| メモ | テーブルメモ（アレルギー情報、VIP等） |

### 食べ放題プラン選択時の動作

着席時に食べ放題プランを選択すると、以下が自動計算されます。

- **有効期限** = 着席日時 + プランの制限時間（例：90分）
- **ラストオーダー期限** = 着席日時 + （制限時間 − ラストオーダー閾値）（例：90分 − 15分 = 75分後）

フロアマップのテーブルカードでは残り時間がカウントダウン表示され、制限時間を超えると「時間超過」ステータスになります。

### コース料理選択時の動作

着席時にコース料理を選択すると、以下が自動で行われます。

1. 現在のフェーズが「フェーズ1」に設定されます
2. フェーズ1に含まれる料理が自動的に注文されます（数量はコース定義に従い、価格は0円）
3. KDS（キッチンディスプレイ）にフェーズ1の料理が表示されます
4. フェーズの進行はKDS画面から「次フェーズ」ボタンで操作します

::: info プランとコースの排他制御
食べ放題プランとコース料理は同時に選択できません。着席時にどちらか一方を選択するか、どちらも選択しない（通常のアラカルト注文）かを選びます。
:::

## 5.4 フロアマップのデータ構造

各テーブルは以下のフロアマップ用データを持っています。APIのPATCHリクエストで更新できます。

| フィールド | 説明 | 初期値 |
|-----------|------|--------|
| pos_x | X座標（ピクセル） | 0 |
| pos_y | Y座標（ピクセル） | 0 |
| width | 幅（ピクセル） | 80 |
| height | 高さ（ピクセル） | 80 |
| shape | 形状（rect / circle / ellipse） | rect |
| floor | フロア名（1F / 2F / テラス 等） | 1F |

::: info フロアマップのドラッグ配置
現在のフロアマップはカード一覧形式で表示されています。ドラッグ＆ドロップによるビジュアルなテーブル配置機能は、上記のデータ構造（座標・サイズ・形状）が準備されており、今後のアップデートで実装予定です。
:::
