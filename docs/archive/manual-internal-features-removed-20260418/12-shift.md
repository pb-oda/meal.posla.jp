# 12. シフト管理

シフト管理機能は**proプラン以上**で利用できます。管理ダッシュボードの「シフト管理」タブから利用します。

マネージャーはシフトの作成・割当・集計の全機能を使用でき、スタッフは自分のシフト確認・希望提出・勤怠打刻のみ操作できます。

## 12.1 サブタブ一覧

| サブタブ | 対象ロール | 説明 |
|---------|-----------|------|
| 週間カレンダー | マネージャー | シフトの割当・編集メイン画面 |
| マイシフト | スタッフ | 自分の確定シフト閲覧 |
| 希望提出 | スタッフ | 希望シフトの提出 |
| 勤怠一覧 | マネージャー | 全スタッフの出退勤記録 |
| テンプレート | マネージャー | シフトパターンの定義 |
| シフト設定 | マネージャー | 各種設定 |
| 集計サマリー | マネージャー | 労働時間・人件費・労基法警告の集計 |
| ヘルプ | マネージャー | 他店舗へのスタッフ派遣要請（enterpriseプラン） |

---

## 12.2 シフトテンプレート

シフトテンプレートは、繰り返し使用するシフトパターン（早番・遅番・通しなど）を定義する機能です。

### テンプレートの作成

| フィールド | 種類 | 説明 |
|-----------|------|------|
| 曜日 | セレクト | 月〜日のいずれか |
| 開始時刻 | 時刻入力 | シフト開始時間 |
| 終了時刻 | 時刻入力 | シフト終了時間 |
| 役割ヒント | テキスト | 役割名（例：ホール、キッチン） |
| 必要人数 | 数値 | その時間帯に必要なスタッフ数 |

### テンプレートの一括適用

作成したテンプレートを指定した期間に一括適用できます。

1. テンプレート一覧から適用するテンプレートを選択します
2. 適用期間（開始日〜終了日）を指定します
3. 「適用」ボタンをクリックすると、期間内の該当曜日にシフト枠が自動生成されます

---

## 12.3 週間カレンダー

シフトの割当・編集を行うメイン画面です。

### カレンダーの表示

- 横軸：曜日（月〜日の1週間）
- 縦軸：スタッフ名
- 各セルにシフト情報（開始〜終了時刻、休憩、役割）が表示されます
- セルの人件費（時給 × 時間）が計算されて表示されます

### シフトの割当

1. カレンダーのセルをクリックします
2. シフト入力フォームが表示されます
3. 以下を入力します

| フィールド | 説明 |
|-----------|------|
| 開始時刻 | シフト開始時間 |
| 終了時刻 | シフト終了時間 |
| 休憩時間（分） | 休憩の長さ |
| 役割 | ホール、キッチンなど |
| ステータス | draft（下書き）/ published（公開） |

4. 「保存」ボタンをクリックします

### AIシフト提案

カレンダー画面に紫色の「AI提案」ボタンがあります。クリックすると、AIが以下のデータを分析して最適なシフトを提案します。

- テンプレート定義
- スタッフの希望提出状況
- 過去4週間のスタッフ配置実績
- 各スタッフの時給
- 直近の出勤状況（出勤日数、遅刻回数）
- 連続勤務日数
- 曜日・時間帯別の平均売上

---

## 12.4 スタッフの希望提出

スタッフは「希望提出」サブタブから、シフトの希望を提出します。

### 提出方法

1. カレンダーで日付を選択します
2. 各日付について以下を入力します

| フィールド | 説明 |
|-----------|------|
| 可否 | 出勤可能 / 希望 / 出勤不可 |
| 希望開始時刻 | — |
| 希望終了時刻 | — |
| メモ | 特記事項 |

3. 「提出」ボタンをクリックします
4. 最大31日分を一括提出できます

**更新方式：** 同じ日に再提出すると上書きされます（UPSERT方式）。

## 12.6 勤怠打刻

### 出勤・退勤ボタン

スタッフロールでログインすると、管理ダッシュボードのヘッダー付近に**出勤ボタン**と**退勤ボタン**が常時表示されます。全てのタブで表示されるため、いつでも打刻できます。

### 出勤打刻の流れ

1. 「出勤」ボタンをタップします
2. GPS検証が行われます（GPS必須が有効な場合）
   - 店舗の座標と現在地のGPS距離をハーバーサイン公式で計算
   - 設定された許容半径内でなければエラー
3. 二重打刻のチェック（既に出勤中の場合はエラー）
4. 予定シフトとの照合
5. 早出警告（設定された時間より前に出勤した場合）
6. 出勤が記録されます

### 退勤打刻の流れ

1. 「退勤」ボタンをタップします
2. 退勤時刻が記録されます
3. 休憩時間が自動計算されます
   - 勤務時間が6時間以上：デフォルト60分の休憩
   - 勤務時間が6時間未満：休憩なし（0分）

### 勤怠データの記録内容

| フィールド | 説明 |
|-----------|------|
| 打刻方法 | manual（手動）、biometric、RFID等 |
| ステータス | working（勤務中）→ completed（完了） |
| 出勤時刻 | — |
| 退勤時刻 | — |
| 休憩時間 | — |

## 12.8 シフト設定

「シフト設定」サブタブで以下の項目を設定します。

| 設定項目 | 説明 |
|---------|------|
| デフォルト時給 | 店舗全体のデフォルト時給（スタッフ個別の時給が未設定の場合に使用） |
| デフォルト休憩時間 | — |
| 残業閾値（分） | この時間を超えた分が残業として計算されます |
| 早出許容時間（分） | シフト開始の何分前から出勤を許可するか |
| GPS必須 | ONにすると、出退勤時にGPS位置情報の検証が必要になります |
| 店舗緯度 | GPS検証用の店舗の緯度 |
| 店舗経度 | GPS検証用の店舗の経度 |
| GPS許容半径（メートル） | 店舗からこの距離以内でのみ打刻を許可 |

## 12.9 集計サマリー

「集計サマリー」サブタブで、労働時間と人件費の集計を確認します。

### 集計期間

- 週単位 / 月単位 で選択
- 日付範囲のカスタム指定も可能

### スタッフ別集計

| 項目 | 説明 |
|------|------|
| 合計勤務時間 | 期間内の総勤務時間 |
| 出勤日数 | — |
| 残業時間 | 残業閾値を超えた時間の合計 |
| 時給 | スタッフ個別の時給（未設定の場合は店舗デフォルト） |
| 人件費 | 時給 × 勤務時間（深夜割増含む） |

### 深夜割増計算

22:00〜05:00の勤務は深夜割増（25%増）が自動計算されます。

### 労基法コンプライアンスチェック

以下の警告が自動で検出・表示されます。

| 警告 | 条件 |
|------|------|
| 週40時間超過 | 1週間の勤務合計が40時間を超えた場合 |
| 休憩不足（8時間以上） | 8時間以上の勤務で60分未満の休憩 |
| 休憩不足（6時間以上） | 6時間以上の勤務で45分未満の休憩 |
| 連続勤務 | 連続で勤務日が続いている場合 |

### 日別詳細

各スタッフの日別詳細データも確認できます。

| 項目 | 説明 |
|------|------|
| 日付 | — |
| 実労働時間（分） | — |
| 休憩時間（分） | — |

## 12.10 ヘルプ要請（enterpriseプラン）

enterpriseプランでは、同じテナント内の他店舗にスタッフの派遣を要請できます。

### ヘルプ要請の送信

1. 「ヘルプ」サブタブを開きます
2. 「ヘルプ要請を送信」ボタンをクリックします
3. 以下を入力します

| フィールド | 説明 |
|-----------|------|
| 派遣先店舗 | ヘルプを要請する店舗 |
| 日付 | 派遣が必要な日 |
| 開始時刻 | — |
| 終了時刻 | — |
| 必要人数 | — |
| 役割ヒント | 必要な役割（ホール、キッチン等） |
| メモ | 補足情報 |

4. 「送信」ボタンをクリックします

### ヘルプ要請のステータス

| ステータス | 意味 |
|-----------|------|
| pending | 要請中（承認待ち） |
| approved | 承認済み（スタッフ割当済み） |
| rejected | 拒否 |
| cancelled | キャンセル |

### ヘルプ要請の承認

派遣先店舗のマネージャーが要請を確認し、スタッフを割り当てます。

1. 「ヘルプ」サブタブの「受信」タブで要請を確認します
2. 「承認」ボタンをクリックします
3. 派遣するスタッフを選択します
4. 回答メモを入力します（任意）
5. 承認するとスタッフのシフトが自動作成されます

### 送信/受信タブ

- **送信タブ：** 自店舗から送信したヘルプ要請の一覧
- **受信タブ：** 他店舗から受信したヘルプ要請の一覧
