# 6. 注文管理（ハンディPOS）

ハンディPOS（`/handy/`）は、スタッフがスマートフォンやタブレットから注文を入力するための画面です。店内飲食の注文とテイクアウトの注文の両方に対応しています。全てのロール（スタッフ・マネージャー・オーナー）で利用できます。

管理ダッシュボードのヘッダーにある「ハンディ」リンクをクリックすると、新しいタブで開きます。

## 6.1 画面構成

### ヘッダー

| 位置 | 要素 | 説明 |
|------|------|------|
| 左 | タイトル「ハンディ」 | — |
| 右 | 店舗名 | 現在操作中の店舗名 |
| 右 | 店舗切替リンク | 複数店舗に所属している場合のみ表示 |
| 右 | ユーザー名 | ログイン中のスタッフ名 |
| 右 | ログアウトボタン | — |

### メインタブ（4つ）

| タブ名 | 説明 |
|--------|------|
| 店内注文 | 店内飲食の注文入力（デフォルト） |
| テイクアウト | テイクアウト注文の入力 |
| テーブル状況 | テーブルの着席・メモ・ステータス管理 |
| 注文履歴 | 当日の注文一覧と編集 |

### 店舗選択

複数店舗に所属しているスタッフの場合、初回アクセス時に店舗選択オーバーレイが表示されます。

1. 所属店舗の一覧が表示されます
2. 操作する店舗をタップします
3. 選択した店舗はブラウザに記憶されます（localStorage）
4. 次回アクセス時は同じ店舗が自動選択されます

---

## 6.2 店内注文の流れ

### ステップ1：テーブルを選択する

1. 「店内注文」タブが選択されていることを確認します
2. 画面上部の**テーブル選択ドロップダウン**からテーブルを選びます
   - 表示形式：「A-1 (4名)」（テーブルコード + 座席数）
   - テーブルは「テーブル管理」で登録されたものが表示されます

### ステップ2：カテゴリを選択する

テーブル選択の下に、メニューカテゴリのタブが横スクロールで表示されます。

- タップするとそのカテゴリのメニューが表示されます
- 選択中のカテゴリは青色（#536dfe）でハイライトされます

### ステップ3：メニューを選択する

カテゴリタブの下に、メニュー品目がグリッド形式で表示されます。

**レスポンシブレイアウト：**
- スマートフォン（600px未満）：2列
- タブレット（600〜900px）：3列
- PC（900px以上）：4列

**メニューカードの表示内容：**

| 要素 | 説明 |
|------|------|
| 画像 | メニュー画像（高さ80px） |
| 品名 | メニュー名。オプションがある場合はアイコン付き |
| 価格 | 緑色テキスト（#4caf50）で表示 |
| 限定バッジ | 限定メニューの場合、ピンク色の「limited」バッジ |
| カート数量 | カートに入っている場合、右上に数量が丸で表示 |
| 品切れ | 品切れの場合、半透明（opacity 0.4）になり「品切れ」オーバーレイ表示 |

### ステップ4：オプションを選択する（該当品目のみ）

オプショングループが設定されているメニューをタップすると、オプション選択モーダルが開きます。

**モーダルの構成：**

1. **ヘッダー：** メニュー名 + 閉じるボタン（×）
2. **本体：** オプショングループごとにセクションが表示されます

| 要素 | 説明 |
|------|------|
| グループ名 | オプショングループ名 |
| 必須バッジ | 必須グループには赤色「必須」バッジ |
| 任意バッジ | 任意グループには灰色「任意」バッジ |
| 選択肢 | ラジオボタン形式。選択肢名 + 追加料金（+¥200 等） |

3. **フッター：** 合計金額表示 + 「カートに追加」ボタン

**バリデーション：**
- 必須グループで何も選択していない場合、「カートに追加」ボタンは無効化（グレーアウト）されます
- 価格は基本価格 + 選択した全オプションの追加料金で自動計算されます

**オプション選択のない品目の場合：**
タップするだけでカートに追加されます（数量1）。既にカートにある場合は数量が1増えます。

### ステップ5：カートを確認する

画面下部に固定表示される**カートバー**でカートの内容を確認できます。

**カートバー（折りたたみ時）：**
- 「カート (Nアイテム)」バッジ
- 合計金額（「¥XXXX」）
- 「注文送信」ボタン

**カートバー（展開時）：** カートバーのサマリー部分をタップすると展開されます。

| 要素 | 説明 |
|------|------|
| 品目一覧 | 品名（省略表示）、価格、オプション（オレンジ色小文字） |
| 数量操作 | 各品目に [-] [数量] [+] ボタン |
| メモ欄 | テキスト入力（最大200文字、プレースホルダー「ネギ抜き等」） |
| カートクリア | カートを全て空にするボタン |
| 合計金額 | — |
| 注文送信ボタン | — |

**数量の操作：**
- [+] で数量を1増やす
- [-] で数量を1減らす
- 数量が1の状態で [-] を押すと、その品目がカートから削除されます

**同一品目・異なるオプションの扱い：**
同じメニューでもオプションの組み合わせが異なれば、別の品目としてカートに追加されます。例えば「ラーメン（にんにく多め）」と「ラーメン（にんにくなし）」は別のカート行になります。

### ステップ6：注文を送信する

1. カートの内容を確認します
2. 「注文送信」ボタンをタップします
3. 成功すると「注文を送信しました (¥XXXX)」トーストが表示されます
4. カートが自動的にクリアされます
5. メニューの表示が更新されます

**送信前のバリデーション：**
- カートが空の場合：送信できません（「カートが空です」エラー）
- テーブルが未選択の場合：「テーブルを選択してください」エラー

**送信されるデータ：**

| フィールド | 値 |
|-----------|-----|
| store_id | 現在の店舗ID |
| table_id | 選択中のテーブルID |
| items | 品目リスト（品目ID、品名、価格、数量、オプション） |
| order_type | "handy" |
| memo | メモ欄の内容（空でもOK） |

**注文送信後：**
- 注文はKDS（キッチンディスプレイ）にリアルタイムで表示されます
- 注文ステータスは「pending」（受付）で作成されます
- スタッフIDが注文に記録されます（誰が注文を入力したか追跡可能）

## 6.3 テイクアウト注文の流れ

### 入力フィールド

「テイクアウト」タブを選択すると、テーブル選択の代わりに以下のフィールドが表示されます。

| フィールド | 種類 | 必須 | 説明 |
|-----------|------|------|------|
| お客さん名 | テキスト | ✅ | 受取人の名前 |
| 電話番号 | 電話番号 | ✅ | 連絡先電話番号 |
| 受取時間 | 日時選択 | ✅ | 受取日時（初期値は現在時刻） |

### 注文の流れ

メニュー選択・オプション選択・カート操作は店内注文と同じです。

**送信されるデータ（店内注文との違い）：**

| フィールド | 値 |
|-----------|-----|
| store_id | 現在の店舗ID |
| items | 品目リスト |
| order_type | "takeout" |
| customer_name | お客さん名 |
| customer_phone | 電話番号 |
| pickup_at | 受取日時（MySQL形式：YYYY-MM-DD HH:mm:ss） |
| memo | メモ欄の内容 |

**テイクアウト注文のバリデーション：**
- お客さん名が空：送信できません
- 電話番号が空：送信できません
- 受取時間が未設定：「受取時間を入力してください」エラー

---

## 6.4 テーブル状況タブ

「テーブル状況」タブでは、全テーブルの現在の状態をカード形式で確認できます。フロアマップと同様のデータを使用しますが、ハンディPOS画面に最適化されたレイアウトです。

### テーブルカードの表示

| ステータス | バッジ | 色 |
|-----------|--------|-----|
| 空席 | 空席 | グレー |
| 未注文 | 未注文 | 黄色 |
| 食事中 | 食事中 | 緑色 |
| 会計待ち | 会計待ち | 赤色 |
| 超過中 | 超過中 | 赤色（時間超過） |

**カードの表示内容：**
- テーブルコード
- ステータスバッジ
- ゲスト数（セッションがある場合）
- 経過時間（セッションがある場合）
- 残り時間（食べ放題プランがある場合）
- プラン/コース情報
- 注文金額合計
- 調理進捗バー（品目のステータス割合）
- メモ（設定されている場合は黄色ボックス）

### 着席操作

空きテーブルの「着席」ボタンをタップすると、着席モーダルが開きます。

**モーダルのフィールド：**

| フィールド | 種類 | 必須 | 初期値 | 説明 |
|-----------|------|------|--------|------|
| 人数 | 数値入力 | — | 2 | 最小値1 |
| 食べ放題プラン | ドロップダウン | — | プランなし（通常） | 登録済みプラン一覧。表示形式：「プラン名 (60分 ¥2,980)」 |
| コース | ドロップダウン | — | コースなし（通常） | 登録済みコース一覧。表示形式：「コース名 (¥5,000)」 |
| メモ | テキストエリア（2行） | — | — | プレースホルダー「アレルギー・VIP等」 |

**注意：** 食べ放題プランとコースは排他的です。両方同時には選択できません。

「着席する」ボタンをタップするとセッションが開始されます。

### メモ編集

使用中テーブルの「メモ編集」ボタンをタップすると、メモモーダルが開きます。フロアマップのメモ機能と同じプリセットタグ（アレルギー注意、VIP、お子様連れ、誕生日、接待、車椅子）とテキストエリアが利用できます。

### 着席キャンセル

使用中テーブルの「着席キャンセル」ボタンをタップすると、セッションが強制終了されます。未会計の注文は全てキャンセルされ、セッショントークンが再生成されます。

## 6.5 注文履歴タブ

「注文履歴」タブでは、当日の全注文を一覧で確認・編集できます。

### 注文カードの表示

各注文はカード形式で表示されます。

| 要素 | 説明 |
|------|------|
| 左ボーダー色 | 店内注文：青色（#536dfe）、テイクアウト：オレンジ色（#ff6f00） |
| ヘッダー | 「テーブルA-1」または「TO: 顧客名」 |
| 時刻 | 注文時刻（「14:30」） |
| ステータスバッジ | 色分けされたステータス（下表参照） |
| 品目一覧 | 「2x ハンバーグ (+チーズ)」「1x ポテト」形式 |
| 合計金額 | 緑色で表示（「¥1,960」） |
| 編集ボタン | 「編集」（未会計の注文のみ） |

### 注文ステータスの色分け

| ステータス | アイコン | 背景色 |
|-----------|---------|--------|
| pending（受付） | ⏳ | オレンジ（#e65100） |
| preparing（調理中） | 👨‍🍳 | 黄色（#f9a825） |
| ready（完成） | ✅ | 緑色（#2e7d32） |
| served（提供済） | 🍽️ | グレー（#455a64） |
| cancelled（キャンセル） | ❌ | 赤色（#c62828）、取り消し線 |

### 注文の編集

会計済み（paid）またはキャンセル済み（cancelled）でない注文は編集できます。

1. 注文カードの「編集」ボタンをタップします
2. 画面上部にオレンジ色のバナーが表示されます：「修正中: [テーブルコード or 顧客名]」
3. 注文の品目がカートに読み込まれます
4. 注文タイプに応じたタブ（店内注文/テイクアウト）に自動切替されます
5. カートの内容を変更します（品目の追加・削除・数量変更）
6. 「注文送信」ボタン（オレンジ色に変化）をタップして更新を送信します
7. 成功すると注文が更新されます

**編集のキャンセル：**
バナーの「キャンセル」ボタンをタップすると、編集モードを終了し、カートがクリアされます。

## 6.6 スタッフ呼び出しアラート

ハンディPOS画面では、お客さんからのスタッフ呼び出しアラートがリアルタイムで表示されます。5秒ごとに自動チェックされます。

### アラートの種類

| 種類 | アイコン | 表示内容 | 対応ボタン |
|------|---------|---------|-----------|
| staff_call（スタッフ呼び出し） | 🔔 | 「テーブルA-1 呼び出し：「お水ください」」 | 「対応済み」 |
| product_ready（料理完成通知） | 🍳 | 「テーブルA-1 パスタ」 | 「配膳完了」 |
| kitchen_call（キッチンからの連絡） | 👨‍🍳 | アラート理由（例：「塩辛すぎた」） | 「対応済み」 |

### アラートの表示

- ヘッダーの下に赤色バナーとして表示されます
- 経過時間が表示されます（「5秒前」「2分前」）
- 種類に応じた効果音が鳴ります
  - staff_call：高周波ビープ音
  - product_ready：低〜中周波ビープ音
  - kitchen_call：高〜中〜高の3連ビープ音（×2セット）

### アラートの対応

1. アラートの「対応済み」（または「配膳完了」）ボタンをタップします
2. アラートが消えます（ステータスが「acknowledged」に更新されます）

---

## 6.7 メニューの自動更新

ハンディPOS画面では、メニューの変更を自動的に検知します。

- **3分ごと**にメニューバージョンをチェックします
- 管理ダッシュボードでメニューが変更された場合、次のバージョンチェックでメニューが自動更新されます
- 手動リフレッシュは不要です
- 更新はバックグラウンドで行われ、現在のカートの内容には影響しません

---

## 6.8 品切れ品目の扱い

品切れに設定されたメニューは以下のように表示されます。

- メニューカードが半透明（opacity 0.4）になります
- 「品切れ」テキストがオーバーレイ表示されます
- タップしてもカートに追加できません

**注文送信時の品切れチェック：**
注文送信時にサーバー側で品切れチェックが行われます。カートに入れた後にメニューが品切れになった場合、送信時にエラーが返されます。

- HTTPステータス409
- エラーメッセージ：「品切れの品目が含まれています: [品目名]」

この場合、該当品目をカートから削除して再送信してください。
