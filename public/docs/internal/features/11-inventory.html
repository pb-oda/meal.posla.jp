11. 在庫・レシピ管理 ​

在庫管理機能はproプラン以上で利用できます。管理ダッシュボードの「在庫・レシピ」タブから、食材の在庫管理、レシピ設定、棚卸しを行います。マネージャーのみ操作できます。

11.1 原材料一覧 ​

「在庫・レシピ」タブ → 「原材料一覧」サブタブで、食材（原材料）の登録・管理を行います。

原材料の登録 ​

  1. 「原材料追加」ボタンをクリックします
  2. モーダルが開きます
  3. 以下のフィールドに入力します
フィールド種類必須説明
原材料名テキスト✅食材の名前（例：「鶏もも肉」「トマト」）
単位セレクト✅g / ml / 個 / 枚 から選択
在庫数量数値—現在の在庫量
単価数値—1単位あたりの原価（円）
低在庫閾値数値—この数量を下回ると低在庫警告が表示されます
  1. 「保存」ボタンをクリックします

原材料一覧の表示 ​

各原材料には以下が表示されます。

表示要素説明
原材料名—
単位g、ml、個、枚
在庫数量現在の在庫量。低在庫時はハイライト表示
単価1単位あたりの原価
低在庫閾値—
レシピ使用数この原材料を使用しているレシピの数
ステータス有効/無効

在庫レベルの色分け：

  • 通常：標準表示
  • 低在庫（閾値以下）：黄色ハイライト
  • 在庫切れ（0以下）：赤色ハイライト

原材料の編集・削除 ​

  • 編集： 全フィールドの変更、有効/無効の切替
  • 削除： 論理削除（データは保持されます）

CSV入出力 ​

  • エクスポート： 原材料一覧をCSVファイルとしてダウンロード
  • インポート： CSVファイルから一括で原材料を登録・更新

11.2 レシピ設定 ​

「在庫・レシピ」タブ → 「レシピ設定」サブタブで、メニューと原材料の紐づけ（レシピ）を管理します。

レシピとは ​

レシピは「メニュー1品あたりに使用する原材料と数量」の定義です。

例：「チキンカレー」のレシピ

  • 鶏もも肉：200g
  • 玉ねぎ：100g
  • カレールー：50g
  • ごはん：250g

レシピの設定 ​

  1. メニュー品目を選択します
  2. 「レシピ編集」ボタンをクリックします
  3. 原材料を追加します
フィールド説明
原材料登録済み原材料から選択（未使用・有効なもののみ表示）
使用量1品あたりの使用量
単位原材料の単位が自動表示
現在の在庫原材料の現在の在庫量を参考表示
  1. 必要な原材料を全て追加したら「保存」ボタンをクリックします

レシピのCSV入出力 ​

  • エクスポート： レシピデータをCSVファイルとしてダウンロード
  • インポート： CSVファイルから一括でレシピを登録・更新

11.3 棚卸し ​

棚卸し機能で、実際の在庫数量を確認・修正します。

棚卸しの実行 ​

  1. 「棚卸しモード」をONにします
  2. 各原材料の在庫数量が編集可能になります
  3. 実際の在庫を数えて数量を入力します
  4. 「一括保存」ボタンをクリックします
  5. 全ての変更がトランザクションで一括保存されます
  6. 更新された件数が表示されます

定期的な棚卸し

在庫の正確性を保つため、週1回または月1回の棚卸しを推奨します。棚卸し結果は在庫管理の基礎データとなり、AI需要予測の精度にも影響します。

11.4 AI需要予測 ​

proプラン以上で在庫管理が有効な場合、AIによる需要予測機能が利用できます。

予測の仕組み ​

  1. 過去7日間の売上データを収集します
  2. 品目別の販売数量、曜日別パターンを分析します
  3. レシピ情報と組み合わせて、必要な原材料量を予測します

AIからの出力 ​

セクション内容
日別需要予測今後の日ごとの品目別販売予測
原材料発注推奨不足が見込まれる原材料と推奨発注量
低在庫アラート閾値を下回る見込みの原材料
ポーション調整使い切りのためのポーション調整提案
