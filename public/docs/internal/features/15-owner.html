15.4 店舗管理 ​

テナント内の店舗を管理します。

店舗の追加 ​

フィールド種類必須説明
店舗名テキスト✅日本語名
店舗名（英語）テキスト—英語名
スラッグテキスト✅URL用の識別子。英小文字・数字・ハイフンのみ

店舗の編集 ​

  • 店舗名の変更
  • 有効/無効の切替（無効にすると店舗の全機能が停止します）

店舗の削除 ​

注意

店舗を削除すると、その店舗に紐づく全てのデータ（メニュー、注文、シフト等）が影響を受けます。通常は削除ではなく無効化を推奨します。

15.5 ユーザー管理 ​

テナント内の全ユーザーアカウントを管理します。

ユーザーの作成 ​

フィールド種類必須説明
表示名テキスト✅スタッフの表示名
ユーザー名テキスト✅ログインID
メールテキスト—任意
パスワードパスワード✅初期パスワード
ロールセレクト✅owner / manager / staff
所属店舗チェックボックス✅複数店舗を選択可能（ownerは全店舗に暗黙アクセス）

スタッフ別の表示ツール設定 ​

各スタッフに対して、利用可能な画面（ツール）を店舗ごとに設定できます。

ツール説明
handyハンディPOS画面
kdsKDS画面
registerPOSレジ画面

スタッフ別の時給設定 ​

スタッフごと・店舗ごとに個別の時給を設定できます。未設定の場合はシフト設定のデフォルト時給が使用されます。

パスワード変更 ​

オーナーは全ユーザーのパスワードを変更できます。

  1. ユーザー一覧で対象ユーザーの「編集」ボタンをクリック
  2. 「パスワード」欄に新しいパスワードを入力
  3. 「保存」をクリック
  4. 新しいパスワードをスタッフに口頭で伝える

15.6 決済設定 ​

Stripeで決済を受け付けるための設定を行います（proプラン以上）。2つの方式のいずれかを利用できます。詳細は第21章「Stripe決済の全設定」を参照してください。

設定方式 ​

方式説明
パターンA：自前Stripeキーテナントが自分のStripeアカウントを持っている場合。tenants.stripe_secret_key に Secret Key を保存して直接決済。application_fee なし
パターンB：POSLA経由Connectテナントが自分のStripeアカウントを持っていない場合。tenants.stripe_connect_account_id を保存。POSLAが決済を代行（application_fee 1.0%）

優先順位 ​

両方設定されている場合はパターンB（Connect）が優先される。/api/connect/status.php が terminal_pattern: 'A' | 'B' | null を返し、レジアプリの Stripe Terminal 初期化経路が決まる。

設定内容 ​

  • 使用する決済サービス（none / stripe）の切替
  • パターンA：Stripe Secret Key の入力・更新・削除（tenants.stripe_secret_key）
  • パターンB：Stripe Connectへの接続・状態確認・解除
  • 接続中はアカウントID・charges_enabled・payouts_enabled・details_submitted を表示

Stripe Connectの解除（P1-1c） ​

パターンBからパターンAに切り替えたい場合は、「Stripe Connectを解除」ボタンで解除できる。

  • エンドポイント: POST /api/connect/disconnect.php
  • 処理: stripe_connect_account_id = NULL, connect_onboarding_complete = 0, charges_enabled = 0, payouts_enabled = 0 にリセット
  • Stripe API は呼ばない。Stripe ダッシュボード側のアカウントは残るため再接続可能
  • audit_log に記録
