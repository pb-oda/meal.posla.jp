Part 2 — スタッフ管理 ​

このページでは、スタッフのアカウント作成・シフト管理・勤怠打刻を、指の動きを 1 つずつ 順番に説明します。

POSLA の鉄則: 1 人 1 アカウント

絶対に「キッチン」「ホール」のような共有アカウントを作らないでください。以下が全部壊れます。

  • 勤怠打刻（誰が出勤したか分からない）
  • 監査ログ（誰が操作したか記録されない）
  • スタッフ評価レポート（全員のデータが混ざる）
  • シフト管理（個別割当不可）

必ず スタッフ 1 人につき 1 つのアカウント を作成してください。

個人スマホは業務に使わない

POSLA の運用前提は、ハンディ・KDS・レジは 全て店舗が用意する備品端末 です。スタッフ個人のスマートフォンを業務に使う BYOD 運用は推奨しません（労務・セキュリティ・紛失時補償の問題を避けるため）。ハンディ端末は店舗が複数台購入して device アカウントでログインしっぱなしで運用するのが標準です。

2-0. 作成できるアカウントの種類（4 種類） ​

ロール役割作成可能者
オーナー (owner)代表者・全店舗統括通常は POSLA 運営が契約時に発行。既存 owner から追加作成も可
マネージャー (manager)店長・店舗運営オーナーのみ作成可
スタッフ (staff)アルバイト・社員オーナーまたはマネージャーが作成可
デバイス (device)端末専用アカウント（KDS / レジ / ハンディ）オーナーまたはマネージャーが作成可

2-1. 新しいスタッフのアカウントを作る ​

2-1-1. 手順（マネージャーの場合） ​

  1. 管理画面にマネージャーでログイン
  2. 上部タブ 【店舗設定】 → サブタブ 【スタッフ管理】
  3. 画面右上の 【+ 追加】 ボタン
  4. 「スタッフ追加」モーダルが開く
  5. 以下を入力:
項目必須説明
表示名—「田中 太郎」等（後でも変更可）
ユーザー名✅半角英数・ハイフン・アンダースコア 3〜50 文字（例: tanaka01）
メール—任意（通知送信用）
パスワード✅8 文字以上 + 英字 + 数字必須
時給—人件費レポート用（円 / 時、1〜10000）
ビジブルツール—通常は空欄（staff には不要）
  1. 【保存】 ボタン
  2. スタッフに ユーザー名 + パスワード を口頭またはメモで伝える
  3. スタッフは初回ログイン後、自分で好きなパスワードに変更（推奨）

2-1-2. 手順（オーナーの場合） ​

オーナーはオーナー管理画面からマネージャー・スタッフを作成できます。

  1. オーナーでオーナー管理画面にログイン
  2. 上部タブ 【ユーザー管理】
  3. 【+ ユーザー追加】 ボタン
  4. モーダル:
    • 表示名
    • ユーザー名
    • メール（任意）
    • パスワード
    • ロール: manager または staff のどちらか選択
    • 担当店舗: 複数チェックボックス（所属店舗）
  5. 【保存】

2-1-3. ユーザー名の命名ルール ​

  • 半角英数字 + ハイフン + アンダースコア のみ
  • 3〜50 文字
  • POSLA 全体で一意。同じ店舗内だけでなく、他店舗・他テナントと重複する場合も登録できない

ユーザー名はPOSLA全体で一意です

他社・他店舗で同じユーザー名が使われている場合も登録できません。店舗名や会社名の略号を付けると重複しにくくなります（例: matsunoya_tanaka01, momonoya_suzuki-hall）。

推奨パターン:

  • <テナント略号>_tanaka01, <テナント略号>_suzuki-hall（人間の場合、名前 + 番号 or 役割）
  • kds-shibuya-01, register-ginza-01, handy-shinjuku-02（device の場合、役割 + 店舗 + 番号）

2-1-4. 失敗時のエラー ​

メッセージ原因対処
「ユーザー名は既に使用されています」同名アカウントあり別のユーザー名（例: tanaka01 → tanaka02）
「パスワードは8文字以上で入力してください」短い8 文字以上にする
「パスワードには英字を1文字以上含めてください」数字のみ英字を混ぜる
「パスワードには数字を1文字以上含めてください」英字のみ数字を混ぜる
「ユーザー名は半角英数字・ハイフン・アンダースコア（3〜50文字）で入力してください」形式違反正しい形式に
「メールアドレスの形式が正しくありません」メール入力ミス正しいメール形式に（任意なので空欄 OK）

2-1-b. スタッフのレジ担当 PIN を設定する ​

POSレジで「誰が操作したか」を特定するため、各スタッフに 4〜8 桁の PIN を設定します。

マネージャーも全員 PIN 設定が必須

マネージャー / オーナーであっても自分の PIN 設定が必須になりました。レジ金庫操作（レジ開け / レジ閉め / 入金 / 出金）にも PIN が要求されるため、PIN を設定していないマネージャーは開店作業ができません。

「会計用だけ」と思って未設定のままにしていたマネージャーは、今すぐ自分の PIN を設定してください。

PIN が必要な操作の一覧 ​

操作PIN 必須
会計確定（現金 / カード / QR / Terminal）✅
返金処理（部分・全額）✅
レジ開け✅
レジ閉め✅
入金（Cash In）✅
出金（Cash Out）✅

設定手順（店長向け） ​

  1. 管理画面 → 【店舗設定】 → 【スタッフ管理】
  2. 対象スタッフの 【編集】 ボタン
  3. 編集モーダル内の 「レジ担当 PIN」 欄に 4〜8 桁の数字を入力
  4. 【保存】 ボタン
  5. スタッフに PIN を口頭で伝える
  6. スタッフは初回ログイン時に自分で別の PIN に変更推奨

スタッフ自身による PIN 変更 ​

  1. スタッフ（マネージャー含む）が管理画面にログイン
  2. 同じ手順で自分の 【編集】 ボタン
  3. 「レジ担当 PIN」欄に新しい PIN を入力 → 【保存】

注意事項 ​

  • ✅ 退勤時は必ず退勤打刻: 退勤したスタッフの PIN は 自動無効化 される（不正悪用防止）
  • ✅ マネージャーも開店前に出勤打刻が必須（出勤打刻していないと自分の PIN でレジ開けできない）
  • ✅ 弱い PIN は登録不可（1234、0000、ゾロ目などはエラー）
  • ✅ PIN は他人に教えない、付箋でレジに貼らない
  • ✅ 半年〜1 年で変更推奨

詳細は 2 章 2.6 担当スタッフ PIN / 8 章 8.6 レジ開閉操作 参照。

2-2. スタッフのパスワードをリセットする ​

スタッフが「パスワードを忘れた」と言ってきた時。

手順 ​

  1. 管理画面 → 【店舗設定】 → 【スタッフ管理】
  2. 該当スタッフの行を探す
  3. 【編集】 ボタン
  4. 編集モーダルが開く
  5. 【新しいパスワード】 欄に新しいパスワード（8 文字以上 + 英字 + 数字）を入力
  6. 【保存】
  7. スタッフに新パスワードを口頭で伝える
  8. スタッフは次回ログイン後、自分で変更してもらう

新パスワードを平文でメモ・メール送信しない

セキュリティリスクがあるので、口頭で伝えるのが基本。メモでも必ず破棄。メール送信時は暗号化メッセージで。

2-3. スタッフを削除する（退職時） ​

推奨: 削除ではなく 無効化 ​

監査ログの履歴を維持するため、アカウントを物理削除せず、無効化する運用 を推奨します。

2-3-1. 無効化の手順 ​

  1. 【スタッフ管理】 で該当スタッフの 【編集】
  2. 「有効/無効」トグル を 【無効】 にする
  3. 【保存】
  4. 以降、このスタッフはログインできなくなる
  5. 過去の注文・勤怠・操作履歴はすべて保持される

2-3-2. 完全削除（非推奨） ​

どうしても完全削除したい場合:

  1. 編集画面で 【削除】 ボタン（赤）
  2. 確認ダイアログで 【OK】
  3. アカウントが削除される

注意

完全削除すると、過去の注文・勤怠・操作履歴とのつながりが確認しづらくなる場合があります。税務監査・労務管理の観点から 無効化を推奨 します。

2-4. デバイスアカウント（KDS / レジ / ハンディ端末専用）の作り方 ​

「人」ではなく「端末」のためのアカウントです。

2-4-1. デバイスアカウントの特徴 ​

  • ログインしっぱなし前提（端末再起動時のみ再ログイン）
  • シフト・人件費・スタッフ評価レポートの対象外（人間ではないため）
  • 監査ログには端末用アカウントとして記録される
  • 管理画面には入れない（KDS / レジ / ハンディのいずれかに直接遷移）

2-4-2. 作成手順 ​

  1. 管理画面 → 【店舗設定】 → 【スタッフ管理】
  2. 画面を下にスクロール → 「デバイスアカウント」セクション
  3. 【+ デバイス追加】 ボタン
  4. モーダル:
項目必須説明
表示名✅「厨房KDS 1番」「レジ-渋谷-01」等の人間向けの名前
ユーザー名✅kds-shibuya-01 等（英数字・ハイフン、3〜50 文字）
パスワード✅8 文字以上 + 英字 + 数字
用途✅チェックボックス: KDS / POSレジ / ハンディ から 少なくとも 1 つ
  1. 【保存】
  2. 一覧に追加される

2-4-3. 用途の組み合わせによる遷移先 ​

デバイスアカウントがログインした後の遷移先は、作成時に選んだ用途で決まります。

用途ログイン後の画面
ハンディハンディ画面
POSレジPOSレジ画面
KDSKDS画面
複数用途を選んだ場合ハンディ、POSレジ、KDSの順で優先表示

1 端末 1 用途を強く推奨

複数選択は技術的に可能ですが、運用上は 1 端末 1 用途（KDS のみ / POSレジ のみ / ハンディのみ）にしてください。KDS 画面表示中に会計が来て切替えたらピックアップ画面が消える、といった事故が起きます。

2-4-4. ワンタイムトークンで端末を自動セットアップする ​

新しい KDS / レジ / ハンディタブレットを設置する時、手動でユーザー名・パスワードを入力する代わりに ワンタイムトークンで自動ログインできます。

手順 ​

  1. 管理画面 → 【スタッフ管理】 → デバイスアカウントセクション
  2. 【トークン発行】 ボタン
  3. ダイアログで端末の「表示名」と「用途」を入力
  4. セットアップ用URLが表示される
  5. URL を新しい端末で開く（手動コピー、または QR にして読み取り）
  6. POSLAが端末用アカウントを自動作成する
  7. 用途に合わせて、KDS / レジ / ハンディの画面へ自動で移動する

注意点 ​

  • セットアップURLは1回だけ使えます
  • 期限が切れた場合は、もう一度【トークン発行】を押してください
  • URLを店外や関係者以外に共有しないでください

2-4-5. スタッフアカウントとの違い ​

項目スタッフアカウントデバイスアカウント
想定用途人間 1 人端末 1 台
シフト管理対象✅❌
人件費計算対象✅❌
スタッフ評価レポート✅❌
GPS 打刻必須店舗設定による常に必須（セキュリティ）
管理画面アクセス限定（予約・シフト 2 タブのみ）✗ 不可

2-5. シフトの作成と公開 ​

シフトの作成は 「シフト管理」タブ から行います（manager 以上）。

2-5-1. シフトを作る手順 ​

  1. 管理画面 → 上部タブ 【シフト管理】
  2. サブタブ 【週間カレンダー】
  3. 画面上部の日付切替ボタンで該当週を表示
  4. スタッフ名 × 日付のグリッドが表示される
  5. セル（スタッフ × 日）をタップ
  6. シフト入力モーダル:
項目説明
開始時刻例: 10:00
終了時刻例: 18:00
ポジションホール / キッチン / レジ / 清掃 等（カスタマイズ可）
メモ任意（例: 「研修日」）
  1. 【保存】
  2. セルに色付きでシフトが表示される

2-5-2. 全員分入力できたら公開 ​

  1. 画面上部の 【週間公開】 ボタン
  2. 確認ダイアログ → 【OK】
  3. スタッフ全員に通知される（「マイシフト」で閲覧可能）
  4. 以降、スタッフは自分のシフトを確認できる

2-5-3. AI シフト提案を使う ​

過去の売上データとスタッフの希望から、最適なシフトを AI が自動生成する機能。

  1. カレンダー画面上部の 【🤖 AI 提案】 ボタン
  2. AI が数秒で全スタッフのシフト案を生成
  3. 内容を画面で確認
  4. 【適用】 ボタンで一括反映（既存の手動シフトは上書きされない）
  5. 確定したら 【週間公開】 で公開

2-6. 出退勤の打刻方法と確認 ​

2-6-1. スタッフの打刻 ​

スタッフ側の操作。

  1. 管理画面（/admin/）に staff アカウントでログイン
  2. デフォルト表示される「スタッフ画面」の 【🟢 出勤】 ボタンをタップ
  3. GPS 許可ダイアログ が出たら 【許可】
  4. 店舗から半径 200m 以内（デフォルト）にいるか判定
  5. OK なら 「出勤しました 10:05」 とメッセージ
  6. 退勤時は 【退勤】 ボタン（赤）をタップ

2-6-2. 店長の勤怠確認 ​

  1. 管理画面 → 【シフト管理】 → サブタブ 【勤怠一覧】
  2. 日別にスタッフの打刻記録が一覧表示
  3. 予定（シフト）vs 実績（打刻）の比較
  4. 遅刻・早退・残業が自動計算
  5. 打刻漏れがある場合は 【編集】 で手動修正

2-7. GPS 打刻の設定 ​

2-7-1. 店舗の GPS 設定 ​

  1. 管理画面 → 【シフト管理】 → 【シフト設定】 サブタブ
  2. 以下を入力:
項目説明
店舗緯度Google Maps で店舗の住所を検索 → 右クリック → 座標をコピー（例: 35.6585）
店舗経度同上（例: 139.7454）
GPS 許容半径デフォルト 200m
GPS 打刻必須トグル ON/OFF
  1. 【保存】

2-7-2. 店舗座標の調べ方（Google Maps） ​

  1. Google Maps を開く
  2. 店舗の住所を検索
  3. 地図上の店舗アイコンを右クリック
  4. 「座標をコピー」 または表示された緯度経度の数字をメモ
  5. 例: 35.6585, 139.7454 → 緯度 35.6585 / 経度 139.7454

2-7-3. device ロールは常に GPS 必須 ​

スタッフは店舗設定により GPS 打刻の必須/任意が変わりますが、デバイスアカウントは常に GPS 確認が必要です。店舗備品の端末が店外に持ち出された場合のリスクを下げるためです。

2-7-4. GPS がずれる場合 ​

  • 室内だと GPS 精度が落ちる（5〜50m ずれることも）
  • ビル内の店舗は特に誤差が大きい
  • 対処: 許容半径を 300〜500m に広げる
  • それでもダメなら GPS 必須を OFF にして、代わりに監査ログで出退勤を確認

2-8. AI シフト提案の詳細 ​

2-8-1. 仕組み ​

  • 過去 30〜90 日の売上データ分析
  • 各時間帯の必要人員を推定
  • スタッフごとの希望シフト（シフト希望提出機能）との合致
  • 労基法遵守（週40時間以下、1日8時間以下）

2-8-2. 使い方 ​

  1. シフト希望を集める（スタッフが 【希望提出】 サブタブで送信）
  2. 【週間カレンダー】で 【🤖 AI 提案】 ボタン
  3. 提案内容を確認
  4. 修正が必要なら手動調整
  5. 【週間公開】 で確定

2-8-3. 注意点 ​

  • AI 提案は「あくまで案」。最終判断はマネージャー
  • 新人スタッフの研修シフト等、AI には見えない要素は手動調整
  • 既存の手動シフトは上書きされない

2-9. 複数店舗のシフトヘルプ ​

owner / manager のみが使える機能。他店舗から人員を借りる仕組み。

2-9-1. ヘルプ要請を送る ​

  1. 管理画面 → 【シフト管理】 → 【ヘルプ】 サブタブ
  2. 【+ ヘルプ要請】 ボタン
  3. 入力:
    • 送信先店舗（同テナント内の他店舗から選択）
    • 希望日時
    • ポジション（ホール / キッチン等）
    • 必要人数
    • 理由・メモ
  4. 【送信】
  5. 相手店舗のマネージャーに通知される

2-9-2. ヘルプ要請を受ける / 承認する ​

  1. 相手店舗のマネージャーが【ヘルプ】サブタブで要請を確認
  2. 承認できる場合 → スタッフを選択 → 【承認】
  3. 選ばれたスタッフは要請元の店舗で勤務できるようになる（その日だけ）

2-9-3. スタッフが他店舗で働く ​

  1. スタッフ本人は管理画面で自分のシフトを確認する
  2. 派遣先店舗のハンディ、KDS、POSレジは、派遣先店舗が用意したデバイスアカウント端末を使う
  3. 会計など人の責任を記録する操作は、スタッフ本人の担当スタッフ PIN を入力する
  4. その日は派遣先店舗のシフトで稼働

2-10. シフト希望の提出（スタッフ側） ​

スタッフが自分の希望シフトを提出する機能。

手順 ​

  1. スタッフが管理画面にログイン
  2. 【シフト管理】 → サブタブ 【希望提出】
  3. カレンダー表示
  4. 希望日をタップ → 時間帯入力（例: 10:00〜15:00 希望）
  5. 【保存】
  6. マネージャーがシフト作成時に参考にする

2-13. よくある質問 ​

Q1. 既存のスタッフアカウントを KDS タブレットでログインしっぱなしにしてはダメ？

A1. 強く非推奨。シフト・勤怠・スタッフ評価レポートが全部そのスタッフ名義になってしまい、データが壊れます。必ず device アカウントを作って運用してください。

Q2. device アカウントは何個まで作れる？

A2. 上限なし。店舗の KDS 台数 + レジ台数 + ハンディ台数分作ってください。例: 3 店舗 × 各 KDS 2 台・レジ 1 台・ハンディ 3 台 = 18 個。

Q3. device アカウントはシフト管理に影響しない？

A3. 影響しません。デバイスアカウントはシフト・勤怠・人件費・スタッフ評価の対象外です。

Q4. 退職したスタッフのアカウント、削除すべき？

A4. 無効化推奨（トグル OFF）。過去の勤怠・注文履歴を守るため。完全削除は非推奨。

Q5. GPS 打刻の精度が悪い

A5. 室内では GPS が 5〜50m ずれることがあります。許容半径を 300〜500m に広げるか、GPS 必須を OFF にして監査ログで確認してください。

Q6. スタッフが勤怠打刻を忘れた

A6. マネージャーが【勤怠一覧】の該当日を 【編集】 で手動追加できます。ただし監査ログには「編集されたシフト」と記録されます。

Q7. AI シフト提案の質が悪い

A7. 過去データの蓄積が少ないと AI 精度が低くなります。最低 30 日分のデータが溜まってから使うのが現実的です。
