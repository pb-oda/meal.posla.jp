Part 0 — POSLA を使う前の準備 ​

このページでは、POSLA をまだ一度も触ったことがない方が、お店で使えるようになるまで の流れを、ひとつずつ順番に説明します。

このページの読み方

  • 上から順番に読んでください。 途中を飛ばすと後で分からなくなります。
  • 「クリック」「タップ」「押す」は全部同じ意味です。 タッチパネル（タブレット・スマホ）では指で触る、パソコンではマウスで押す動作のことです。
  • 知らない言葉が出てきたら、そのすぐ下に必ず説明があります。分からないまま進まないでください。
  • 画面の写真（スクリーンショット） は各ステップに入れる予定です（準備中）。今は文字で説明しています。
  • 困ったら: 各節の最後にある「うまくいかないとき」を読むか、POSLA 運営（info@posla.jp）に連絡してください。

0-0. そもそも POSLA とは？ ​

POSLA（ポスラ） は、飲食店で使う 「お店の道具セット」 です。具体的には次のようなことができます。

  1. お客様が席に座ってスマホで注文できる（セルフオーダー）
  2. 注文がキッチン（厨房）の画面に自動で流れる（KDS = キッチンディスプレイ）
  3. お会計（レジ打ち）ができる（クレジットカード・現金・QR決済）
  4. スタッフが注文を取るハンディ端末として使える（タブレット POS）
  5. 予約の受付・管理（お客様が自分で予約を取れる）
  6. シフト管理・勤怠打刻（スタッフの出退勤を記録）
  7. 売上レポート・分析（何がどれくらい売れたか）

POSLA の特徴

  • 専用機械を買う必要がない — 手持ちのタブレット・スマホでほぼ全部動きます
  • インターネット（Wi-Fi）につながっていれば使える — クラウドに情報が保存される仕組み
  • 月額料金: 基本 ¥20,000 / 月（1店舗目）、2店舗目以降は ¥17,000 / 月
  • 最初の 30 日間は無料で試せる（クレジットカード登録は必要）

POSLA を使うには、お店に以下のものを用意してください。最初から全部そろえる必要はありません。 まずは最小構成で始めて、営業しながら買い足すのが現実的です。

⚠ 買ってはいけないもの ​

  • 業務専用の超高額 POS 機械（20〜50 万円するもの） — POSLA はタブレット 1 枚で動くので不要
  • プリンタと一体型の POS 機 — POSLA は別々のほうが柔軟

⚠ スタッフ個人のスマホ・タブレットを業務に使わない ​

POSLA では ハンディ端末は必ず店舗が用意してください。スタッフ個人のスマホを業務に使うと以下の問題が起きます。

  • スタッフが退職したときログアウトし忘れる → セキュリティ問題
  • 端末が壊れた・紛失したときの補償責任が曖昧
  • 労務管理で「会社の業務で私物を使わせるのか」問題
  • 個人情報がスタッフのスマホに残る

店舗で中古 iPhone / 中古 iPad mini 3〜5 台を買って、ハンディ専用にしてください。

機材をどこで買うか ​

Apple 製品（iPad / iPhone）:

  • Apple 公式サイト（apple.com/jp）
  • ビックカメラ / ヨドバシカメラ
  • 中古は「iOsys（アイオシス）」「じゃんぱら」「ラクマ」などで

Android タブレット:

  • Amazon.co.jp
  • 家電量販店

Wi-Fi ルーター:

  • 家電量販店（Buffalo、NEC、I-O DATA などのメーカーが代表的）
  • 店舗なら 業務用ルーター（YAMAHA、Cisco など）も選択肢

カードリーダー Stripe Reader S700:

  • Stripe 公式サイト（https://stripe.com/jp/terminal）からのみ購入可
  • 他社のカードリーダーは POSLA では動きません

レシートプリンタ:

  • Star マイクロニクス公式（https://star-m.jp/）
  • エプソン販売（https://www.epson.jp/）
  • Amazon.co.jp でも購入可

うまくいかないとき ​

  • タブレット選びで迷う → iPad（無印）11 インチを買っておけば間違いありません（約 7〜8 万円）
  • 中古で十分か新品が必要か → 中古で OK。バッテリーが弱っていないか確認すること
  • Wi-Fi ルーターの性能 → 「11ax（Wi-Fi 6）対応」「同時接続数 30 台以上」がキーワード

0-2. Wi-Fi（ワイファイ）を準備する ​

Wi-Fi とは ​

Wi-Fi（読み方: ワイファイ）とは、ケーブルを使わずにインターネットに繋ぐ仕組みです。お店の天井や壁に「Wi-Fi ルーター」という箱を置くと、店内のタブレット・スマホが無線でインターネットにつながります。

POSLA には Wi-Fi が必須です

POSLA は インターネットに繋がっていないと動きません。注文情報・メニュー・在庫情報などが全部インターネットの先のコンピューター（サーバー）に保存されているためです。

Wi-Fi が切れている間は:

  • 新しい注文が入力できません
  • レジで会計できません
  • キッチン画面に注文が表示されません

そのため Wi-Fi ルーターは 絶対に止めない 運用が必要です。

Wi-Fi ルーターの設置 ​

  1. 家電量販店で Wi-Fi ルーター を 1 台買ってくる（5,000〜20,000 円）
  2. NTT（フレッツ光）やソフトバンク光などの光回線を契約（月額 5,000〜7,000 円程度）
  3. 工事の人が来て、お店の壁に光回線の終端装置を設置してくれる
  4. Wi-Fi ルーターを終端装置と LAN ケーブル（工事時についてくる）で繋ぐ
  5. Wi-Fi ルーターの電源を入れる
  6. 側面や底面のラベルに書かれた 「SSID」（ネットワーク名）と 「暗号化キー」（パスワード）をメモする

「SSID」「暗号化キー」とは

  • SSID（エスエスアイディー）= Wi-Fi の名前。例: Buffalo-A-1234
  • 暗号化キー（パスワード）= Wi-Fi にログインするためのパスワード。例: a1b2c3d4e5f6g7h8

この 2 つを後でタブレットに入力します。ラベル写真を撮って保存しておく と安心。

タブレットを Wi-Fi に繋ぐ（iPad の場合） ​

  1. iPad のホーム画面にある 【設定】 アイコン（灰色の歯車マーク）をタップ
  2. 左側の一覧から 【Wi-Fi】 をタップ
  3. 画面上の 「Wi-Fi」スイッチ が 緑 になっているか確認。グレーならタップして ON に
  4. 「ネットワーク」の一覧が表示される。自分のお店の SSID（例: Buffalo-A-1234）をタップ
  5. パスワード入力画面が出る。ラベルに書かれた 暗号化キー を入力
  6. 【接続】ボタンを押す
  7. 接続に成功すると、SSID の左に 青いチェックマーク ✓ が表示される
  8. 画面右上に 扇形の Wi-Fi マーク が 3 本線で表示されれば OK

タブレットを Wi-Fi に繋ぐ（Android の場合） ​

  1. 画面上から下にスワイプ（指でなぞる）して クイック設定パネル を開く
  2. Wi-Fi アイコン（扇形のマーク）を長押し
  3. 「ネットワーク設定」画面に切り替わる
  4. Wi-Fi のスイッチを ON にする
  5. SSID 一覧から自分のお店のネットワークをタップ
  6. パスワードを入力して【接続】

Windows パソコンの場合 ​

  1. 画面右下の Wi-Fi アイコン（地球儀 or 扇形）をクリック
  2. 利用可能なネットワーク一覧が出る
  3. 自分のお店の SSID を選んで【接続】
  4. パスワードを入力

うまくいかないとき ​

症状原因対処
SSID が一覧に出てこないルーターの電源が入っていない / 距離が遠いルーターの電源ランプを確認。タブレットをルーターの近くに持っていく
パスワードを入れても弾かれる大文字小文字の入力間違いラベルを目で確認。「I（アイ）と l（エル）」「0（ゼロ）と O（オー）」を混同しがち
接続できたのにネットが繋がらないルーターと光回線の配線不良光回線の終端装置とルーターの間のケーブル接続を確認
店内の奥の席で電波が弱いルーター 1 台で届く範囲を超えているWi-Fi 中継機（メッシュ Wi-Fi）を追加

0-3. ブラウザ（ブラウザーソフト）の開き方 ​

ブラウザとは ​

ブラウザ とは、インターネット上のページを見るためのソフトです。POSLA は ブラウザから 使います。代表的なブラウザ:

  • Google Chrome（グーグル クローム）← POSLA は Chrome 推奨
  • Safari（サファリ）— iPad / iPhone に最初から入っている
  • Edge（エッジ）— Windows パソコンに最初から入っている
  • Firefox（ファイヤーフォックス）

POSLA は Google Chrome で使うのが一番安定します

特にキッチンディスプレイ（KDS）で音声コマンドを使う場合、Chrome でないと動きません。 iPad でも Chrome アプリをインストールして使ってください（App Store から無料でダウンロードできます）。

Chrome のインストール（iPad の場合） ​

  1. iPad のホーム画面で 【App Store】 アイコン（青地に白の A）をタップ
  2. 右下の【検索】をタップ
  3. 検索窓に chrome と入力
  4. 「Google Chrome - Google が提供する Web ブラウザ」が出てきたら【入手】ボタン
  5. Face ID（顔認証）または Apple ID パスワードで認証
  6. インストール完了後、ホーム画面に Chrome のアイコン（赤・黄・緑・青の円）が表示される

Chrome の開き方 ​

  • iPad: ホーム画面の Chrome アイコン（丸いカラフルなマーク）をタップ
  • Android: ホーム画面の Chrome アイコンをタップ（大体最初から入っています）
  • Windows: スタートメニュー → Chrome、またはデスクトップのアイコンをダブルクリック

Chrome が起動すると、画面上部に 「検索または URL を入力」 と書かれた長い四角（アドレスバー）が表示されます。

0-4. ログイン画面を開く ​

URL（ユーアールエル）とは ​

URL とは、インターネット上の住所のことです。お店の郵便番号みたいなものです。POSLA の URL は以下です。

  • 本番環境の URL: https://eat.posla.jp/public/admin/
  • （30 日無料トライアル用の URL は契約時に POSLA からメールで届きます）

URL を入力する ​

  1. Chrome の上部にあるアドレスバーをタップ
  2. 今入っている文字（あれば）を削除
  3. 次の URL を 半角英数字 で正確に入力:
    https://eat.posla.jp/public/admin/
  4. キーボードの Enter（改行キー）または【開く】ボタンを押す
  5. 数秒待つと「管理画面」と書かれたログイン画面が表示される

URL 入力のコツ

  • 「半角」 で入力（日本語入力を切っておく）
  • 「https://」を忘れない（セキュリティのため必要）
  • 「/」 スラッシュは 日本語キーボードの右下 にあります
  • タイプミスすると別のサイトにいってしまうので注意

お気に入り（ブックマーク）に登録する ​

毎回 URL を入力するのは面倒なので、ブックマーク（お気に入り）に登録しておきましょう。

iPad Chrome の場合:

  1. ログイン画面を開いた状態で、画面右上の 【︙】 メニュー（縦三点リーダー）をタップ
  2. 【ブックマーク】→【ブックマークを追加】をタップ
  3. 名前を「POSLA ログイン」などに変更して保存

次回以降:

  1. Chrome を開く
  2. アドレスバーをタップ
  3. 下に出るブックマークから「POSLA ログイン」をタップ

ホーム画面にアイコンを追加する（iPad Safari の場合のみ） ​

Safari を使う場合は、ログイン画面を ホーム画面にアプリのように追加 できます。

  1. Safari で https://eat.posla.jp/public/admin/ を開く
  2. 画面上部の 共有ボタン（□に↑のマーク）をタップ
  3. メニューから【ホーム画面に追加】をタップ
  4. 名前を「POSLA」などにして【追加】
  5. ホーム画面に POSLA アイコンが表示される → 次回からタップで起動

0-5. ログインする（ユーザー名・パスワードの入力） ​

ログイン情報の取得 ​

POSLA を契約すると、オーナー（代表者）のユーザー名とパスワードが POSLA 運営から発行されます。メールで届きます。

  • ユーザー名（例: tanaka-owner）
  • 初期パスワード（例: Posla2026abc）

メールアドレスは必要ありません

POSLA のログインは ユーザー名 + パスワード だけです。メールアドレス入力欄はありません。アルバイトスタッフがメールアドレスを持っていなくても POSLA は使えます。

ログインの手順 ​

  1. https://eat.posla.jp/public/admin/ を開く
  2. 「管理画面」と書かれたログイン画面が表示される
  3. 「ユーザー名」 の入力欄に、発行されたユーザー名を 半角 で入力
  4. 「パスワード」 の入力欄に、発行された初期パスワードを 半角 で入力
  5. 【ログイン】ボタン（青いボタン）をタップ
  6. 成功すると次の画面（オーナーなら「本部管理画面」、マネージャー/スタッフなら「管理ダッシュボード」）に切り替わる

ログインできないときのエラーメッセージと対処法 ​

表示されるメッセージ原因対処
「ユーザー名またはパスワードが正しくありません」入力ミスの可能性が高い大文字小文字、「I」と「l」、「O」と「0」の見間違いを確認
「このアカウントは無効化されています」オーナー/店長がアカウントを停止したオーナーに連絡してアカウントを再有効化してもらう
「このテナントは無効化されています」契約が停止されているPOSLA 運営（info@posla.jp）に連絡
「通信エラーが発生しました」インターネット（Wi-Fi）が切れているWi-Fi が繋がっているか確認。ルーターの電源を確認
「サーバーが空のレスポンスを返しました」POSLA サーバー側の一時的な不具合数分待ってから再度お試しください。続く場合 POSLA 運営に連絡
「ユーザー名とパスワードは必須です」入力欄が空のままログインを押した両方を入力してからログインを押す

パスワードを何度も間違えてもロックされません（2026-04-18 時点）

現状、POSLA にはログイン失敗回数による自動ロック機能はありません。何度でも試せます。

ただし POSLA 運営は監査ログ（誰がいつログインしたか）を記録しているので、不審なアクセスは検知できます。

0-6. パスワードを変更する（初回ログイン後の必須作業） ​

なぜ変更するか ​

初期パスワードは POSLA 運営が発行した仮パスワードなので、自分だけが知っているパスワードに変更してください。これが最低限のセキュリティです。

パスワードの条件（必ず守るルール） ​

POSLA のパスワードは以下の条件をすべて満たす必要があります（実装: api/lib/password-policy.php）。

  • 8 文字以上
  • 英字（A-Z または a-z）を 1 文字以上含む
  • 数字（0-9）を 1 文字以上含む

良いパスワードの例 ​

  • tanaka2026 ✅（10 文字・英字あり・数字あり）
  • Shibuya0428 ✅（11 文字・大文字あり・数字あり）
  • Matsunoya1234 ✅

悪いパスワードの例（禁止） ​

  • password ❌（数字なし）
  • 12345678 ❌（英字なし）
  • abc123 ❌（7 文字しかない）
  • 123 ❌（短すぎ）

やってはいけないパスワードの付け方

  • ユーザー名と同じ にしない
  • 店舗名をそのまま 使わない
  • 誕生日や電話番号 をそのまま使わない（推測されやすい）
  • 複数のスタッフで共有 しない
  • 付箋でレジに貼らない

パスワード変更の手順 ​

  1. ログイン後、画面右上（ユーザー名の横あたり）に 「パスワード変更」 という小さな白いボタンがあります。タップ
  2. 「パスワード変更」というモーダル（小さなウィンドウ）が表示される
  3. 3 つの入力欄がある:
    • 現在のパスワード: 今使っているパスワード
    • 新しいパスワード: これから使うパスワード（上記条件を満たすもの）
    • 新しいパスワード（確認）: 同じパスワードをもう一度入力
  4. 【変更する】ボタンをタップ
  5. 成功すると「パスワードを変更しました」と緑色のメッセージが出る
  6. 他の端末でログインしている場合、それらのセッションは自動でログアウトされます（あなたが今操作中の端末だけは残ります）

変更時のエラー ​

メッセージ原因対処
「パスワードは8文字以上で入力してください」新パスワードが短い8 文字以上にする
「パスワードには英字を1文字以上含めてください」新パスワードに英字がない英字を混ぜる
「パスワードには数字を1文字以上含めてください」新パスワードに数字がない数字を混ぜる
「現在のパスワードが正しくありません」今使っているパスワードが違う初期パスワードをもう一度確認
「新しいパスワードが現在のパスワードと同じです」新旧を同じにしようとした別のパスワードにする

パスワードを忘れたとき ​

自分でパスワードを変更した後、自分でリセットする手段はありません。以下のどれかで対応します。

  • スタッフ（staff）が忘れた: オーナーまたはマネージャーに依頼してリセットしてもらう
  • マネージャーが忘れた: オーナーに依頼
  • オーナーが忘れた: POSLA 運営（info@posla.jp）に連絡

0-7. 画面の見方（タブ・ボタン・アイコンの場所） ​

初めて POSLA にログインすると、情報がたくさんあって戸惑うかもしれません。画面の基本パーツの名前と位置を覚えましょう。

店舗切替（複数店舗を持っている人だけ） ​

お店を 2 つ以上経営している場合、ヘッダーに 「店舗切替」 というリンクが表示されます。クリックすると店舗を選ぶドロップダウンが出てきて、操作対象のお店を変えられます。

「📖 取扱説明書」ボタン ​

ヘッダー右側に 「📖 取扱説明書」 という白いボタンがあります。押すと このマニュアル（あなたが今読んでいるやつ） が新しいタブで開きます。分からなくなったらここから調べてください。

0-8. ロール（権限）の違い ​

POSLA のユーザーには 4 種類のロール（役職のようなもの） があります。ロールごとに できること・見られる画面 が違います。

ロール役職のイメージできること
オーナー（owner）社長・代表者全店舗の管理、ユーザー作成、契約管理、本部レポート、決済設定
マネージャー（manager）店長担当店舗の日常運営、メニュー編集、スタッフ作成、シフト作成、レポート閲覧
スタッフ（staff）アルバイト店員・ホール・キッチンシフト確認・希望提出、勤怠打刻（予約閲覧はハンディ POS で）
デバイス（device）店舗備品端末そのものKDS / POSレジ / ハンディのいずれかの画面に直接ログインしっぱなし

具体例 ​

田中さん（オーナー） の場合:

  • ログインすると「本部管理画面」が開く
  • 3 店舗（渋谷・新宿・池袋）全部のデータが見える
  • 新しいマネージャーを作って、それぞれの店舗の店長にできる

鈴木さん（店長 = マネージャー） の場合:

  • ログインすると「管理ダッシュボード」が開く
  • 自分が担当する渋谷店のデータだけ見える
  • アルバイトの新人アカウント（staff）を自分で作れる
  • 新宿店や池袋店のデータは見えない

山田さん（アルバイトスタッフ） の場合:

  • ログインすると「管理ダッシュボード」が開くが、見えるタブは「予約管理」と「シフト管理」の 2 つだけ
  • シフトを確認したり、希望を出したりできる
  • メニュー編集・売上レポート・決済設定は見えない

KDS-1（キッチンの画面に常時ログインしているデバイスアカウント） の場合:

  • kds-shibuya-01 というアカウント名・パスワードでログイン
  • ログイン後すぐ KDS 画面（カンバン）が開く
  • 管理画面には入れない
  • シフト・勤怠には影響しない（人間ではないため）

各ロールをどうやって使い分けるか ​

  1. 最初は 1 人のオーナーアカウントから始める（POSLA 運営が発行）
  2. 店長を雇ったらマネージャーアカウントを作る（owner-dashboard の「ユーザー管理」タブ）
  3. アルバイトを雇ったら staff アカウントを作る（店長なら dashboard.html の「スタッフ管理」サブタブ）
  4. キッチンタブレットやレジタブレットを設置したら device アカウントを作る（同じく「スタッフ管理」サブタブの「+デバイス追加」）

1 人 1 アカウント（絶対守る）

「キッチン」「ホール」みたいな共有アカウントは絶対に作らないでください。以下の機能が壊れます。

  • 勤怠打刻: 誰が出勤したか分からなくなる
  • 監査ログ: 誰が何をしたか追跡できない
  • スタッフ評価レポート: 全員の成績が混ざる
  • シフト管理: 個別割当ができない

必ず スタッフ 1 人に 1 アカウント を作ってください。

0-9. まずは何から始める？（オーナー向け、1 日目のチェックリスト） ​

POSLA を契約した最初の日にやることリストです。上から順番に どうぞ。

事前準備（契約前） ​

  • [ ] Wi-Fi ルーターをお店に設置し、動作確認
  • [ ] タブレットを少なくとも 1 台購入
  • [ ] クレジットカードを準備（POSLA 月額料金の引き落とし用）

POSLA 契約 ​

  • [ ] https://meal.posla.jp/ からサインアップフォームに入力
  • [ ] Stripe 決済画面でクレジットカード情報を入力（30 日間は無料、31 日目から課金）
  • [ ] 契約完了メールが届くのを待つ（5 分以内）
  • [ ] メール内のユーザー名・初期パスワードをメモ

初回ログイン ​

  • [ ] タブレットの Chrome で https://eat.posla.jp/public/admin/ を開く
  • [ ] オーナーのユーザー名・初期パスワードでログイン
  • [ ] 画面右上の「パスワード変更」から自分だけが知るパスワードに変更

店舗と初期設定 ​

  • [ ] owner-dashboard の「店舗管理」タブで最初の店舗を作成（店舗名・住所・電話番号など）
  • [ ] 「店舗設定」タブで営業時間・税率（通常 10%、テイクアウト 8%）を設定
  • [ ] 「店舗設定」タブでウェルカムメッセージ（セルフメニュー上部に出る文）を入力

メニュー登録 ​

  • [ ] 「メニュー管理」タブ →「カテゴリ管理」で前菜・メイン・ドリンクなどのカテゴリを作成
  • [ ] 「店舗メニュー」または「限定メニュー」で実際のメニューを登録（料理名・価格・写真・カロリー・アレルギー）
  • [ ] CSV 一括登録も可能

テーブル登録 ​

  • [ ] 「テーブル・フロア」タブ →「テーブル管理」でお店のテーブルを 1 つずつ登録（T01, T02, …）
  • [ ] 各テーブルの QR コード印刷 ボタンで紙に印刷してテーブルに設置

スタッフ登録（従業員がいる場合） ​

  • [ ] 「ユーザー管理」タブ（owner-dashboard）で店長のマネージャーアカウントを作成
  • [ ] 店長に dashboard.html へのログイン方法を伝え、パスワード変更を依頼
  • [ ] 店長が各スタッフの staff アカウントを作成

決済設定（カード決済を受け付ける場合） ​

  • [ ] 「決済設定」タブから Stripe 設定（パターン A: 自前 Stripe キー / パターン B: POSLA Connect 経由）を選ぶ
  • [ ] カードリーダー Stripe Reader S700 を購入（約 5 万 2 千円）
  • [ ] カードリーダーを Wi-Fi に繋いで POSLA 管理画面でペアリング

運用開始 ​

  • [ ] 試しに 1 つ注文を入れてみる（自分が客役になってスマホで QR を読む）
  • [ ] KDS 画面で注文が表示されるか確認
  • [ ] レジ画面でテスト会計（現金）を実行
  • [ ] レポートタブで売上が記録されているか確認

ここまで完了したら、営業日のオペレーションは Part 1 以降で詳しく説明します。

0-10. この章のまとめ ​

  • POSLA は インターネットにつながれば動く クラウド型 POS
  • お店に必要なのは Wi-Fi + タブレット + お客様用 QR が最低構成
  • 業務端末は全て店舗備品で用意する（個人スマホは使わない）
  • ログインは ユーザー名 + パスワード（メール不要、8 文字以上・英字数字混在）
  • ロールは オーナー / マネージャー / スタッフ / デバイス の 4 種類
  • 1 人 1 アカウント が鉄則（共有禁止）
  • 初期パスワードは必ず 初回ログインで変更

0-11. よくある質問（Part 0 向け） ​

Q1. タブレットとスマホ、どっちが必要？

A1. タブレット（10 インチ以上）が望ましいです。キッチン画面・レジ画面は情報量が多いので、小さいスマホだと操作しづらいです。ただし、ハンディ POS（スタッフが持ち歩く端末）はスマホサイズでも OK です。

Q2. iPad と Android、どっちが良い？

A2. iPad 推奨。理由は以下:

  • 動作が安定していて壊れにくい
  • バッテリーが持つ
  • 中古市場が大きく、壊れたときの代替入手が楽
  • Apple サポートが受けられる

Android でも動きますが、メーカー・機種によって挙動が違う場合があります。

Q3. Wi-Fi が不安定な店舗でも使える？

A3. 使えません。POSLA は常時インターネット接続が必要です。電波が弱い席がある場合は Wi-Fi 中継機やメッシュ Wi-Fi で全席カバーしてください。

Q4. 最低何台のタブレットが必要？

A4. 最小構成なら 1 台で OK。その 1 台で KDS・POSレジ・ハンディ全部を兼ねて運用できます。ただし営業中に切り替えるのは大変なので、以下を推奨:

  • 1 人店舗: 1 台
  • 小規模（5〜10 席）: 2 台（レジ用 + ハンディ用）
  • 中規模（15 席以上）: 3〜5 台（レジ・KDS・ハンディ 2〜3 台）

Q5. パソコン（PC）は必要？

A5. 必須ではありません。タブレットですべて完結します。ただし、オーナーが 売上レポートを印刷したり、スタッフ評価をじっくり見る にはパソコンのほうが作業しやすいかもしれません。

Q6. インターネット回線はどのくらいの速度が必要？

A6. 光回線（100Mbps 以上） 推奨。家庭用の ADSL やモバイル Wi-Fi でも動きますが、複数端末の同時利用で遅くなることがあります。

Q7. 契約したらすぐ使える？

A7. 契約後 5 分程度 でオーナーアカウントの発行メールが届きます。そこからログインして初期設定を進める流れです。ただしメニュー登録などの初期準備に数時間〜1 日かかります。

Q8. 30 日無料トライアルで何ができる？

A8. 全機能が試せます。機能制限はありません。30 日以内にキャンセルすれば料金はかかりません。Stripe にクレジットカード情報を登録する必要がありますが、期間内キャンセルなら請求は発生しません。

Q9. 途中でプランを変えられる？

A9. POSLA は α-1 以降、単一プラン + アドオン構成 です。選べるのはアドオン（本部一括メニュー配信、+3,000 円/店舗）のオン/オフだけです。基本料金は固定です。

Q10. スマホからでも管理画面を使える？

A10. 使えますが、画面が小さいのでレジ操作・メニュー編集などは PC・タブレットを推奨します。シフト確認・注文履歴確認ぐらいならスマホでも OK です。

0-12. ホーム画面追加でアプリ化する（PWA） ​

POSLA のスタッフ向け業務画面（管理ダッシュボード／KDS／レジ／ハンディ POS）は、iPad / iPhone / Android タブレットのホーム画面に「アプリ」として追加 できます。1 タップで全画面起動できるようになり、毎回 Chrome / Safari でアドレスバーに URL を入れる手間がなくなります。

専用アプリのインストールは不要

App Store / Google Play から POSLA アプリを ダウンロードする必要はありません。Chrome / Safari の標準機能（PWA = Progressive Web App）でホーム画面追加するだけです。

0-12.1 何をホーム画面に追加するか ​

スタッフ用の 3 画面（用途別にアイコン色が違います）を追加できます。

画面URL の例アイコンこんな人が追加
管理ダッシュボードhttps://meal.posla.jp/public/admin/dashboard.html青「管」オーナー・店長・事務スタッフ
KDS（キッチンディスプレイ）https://meal.posla.jp/public/kds/index.html緑「厨」キッチン担当
レジ（POSレジ）https://meal.posla.jp/public/kds/cashier.html緑「厨」レジ担当
ハンディ POShttps://meal.posla.jp/public/handy/index.html橙「配」ホール担当

3 つとも 1 台のタブレットに追加することも可能です。アイコンの色と文字（管／厨／配）で取り違えを防げます。

お客様のスマホには追加しません

お客様用のセルフオーダー画面（テーブルの QR から開くメニュー画面）は 意図的にホーム画面追加できないようにしています。お客様には「ブラウザでそのまま開いてもらう」運用です。理由は 第 1 章 1.G.2 を参照してください。

0-12.2 iPhone / iPad（Safari）の手順 ​

  1. Safari（青地に白の矢印コンパスのアイコン）を開く
  2. アドレスバーに対象 URL を入力（例: https://meal.posla.jp/public/admin/dashboard.html）
  3. ページが表示されたら、画面下部（iPad は上部）の 共有ボタン（□ から ↑ が出ているマーク）をタップ
  4. メニューを下にスクロールし、「ホーム画面に追加」 をタップ
  5. 名前を確認（例: POSLA管理）→ 右上の 「追加」 をタップ
  6. ホーム画面に 青いアイコン「管」（KDS なら緑、ハンディなら橙）が並びます
  7. アイコンをタップすると アドレスバーなしの全画面 で POSLA が起動します

Chrome（iOS 版）からは追加できません

iOS / iPadOS では Safari 限定 の機能です。iPhone の Chrome アプリにはこの機能がありません（Apple のセキュリティ仕様）。必ず Safari で開いてから追加してください。

0-12.3 Android タブレット / スマホ（Chrome）の手順 ​

  1. Chrome（赤・黄・緑・青の円アイコン）を開く
  2. アドレスバーに対象 URL を入力（例: https://meal.posla.jp/public/handy/index.html）
  3. ページが表示されると、画面下部に 「ホーム画面に追加」または「アプリをインストール」 のバナーが自動で出る場合があります → そのままタップ
  4. バナーが出ない場合は、画面右上の 「︙」（縦三点リーダー）→ 「ホーム画面に追加」 または 「アプリをインストール」 をタップ
  5. 名前を確認 → 「追加」 または 「インストール」 をタップ
  6. ホーム画面に アイコン（管／厨／配のいずれか）が並びます
  7. アイコンをタップすると全画面で POSLA が起動します

0-12.4 注意：お客様（来店客）のスマホには追加しないでください ​

接客中に「これホーム画面に追加できますか？」とお客様に聞かれることがあるかもしれません。追加しないように案内してください。理由:

  • セルフオーダー画面の URL には テーブル番号やセッション ID が入っているため、別のテーブルや別の日に開くと正しく動きません
  • お客様のスマホに古いメニュー情報が残ると、次回来店時に 古い価格・古いメニューで注文される事故 が起きます
  • POSLA はお客様セルフオーダー画面はあえて PWA 化（ホーム画面追加機能）を 無効 にしています（追加しても動きません）

お客様には「お帰りの際にブラウザのタブを閉じてください」と一言添えてください。

0-12.5 困ったとき ​

症状原因対処
「ホーム画面に追加」が出てこないURL が違う / Safari 以外で開いている (iOS)必ず Safari で開く。Android は Chrome で開く
古い画面が表示されるキャッシュが残っているアイコンを削除して再追加 すれば直ります（最も確実）
起動するとアドレスバーが出るPWA 化されていない普通のブックマーク登録になっている一度削除して、上記の正しい手順で「ホーム画面に追加」をやり直す
全画面でなく Chrome / Safari の中で開くiOS で「ホーム画面に追加」ではなく「ブックマーク」になっているアイコンを削除し、共有ボタン →「ホーム画面に追加」を選ぶ
アイコンが白い四角になるアイコン画像読み込みに失敗一度削除して再追加（Wi-Fi 安定環境で）

それでも直らないとき

端末の再起動 → 再追加 で大半が解決します。それでもダメな場合は POSLA 運営（info@posla.jp）に連絡してください（OS バージョンとブラウザ名を伝えると早く解決できます）。
