Part 5 — 予約管理 ​

このページでは、お客様からの予約受付・管理を、実際にマウスで押すボタン を 1 つずつ順番に説明します。

この章の対象

  • 飲食店で 電話予約を受ける / お客様がオンラインで予約を入れる / 予約客が来店したときの着席 / 無断キャンセル管理 の一連を確認できます。
  • 予約管理の全体像を詳しく確認したい場合は、24 章「予約管理」 も参照してください。

5-0. 予約管理とは？ ​

POSLA の予約管理は、お客様が来店する前日までに「何時に何人で行きます」と申し込む仕組みです。以下の機能があります。

  1. オンライン予約: お客様が自分のスマホで予約 LP（ランディングページ）から予約できる
  2. 電話予約: マネージャー以上がハンディまたは管理画面から手動で予約を登録できる
  3. AI 予約: お客様が自由文（例: 「明日 19 時 2 名田中」）で予約できる、AI が解析
  4. 予約金徴収: お客様にカードを登録してもらい、来店したら解除、無断キャンセル時に請求
  5. 自動着席: 予約客が QR を読むとスタッフ操作なしで自動着席
  6. ガント台帳: 時間 × テーブルのマトリクスで予約状況を見える化

5-1. 予約 LP（お客様向けページ）の URL 取得と掲載 ​

5-1-1. URL / QR を確認する ​

  1. 管理画面にログイン
  2. 上部タブから 【店舗設定】 をクリック
  3. サブタブから 【店舗設定】 を選択
  4. 画面を下にスクロール
  5. 「📅 予約ページ」 セクションが見える
  6. 以下の 4 つのボタンと情報が表示されている:
要素用途
URL 表示お客様に案内する予約ページの URL
QR コード画像上記 URL を QR に変換したもの
【URL をコピー】クリップボードに URL を保存（LINE / メールで送信用）
【新タブで開く】動作確認用
【QR 画像コピー】Instagram 等に貼る用
【🖨 印刷】A4 ポスターとして印刷

5-1-2. どこに掲載する？ ​

  • Instagram / X / Facebook などの SNS プロフィール欄に URL 貼り付け
  • Google マイビジネス の「予約」リンク
  • お店の公式ホームページ
  • 店頭ポスター（印刷ボタンで出力して店頭に貼る）
  • LINE 公式アカウント のメッセージ
  • 食べログの口コミ返信 に予約 URL を入れる

5-1-3. お客様が URL から予約する流れ（5 ステップ） ​

  1. お客様が URL / QR から予約 LP を開く
  2. 日付選択（カレンダーから希望日をタップ）
  3. 人数選択（1〜20 名程度）
  4. 時刻スロット選択（空きのある時刻のみ緑で表示）
  5. お客様情報入力（名前・電話・メール・特記事項）
  6. 確認画面 → 【予約する】 ボタン
  7. 予約金を設定している店舗なら Stripe Checkout で決済へ
  8. 予約完了メールが自動送信される

5-2. ガント台帳の見方 ​

5-2-1. 開く手順 ​

  1. 管理画面の上部タブから 【予約管理】 をタップ
  2. サブタブから 【予約台帳】 を選択
  3. 画面が開く
  4. 画面上部に 「今日」「‹」「日付入力」「›」 のボタンで日付切替
  5. 本体は 時間（縦軸）× テーブル（横軸） のマトリクス

5-2-2. 画面構成 ​

│ 17:30 │  (2名) │  🟦鈴木 │        │        │

5-2-3. 予約ブロックの色分け ​

色意味状態
🟦 青確定通常の予約
🟨 黄予約金決済待ち予約金の決済がまだ完了していない予約
🟩 緑着席中QR 読み込み済み、またはスタッフが着席処理済み
🟥 赤無断キャンセル予約時刻を過ぎても来店がない予約
⚫ グレーキャンセルお客様または店舗がキャンセルした予約

5-2-4. ブロックをクリックすると？ ​

予約の詳細モーダルが開き、以下の操作ができます:

  • 【🪑 着席】 — 手動で着席状態にする
  • 【無断キャンセル】 — 無断キャンセルとして記録（予約金があれば請求）
  • 【✏️ 変更】 — 時刻・人数・メモ変更
  • 【🗑 キャンセル】 — 予約を取り消し

5-3. 電話で予約を受けたときの登録手順 ​

お客様が電話で予約を入れたい場合、スタッフが手動で登録します。

5-3-1. ガント画面から登録 ​

  1. 管理画面 → 【予約管理】 → 【予約台帳】
  2. 画面上部の 【+ 新規予約】 ボタンをクリック
  3. モーダルが開く
  4. 以下を入力:
項目必須内容
名前✅電話で聞いた氏名
電話番号✅10〜11 桁、ハイフンなし
メール—任意。予約確認メール送信用
日時✅予約日と時刻
人数✅来店予定人数
テーブル割当—チェックボックスで指定テーブル選択（未指定なら後で）
メモ—「アレルギー: 卵」「誕生日」等
タグ—VIP / 大事な接待 等のマーカー
  1. 「予約金徴収をスキップ」 チェックボックス（電話受付では通常 ON）
  2. 【作成】 ボタン
  3. 予約が作成され、ガント台帳にブロックが表示される

電話予約では予約金徴収をスキップする理由

電話越しにお客様にカード番号を伝えてもらうのは現実的でないため、電話予約では予約金徴収をスキップするのが普通です。ただし大型パーティー（10 名以上）などは、念のため予約金を徴収するよう検討してください。

5-3-2. ハンディから登録 ​

外出先や混雑時はハンディからも予約登録できます。

  1. ハンディ画面の 【📅 予約】 タブ
  2. 右下の 【+ 新規】 ボタン
  3. 同じくモーダルで入力
  4. 保存

5-4. 予約なしで来店したお客様の登録 ​

予約なしのお客様をすぐに着席・注文開始させる場合。

手順 ​

  1. ハンディ画面の 【テーブル状況】 タブ
  2. 空きテーブル（T03）をタップ
  3. 画面に 【+ 予約なし来店】 ボタン → タップ
  4. 人数入力（例: 2 名）
  5. 【着席】
  6. テーブルがセッション確立、メニュー注文開始できる

予約なしのお客様として記録され、通常の来店と同じように注文・会計できます。

5-5. 予約客が来店したとき（自動着席の仕組み） ​

POSLA では予約客が店に来て QR を読むと、条件が合えば スタッフ操作なしで自動的に着席状態になります。

5-5-1. 自動着席の条件（3 つすべて必要） ​

  1. テーブル割当済み（予約登録時に特定のテーブルを指定してあること）
  2. 予約時刻 ±30 分以内（18:00 予約なら 17:30〜18:30 の間に QR を読む）
  3. 予約が確定済み、または予約金の決済待ちであること

5-5-2. 流れ ​

  1. 予約客が来店
  2. スタッフが該当テーブル（T03）に案内
  3. お客様が T03 の QR をスマホで読む
  4. POSLA側で自動マッチング（予約情報と照合）
  5. マッチすれば即セッション確立 → お客様のメニュー画面が開く
  6. ガント台帳のブロックが 🟦 青 → 🟩 緑（着席中）に変わる

お客様が注文を始められるまでスタッフの作業は 0 秒です。

5-6. 自動マッチングが成立しないときの手動着席 ​

以下の場合、自動着席が失敗します:

  • テーブル未割当で予約された
  • 予約時刻を 30 分以上過ぎて来店
  • お客様が違うテーブルの QR を読んだ

手動着席の手順 ​

  1. ガント台帳で該当予約ブロックをタップ
  2. モーダルに 【🪑 着席】 ボタン
  3. タップ
  4. テーブルを選択（未割当の場合）
  5. 着席状態に変わる

5-7. 無断キャンセルの登録 ​

予約時刻を過ぎても来店がない場合の対応。

5-7-1. タイミング ​

通常、予約時刻から 30 分経過しても来店がない場合に無断キャンセルとして処理します。

5-7-2. 手順 ​

  1. ガント台帳で該当予約ブロックをタップ
  2. モーダルに 【無断キャンセル】 ボタン
  3. タップ
  4. 確認ダイアログで 【OK】
  5. 予約が赤色の無断キャンセル表示に変わる

5-7-3. 予約金の自動請求 ​

予約金ありの店舗で無断キャンセルを登録すると、設定に従って予約金が請求されます。お客様のカードに反映される日はカード会社の処理状況により異なります。

5-8. 予約の変更・キャンセル ​

5-8-1. お客様側からの変更 ​

お客様が予約確認メールに記載された URL を開くと、自分で変更・キャンセルできます。

  • 時刻変更
  • 人数変更
  • メモ変更
  • キャンセル

5-8-2. 店舗側からの変更 ​

  1. ガント台帳で予約ブロックをタップ
  2. 【✏️ 変更】 または 【🗑 キャンセル】
  3. 変更内容を入力 or キャンセル確認

5-9. コース予約の管理 ​

コース料理（懐石・フランス料理のプリフィックス等）の予約設定。

5-9-1. コース定義 ​

  1. 【予約管理】 → 【コース管理】サブタブ
  2. 【+ コース追加】
  3. 以下を入力:
    • コース名（例: 「スペシャルディナーコース」）
    • 価格（例: ¥8,800）
    • 所要時間（例: 120 分）
    • 人数範囲（例: 2〜8 名）
    • 含まれる品目（カテゴリ別に設定）
  4. 【保存】

5-9-2. お客様視点 ​

予約 LP でコース選択肢として表示される → お客様が予約時に選択 → 来店時はコースの順番通りに料理が流れる（KDS でコース自動発火）

5-10. 顧客台帳（リピーター管理） ​

過去に来店したお客様の情報を記録する機能。

手順 ​

  1. 【予約管理】 → 【顧客台帳】サブタブ
  2. 検索フィルタ（名前・電話・タグ）
  3. 顧客カードをタップ → 詳細画面
  4. 表示される情報:
    • 来店履歴（直近 50 件）
    • 好み / アレルギー
    • タグ（VIP / リピーター / ブラックリスト）
    • 内部メモ（スタッフ間の申し送り）
    • 累計来店回数・累計金額

5-11. VIP / ブラックリストの設定 ​

5-11-1. VIP 設定 ​

  1. 顧客詳細画面 → VIP チェックボックス ON
  2. 保存
  3. 予約時にガントで ★VIP バッジ表示
  4. スタッフが特別対応しやすくなる

5-11-2. ブラックリスト（manager 以上のみ） ​

2 回以上無断キャンセルをしたお客様や、店舗運営に支障があるお客様をブロックできます。

  1. 顧客詳細画面 → ブラックリスト チェックボックス ON
  2. 理由を入力（例: 「過去 3 回無断キャンセル」）
  3. 保存
  4. 以降、この電話番号からの予約 LP からの申込は自動拒否

5-12. 予約金（デポジット）の設定 ​

大人数予約や、無断キャンセルが多い店舗向けの設定です。

5-12-1. 有効化 ​

  1. 【予約管理】 → 【予約設定】サブタブ
  2. 「予約金を徴収する」 チェックボックス ON
  3. 1 人あたりの金額を入力（例: ¥2,000 / 人）
  4. 徴収開始人数を入力（例: 6 名以上）
  5. 【保存】

Stripe 設定済みであることが前提

予約金徴収は Stripe 経由で処理するため、先に Stripe 設定（21 章）を完了してください。Stripe 未設定だと予約金 ON にできません。

5-12-2. 予約金の動き ​

イベントPOSLA の動作
予約完了時カードの有効性を確認し、予約金枠を確保
来店・着席時予約金枠を解除し、課金なし
無断キャンセル時予約金を請求
期限内キャンセル予約金枠を解除し、課金なし
期限外キャンセル予約金を請求

5-13. AI 予約（お客様の自由文入力） ​

お客様が「明日 19 時 2 名田中」のように自由文で予約できる機能。

使い方 ​

  1. お客様が予約 LP を開く
  2. 画面右下の 【💬 AI で予約】 ボタン
  3. 自由文で希望を入力
  4. Gemini AI が解析 → 日時・人数・名前を抽出
  5. 確認画面で間違いがないかチェック
  6. 【予約する】

店舗側設定 ​

  • 【予約管理】 → 【予約設定】 → 「AI チャット予約」 トグル ON/OFF で有効化

5-14. 予約リマインダーメール ​

予約日時の前にお客様にメール通知。

設定 ​

  1. 【予約管理】 → 【予約設定】
  2. 24 時間前リマインダー トグル ON
  3. 2 時間前リマインダー トグル ON
  4. 【保存】

動作 ​

設定を有効にすると、指定したタイミングで自動送信されます。初回設定に不安がある場合は POSLA 運営にご相談ください。

5-15. よくある質問 ​

Q1. お客様が予約後に来ないとき、どう対応する？

A1. 予約時刻から 30 分経ったらガントで予約ブロックをタップ → 【無断キャンセル】。予約金設定があれば、設定に従って請求されます。

Q2. 大人数予約（10 名以上）はどうする？

A2. 複数テーブル連結予約が可能（最大 3 テーブル）。予約登録時にテーブル複数チェックで割り当てます。無断キャンセルリスクが高いため、予約金を設定する運用がおすすめです。

Q3. AI 予約で誤解析されたら？

A3. お客様が確認画面で修正できます。送信後は店舗側で修正も可。

Q4. 電話予約でテーブルを決めずに予約受けた。後からどう割当てる？

A4. ガント画面で予約ブロック →【✏️ 変更】でテーブル追加。または、お客様が来店時に手動着席でテーブル指定。

Q5. 予約金を取りたくないが、無断キャンセルが多くて困る

A5. ブラックリスト機能で問題のあるお客様をブロックできます。2 回以上無断キャンセルしたお客様は、電話番号をもとにブラックリスト登録を検討してください。
