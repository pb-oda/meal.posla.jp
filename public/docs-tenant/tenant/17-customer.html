17. セルフオーダー画面 ​

セルフオーダー画面は、お客様がテーブルQRコードから開く注文画面です。店舗スタッフは、表示内容と注文の流れを理解しておくと問い合わせ対応がしやすくなります。

お客様の注文手順 ​

  1. テーブルのQRコードをスマートフォンで読み取ります。
  2. メニューを選び、数量やオプションを選択してカートに入れます。
  3. カート内容を確認します。
  4. ゲスト名（任意）: 注文送信前に、注文者の名前（ニックネーム等）を入力できます。入力すると、注文履歴やレジ画面で誰の注文かが判別しやすくなります。
  5. 注文を送信します。
  6. 必要に応じてスタッフ呼び出しやAIウェイターを利用します。

注文履歴とセルフ会計 ​

注文履歴の二層表示 ​

注文履歴画面では、以下の2つのセクションに分かれて表示されます。

  • 未払い注文: 今回の滞在でまだお会計が済んでいない注文です。
  • 会計済履歴: すでにお支払いが完了した過去の注文です。

ゲスト名（任意入力） ​

カートモーダルで、注文送信前に「ゲスト名」を入力できます。入力した名前は以下の場所で利用され、自分や同席者の注文を区別しやすくなります。

  • 注文トラッカー: 誰が何を注文したかを確認。
  • 注文履歴: 過去の注文を名前で整理。
  • セルフ会計画面: 支払うべき注文の確認。

個人負担目安（割り勘計算） ​

お会計前に、卓全体の合計金額をどのように分担するかを「手入力」でシミュレーションできる計算ツールです。以下の3つのモードから選べます。

  • 均等に切り上げ: 合計金額を人数で割り、端数を切り上げた「一人あたりの支払額」を表示します。
  • 一人が端数を負担: 一人あたりの金額を指定し、残りの「端数を含めた金額」を特定の一人が支払う目安を表示します。
  • 自分の分 ＋ 残りを人数割り: 自分の支払額を手入力し、残りの金額を同席人数で割った目安を表示します。

::: note この計算機能は入力の手助けをするためのものであり、guest_alias と連動した自動計算や、品目を選んでの個別決済を行うものではありません。 :::

決済の制限事項

セルフ決済（Stripe Checkout）は、「テーブル全体の合計金額を一括で支払う」 仕組みです。特定の商品だけを選んで支払う「品目別の個別決済」には対応していません。厳密な個別会計が必要な場合は、店舗のレジスタッフへお申し付けください。

会計後の継続注文 ​

セルフ決済が完了した後、画面に「続けて注文する」ボタンが表示されます。これを利用すると、同じテーブルセッションを維持したまま、追加の注文（デザートや飲み物など）をスムーズに行うことができます。

AIウェイター ​

お客様はメニューについて質問できます。アレルギー、提供可否、価格など重要な内容は、店舗スタッフが最終確認してください。

スタッフ呼び出し ​

お客様から呼び出しが入った場合は、店舗のルールに従って対応してください。呼び出し内容が分かる場合は、先に内容を確認してから向かうとスムーズです。

困ったとき ​

症状対処
メニューが表示されないQRコード、店舗設定、メニュー公開状態を確認してください。
注文できないWi-Fi、売り切れ状態、テーブル開放状態を確認してください。
表示言語が違う画面上の言語切替を確認してください。
金額が違う商品価格、オプション、税区分を確認してください。
