3. 管理ダッシュボード ​

管理ダッシュボードは、店長・副店長・店舗責任者が毎日使う店舗運営の入口です。POSLA の主要機能は、左側メニューの大分類と、上部のサブタブに分かれています。

この章では、画面全体の見方と、日々の運営で最初に確認する 運営管理 の使い方を説明します。

画面の基本構成 ​

場所内容
左側メニュー運営管理、メニュー管理、プラン・コース、テーブル・フロア、在庫・レシピ、レポート、店舗設定、テイクアウト、AIアシスタント、予約管理、シフト管理を切り替えます
上部サブタブ選択した大分類の中で、細かい画面を切り替えます
ヘッダー店舗名、店舗切替、ログイン中ユーザー、取扱説明書、パスワード変更、ログアウトを表示します
業務ショートカットハンディPOS、KDS、POSレジ、お客様注文画面を別タブで開きます
右下のサポートガイドFAQ、症状別案内、エラー番号検索を開きます
右下のキャッシュリセット画面表示が古い、ボタンが反応しないなどのときにブラウザ側キャッシュをリセットします

複数店舗を管理している場合は、操作前に必ずヘッダーの店舗名を確認してください。選択中の店舗に対して、メニュー、予約、シフト、売上、設定が読み書きされます。

左側メニューの役割 ​

メニュー主な用途
運営管理今日の要注意、全体検索、売上ドリル、初期設定、端末稼働、カート離脱を確認
メニュー管理カテゴリ、オプション、店舗メニュー、限定メニュー、翻訳、CSV一括更新を管理
プラン・コース食べ放題プラン、プラン専用メニュー、コース料理を管理
テーブル・フロア卓、QR、個別QR、フロアマップを管理
在庫・レシピ原材料、レシピ、低在庫、注文時の在庫減算を管理
レポート売上、レジ、注文履歴、回転率、スタッフ評価、満足度、併売分析を確認
店舗設定営業日区切り、税率、レジ、プリンタ、テイクアウト、予約URL、外観を設定
テイクアウトテイクアウト注文、SLA警告、梱包、通知、受取遅れ、返金対応を管理

スタッフロールでログインした場合は、店舗運営メニューの代わりにスマホ向けのスタッフ画面が中心になります。スタッフ画面では、今日の勤務、勤怠打刻、未対応、マイシフト、希望提出、交代/欠勤申請を扱います。

運営管理 ​

運営管理は、店長が「今日まず何を見るか」を確認するための集約画面です。複数の詳細画面を開く前に、ここで異常や未対応を拾います。

サブタブ確認できること主な使い方
運営コックピット未会計、未締め、予約、シフト不足、品切れ/低在庫、低評価、障害/エラー、テイクアウト遅延開店前、ピーク前、閉店前に警告件数を確認
全体検索注文、予約、顧客、商品、スタッフ、エラーを横断検索電話問い合わせ、クレーム、注文確認、エラー番号調査
売上ドリル日別 → 商品 → 注文 → 会計履歴「どの商品が、どの注文で、どう会計されたか」を追跡
初期設定店舗、カテゴリ、メニュー、卓・QR、スタッフ、KDS、予約、テイクアウト、在庫、レシピ、プラン、コース導入時や新店舗追加時の抜け漏れ確認
端末稼働KDS、ハンディ、レジ、セルフメニューの最終通信時刻端末が止まっていないか、古い端末が残っていないか確認
カート離脱一度カートに入れて外された商品、注文内キャンセル商品写真、価格、説明、オプション、売り切れ表示の改善判断

運営コックピットで見る警告 ​

カード見ているデータ店舗側の対応
未会計paid / cancelled 以外の注文レジ画面で会計待ちの卓・注文を確認
未締め当日のレジ開局と締めの記録閉店前にレジ締め、締め忘れ警告を確認
予約当日の保留・確定・着席中予約と遅刻注意予約台帳で遅刻、no-show、席割当を確認
シフト不足当日の未打刻と未対応の交代/欠勤申請シフト管理の当日・ヘルプで対応
品切れ/低在庫店舗メニューの売り切れ、原材料の低在庫メニュー管理、在庫・レシピで更新
低評価本日の1〜2点評価満足度分析で理由、品目、時間帯を確認
障害/エラー直近24時間のAPIエラーエラー監視またはサポートへ連絡
テイクアウト遅延受取時刻15分以内または超過リスクの未完了注文テイクアウト管理で調理・梱包・通知を確認

運営コックピットは、各詳細画面の代替ではありません。件数が出ている場合は、該当する詳細画面に移動して対応してください。

全体検索 ​

全体検索は、2文字以上のキーワードで、店舗内の主要データをまとめて検索します。

検索対象:

  • 注文ID、注文内容、注文者名、電話番号
  • 予約者名、電話番号、メール、予約メモ
  • 顧客台帳の名前、電話番号、好み、注意点
  • 本部メニュー、店舗限定メニュー
  • スタッフ名、ユーザー名
  • エラー番号、エラーコード、エラーメッセージ

電話問い合わせやクレーム対応では、まず名前、電話番号、注文ID、予約者名のいずれかで検索してください。検索結果は確認用であり、編集や返金などの操作は各詳細画面で行います。

売上ドリルダウン ​

売上ドリルダウンでは、期間を選んで次の順に深掘りできます。

  1. 日別売上
  2. その日の商品別売上
  3. その商品を含む注文一覧
  4. 注文詳細と会計履歴

売上ドリルは、商品改善や問い合わせ対応に使います。売上集計は会計済み注文を中心に確認します。返金、取消、外部決済控えの詳細は、必要に応じてレジ分析や取引ジャーナルも確認してください。

卓・QR有効なテーブルが1卓以上ある
スタッフmanager または staff の所属ユーザーがいる
KDS有効なKDSステーションがある
予約予約設定が作成されている
テイクアウトテイクアウト設定があり、受付が有効になっている

端末稼働状況 ​

端末稼働状況では、KDS、ハンディ、レジ、セルフメニューの最終通信時刻を確認します。

見るポイント:

  • KDSやレジの最終通信が古い場合、端末の電源、Wi-Fi、ログイン状態を確認する
  • ハンディ端末が複数ある場合、使っていない端末が残っていないか確認する
  • セルフメニューの最終通信がない場合、QR掲示、テーブル開放、注文画面URLを確認する
  • 端末専用アカウントは人件費・勤怠の対象ではないため、人のスタッフアカウントと混ぜない

カート離脱・キャンセル分析 ​

カート離脱では、お客様が商品を一度カートに入れた後、外した商品を確認できます。注文履歴に残る注文内キャンセルも、環境に記録がある場合は同じ画面に表示されます。

活用例:

  • 写真と実物の印象が違う商品を見つける
  • 価格を見て外されやすい商品を見つける
  • 必須オプションが分かりにくい商品を見つける
  • 売り切れや説明不足で注文をやめた可能性を確認する
  • セット化、写真差し替え、説明文追加の優先順位を決める

ロール別の表示 ​

ロール主に表示される内容
オーナー複数店舗管理、本部機能、契約・決済設定、ユーザー管理
マネージャー担当店舗のメニュー、注文、会計、在庫、シフト、レポート、店舗設定
スタッフ勤怠打刻、マイシフト、希望シフト提出

表示されないタブがある場合は、権限または所属店舗が不足している可能性があります。

店舗を切り替える ​

複数店舗に所属している場合は、画面上部の店舗セレクターから操作対象の店舗を選びます。選択した店舗に対して、メニュー、注文、レポートなどの操作が行われます。

サポートガイドを使う (右下の 📚 ボタン) ​

管理画面右下の 「📚 サポートガイド」 (オレンジ色の丸ボタン) から、操作で困った時の情報を検索できます。

5 つの入口:

入口用途
ホームカテゴリ別ショートカット + 人気の質問 + エラー番号入力欄
よくある質問 (FAQ)カテゴリ別に厳選した Q&A (ログイン・注文・会計・印刷・予約・KDS・メニュー・シフト・テイクアウト・設定)
キーワード検索「会計」「パスワード」「予約」「プリンタ」などで FAQ / 案内記事 / エラーコードを横断検索
症状からたどる「ログインできない」「KDSに出ない」等の症状からステップバイステップの対処手順
エラー番号E3024 等を入力すると意味と対処を即座に表示

まずはサポートガイドをご利用ください

サポートガイドには、よくあるお問い合わせと基本的な対処方法をまとめています。該当する案内が見つからない場合は、POSLAサポートへお問い合わせください。
