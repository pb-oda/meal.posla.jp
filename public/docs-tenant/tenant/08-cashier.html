8. 会計・POSレジ ​

POSレジでは、店内注文やテイクアウト注文の会計、割引、個別会計、領収書、返金、レジ開閉を行います。現金が動く操作では、担当スタッフのPIN入力が必要です。

推奨端末

POSレジはAndroidタブレット + Google Chrome最新版を 標準環境 としています。PC + Chrome最新版でもご利用いただけます。iPad/Safariは決済端末（Stripe Reader）連携やレシート印刷の挙動が不安定なためサポート対象外です。

会計・決済の運用 ​

POSLAでは、お客様自身がスマートフォンで決済する「セルフ決済」と、スタッフがレジ端末で操作する「レジ決済」を併用できます。

セルフ決済とレジ決済の使い分け ​

決済方法特徴・制限推奨シーン
セルフ決済テーブル全体の合計金額を Stripe で一括決済します。個別会計には対応していません。スムーズな退店を優先する場合、グループ一括支払いの場合。
レジ決済現金、クレジットカード、個別会計、分割支払いなど、柔軟な決済が可能です。個別会計（割り勘）を希望される場合、現金支払いの場合。

責務の境界

お客様のスマートフォン画面（セルフ決済）には、常に「その時点で未払いの商品すべて」の合計金額が表示されます。レジで一部の商品を精算すると、セルフ決済の対象金額も自動的に減算されます。

会計の基本手順 ​

  1. POSレジ画面を開きます。
  2. 会計するテーブルまたは注文を選びます。
  3. 注文内容、数量、割引、合計金額を確認します。
  4. 支払い方法を選びます。
  5. 必要に応じて担当スタッフのPINを入力します。
  6. 会計を完了し、レシートまたは領収書を発行します。

支払い方法 ​

支払い方法確認事項
現金預かり金、お釣り、ドロア操作を確認してください。
クレジットカード決済端末の接続と決済完了表示を確認してください。
QR決済店舗の運用ルールに従って入金確認を行ってください。
その他店舗で定めた支払い方法名と集計方法を確認してください。

個別会計 ​

同じテーブル内で別々に支払う場合は、個別会計を利用します。お客様から「品目ごとに分けて支払いたい」と要望があった場合は、レジ端末で Partial（個別）モード を有効にし、精算する品目を選択してください。

対象商品、数量、割引の適用範囲を確認してから会計してください。

返金 ​

返金は金額と理由を確認し、店舗責任者のルールに従って実施してください。返金後は、レジ締め時の差額確認も行ってください。

レジ開閉 ​

営業開始時にレジを開け、営業終了時にレジを閉めます。現金残高、入金、出金、売上金額を確認し、差額がある場合は理由を記録してください。

困ったとき ​

症状対処
会計対象が出てこない注文が提供済みになっているか確認してください。
金額が合わない注文内容、割引、個別会計、返金履歴を確認してください。
決済端末が反応しない端末の電源、Wi-Fi、ペアリング状態を確認してください。
PINが通らない出勤打刻、PIN設定、入力ミスを確認してください。
通常会計ボタンが押せない (黄色バナー)緊急レジモード を使ってください (下記)

緊急レジモード (通信障害時の会計記録) ​

通信が不安定な時・POSLA の通常会計が使えない時に、会計した事実を端末内に記録して後から POSLA へ同期する モードです。

この機能が代替するのは「記録」だけです

  • POSLA がカード決済を代替するわけではありません。カード決済は必ず外部決済端末 (Airpay / Square / Stripe Terminal / 電子決済端末等) で処理してください。
  • POSLA には カード番号・有効期限・セキュリティコードを絶対に入力しないでください。入力欄そのものを用意していません。
  • 緊急レジで記録した会計は「POSLA 未同期」です。通常の売上レポートには、管理者が確認してから反映されます。

いつ使うか ​

  • レジ画面上部に黄色いバナー「古いデータを表示中」が出て、通常会計ボタンが押せない時
  • 通常会計を押したら「通信エラー」「最新の注文状態を取得できませんでした」と出た時
  • お店の Wi-Fi / モバイル回線が不安定で、会計だけは記録を残したい時

起動方法 ​

レジ画面ヘッダ右上の 🔴 「緊急レジ」 ボタンを押してください。

  • バッジ (赤丸の中の数字) が出ている時は、未同期の緊急会計がその件数ぶん端末内に残っています。
  • 通常運用でも店長判断で手動起動できます (検証・訓練用)。

使える支払方法 ​

選択肢意味
現金お客様から現金を受領済み。受領額とお釣りを入力
外部カード端末決済端末 (Airpay 等) でカード決済を完了させてから、控え番号 / 承認番号を記録
外部 QR / 電子決済外部の QR 決済端末 / 電子マネー端末で完了させてから記録
その他外部決済上記以外 (商品券、事前振込 等)。メモで詳細を残す

カード決済の流れ

  1. お客様のカードは外部決済端末に通す (POSLA では処理しない)
  2. 外部端末で承認 → 承認番号・控え番号が出る
  3. 外部端末の 控えを必ず保管 (紙 or 写真)
  4. POSLA の「緊急レジ」で支払方法 = 「外部カード端末」を選び、控え番号 / 承認番号を記録
  5. 通信復帰後、POSLA 側で自動同期される

記録後の画面 ​

記録すると「POSLA 未同期の緊急会計控え」が画面に表示されます。これには:

  • 緊急会計番号 (端末内一意 ID)
  • テーブル名 / 合計金額 / 支払方法
  • 外部端末の控え番号・承認番号 (入力した場合)
  • 担当者名 / 記録時刻

印刷ボタンで紙に残すこともできます。この控えは「POSLA 未同期」と明示されている点が、通常のレシートとの違いです。

未同期件数の確認と同期 ​

  • ヘッダの 🔴 ボタン横のバッジ = 未同期 + 要確認件数の合計
    • 3 のように数字だけ → 未同期のみ
    • !3 のように ! 付き → 要確認の会計が含まれています。管理者に連絡してください
    • tooltip (長押し / マウスホバー) で「未同期: N 件 / 要確認: M 件」の内訳表示
  • 🔴 ボタン → 「未同期会計一覧」で状態を確認
  • 状態の意味:
    • 未送信 (pending_sync, オレンジ): 端末にあり、まだ送信していない状態
    • 送信中 (syncing, 水色): 同期を試している状態
    • 同期済み (synced, 緑): 正常に反映された状態
    • 要確認: 重複の可能性 (conflict, 赤): 同じ注文が通常会計でも処理されている可能性があります。管理者確認必須
    • 要確認: 差異あり (pending_review, 紫): 金額や内容に差異があり、手動確認が必要です
    • 送信失敗 (failed, 濃赤): 通信エラー等で送信できていません。同期再試行を行ってください

未同期会計があるときにやってはいけないこと ​

未同期の間は絶対やらないでください

  • ❌ ブラウザのキャッシュ削除
  • ❌ ブラウザデータ削除
  • ❌ ログアウト
  • ❌ POSLA アプリの再インストール
  • ❌ 端末の初期化

→ これらをすると ブラウザ内の保存データが消え、未同期の会計記録が復元できなくなる おそれがあります。

通信復帰後、未同期件数が 0 になってから行ってください。

通信復帰後の確認 ​

  1. 通信が復帰すると自動で同期が試みられます (上部の緑バナー「通信が復帰しました」)
  2. 🔴 ボタンのバッジが 0 になれば同期完了
  3. 0 にならない件がある場合は、🔴 → 「未同期会計一覧」で状態を確認
  4. 要確認の会計が残っている場合は、管理者に連絡して精査してもらってください

管理者: 「緊急会計台帳」で確認する ​

店長・オーナーの方は、管理画面で店舗単位の緊急会計を一覧確認し、判断記録を残せます。

場所: 管理画面 → レポート → 緊急会計台帳

見える内容:

  • 期間フィルタ（開始日 / 終了日、最大 92 日）
  • 状態フィルタ（すべて / 同期済み / 要確認 / 送信失敗 など）
  • 集計カード: 件数・合計金額・要確認件数・PIN 未確認件数・判断状況の内訳
  • 一覧表（日付時刻 / 状態 / 判断 / テーブル / 金額 / 支払方法 / 外部端末控え番号 / 担当者 / PIN 確認状況 / 判断者・判断日時 / 備考）
  • 行をクリックすると詳細画面（明細 / 関連注文 / メモ / 管理者判断）

現時点でできること / できないこと

  • ✅ 管理者判断の記録: 「有効確認 / 重複扱い / 無効扱い / 保留」ボタンで判断を残せます (詳細は次節)
  • ❌ 売上レポートへの反映: 判断を記録しても、売上レポートには自動反映されません
  • ❌ 修正・取消: 一度記録した「確定判断 (有効確認 / 重複扱い / 無効扱い)」は、その場では変更できません
  • 要確認の会計があれば、外部決済端末の控え（レシート紙・端末画面）と必ず突き合わせてから判断してください
  • カード決済の真偽は POSLA では確定できません。外部端末の承認記録が正です

よくあるケース:

  • すでに通常会計で処理済みだった → 緊急会計側を無効扱いにするか確認
  • 別端末で同じ注文を記録していた → どちらを正とするか管理者が判断
  • テーブルを選ばずに手入力で記録していた → 外部端末控えと突き合わせて精査
  • PIN 入力後に送信失敗していた → 担当者本人へ確認

通常会計も取り消せます ​

POS レジ（kds/cashier.html）から普段どおりに行った会計も、取引履歴パネルから管理者が論理取消できます。誤会計の 記録だけを無効化する操作で、調理や卓状態は動かしません。

操作手順:

  1. POS レジ > 右上「取引履歴」ボタンで取引ジャーナルを開く
  2. 対象行の右側に表示される 「会計取消」 ボタンを押す（表示条件は下記）
  3. 取消理由（必須） を入力して実行

取消ボタンが表示される条件:

  • まだ取消していない
  • まだ返金していない
  • 分割会計ではない
  • 外部決済サービスを通していない会計である
    • 対象の支払方法: 現金 / 手入力カード / 手入力 QR
    • 対象外: Stripe Connect 経由のカード決済など（返金で対応）
  • 対象の注文種別: 店内注文 / ハンディ注文 / POS レジで会計したテイクアウト
    • 対象外: オンライン決済済みテイクアウト（オンラインで決済完了しているもの）

取消後の挙動:

  • 取引履歴: 「✗ 取消済」赤バッジが表示され、取消分を差し引いた合計が表示されます
  • 現金の場合のみ: レジ分析に現金返却分の記録が自動で追加され、期待残高に反映されます
  • 売上レポート: 取消済みの会計は集計から除外されます
  • 注文状態そのものは変更されません。卓状態・レシート・KDS 表示もそのままです
  • レシート再印刷: 既発行分は引き続き閲覧できます
  • 取消済みの会計には、新しい領収書発行や返金を重ねて行えません

この機能ではまだ取り消せないもの（取消ボタンが出ません）:

  • 分割会計
  • Stripe Connect など外部決済サービス経由のカード / QR 決済 — 取消ではなく返金で処理します（テイクアウト章「オンライン決済テイクアウトの返金」を参照）
  • オンライン決済済みテイクアウト — 取消ではなく返金で処理します
  • 返金済みの会計
  • 緊急会計転記由来（手入力計上 / 注文紐付き転記）— 既存の緊急会計台帳から取消

注意:

  • 取消理由は必須・監査ログに残ります
  • 同じ会計を二重に取り消さないよう、安全に処理されます
  • 現金会計ではレジ上の残高は自動調整されますが、お客様への現金返却は別途手渡しで行ってください

注文紐付きの緊急会計転記も取り消せます ​

「売上へ転記する」（青ボタン）で通常会計に反映した記録も、後から取り消せるようになりました。

操作手順:

  1. 緊急会計台帳で対象の行をクリック
  2. 「売上転記」セクションに 「緊急会計転記を取り消す」 ボタン（オレンジ色）が表示される
  3. 取消理由（必須） を入力
  4. 「緊急会計転記を取り消す」を押す → 確認ダイアログ → 取消実行

取消後の挙動:

  • 取引ジャーナル: 「取消済み」バッジ
  • レジ分析 > 現金売上 / 期待残高: 現金返却分の調整記録が自動で入り、期待残高との差分は 0 になります
  • レジ分析 > 外部分類別: voucher 等の集計から除外
  • 緊急会計台帳: 「✗ 取消済み」バッジ + 取消日時 / 理由

手入力計上の取消と異なる点:

  • 売上レポート（注文ベース集計）に「残ります」。通常会計と同じく「会計済み注文」として表示されます
  • 回転率レポートにも残ります
  • テーブル状態は戻りません。お客さんが退店済み・卓クローズ済みの後で取り消す前提です
  • QR コードは元に戻りません。新しいお客様が座る前に、通常の卓オープン手順で再開してください

いつ使うか:

  • 外部端末控えとの不一致が後から判明したとき
  • 誤って二重に転記してしまったとき
  • 本来 other_external で分類すべきだったが先に external_card で転記してしまったとき（取消後に正しい分類で再転記）

いつ使わないか:

  • 通常会計（POS レジから直接会計したもの）→ 取引ジャーナル（取引履歴パネル）の「会計取消」ボタンから取り消します（§「通常会計も取り消せます」参照）。本ボタン（緊急会計転記の取消）ではありません。
  • 営業中で次のお客さんが座っている卓 → ❌ テーブルセッションは戻らないので、運用上の問題が発生

注意:

  • 取消済の注文は売上集計から自動で除外されます
  • 取消理由は必須で記録に残ります
  • 同じ会計を二重に取り消さないよう、安全に処理されます

手入力計上は取り消せます ​

誤って「売上として計上する（手入力）」を押してしまった場合、管理者が後から取り消せます。

操作手順:

  1. 緊急会計台帳で対象の行をクリック（詳細モーダル）
  2. 「売上転記」セクションの下に 「手入力計上を取り消す」 ボタン（赤）が出る
  3. 取消理由（必須） を入力
  4. 「手入力計上を取り消す」を押す → 確認ダイアログ → 取消実行

取消後の挙動:

  • 取引ジャーナル: 「取消済み」バッジが付く（行自体は残る）
  • レジ分析 > 外部分類別: 商品券や振込などの集計から除外されます
  • レジ分析 > 現金売上 / 期待残高: 現金の場合は返却分が反映され、期待残高との差が合うようになります
  • 台帳: 緊急会計記録には「取消済み」バッジが付く、取消日時と理由が表示される

この「手入力計上の取消」では取り消せないもの（それぞれ別の経路で取り消せます）:

  • 注文紐付きで転記された通常会計（売上として計上する青いボタン経由）→ §「注文紐付きの緊急会計転記も取り消せます」
  • 通常の POS レジで会計した payments → §「通常会計も取り消せます」（POS レジ > 取引履歴の「会計取消」ボタン）
  • ❌ 取消そのものの取り消し（= 再計上）は全系統で不可

注意:

  • 取消理由は必須です
  • 同じ記録を 2 回取り消そうとしても安全です（2 回目は何もせず成功扱い）
  • 取消操作は元に戻せません。外部端末控えと金額を必ず再確認してから押してください

手入力売上は売上レポートには出ません ​

管理者が台帳から「売上として計上する（手入力）」を押した記録は、以下に反映されます。

画面反映
取引ジャーナル✅ 出ます
レジ分析 > レジログ・現金売上・期待残高✅ 現金の手入力だけ反映
レジ分析 > 外部分類別（商品券・事前振込等）✅ 外部分類済みは反映
売上レポート❌ 出ません
領収書△ 明細が入っている記録のみ発行可能
返金（レジ画面）❌ 構造上システム返金できません。手渡し返金 or 外部端末返金で運用

大事なポイント:

  • 売上レポート（注文ベース集計）には出ないため、手入力売上を含めた日次売上の確認はレジ分析タブを併用してください
  • 計上ボタンを押す前に、外部端末の控えと金額が合っているかを必ず確認してください

その他外部決済は分類を選んでください ​

「その他外部決済」を選んだ場合、5 つの分類から必ず 1 つ選択してください。未選択のまま記録ボタンを押しても止まります。

分類いつ使うか
商品券紙の商品券、電子ギフト券、クーポン券を使ったとき
事前振込予約時の銀行振込、団体予約・ケータリングの前金が入っていたとき
売掛接待・固定客の付け払い、法人請求で後日集金するとき
ポイント充当店舗独自ポイントで全額または一部を相殺したとき
その他上記に当てはまらない特殊ケース (メモ欄に詳細を残す)

どれに当てはまるかわからないとき:

  • 最初の 4 つに当てはまりそうならそれを選ぶ
  • どうしても当てはまらない場合だけ「その他」を選び、メモ欄 に理由を書く
  • 台帳を確認する管理者が後追いで判断できるように詳細を残すのが安全です

注意:

  • 選んだ分類は、台帳表示と管理者確認のために使われます
  • 以前に登録した記録は「その他外部（未分類）」と表示される場合があります

緊急レジの PIN は任意です ​

緊急レジの「担当スタッフ PIN」欄は 任意 です。空欄でも緊急記録は残せます。

PIN 欄の使い方:

  • スタッフ本人の PIN を入力して記録 → 台帳 PIN 列が「✓ 確認済」（緑）になります
  • 空欄のまま記録 → 台帳 PIN 列は「入力なし」（グレー）になります
  • 間違った PIN を入力 → 台帳 PIN 列は「入力不一致」（オレンジ）になります
  • 古い記録 (2026-04-20 以前) → 台帳 PIN 列は「未確認」（グレー） のままになります

「確認済」と「自己申告」の違い:

  • 確認済 = PIN 検証が通り、担当スタッフを確認できている状態
  • 自己申告 = 担当欄にユーザー名は入っているが、PIN 検証なしでクライアントが自己報告したもの。台帳上で「担当」列に (自己申告) 〇〇 と表示され、紐付け保証はありません

PIN を運用したい場合はオーナーダッシュボード「スタッフ管理」タブで各スタッフに PIN を事前登録 + 勤怠打刻しておいてください。

管理者の判断を記録する ​

manager / owner は緊急会計台帳の詳細モーダルから 判断結果 を記録できます。

記録できる判断 (4 種類):

ボタン意味note
有効確認この緊急会計は正しい会計記録と確認した任意
重複扱い通常会計 / 別の緊急会計と重複していたため売上計上対象外必須
無効扱い誤入力・テスト・無効記録必須
保留にするまだ判断できない (外部端末の日締め待ち等)任意

操作手順:

  1. 緊急会計台帳で対象の行をクリック → 詳細モーダルが開く
  2. 下部の「管理者解決」エリアで note を入力 (重複扱い / 無効扱いは必須)
  3. ボタンをクリック → 確認ダイアログ → 記録

売上レポートには反映されません

  • 「有効確認」を記録しても、売上レポートには自動で反映されません
  • 管理者判断は台帳に残りますが、売上集計への自動反映は行われません
  • 店舗の日次売上に緊急会計を含める場合は、外部決済端末の控えとあわせて管理してください

同じ判断の再送はエラーになりません

  • 同じ判断を誤って 2 回押しても「既に記録済み」として正常に処理されます
  • ただし異なる判断に変更する操作はできません。変更したい場合は POSLAサポートへご相談ください

売上へ転記する ​

2026-04-20 以降、「有効確認」済みの緊急会計は、管理者が「売上へ転記」ボタンを押すと通常の売上に計上されます。

前提条件 (すべて満たすとき転記ボタンが出ます):

  • 管理者解決が「有効確認 (confirmed)」
  • まだ転記していない
  • 同期失敗ではない
  • 対象の注文が 1 件以上ある
  • 支払方法が「その他外部決済」ではない (現金 / 外部カード / 外部 QR のみ)

操作手順:

  1. 緊急会計台帳で有効確認済みの行を開く
  2. 詳細モーダル下部の「売上転記」ブロックに「売上へ転記する」ボタンが表示される
  3. メモ (任意) を入力
  4. ボタンを押すと確認ダイアログで「合計・支払方法・テーブル」が表示される
  5. 外部決済端末の控えと金額・承認番号が一致することを必ず確認
  6. 確定 → 通常の売上記録として登録され、売上レポート・レジ締めに反映

転記後の扱い:

  • 詳細モーダルに「✓ 転記済み」と表示
  • 通常の売上レポートで「緊急会計由来の売上」として集計される
  • 同じ緊急会計に対して再度ボタンを押しても二重転記されません

転記取消はまだできません

  • 一度「売上へ転記」した緊急会計は、その場では取り消せません
  • 誤転記した場合は POSLAサポートへご相談ください
  • 転記前の外部端末控えとの突合を必ず行ってください

支払方法の違い

  • 現金: 現金残高と一致する想定 → 通常レポートで「現金売上」に計上
  • 外部カード端末: 通常レポートで「カード売上」に計上
  • 外部 QR / 電子マネー: 通常レポートで「QR 売上」に計上
  • その他外部決済: 転記ボタンは表示されません

手入力モードの売上反映 ​

緊急会計の「手入力モード」(テーブル未選択で記録した会計) は、現時点で売上転記できません。

理由: 注文と紐づかない手入力の会計は、通常の売上記録へ自動反映しにくいためです。

対応:

  • それまでは、手入力モードの緊急会計は POSLAサポートへご相談の上で対応 してください

管理者が未同期会計がある端末を扱う時の注意 ​

スタッフが未同期件数バッジを「!N」表示 (要確認あり) と報告してきた場合:

  1. 端末の 🔴 → 「未同期会計一覧」で状態を確認
  2. pending_sync / syncing / failed が残っていれば通信復帰を促す
  3. 要確認の会計があれば、管理画面の「緊急会計台帳」で精査
  4. 端末のブラウザ内保存データを絶対に消さないよう、スタッフに再周知

よくある質問 ​

Q. 緊急レジで記録した会計は、自動的に売上レポートに乗りますか？ A. いいえ。まずは台帳に保存され、必要に応じて管理者が確認・対応します。

Q. 同じテーブルで同じ会計を 2 回記録してしまったら？ A. 各緊急会計には端末内で一意の番号が付くため、同じ記録ボタンを連打しても二重登録はされません。ただし手順自体を 2 回繰り返した場合は 2 件記録されます。未同期一覧で確認し、不要な方を管理者に相談してください。

Q. 複数端末で同じテーブルを同時に緊急会計したら？ A. 同じ注文が重複して記録された可能性があるとして「要確認」表示になります。管理者が内容を確認してください。

Q. 外部端末の控え番号を間違えて記録してしまった A. 現時点では緊急会計の修正はできません。メモ欄に訂正情報を書き、管理者へ連絡してください。

Q. カード番号を入力したい A. 入力しないでください。POSLA の緊急レジではカード情報を扱いません。カード決済は必ず外部端末で完了し、その控え番号のみを記録してください。
