Part 8 — トラブルシューティング（困ったとき） ​

このページでは、営業中によく起きるトラブルと、その場での対処法 を順序立てて説明します。

基本の復旧 3 ステップ

何が起きても、まず以下の 3 つを試してください。

  1. ブラウザをリロード（Cmd+Shift+R / Ctrl+Shift+F5）
  2. Wi-Fi 接続を確認（画面右上のアイコン）
  3. 1 分待って再試行

これで 6 割のトラブルは解消します。それでもダメなら下の詳細手順へ。

通知が来ないときの確認（PWA 通知関連）

業務端末の Push 通知 (呼び出し / 低評価 / 重要エラー) が届かない場合は以下を順に確認してください。

  1. Android Chrome 最新版か（iPhone / iPad / Safari は業務端末の標準環境外）
  2. PWA をホーム画面に追加済みか（追加していない場合は通知が届かない可能性あり）
  3. 通知許可が ON か（画面ヘッダの「通知: 未設定」をタップして許可、または Android 設定 → アプリ → POSLA → 通知）
  4. 省電力モード / バッテリーセーバー OFF（端末側で通知を抑制していないか）
  5. 端末がオンラインか（オフラインだと通知は届かない / 復帰後に届く可能性はある）
  6. POSLA 運営側の VAPID キー設定が完了しているか（管理画面の「通知: 未設定」が「通知: サーバー未設定」になっている場合は POSLA 運営にお問い合わせください）

通知は補助機能です。届かなかった場合に備えて KDS / ハンディ / 管理画面の画面表示自体は引き続きご確認ください。

表示が古い・崩れているときの復旧 3 ステップ（PWA 関連）

新しいバージョンに更新されない、画面表示が壊れているときは以下を順に試してください。

  1. 画面下部の「更新する」バナー が出ていればタップ（最新版 Service Worker に切替）
  2. 画面右下の「⟳ キャッシュリセット」ボタン をタップ（このスコープのキャッシュだけ削除して再読込。ログイン情報・店舗選択は保持されます）
  3. PWA を再インストール（ホーム画面のアイコンを長押しして削除 → ブラウザで対象 URL を開き直し → 「ホーム画面に追加」/「アプリをインストール」）

それでも直らない場合は POSLA 運営にエラー番号と発生時刻をお伝えください。

8-0. エラーが出たら ​

POSLA でエラーが出ると、画面に **以下のような形式で「番号付きメッセージ」**が出ます。

[E3024] PIN が一致しません

8-0-1. やること（バイトスタッフ向け） ​

  1. 番号（Exxxx）を控える（紙にメモ・スクショ・声に出す）
  2. マネージャーに伝える（口頭でOK）
  3. マネージャーが管理画面の オレンジ色 📚 サポートガイドボタン（右下） で番号を入力して対処方法を確認する

8-0-2. スタッフは サポートガイドを使わない ​

スタッフ用端末（KDS / レジ / ハンディ）にはサポートガイドのボタンはありません

  • スタッフが操作で困ったときは 必ずマネージャーに聞く のが POSLA の運用ルールです
  • サポートガイドは管理画面にだけ設置されています
  • 業務中の操作集中度を保つため、KDS・レジ・ハンディの画面には何も置きません

8-0-3. マネージャー向け：エラー番号の調べ方 ​

2026-04-23 以降、管理画面右下のボタンは AIチャットから非AIのサポートガイドに変更されています。以下の順で調べます:

  1. 管理画面右下の オレンジ色 📚 サポートガイドボタン をタップ
  2. 上部タブの 「エラー番号」 をタップ
  3. E3024 のように入力 → 代表的な 15 件については即座に意味と対処を表示
  4. 収録外の番号なら、タブ内の「全エラーカタログを開く」リンク (tenant/99-error-catalog) で確認
  5. それでも解決しない場合は POSLA サポートへ連絡 (番号・画面・発生時刻を伝える)

エラー番号の桁の意味（参考）:

範囲系統
E1xxx認証・セッション
E2xxx入力ミス
E3xxx会計・PIN・決済
E4xxx注文・KDS
E5xxx在庫・メニュー
E6xxx予約
E7xxx印刷
E8xxx外部連携
E9xxxPOSLA側で確認が必要なエラー

8-0-4. 障害報告窓口を使うタイミング ​

注文、会計、KDS、予約、ログインなど営業に影響する問題は、通常のお問い合わせではなく 障害報告窓口 から送信してください。

送信前に控えるもの:

  1. お問い合わせコード
  2. 店舗名
  3. 画面名
  4. 発生時刻
  5. 操作内容
  6. 支払い方法や注文番号など、原因調査に必要な補足
  7. スクリーンショット

障害報告が送られると、POSLA運営側の運用支援基盤で対象店舗、対象cell、エラーログ、直近の監視イベントを確認します。 営業中に会計や注文が止まる場合は、紙伝票や手書き会計などの代替運用へ切り替え、営業継続を優先してください。

8-1. ログインできない ​

8-1-1. エラーメッセージ別の対処 ​

エラー文言原因対処法
「ユーザー名またはパスワードが正しくありません」入力ミスの可能性が最も高い大文字小文字、スペース、似た文字（I/l/1、O/0）の見間違いを確認。何度も失敗する場合は店長またはPOSLA運営に相談
「このアカウントは無効化されています」店長がアカウントを停止店長に連絡して有効化してもらう
「このテナントは無効化されています」契約が止まっているPOSLA 運営（info@posla.jp）に連絡
「ユーザー名とパスワードは必須です」片方が空欄両方を入力
「通信エラーが発生しました」Wi-Fi 切れ or サーバー障害Wi-Fi 確認、1 分後再試行
「サーバーが空のレスポンスを返しました」POSLA側の一時的な問題5 分待って再試行。繰り返す場合 POSLA 運営に連絡
画面真っ白ブラウザの一時的な不具合Cmd+Shift+R で強制リロード
URL が違うURL タイプミス正しい URL: /admin/

8-1-2. パスワードを忘れた場合 ​

スタッフ（staff）が忘れた:

  1. 店長に依頼
  2. 店長が管理画面の 【スタッフ管理】 で 【パスワードリセット】
  3. 新しい仮パスワードをスタッフに伝える
  4. スタッフは次回ログイン後に自分で好きなパスワードに変更

店長（manager）が忘れた:

  1. オーナーに依頼
  2. オーナーがオーナー管理画面 → ユーザー管理 で 【パスワードリセット】

オーナーが忘れた:

  1. info@posla.jp に連絡
  2. 本人確認のため 契約時の代表者名 + 契約時のメール を送る
  3. POSLA 運営が 24 時間以内にリセット対応

8-1-3. ログイン画面が開かない ​

症状: URL を開いても「管理画面」と表示されない

対処:

  1. URL を再確認（/admin/）
  2. Wi-Fi 接続確認
  3. Chrome を完全終了してもう一度起動
  4. 別の Android 端末で試す（問題の切り分け）。Safari/iPad で試すのは切り分け用途のみ・本番運用は Android Chrome で行う
  5. 全部ダメなら POSLA サーバー障害の可能性 → info@posla.jp に連絡

8-2. 注文が入らない / 画面が動かない ​

8-2-1. ハンディでの注文送信トラブル ​

症状原因対処
ボタンが反応しないWi-Fi切れ、またはブラウザの一時的な不具合Wi-Fi確認 → Cmd+Rで再読み込み
「カートが空です」エラーメニュー未選択で送信を押したメニューをタップしてカートに追加してから送信
「テーブルを選択してください」テーブル未選択画面上部のドロップダウンからテーブル選択
「品切れが含まれています」カート内に品切れ品目がある該当品目をカートから削除して再送信
「セッションが無効です」4 時間経過で切れた再ログイン
「注文上限に達しています」1 時間 / 1 テーブルの上限（20 件/10 分）超過10 分待って再送信
タイムアウト通信遅い1 分待って再試行、別端末で試す

8-2-2. セルフオーダーでお客様が注文できない ​

症状: お客様から「メニューで注文ボタンが押せない」と言われた

症状原因対処
「ラストオーダー時刻を過ぎました」設定されたラストオーダー時刻超過マネージャーが店舗設定で LO 時刻を確認、必要なら延長
「セッションが期限切れです」QR 読み込みから 4 時間経過スタッフが新しい QR を発行（ハンディで 📲 QR 開放）
QR コードが読めないQR が汚れ / 折れている管理画面で QR 再印刷して貼り直し
画面が真っ白スマホブラウザのバグお客様のスマホで Chrome / Safari を再起動してから QR を読み直す

8-2-3. 注文の送信後、キッチンに届いていない ​

症状: ハンディで注文送信したのに KDS 画面に現れない

対処:

  1. KDS 画面をリロード（Cmd+Shift+R）
  2. ステーション設定で 「全品目」 に切替（ステーション別に絞られている可能性）
  3. ハンディの 【注文履歴】 タブで該当注文があるか確認
  4. あるのに KDS に出ない → POSLA側の問題 → 運営に連絡

8-3. KDS に注文が表示されない ​

8-3-1. 症状別対処 ​

症状原因対処
新規注文が全く入らないポーリング停止ブラウザリロード。音声の「タップして開始」を実行していないと通知音が鳴らないので見落としている可能性
一部品目だけ表示されないステーション設定が絞り込んでいる画面上部のステーションバーで 【全品目】 をタップ
音が鳴らない音声未許可初回 KDS 起動時の 【タップして開始】 が実行されていない。再ログインして必ずタップ
更新が遅い3 秒間隔のポーリング仕様通り。手動リフレッシュは不要だが、心配なら Cmd+R
調理ボタンが反応しない通信エラー3 秒待って状態が戻るか確認。戻らないならリロード
カードが古い情報のままブラウザキャッシュCmd+Shift+R で強制更新

8-3-2. KDS 画面が赤く点滅 ​

症状: 注文カードが赤く点滅している

意味: 注文から 20 分以上経過しています。

対処:

  1. 急いで調理・提供
  2. お客様にお詫び
  3. 遅延が続くなら人員配置・調理フローの見直し

8-4. カードリーダーが反応しない ​

8-4-1. 手順で確認 ​

  1. 電源ランプ（本体正面）が緑色点灯しているか確認
    • 消えている → 本体の電源ボタンを 2 秒長押し
  2. Wi-Fi ランプ（青色）を確認
    • 消えている → ルーターの電源を確認、必要なら再起動
  3. POSレジ画面右上の 「Terminal 接続中」 表示を確認
    • 未接続の場合 → 画面右上の 【再検出】 ボタン
  4. それでも無反応 → USB-C ケーブルでコンセントに接続（バッテリー切れ）
  5. すべてダメなら Stripe 側の障害の可能性 → 店長またはPOSLA運営に確認

8-4-2. カードが通らない ​

症状: お客様のカードを当てても「決済失敗」

対処:

  1. 別のカード で試す（カード側の問題切り分け）
  2. 挿入式カードなら IC チップ面を正しい向き で挿入
  3. タッチ式なら カードをリーダーから離して再タッチ
  4. それでも失敗: 現金 / QR 決済に切り替え
  5. お客様に無理強いしない（失敗は Stripe 側の判断）

8-4-3. 「[STRIPE_MISMATCH]」エラーが出た ​

症状: カード決済確定時に画面に [STRIPE_MISMATCH] または「決済結果が一致しません」と表示

意味: カード決済の結果とPOSLA側の記録が 一致しなかった 状態です。二重請求や売上集計ズレを防ぐため、POSLAが会計確定を止めています。

対処（必須順序）:

  1. 会計を確定しない（現金 / QR 等への切り替えも一旦止める）
  2. 店長またはオーナーに報告
  3. POSLA 運営（info@posla.jp）に連絡:
    • 店舗名・日時
    • お客様への請求金額（カード末尾 4 桁）
    • 画面に表示されたエラー文
  4. POSLA 運営の確認後、改めて会計可否を判断

このエラーが出たら必ず手を止める

決済結果とPOSLA側の記録が食い違ったまま会計を確定すると、二重請求や売上集計ズレにつながる可能性があります。STRIPE_MISMATCH が出たら必ず手を止めてPOSLA運営に連絡 してください。

8-5. Wi-Fi が切れた / インターネットに繋がらない ​

8-5-1. 症状の確認 ​

  1. タブレット画面右上の Wi-Fi アイコン確認
    • 扇形 3 本 → 強い
    • 扇形 1 本 → 弱い
    • バツマーク → 切れている
  2. ルーターのランプ確認（緑点灯が正常、赤は異常）

8-5-2. 対処（順番に試す） ​

  1. タブレットの Wi-Fi を OFF → ON（設定アプリから）
  2. ルーターを再起動:
    • ルーターの電源コンセントを 30 秒間抜く
    • 再度挿す
    • 緑ランプが全部点灯するまで待つ（約 1 分）
  3. 違う端末（スマホ等）で Wi-Fi に繋いでみる
  4. ブラウザで google.com を開いてみる（Wi-Fi は繋がっているけどインターネットがない場合）
  5. それでもダメなら プロバイダ（NTT / ソフトバンク等）に連絡

8-5-3. オフライン時の POSLA の動き ​

Wi-Fi が切れている間の POSLA の挙動:

  • ハンディの注文: 自動的に端末内へ一時保存 → 再接続時に自動送信
  • KDS: 画面上部にオフラインバナー表示 + 30 秒ポーリング
  • POSレジ: 会計は失敗する（サーバーに届かない）
  • セルフオーダー: お客様のスマホにも同様に保留注文が残る（24 時間以内に接続復旧すれば自動送信）

8-6. お客様の QR コードが読めない ​

8-6-1. 対処フロー ​

  1. QR コードの物理確認:
    • 汚れていないか（拭く）
    • 折れ目・シワがないか
    • 印字がかすれていないか
  2. QR 再印刷:
    • 管理画面 → 【テーブル・フロア】 → 該当テーブルの【QR 印刷】
    • 新しい QR をテーブルに貼る
  3. それでも読めない場合のフォールバック:
    • スタッフがハンディで 【+ 予約なし来店】 から手動着席
    • スタッフがお客様の注文を代わりに取る

8-6-2. お客様のスマホ側の問題 ​

症状原因対処
カメラが QR 認識しない古い iOS / AndroidiOS 14+, Android 9+ 推奨
QR を読んでも URL が出ないQR リーダーアプリの問題カメラアプリで直接読み取り、または Chrome で読み取り
URL をタップしても開かないブラウザのポップアップブロックChrome のプライバシー設定を確認

8-7. 間違えて注文を入れてしまった ​

8-7-1. 注文履歴から編集（会計前） ​

  1. ハンディ画面の 【注文履歴】 タブ
  2. 該当注文をタップ
  3. 【編集】 ボタン
  4. 品目を追加・削除・数量変更
  5. 【注文送信】 で変更内容を反映

8-7-2. 注文をキャンセル（会計前） ​

  1. KDS 画面で該当注文カードをタップ
  2. 【取消】 ボタン（赤）
  3. 確認ダイアログ → 【OK】
  4. 注文がキャンセル済みになる

8-7-3. 会計済みの注文の修正（返金対応） ​

会計後は編集不可。8-8 返金を参照。

8-8. 返金が必要なとき ​

8-8-1. 対象者 ​

マネージャー以上のロールで操作します。スタッフは返金操作を実行できません。返金担当者は出勤打刻済 + レジ担当 PIN 設定済であることを確認してください。

8-8-2. 手順 ​

  1. POSレジ画面ヘッダーの「📋 取引履歴」ボタン押下 → 取引履歴パネルが開く
  2. 取引一覧から該当取引を探す（日時・金額・支払方法で識別）
  3. 該当取引の右側にある「返金」ボタン（赤い枠線 + 赤文字、塗りつぶしではない）を押下
  4. 確認ダイアログ「この取引 (¥XXX) を返金しますか？この操作は取り消せません。」→ OK
  5. 担当 PIN 入力モーダル（テンキー）が表示される
  6. 出勤中スタッフの**レジ担当 PIN（4〜8 桁）**を入力 → 確定
  7. 「返金が完了しました」アラートが表示されれば成功

ボタンが見つからないとき

返金ボタンは取引履歴パネルを開かないと表示されません。POSレジのメイン画面（会計ボタンが並んでいる画面）には返金ボタンはありません。 また、既に返金済みの取引には返金ボタンが出ません（「返金済」バッジが付く）。

8-8-3. 返金処理の中身 ​

  • Stripe カード決済: Stripe の返金処理で自動返金。お客様のカードへ 3〜5 営業日で戻る
  • 現金: POSLA 上では自動返金できません。お客様へ手渡しで返却し、レジの出金記録を残してください。
  • QR 決済: PayPay / d払いの管理画面から別途返金（POSLA は記録のみ）

8-8-4. 部分返金 — 未対応 ​

全額返金のみ対応

現在は 全額返金のみ に対応しています。金額を指定した部分返金はできません。

  • 一部だけ返したい場合は、全額返金したうえで差額を新規会計として取り直してください
  • 部分返金が必要な運用が多い場合は POSLA 運営にご相談ください

8-8-5. 担当 PIN を持っていない / 出勤打刻していない場合 ​

返金ボタン押下後の PIN モーダルで以下のエラーが出るときの対処:

メッセージ原因対処
[E3024] PIN が一致しないか、対象スタッフが出勤中ではありませんPIN 間違い or 未出勤スタッフ管理から PIN 再設定、または出勤打刻
[E3025] 返金には担当スタッフ PIN の入力が必要です空欄送信PIN を入力
[E3038] 担当 PIN は 4〜8 桁の数字で入力してください桁数 / 文字種違反数字 4〜8 桁で入力

担当 PIN 設定方法は「Part 2: スタッフ管理 → 2-1-b スタッフのレジ担当 PIN を設定する」を参照。

8-9. パスワードがわからなくなった ​

8-1-2 を参照。

8-10. 画面が真っ白 / エラーが出る ​

8-10-1. 対処（順番に試す） ​

  1. ブラウザリロード（Cmd+Shift+R / Ctrl+Shift+F5）
  2. ブラウザのキャッシュクリア:
    • Chrome: 設定 → プライバシー → 閲覧履歴データの削除
    • Safari: 設定 → Safari → 履歴と Web サイトデータを消去
  3. 違うブラウザで試す（Chrome 推奨）
  4. タブレットを再起動（電源ボタン長押し → 再起動）
  5. 違う端末で同じ操作を試す（問題の切り分け）
  6. それでもダメなら info@posla.jp に連絡
    • エラー画面のスクリーンショット添付
    • 使用ブラウザ + 端末情報（iPad Pro 12.9, iOS 17 等）

8-11. プリンタが印刷しない ​

8-11-1. チェックリスト ​

  1. プリンタの電源ランプ 緑点灯か確認
  2. 用紙残量 確認（紙がなくなっていないか）
  3. 紙詰まり 確認（紙送りゼロだと印字できない）
  4. プリンタ本体のカバー が正しく閉まっているか
  5. Wi-Fi 接続 確認（プリンタ画面の Wi-Fi マーク）
  6. POSLA 管理画面で 「プリンタ設定」→ 【🖨 テスト印刷】 を実行

8-11-2. テスト印刷で何も出ない場合 ​

  1. プリンタ電源を切る → 10 秒待つ → 再度電源
  2. プリンタの IP アドレスが変わっていないか確認（FEED ボタン長押しで再確認）
  3. POSLA 管理画面でプリンタ設定の IP を更新

詳細は 26 章 参照。

8-12. 予約が反映されない / 自動着席しない ​

8-12-1. お客様が QR を読んでも自動着席しない ​

原因の切り分け:

  1. ガント台帳でその予約を確認
    • テーブル割当済みか？ → 未割当ならスタッフが手動割当
    • 予約時刻 ±30 分以内か？ → 範囲外なら手動着席
    • ステータス（confirmed / pending）か？ → キャンセルされていないか
  2. お客様が読んだ QR が 正しいテーブル のものか？
  3. 上記クリアなら手動着席ボタン（ガントブロック → 🪑 着席）

8-12-2. 予約が LP で作れない ​

原因:

  • 予約設定で 「予約受付」 が OFF になっている
  • スロット容量を使い切っている（その時刻の席が埋まっている）
  • 予約金設定ありで Stripe 未設定（決済画面で止まる）

対処:

  1. 管理画面の【予約管理】→【予約設定】で受付 ON を確認
  2. Stripe 設定が完了しているか確認（21 章）

8-17. 付録 D — サポートガイドの使い方 (非AI、2026-04-23 以降) ​

わからないことは管理画面の サポートガイド で素早く調べます。マネージャー以上が管理画面で利用します。

  1. 管理画面の 右下にあるオレンジ色 📚 ボタン を 1 タップ
  2. サポートガイドパネルが開く (ブラウザ内、通信なしで即表示)
  3. 上部タブから選ぶ:
    • ホーム: カテゴリ別ショートカット + 人気の質問 + エラー番号入力欄
    • よくある質問 (FAQ): ログイン / 注文 / 会計 / 印刷 / 予約 / KDS / メニュー / シフト / テイクアウト / 設定 のカテゴリ別 Q&A
    • キーワード検索: 「会計」「パスワード」「予約」「プリンタ」等で横断検索
    • 症状からたどる: 「ログインできない」「KDSに出ない」等の症状から手順表示
    • エラー番号: E3024 のような番号を入力すると意味と対処を即表示 (代表 15 件)
  4. 収録外の質問は POSLAサポート (info@posla.jp) に連絡

AI ではないので誤回答しません

以前の AI ヘルプデスクと違い、サポートガイドはスタッフが厳選した静的データベースです。画面の回答は常に docs と一致し、曖昧な推測 / ハルシネーションはありません。その代わり収録されていない質問には答えませんので、その場合は POSLA サポートにご連絡ください。

スタッフ用端末にはサポートガイドボタンはありません

KDS・POSレジ・ハンディにはサポートガイドボタンを設置していません（業務集中のため）。スタッフが困ったらマネージャーに聞くフローです。

お客様側（セルフメニュー）の AIウェイター (「このメニューに卵は入っていますか？」などメニュー質問に回答) はこれとは別の機能で継続提供しています。
