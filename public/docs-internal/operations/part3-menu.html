Part 3 — メニュー管理 ​

このページでは、お店のメニューを POSLA に登録・編集する方法を、ボタンの押し方 1 つずつ 説明します。

メニュー管理の考え方（重要）

POSLA のメニューは、複数店舗でも運用しやすいように次の考え方で管理します。

  1. 全店共通メニュー — ブランド全体で共通して販売する商品
  2. 店舗ごとの調整 — 店舗ごとの価格変更・非表示・売切設定
  3. 店舗限定メニュー — その店舗だけで販売する商品

この仕組みにより、同じブランド内で 「全店共通メニュー + 店舗限定メニュー」 を両立できます。

本部一括配信アドオン契約時の特殊運用

本部一括配信アドオンを契約している場合:

  • 全店共通メニューの追加・編集は オーナー管理画面の「本部メニュー」タブ から行います
  • 各店舗の管理画面では、共通メニューの商品名・基本価格・カロリー・アレルゲン・画像などは編集できません
  • 各店舗で変更できるのは、店舗別価格・非表示・売切などの運用項目です

アドオン未契約の場合は、店舗管理画面から通常どおりメニューを追加・編集できます。

3-1. 新しいメニューを追加する ​

3-1-1. 通常（アドオン未契約）の場合 ​

  1. 管理画面にログイン
  2. 上部タブ 【メニュー管理】 → サブタブ 【店舗メニュー】
  3. 画面右上の 【+ メニュー追加】 ボタン
  4. モーダルが開く
  5. 以下を入力:
項目必須説明
品名✅日本語名（例: 鶏の唐揚げ）
品名（英語）—任意（例: Fried Chicken）
カテゴリ✅ドロップダウンから選択
基本価格✅円（例: 900）
カロリー—kcal 単位（例: 380）
アレルゲン—卵 / 乳 / 小麦 / 落花生 / えび / かに / そば のチェックボックス
画像URL—HTTPS の URL（https://...）
  1. 【保存】
  2. セルフメニュー・ハンディに即反映（お客様側は次回リロード、または 3 分キャッシュ）

3-1-2. アドオン契約時の場合 ​

本部一括配信アドオンを契約中のテナントでは、店舗側から本部マスタは追加できません。

  1. オーナー管理画面にオーナーアカウントでログイン
  2. 【本部メニュー】 タブ
  3. 【+ 本部メニュー追加】 ボタン
  4. 本部マスタとして追加（全店共通メニューになる）

店舗側で 「この店舗だけのメニュー」 を追加したい場合は、3-6「限定メニュー」を参照。

3-2. メニューの価格を変更する ​

3-2-1. 全店共通の価格を変える（本部マスタ） ​

アドオン未契約: 管理画面の「店舗メニュー」→ 編集で直接変更 アドオン契約中: オーナー管理画面「本部メニュー」→ 編集

3-2-2. 店舗別の価格を上書きする（店舗オーバーライド） ​

同じ商品を店舗ごとに別価格で売りたい場合。

  1. 管理画面 → 【メニュー管理】 → 【店舗メニュー】
  2. 該当メニューの 【編集】 ボタン
  3. 「店舗価格（上書き）」 欄に新しい価格を入力
    • 例: 本部価格 ¥900 / 渋谷店（価格上書き）¥1,000 / 新宿店（価格上書きなし）→ ¥900
  4. 【保存】
  5. セルフメニュー・ハンディに店舗別価格が反映される

クリアするには: 同じ欄を空白にして保存します。空白に戻すと、全店共通の基本価格が適用されます。

3-2-3. 単純な価格変更の手順（本部価格変更） ​

  1. 【店舗メニュー】で該当メニューの 【編集】
  2. 基本価格 欄を変更
  3. 【保存】

3-3. メニューの写真（画像）を登録する ​

3-3-1. 画像の準備 ​

  • 推奨サイズ: 800×600px または 1000×750px（4:3 比率）
  • フォーマット: JPG または PNG
  • ファイルサイズ: 200KB 以下推奨（軽いほうが表示が早い）
  • 明るく・清潔感のある写真

3-3-2. 画像を公開 URL にアップロード ​

POSLA 側に画像アップロード機能はないため、別サービスで公開アップロードしてから URL を取得してください。

推奨サービス:

  • Imgur（無料、imgur.com）— 一番簡単
  • 自社ホームページや画像管理サービス
  • Cloudinary（無料枠あり、画像最適化も）
  • 店舗のホームページ（WordPress 等）

手順（Imgur の場合）:

  1. imgur.com にブラウザでアクセス
  2. 「New post」 ボタン → 画像をドラッグ&ドロップ
  3. アップロード完了後、画像を右クリック → 「画像アドレスをコピー」
  4. URL は https://i.imgur.com/xxxxx.jpg のような形式

3-3-3. POSLA に URL を登録 ​

  1. 該当メニューの 【編集】
  2. 画像URL 欄にコピーした URL を貼り付け
  3. 【保存】
  4. プレビュー画像がサムネイル表示される

3-3-4. よくあるエラー ​

症状原因対処
画像が表示されないURL が HTTP（http://）になっているHTTPS（https://）の URL に変更
404 エラーURL が無効 / 画像が削除されたImgur で画像を確認、URL 再取得
崩れて表示極端なアスペクト比4:3 または正方形推奨
画像が重くて遅いファイルサイズが大きすぎ200KB 以下に圧縮（tinypng.com 等で）

3-4. カテゴリの追加・並べ替え ​

3-4-1. カテゴリとは ​

メニューを分類するグループです。セルフメニュー画面で横タブとして表示されます。

例:

  • 前菜 / メイン / ご飯もの / デザート / ドリンク
  • 鮮魚 / 寿司 / 天ぷら / ご飯もの / デザート
  • 串 / 刺身 / サラダ / ご飯 / ドリンク

3-4-2. カテゴリを追加する ​

  1. 管理画面 → 【メニュー管理】 → サブタブ 【カテゴリ管理】
  2. 【+ カテゴリ追加】 ボタン
  3. モーダル:
    • カテゴリ名（例: 「デザート」）
    • カテゴリ名（英語）（例: Dessert）
    • 並び順（数値、小さいほど上に表示。例: 5）
  4. 【保存】

3-4-3. 並べ替え ​

  1. 該当カテゴリの 【編集】
  2. 並び順 の数値を変更
  3. 【保存】

ドラッグ&ドロップ並び替えは今後実装予定

現状は数値入力方式。視覚的な並び替えは将来のアップデートで対応予定。

3-4-4. カテゴリの削除 ​

  1. 該当カテゴリの 【削除】 ボタン
  2. 確認ダイアログ
  3. 削除前の警告: そのカテゴリに紐付く品目がある場合、削除できない（品目のカテゴリを先に変更する必要あり）

3-5. オプション（トッピング等）の設定 ​

3-5-1. オプションとは ​

メニューに追加で選べる項目。例:

  • 「唐揚げ」に対して 【サイズ選択】（小 / 中 / 大、+価格）
  • 「パスタ」に対して 【ソース選択】（トマト / クリーム / ペペロンチーノ）
  • 「ラーメン」に対して 【トッピング】（煮卵 +100 / チャーシュー +200 / コーン +50）

3-5-2. オプショングループを作る ​

  1. 管理画面 → 【メニュー管理】 → サブタブ 【オプション管理】
  2. 【+ オプショングループ追加】 ボタン
  3. モーダル:
項目説明
グループ名例: 「トッピング」
グループ名（英語）例: Toppings
選択必須トグル ON/OFF（必須にするとお客様が必ず選ばなければならない）
複数選択可トグル ON/OFF（ON なら客が複数選べる）
並び順数値
  1. 【保存】
  2. グループが一覧に追加される

3-5-3. グループに選択肢を追加する ​

  1. 作成したグループの行で 【＋ 選択肢追加】 をタップ
  2. モーダル:
    • 選択肢名（例: 「チーズ増し」）
    • 選択肢名（英語）（例: Extra Cheese）
    • 追加価格（例: 200 で +¥200、0 なら価格変動なし）
  3. 【保存】
  4. 繰り返して選択肢を増やす（例: チーズ増し、コーン、煮卵）

3-5-4. 選択肢をメニューに紐付ける ​

  1. 【メニュー管理】 → 【店舗メニュー】
  2. 該当メニューの 【編集】
  3. 「オプショングループ」 セクション
  4. 使いたいグループのチェックボックスを ON
  5. 【保存】
  6. セルフメニューで該当メニューをお客様が選ぶと、オプション選択モーダルが自動で開く

3-6. 限定メニュー（店舗独自品）の追加 ​

本部マスタにない、その店舗だけのメニュー。

3-6-1. 手順 ​

  1. 管理画面 → 【メニュー管理】 → サブタブ 【限定メニュー】
  2. 【+ 限定メニュー追加】 ボタン
  3. モーダル:
    • 品名
    • 品名（英語）
    • カテゴリ（本部マスタのカテゴリを流用）
    • 価格
    • カロリー・アレルゲン（任意）
    • 画像URL（任意）
  4. 【保存】
  5. セルフメニューに 「限定」 ピンクバッジ付きで表示される

3-6-2. 限定メニューの使いどころ ​

  • 季節メニュー: 冬の鍋メニュー、春のお花見弁当など
  • 店舗独自開発品: 渋谷店シェフのオリジナル料理
  • 試作メニュー: 特定店舗でテスト販売してから本部マスタに昇格

3-7. 多言語翻訳（英語・中国語・韓国語等） ​

3-7-1. 自動一括翻訳 ​

メニュー全体を一度に翻訳。

  1. 管理画面 → 【メニュー管理】 → 【店舗メニュー】
  2. 画面右上の 【🌐 一括翻訳】 ボタン
  3. モーダルで翻訳先言語をチェック:
    • ☑ English
    • ☑ 中文（簡体字）
    • ☑ 繁體中文
    • ☑ 한국어
  4. 【翻訳開始】
  5. AI（Gemini）が数十秒で全品目・カテゴリを一括翻訳
  6. 「翻訳完了」トースト表示

3-7-2. 強制再翻訳 ​

既に翻訳済みのものを上書きしたい時。

  1. 一括翻訳ダイアログで 「🔄 強制再翻訳」 チェックボックス ON
  2. 【翻訳開始】
  3. 既存の翻訳を上書き

3-7-3. 個別修正 ​

AI の訳が気に入らない場合。

  1. 該当メニューの 【編集】
  2. 品名（英語） 欄に手動入力（例: Fried Chicken Curry Rice）
  3. 【保存】

3-7-4. お客様側での言語切替 ​

  • セルフメニュー画面右上の言語ボタン（🌐）でお客様が切替
  • 日本語 / English / 中文 / 繁體 / 한국어 の 5 言語

3-8. CSV でメニューを一括登録する ​

大量のメニューを一度に登録する場合は CSV 一括登録が便利。

3-8-1. エクスポート（バックアップ・雛形取得） ​

  1. 【店舗メニュー】 → 【CSV エクスポート】 ボタン
  2. ダウンロードフォルダに CSV 保存
  3. Excel / Numbers で開く

3-8-2. CSV の編集 ​

CSV には、画面から入力する内容と同じ項目を入れます。

項目必須説明
品名（日本語）✅メニュー名
価格✅円単位の販売価格
カテゴリ✅既存カテゴリと同じ名前
英語名—多言語表示で使う名称
カロリー—kcal 単位
アレルゲン—卵・乳・小麦など
画像URL—画像を表示するためのURL

3-8-3. インポート（一括登録） ​

  1. 【店舗メニュー】 → 【CSV インポート】 ボタン
  2. CSV ファイルを選択
  3. プレビュー画面で内容確認
  4. エラーがあれば修正指示が表示される
  5. 【インポート実行】
  6. 一括登録完了

3-8-4. 文字化け対処 ​

日本語が文字化けする場合:

  • Windows Excel: 保存時に 「CSV UTF-8（コンマ区切り）」 を選択
  • Mac Numbers: デフォルトで UTF-8 なので OK
  • Google Sheets: ファイル → ダウンロード → カンマ区切りの値（UTF-8）

3-9. スマレジからメニューをインポートする ​

スマレジを使っているお店が POSLA に移行する場合、メニューを自動移行できます。

詳細は 22 章「スマレジ連携の全設定」 参照。

簡易手順 ​

  1. オーナーアカウントでオーナー管理画面 → 【外部POS連携】 タブ
  2. 【OAuth 接続】 → スマレジに既存アカウントでログイン
  3. 許可を与える
  4. 【商品インポート】 ボタン
  5. 非破壊インポート（既存 POSLA メニューは維持、スマレジの商品だけ追加）

3-10. 本部メニューの管理（チェーン店のみ） ​

本部一括配信アドオン契約時のみ利用できます。

3-10-1. 本部メニュータブの場所 ​

  1. オーナーアカウントでオーナー管理画面
  2. 上部タブ 【本部メニュー】

3-10-2. 本部メニューの追加・編集 ​

  1. 【+ 本部メニュー追加】
  2. モーダルで品名・価格・カテゴリ入力
  3. 【保存】
  4. 全店舗に自動配信される

3-10-3. 本部 CSV 一括管理 ​

  1. 【CSV エクスポート】
  2. 【CSV インポート】

3-10-4. カテゴリフィルタ ​

本部メニューが多くなったら、画面上部のカテゴリ選択で絞り込みます。

3-10-5. 店舗側での本部メニュー編集制限 ​

アドオン契約中、各店舗の管理画面では全店共通メニューの基本情報は編集できません。店舗側で変更できるのは次の項目です。

  • 店舗別価格
  • 非表示
  • 売切

3-11. 今日のおすすめ（日替わり推し商品） ​

3-11-1. 機能概要 ​

特定のメニューを「今日のおすすめ」として、セルフメニュー画面の上部に目立つように表示する機能。

3-11-2. 設定手順 ​

  1. 管理画面 → 【メニュー管理】 → 【店舗メニュー】
  2. 画面下部の 「今日のおすすめ」 セクション（折りたたみ式）を展開
  3. メニューを選択（最大 5 品目）
  4. 【保存】
  5. 日付指定で予約設定も可（翌日の「おすすめ」を事前登録）

3-12. 品切れ切替（キッチンから即時反映） ​

3-12-1. 管理画面から ​

  1. 管理画面 → 【店舗メニュー】
  2. 該当メニューの 「売切」トグル を ON
  3. セルフメニューで「品切れ」バッジ表示
  4. 新規注文で該当メニューを選ぶとエラー

3-12-2. KDS 画面から ​

キッチンスタッフが素早く切替できるように KDS 側にも UI あり。

  1. KDS 画面の 【品切れ管理】 ボタン
  2. 品切れにする品目をタップ
  3. 即反映

3-13. よくある質問 ​

Q1. カテゴリを削除できない

A1. そのカテゴリに紐付く品目がある場合は削除できません。先に該当品目のカテゴリを変更してから削除してください。

Q2. 画像URL を公開サービスに置きたくない

A2. 店舗のホームページや画像管理サービスに画像を置き、その画像URLを設定してください。画像アップロード機能は今後対応予定です。

Q3. AI 翻訳が不正確

A3. 固有名詞（「牛丼」→「Gyudon」等）は AI が一律翻訳しきれないので、手動修正してください。

Q4. メニュー画像は必須？

A4. 必須ではないですが、写真があるとお客様の注文率が大幅に上がります。最低でも主要メニューには画像を付けましょう。

Q5. CSV インポートで日本語が文字化け

A5. Excel の保存時に 「CSV UTF-8」 を選択してください（通常の CSV だと Shift-JIS で保存され文字化けの原因）。
