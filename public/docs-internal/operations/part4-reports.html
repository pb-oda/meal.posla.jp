Part 4 — 売上とレポート ​

このページでは、お店の売上・品目別ランキング・スタッフ評価・顧客分析などの各種レポートの見方と活用方法を説明します。

レポートの種類と見る画面

レポートは以下の 2 つの画面に分かれています。

  • 管理画面の「レポート」タブ（マネージャー以上）: 基本レポート全般
  • オーナー管理画面のタブ（オーナー専用）: ABC 分析、クロス店舗レポート

4-1. 日次売上の見方 ​

4-1-1. 売上レポートを開く ​

  1. 管理画面にマネージャー以上でログイン
  2. 上部タブ 【レポート】 → サブタブ 【売上レポート】
  3. 画面が開く

4-1-2. 表示される項目 ​

項目説明
売上合計期間内の税込売上合計
注文件数注文数
客単価売上合計 ÷ 注文件数
時間帯別売上1時間ごとの売上推移（棒グラフ）
日別売上推移過去 7 / 30 / 90 日の折れ線グラフ
支払方法別現金・カード・QR の内訳（円グラフ）

4-1-3. 期間選択 ​

画面上部に期間切替ボタン:

  • 日次: 本日の売上
  • 週次: 今週月〜日
  • 月次: 今月
  • カスタム: 任意の開始日〜終了日

4-1-4. 活用例 ​

  • 毎朝「昨日の売上」確認
  • 週次ミーティングで「先週の合計」報告
  • 月次決算で「月間売上」集計
  • 同じ曜日の比較（例: 先週金曜と今週金曜）

4-2. 商品別ランキング ​

4-2-1. 見方 ​

  1. 【売上レポート】 内の 「商品別ランキング」 セクション（同じタブ内の下スクロール）
  2. 売上順に品目が表示
  3. 各品目の情報:
列説明
商品名メニュー名
販売数期間内の注文数
売上合計円単位
粗利売上 − 原価（レシピ設定時のみ表示）

4-2-2. 活用例 ​

  • 上位 10 品: 看板メニューの把握、在庫発注優先度
  • 下位 10 品: 廃止候補・値引き候補
  • 粗利率ランキング: 利益率の高い品目を積極プッシュ

4-3. ABC 分析（オーナー専用） ​

ABC 分析は オーナー管理画面の専用タブ で確認します。マネージャー・スタッフには表示されません。

4-3-1. 開く手順 ​

  1. オーナーアカウントでオーナー管理画面を開く
  2. 上部タブ 【ABC 分析】

4-3-2. A / B / C の定義 ​

売上累積シェアで 3 ランクに分類:

ランクシェア意味対応方針
A ランク売上上位 70%主力商品品切れ厳禁、メイン訴求に使う
B ランク70〜90%安定商品現状維持、プラスアルファの訴求
C ランク90〜100%下位商品廃止候補 / メニュー見直し / 値下げ検討

4-3-3. 活用例 ​

  • 全 50 品のうち C ランク 25 品を見直し
  • A ランクの原材料を優先発注
  • A ランクの新規バリエーション開発（例: 「醤油ラーメン」が A → 「辛味噌ラーメン」を追加）

4-4. バスケット分析（併売分析） ​

「○○を注文した人は △△ も買う」傾向を分析。

4-4-1. 見方 ​

  1. 管理画面 → 【レポート】 → サブタブ 【併売分析】
  2. 品目の組み合わせがリフト値順に表示

4-4-2. リフト値とは ​

「Aを買った人がBも買う確率 ÷ 全注文でBが買われる確率」

  • リフト値 > 1: 一緒に買われやすい
  • リフト値 = 1: 関係なし（独立）
  • リフト値 < 1: むしろ一緒には買われにくい

例: ラーメン + 餃子 のリフト値 = 2.5 → ラーメンを食べる人は通常の 2.5 倍の確率で餃子も頼む

4-4-3. 活用例 ​

  • セットメニュー提案: 「ラーメン + 餃子」セットを作って単価アップ
  • レコメンド: セルフメニューで「一緒にいかがですか？」で餃子を推薦
  • メニュー配置: 一緒に頼まれる品目を隣接ページに配置

4-6. スタッフ評価レポート ​

4-6-1. 開く手順 ​

  1. 管理画面 → 【レポート】 → サブタブ 【スタッフ評価】

4-6-2. スタッフごとに表示 ​

4-6-3. 活用例 ​

  • 研修対象の発見: 客単価が低いスタッフは接客改善研修
  • 優秀スタッフの昇格判断: 全指標が高いスタッフはマネージャー候補

4-7. 満足度・低評価分析 ​

お客様のセルフ注文画面で送信された星評価（1〜5）を集計し、店舗運営の改善に活用するレポートです。低評価（1〜2）には、お客様が任意で選んだ理由コードと自由コメントが含まれる場合があります。

スタッフ個人評価ではなく、改善材料として扱ってください

  • 顧客のセルフ注文経由の評価では、担当スタッフが特定されないケースが多くあります。
  • 同じ品目に低評価が集中している場合、原因はレシピ・温度・盛り付け・提供時間など店舗オペレーション側にあることが多いです。
  • 「低評価が多い時間帯 = ピーク時の人員不足や提供導線の混雑」のように、運営改善のヒントとして使ってください。
  • 個別のスタッフを責める材料には使わないでください。

4-7-1. 開く手順 ​

  1. 管理画面 → 上部タブ 【レポート】 → サブタブ 【満足度分析】
  2. 期間を選び、【表示】ボタンを押す（今日 / 昨日 / 今週 / 今月のプリセットあり）

4-7-2. サマリーカード ​

画面上部に以下が並びます。

指標説明
評価件数期間内に届いた星評価の総数
平均評価1〜5 の平均値
低評価件数星 1〜2 の件数
低評価率低評価件数 ÷ 評価件数
高評価件数星 4〜5 の件数

4-7-3. 低評価の理由ランキング ​

低評価を選んだお客様が任意で送ってくれた理由を集計します。

理由コード表示ラベル
slow_service提供が遅かった
taste味が好みではなかった
temperature温度が気になった
portion量が気になった
wrong_item注文内容と違った
staff接客が気になった
cleanliness清潔感が気になった
price価格が気になった
otherその他

理由送信は任意です

お客様には負担をかけないよう、理由選択は任意設定です。理由なしで星だけ送られてきたケースもこのテーブルには現れません。理由が集まらないからといって低評価を軽視せず、品目ランキング・時間帯ランキングと併せて確認してください。

4-7-4. 低評価が出た品目ランキング ​

品目ごとの低評価件数・平均評価・最終発生時刻を一覧表示します。

活用例:

  • 同じ品目が常に上位 → レシピ・温度・盛り付けを確認
  • 一時的に上位 → 食材ロット・調理担当の交代タイミング・ピーク時のオペレーションを確認
  • 平均評価が 2 を切っている品目 → 一時的な販売停止や改善試作を検討

4-7-5. 時間帯別 低評価 ​

0〜23 時の各時間帯について、評価件数・低評価数・低評価率を表示します（評価が無い時間帯は 0 件で並びます）。

活用例:

  • ランチピーク（12 時台）に低評価率が突出 → 人員配置・調理キャパシティ・提供順序を見直し
  • 夜営業のラストオーダー前（22 時台等）の低評価が多い → 在庫切れによる代替対応・温度管理を確認
  • 開店直後の低評価が多い → 仕込み完了タイミング・予熱不足を確認

4-7-6. 直近の低評価一覧 ​

最大 50 件の低評価を新しい順に表示します。

列説明
日時評価が送信された日時
品目該当の料理
評価星の数 (1〜2)
理由お客様が選んだ理由ラベル（任意）
コメントお客様の自由記述（任意・最大 255 文字）
注文注文 ID の短縮表示

コメントの扱い

自由コメントは、お客様が「気になったこと」を率直に書いてくれている場合があります。否定的な書き方でも責めの意図ではないことが多いので、改善材料として冷静に読んでください。スタッフ個人を名指しするコメントが入った場合は、内容を確認したうえで該当スタッフと一対一で話し合うなど、適切な配慮をしてください。

4-7-7. 全店舗合算で見る (オーナーのみ) ​

オーナーアカウントでオーナー管理画面 →【全店舗横断レポート】を開くと、末尾に 満足度サマリー が表示されます。

  • 全店舗合算の平均評価・低評価率
  • 店舗別 平均評価・低評価率
  • 低評価が多い品目ランキング（全店舗合算）
  • 低評価の理由ランキング（全店舗合算）

複数店舗を比較して、店舗別の改善優先度を判断するのに役立ちます。

店舗別と全店合算で件数がわずかにずれることがあります

店舗別の満足度分析は 営業日区切り で集計、全店舗横断の満足度サマリーは 暦日 + 余白付き の簡易範囲で集計しています。これは横断レポート全体（売上品目・キャンセル品目・迷い品目）と同じ集計レンジを使うためで、深夜営業の店舗で数件の差が出る可能性があります。

4-9. クロス店舗レポート（オーナー向け） ​

複数店舗の売上・客数・業績を横並びで比較。満足度サマリー（§4-7-7）も同じレポート末尾に表示されます。

4-9-1. 開く手順 ​

  1. オーナーアカウントでオーナー管理画面を開く
  2. 上部タブ 【クロス店舗レポート】

4-9-2. 表示項目 ​

  • 店舗ごとの売上合計
  • 店舗ごとの客数
  • 店舗ごとの客単価
  • 全店平均・最大・最小のベンチマーク
  • 期間別（日次・週次・月次）
  • 満足度サマリー（全店舗合算 + 店舗別 + 低評価品目 / 理由ランキング）

4-9-3. 活用例 ​

  • 全 3 店舗のうち 1 店舗だけ売上低い → 原因調査（立地 / スタッフ / メニュー）
  • 業績トップ店舗の成功要因を他店舗に横展開
  • 低評価が一店舗に集中 → §4-7 と併せて品目・時間帯・運営の改善ポイント特定

4-10. レジ分析（レジ締め精算レポート） ​

4-10-1. 開く手順 ​

  1. 管理画面 → 【レポート】 → サブタブ 【レジ分析】

4-10-2. 表示項目 ​

項目説明
レジ開閉履歴いつ・誰が開けた / 閉めたか、開始時在高、締め時現金、担当者
日別キャッシュログ釣銭補充、現金回収、両替、小口購入、消耗品購入、過不足の履歴
金種内訳レジ開け・レジ締めで入力した紙幣・硬貨の枚数
予想在高 vs 実際差額がプラスなら過多、マイナスなら不足
レジ締め照合締め時刻、担当者、予想現金、実際現金、差額、現金売上、カード売上、QR/電子マネー売上、締めメモ
締め前チェック未会計、取消・返金、外部決済控えメモ未入力、現金差額、締め後操作を確認
閉店引き継ぎメモ差額理由、未処理注文、翌日共有事項を確認
レジ状態未開け、締め未完了、過不足500円以上、締め完了、要確認を表示
会計取消・返金履歴処理日時、取消/返金、支払方法、金額、担当者、理由

過不足アラート閾値（デフォルト ¥500）を超える差額は警告

4-10-3. 活用例 ​

  • 毎日のレジ締めで過不足チェック
  • 外部カード端末・QR決済端末の日締め控えとPOSLA上のカード/QR・電子マネー売上を突き合わせ
  • 締め未完了のままになっている営業日を管理画面から発見
  • 取消・返金された会計を日次精算時に確認
  • 釣銭補充や現金回収の記録漏れをキャッシュログから確認
  • 閉店引き継ぎメモを見て、翌日に調査すべき差額や未処理を確認
  • 外部決済控えメモ未入力の取引を探し、端末控えと照合
  • レジ分析CSVを税理士・本部・店長へ共有
  • 過不足が続くスタッフの現金取扱い研修
  • 入金・出金記録漏れのチェック

4-10-4. レジ締めで問題が出た日の見方 ​

  1. レジ状態 で「要確認」「過不足500円以上」「締め未完了」の日を探す
  2. レジ締め照合 で予想現金、実際現金、差額、締めメモを見る
  3. 日別キャッシュログ で釣銭補充、現金回収、両替、小口購入の記録漏れがないか見る
  4. 会計取消・返金履歴 で、現金返却や取消理由が正しいか見る
  5. 支払い詳細別 / 外部決済控え で、カード端末・QRアプリの日締め控えとPOSLAの記録を突き合わせる
  6. 原因が当日中に確定しない場合は、閉店引き継ぎメモを残して翌日調査に回す

管理画面で見るべき順番

最初に「差額」、次に「現金移動」、最後に「取消・返金」と「外部決済控え」を見ます。いきなり全取引を見るより、原因に近い場所から確認する方が早く終わります。

4-11. 注文履歴 ​

4-11-1. 開く手順 ​

  1. 管理画面 → 【レポート】 → サブタブ 【注文履歴】

4-11-2. 表示項目 ​

  • 全注文の一覧（日時・テーブル・品目・金額・ステータス）
  • 検索フィルタ（日時・テーブル・品目・ステータス）
  • CSV エクスポート可

4-11-3. 活用例 ​

  • 「先週金曜の 19 時の注文を確認したい」等の個別調査
  • クレーム対応: お客様から「○○を頼んだはず」と言われた時の確認
  • 税務監査対応: 過去の取引記録

4-12. CSV エクスポートの方法 ​

各レポートの 【CSV 出力】 ボタンから:

  1. 期間を選択
  2. 【CSV 出力】
  3. ダウンロードフォルダに保存
  4. Excel / Numbers / Google Sheets で開く

レジ分析CSVには、期間内のレジ状態、会計取消・返金履歴、レジ締め照合、支払方法別、支払い詳細別、外部分類別、レジログが含まれます。

文字化け対処 ​

  • Excel の場合: 「データ」→「テキストから」でインポート時に UTF-8 指定
  • Numbers / Google Sheets: デフォルト UTF-8 で OK

4-13. 活用ストーリー例 ​

例 1: 月次決算（毎月 1 日の作業） ​

  1. 先月の 売上レポート（月次） で合計売上を確認
  2. ABC 分析 で主力商品の推移を見る
  3. スタッフ評価 で最優秀スタッフにボーナス
  4. レジ分析 で過不足合計を確認

例 2: 新メニュー導入判断（週次） ​

  1. バスケット分析 で既存メニューの組み合わせ傾向を把握
  2. ABC 分析 で C ランク品目を特定
  3. C ランク廃止 → 新メニュー追加の判断

例 3: スタッフ配置の最適化（月 1 回） ​

  1. 回転率レポート の時間帯別で、ピーク時間を特定
  2. スタッフ評価 で客単価の高いスタッフを把握
  3. ピーク時間に優秀スタッフを配置

4-14. よくある質問 ​

Q1. レポートの数字が合わない

A1. 期間指定のミスが多い。「営業日区切り時刻」（デフォルト 05:00）を確認。深夜 1 時の売上が「翌日」扱いになっていないか。

Q2. 粗利が表示されない

A2. 粗利表示には レシピ設定（品目と原材料の紐付け）が必要。在庫・レシピ管理タブで設定してください。

Q3. スタッフ評価の「担当注文」とは？

A3. ハンディで注文入力した個人スタッフ、または POSレジで会計担当に指定されたスタッフの注文です。

Q4. ABC 分析が見えない

A4. ABC 分析は オーナー専用です。オーナーアカウントでオーナー管理画面にログインしてください。

Q5. CSV ダウンロードしたら改行がおかしい

A5. Excel で直接開くと表示が崩れることがあります。「データ」→「テキストまたはCSVから」で取り込み、文字コードを UTF-8 に指定してください。
