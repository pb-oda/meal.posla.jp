Operations Overview

ダッシュボード

擬似本番で現在見えている全体像を素早く把握するための入口です。テナント数、店舗数、利用状況の偏りをここで確認し、必要なら各管理タブに移動します。

契約構成

単一プラン運用を前提に、標準契約数と本部一括配信オプション契約数を確認します。

最近のテナント

直近で作成または確認対象になったテナントを一覧で確認します。

危険テナント上位

健全性スコアが低いか、直近で未解決異常を抱えているテナントを優先表示します。

オンボーディング要フォロー

卓・メニュー・運営アカウントなどが揃い切っていないテナントを進捗順に確認します。

Cell作成待ち

LP申込・POSLA管理画面追加から登録された、1 tenant / 1 cell provisioning の待ち行列です。

Tenant Operations

テナント管理

契約、Stripe Connect、店舗数、ユーザー数を横断で確認する一覧です。テナント単位の異常や運用差分をここから拾います。

テナント一覧

健全性スコア、導入進捗、直近異常、契約状態を 1 行で追える構成です。

Provisioning

テナント新規作成

擬似本番で新しいテナントを追加し、同時にテスト用の店舗とログインIDまで立ち上げる入口です。作成直後から owner / manager / staff / device で検証を始められます。

半角英小文字・数字・ハイフンのみ
α-1 以降は単一プラン運用です。`tenants.plan` には互換用に `standard` を保存します。

作成時の前提

  • 作成時に既定店舗 `main` と `store_settings`、テスト用ログイン `owner / manager / staff / device` を同時に作成します。
  • 初期パスワードは共通で `Demo1234` です。結果カードに生成済み username を表示します。
  • 作成直後は Stripe・Connect・Google Chat・PWA などの外部連携は未設定でも構いません。
  • 契約差分は `hq_menu_broadcast` オプションだけです。旧 `plan` は表示互換のためだけに保持しています。
Rollout Control

Feature Flags

global / cell / tenant 単位で運用機能の表示・書き込み許可を切り替えます。契約アドオンの `hq_menu_broadcast` とは分離して扱います。

解決状態

選択 tenant に対して default → global → cell → tenant の順で解決した結果です。

Override

cell scope は現在の cell_id に保存します。
Platform Settings

API設定（POSLA共通）

POSLA運営が全テナント共通で使う外部連携設定です。実値は読み出さず、状態だけ表示して変更時のみ新しい値を入力します。運用監視は Google Chat webhook を正系、旧 Slack webhook を後方互換 fallback として扱います。

秘密値充足チェック

実値は表示せず、連携ごとの `設定済 / 未設定 / 不整合` だけを判定します。本番投入前の差分確認用です。

Operations Source

OP連携

codex-ops-platform が POSLA の snapshot を読む入口を管理します。ここで扱う source は顧客cellではなく、顧客cell一覧を返すための制御用入口です。

Source
POSLA control source
`posla_cell_registry` とは分けて管理します。`registry.cells` には tenant 付きの顧客cellだけを出します。
op側で識別する POSLA 入口 ID。顧客cell ID ではありません。
本番では `X-POSLA-OPS-SECRET` を正系にします。ローカル検証では cron secret も許可できます。
Endpoint
opが読むURL
op側には snapshot URL と認証ヘッダーを登録します。ping URL は疎通確認、snapshot URL は cell discovery に使います。

登録済み source

複数環境を管理する場合は source を追加し、op側で接続先を選べるようにします。

Notifications Infrastructure

PWA / Web Push

Web Push の状態確認、VAPID 鍵の生成・取得・退避、購読の全体傾向をここで扱います。鍵を再生成すると既存購読は再登録が必要になります。

注意: 再生成は既存のブラウザ購読を無効化するため、店舗端末・スタッフ端末側で再登録が必要になります。

鍵の取得 / 退避

公開鍵・秘密鍵・退避用 bundle を必要なときだけ取得します。

VAPID 鍵

購読統計

登録済み端末と有効購読数の分布を確認します。

直近 24 時間 送信統計

通知送信の成功率とエラー傾向を把握します。

Admin Identity

POSLA 管理者ユーザー

POSLA運営アカウントの作成、権限、最終ログイン状況を扱います。テナント側ユーザーとは完全に別管理です。

Customer Assist

顧客サポート

顧客側セッションの状態を読み取り専用で確認する運用ビューです。店舗操作や注文送信の代行は行わず、現状把握に限定します。

Support Knowledge

📚 サポートセンター

テナント問い合わせ時の一次回答と誘導先をまとめた運用ナレッジです。AI ではなく固定ナレッジベースとして扱います。

Ops Readonly

🛠️ 運用補助センター

障害切り分けや日次確認のための read-only 支援ビューです。本番修正や強制操作はここでは行いません。
